## Overview

### What is S3? Why Do We Need It?

Imagine you're building a web application and users upload profile photos. Where do you store those photos? If you store them on your EC2 server's disk, they're gone if the server crashes. You also can't share them across multiple servers easily.

**Amazon S3 (Simple Storage Service)** solves this — it's like an **infinite, indestructible USB drive in the cloud**. You upload any file (photos, videos, backups, logs, datasets), and AWS stores it across multiple data centers automatically. Even if hardware fails, your data is safe.

**Simple examples of S3 usage:**
- 📸 A photo-sharing app stores user uploads in S3
- 💾 A company backs up databases to S3 every night
- 🌐 A startup hosts their website's HTML/CSS/JS on S3 (static website)
- 📊 A data team stores terabytes of CSV/Parquet files in S3 for analytics

S3 stores virtually unlimited data as objects in buckets with 99.999999999% (11 nines) durability. It's one of the most fundamental AWS services — used for backups, static websites, data lakes, application assets, and much more.

## Part 1: S3 Fundamentals

```
┌─────────────────────────────────────────────────────────────────────┐
│           S3 CORE CONCEPTS                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                       │
│ What is S3?                                                          │
│ ├── Object storage service (not file system, not block storage)   │
│ ├── Stores data as "objects" in "buckets"                          │
│ ├── Each object: data + metadata + unique key                     │
│ ├── Max object size: 5 TB (single upload: 5 GB)                  │
│ ├── Unlimited total storage per bucket                            │
│ └── 99.999999999% (11 nines) durability                           │
│                                                                       │
│ Bucket:                                                              │
│ ├── Container for objects                                         │
│ ├── Name must be GLOBALLY unique (across ALL AWS accounts)       │
│ ├── Created in a specific region                                  │
│ ├── Flat structure (no real folders — just key prefixes)          │
│ └── Soft limit: 100 buckets per account (can increase to 1000)   │
│                                                                       │
│ Object:                                                              │
│ ├── Key: Full path (e.g., photos/2024/vacation/beach.jpg)        │
│ ├── Value: The actual data (content of the file)                  │
│ ├── Metadata: System + user-defined key-value pairs              │
│ ├── Version ID: If versioning is enabled                          │
│ ├── Tags: Key-value pairs for organization/billing               │
│ └── Max size: 5 TB                                                │
│                                                                       │
│ S3 URL patterns:                                                     │
│ ├── Path-style: https://s3.amazonaws.com/my-bucket/key           │
│ ├── Virtual-hosted: https://my-bucket.s3.amazonaws.com/key       │
│ ├── Region-specific: https://my-bucket.s3.us-east-1.amazonaws.com│
│ └── ⚡ Virtual-hosted style is the recommended default             │
│                                                                       │
│ "Folders" in S3:                                                     │
│ ├── S3 has NO folder/directory concept — it's flat!              │
│ ├── Key "photos/2024/beach.jpg" is just a string                 │
│ ├── Console SHOWS it as folders for convenience                   │
│ ├── The "/" in key names is just a character (prefix)             │
│ └── Prefix-based listing: list all keys starting with "photos/"  │
│                                                                       │
│ Consistency Model:                                                   │
│ ├── Strong read-after-write consistency (since Dec 2020)         │
│ ├── PUT new object → immediately readable                        │
│ ├── Overwrite PUT → immediately see new version                  │
│ ├── DELETE → immediately reflected                                │
│ └── No more eventual consistency headaches!                       │
│                                                                       │
│ Pricing (us-east-1):                                                │
│ ├── Storage: $0.023/GB/month (Standard)                          │
│ ├── PUT/COPY/POST: $0.005 per 1,000 requests                   │
│ ├── GET/SELECT: $0.0004 per 1,000 requests                     │
│ ├── Data transfer OUT: $0.09/GB (first 10 TB)                  │
│ ├── Data transfer IN: FREE                                       │
│ └── ⚡ Free tier: 5 GB Standard, 20K GET, 2K PUT per month       │
│                                                                       │
└─────────────────────────────────────────────────────────────────────┘
```

### Bucket Naming Rules

```
┌─────────────────────────────────────────────────────────────────────┐
│           BUCKET NAMING RULES                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                       │
│ ✅ Must be 3-63 characters                                           │
│ ✅ Lowercase letters, numbers, hyphens only                          │
│ ✅ Must start with letter or number                                  │
│ ✅ Must NOT start with "xn--" (reserved)                             │
│ ✅ Must NOT end with "-s3alias" (reserved)                           │
│ ✅ Must be unique across ALL AWS accounts globally                   │
│                                                                       │
│ ❌ No uppercase letters                                              │
│ ❌ No underscores                                                    │
│ ❌ No periods (technically allowed but causes SSL issues)            │
│ ❌ Cannot be an IP address format (192.168.1.1)                     │
│                                                                       │
│ Good names:                                                          │
│ ├── my-company-prod-assets                                        │
│ ├── acme-app-uploads-us-east-1                                   │
│ └── project-alpha-backups-2024                                    │
│                                                                       │
│ Bad names:                                                           │
│ ├── My_Bucket (uppercase + underscore)                            │
│ ├── my.bucket.com (periods cause SSL issues)                     │
│ └── s3 (too short + likely taken)                                 │
│                                                                       │
└─────────────────────────────────────────────────────────────────────┘
```

## Part 5: Versioning

```
┌─────────────────────────────────────────────────────────────────────┐
│           S3 VERSIONING                                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                       │
│ What: Keep multiple versions of an object in the same bucket.      │
│ Every PUT/POST/COPY/DELETE creates a new version.                  │
│                                                                       │
│ States:                                                              │
│ ├── Unversioned: Default. No version tracking.                   │
│ ├── Enabled: All objects get a version ID.                        │
│ └── Suspended: No new versions, but existing versions remain.    │
│                                                                       │
│ Enable versioning:                                                   │
│ S3 → Bucket → Properties → Bucket Versioning → Edit → Enable    │
│                                                                       │
│ How it works:                                                        │
│                                                                       │
│ Upload file.txt (v1):                                               │
│ ┌─────────────────┐                                                 │
│ │ file.txt        │                                                 │
│ │ Version: abc123 │ ← Current version                              │
│ └─────────────────┘                                                 │
│                                                                       │
│ Upload file.txt again (v2):                                         │
│ ┌─────────────────┐                                                 │
│ │ file.txt        │                                                 │
│ │ Version: def456 │ ← Current version                              │
│ ├─────────────────┤                                                 │
│ │ file.txt        │                                                 │
│ │ Version: abc123 │ ← Previous version (still stored!)             │
│ └─────────────────┘                                                 │
│                                                                       │
│ Delete file.txt:                                                     │
│ ┌─────────────────┐                                                 │
│ │ file.txt        │                                                 │
│ │ Delete marker   │ ← Marker (object appears "deleted")            │
│ ├─────────────────┤                                                 │
│ │ file.txt        │                                                 │
│ │ Version: def456 │ ← Still exists! Can be restored                │
│ ├─────────────────┤                                                 │
│ │ file.txt        │                                                 │
│ │ Version: abc123 │ ← Still exists!                                │
│ └─────────────────┘                                                 │
│                                                                       │
│ Restore deleted object: Delete the "delete marker"                 │
│                                                                       │
│ ⚠️ Versioning costs:                                                 │
│ ├── Every version consumes storage (you pay for all versions)    │
│ ├── 10 versions of a 1 GB file = 10 GB stored                   │
│ └── Use lifecycle rules to expire old versions!                  │
│                                                                       │
│ MFA Delete:                                                          │
│ ├── Requires MFA to delete object versions                       │
│ ├── Extra protection against accidental/malicious deletion       │
│ ├── Can only be enabled via CLI (not console)                    │
│ └── aws s3api put-bucket-versioning --bucket my-bucket \         │
│       --versioning-configuration Status=Enabled,                  │
│       MFADelete=Enabled --mfa "arn:...:mfa/device 123456"       │
│                                                                       │

```

│           CREATE LIFECYCLE RULE                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                   │
│ Lifecycle rule name: [move-to-glacier-after-90d]               │
│                                                                   │
│ Choose a rule scope:                                            │
│   ○ Limit the scope using one or more filters                  │
│   ● Apply to all objects in the bucket                         │
│                                                                   │
│ Filter (if scoped):                                             │
│   Prefix: [logs/]                                              │
│   Tags: Key=[type] Value=[archive]                            │
│   Object size: Min [128] KB  Max [5] GB                       │
│                                                                   │
│ → Prefix: Rule applies only to objects with this key prefix   │
│ → Tags: Rule applies only to objects with matching tags       │
│ → Size: Filter by object size range                            │
│                                                                   │
│ ── Lifecycle rule actions ──                                   │
│                                                                   │
│ ☑ Move current versions of objects between storage classes    │
│ ☑ Move noncurrent versions between storage classes            │
│ ☐ Expire current versions of objects                          │
│ ☑ Permanently delete noncurrent versions                      │
│ ☐ Delete expired object delete markers or incomplete          │
│   multipart uploads                                            │
│                                                                   │
│ ── Transition current versions ──                              │
│ Storage class transition:                                      │
│   After [30] days → Standard-IA                               │
│   After [90] days → Glacier Flexible Retrieval                │
│   After [365] days → Glacier Deep Archive                     │
│                                                                   │
│ → Objects automatically move to cheaper storage over time     │
│ → Waterfall: Standard → IA → Glacier → Deep Archive         │
│                                                                   │
│ ── Transition noncurrent versions ──                           │
│ After [30] days → Glacier Flexible Retrieval                  │
│ Number of newer versions to retain: [3]                       │
│                                                                   │
│ ── Delete noncurrent versions ──                               │
│ After [90] days                                                │
│ Number of newer versions to retain: [3]                       │
│                                                                   │
│ → Keeps 3 most recent versions, deletes older after 90 days  │
│                                                                   │
│                    [Create rule]                                │
└─────────────────────────────────────────────────────────────────┘

Common lifecycle patterns:
├── Log files: Standard (30d) → IA (90d) → Glacier (365d) → Delete (730d)
├── User uploads: Standard (60d) → Intelligent-Tiering
├── Backups: Standard (7d) → Glacier Deep Archive → Delete (2555d/7yrs)
├── Old versions: Delete noncurrent versions after 30 days, keep 3
└── Incomplete uploads: Abort incomplete multipart uploads after 7 days

## Troubleshooting: Common S3 Issues

### "403 Forbidden" When Accessing S3 Objects

```
Checklist:
1. ☐ Is S3 Block Public Access enabled?
   Console → S3 → Bucket → Permissions → Block public access
   If enabled AND you want public access: Disable the relevant option
   (Usually you want it ON and use CloudFront OAC instead)

2. ☐ Does the bucket policy allow the access?
   Console → S3 → Bucket → Permissions → Bucket policy

3. ☐ Does the IAM user/role have s3:GetObject permission?
   Check the IAM policy attached to the identity

4. ☐ Is the object ACL correct? (if using legacy ACLs)
   Most new buckets have ACLs disabled — use bucket policy instead

5. ☐ Is there a VPC endpoint policy restricting access?
   If using a VPC Gateway Endpoint for S3, check its policy
```

### "My S3 bill is unexpectedly high"

```
Common causes:
1. ☐ Incomplete multipart uploads accumulating
   Fix: Add lifecycle rule to abort incomplete uploads after 7 days

2. ☐ Old object versions piling up (versioning enabled)
   Fix: Add lifecycle rule to delete old versions after 30 days

3. ☐ Data in wrong storage class
   Fix: Use lifecycle rules to transition to IA/Glacier

4. ☐ Data transfer charges (downloading from S3)
   Fix: Use CloudFront (cheaper egress) or VPC Gateway Endpoint (free)

5. Check with: Console → S3 → Bucket → Metrics tab → Storage/Requests
```

### Common Mistakes

| Mistake | Impact | Fix |
|---------|--------|-----|
| No lifecycle rules | Old data accumulates forever | Transition to IA after 30d, Glacier after 90d |
| No versioning on important data | Can't recover deleted/overwritten files | Enable versioning + lifecycle for old versions |
| Bucket name with dots | SSL certificate errors | Use hyphens instead of dots |
| Large files without multipart | Upload fails, can't resume | Use multipart upload for files > 100 MB |
| Public bucket for static site | Security risk | Use CloudFront + OAC instead |
