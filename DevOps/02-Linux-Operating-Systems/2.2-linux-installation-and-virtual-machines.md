# 2.2 — Linux Installation & Virtual Machines (VirtualBox, VMware, WSL2)

> **One Line Answer:** You don't need a separate computer to learn Linux — you can run it **inside your current OS** using Virtual Machines (VirtualBox, VMware) or use **WSL2** on Windows to get a real Linux environment in seconds.

## 🤔 Why Do You Need to Install Linux?

You can't learn Linux by just reading about it. You need to **practice**. Every. Single. Day.

But here's the thing — most people use Windows or macOS as their daily operating system. So how do you practice Linux without buying a new computer?

## 🧠 First — What is Virtualization?

Before we jump into VirtualBox and VMware, you need to understand **virtualization**.

### The Simple Explanation

Virtualization lets you run a **computer inside your computer**. You create a "virtual machine" (VM) that thinks it's a real computer, but it's actually just software running on your physical machine.

```
┌─────────────────────────────────────────────┐
│         Your Physical Computer               │
│         (Host Machine)                       │
│  ┌────────────────┐  ┌────────────────┐     │
│  │   Virtual       │  │   Virtual       │     │
│  │   Machine 1     │  │   Machine 2     │     │
│  │   (Ubuntu)      │  │   (CentOS)      │     │
│  │                 │  │                 │     │
│  │  Has its own:   │  │  Has its own:   │     │
│  │  - CPU          │  │  - CPU          │     │
│  │  - RAM          │  │  - RAM          │     │
│  │  - Disk         │  │  - Disk         │     │
│  │  - Network      │  │  - Network      │     │
│  └────────────────┘  └────────────────┘     │
│                                              │
│         Hypervisor (VirtualBox/VMware)        │
│         (manages all VMs)                    │
│                                              │
│         Host OS (Windows/macOS)              │
│         Hardware (CPU, RAM, Disk)            │
└─────────────────────────────────────────────┘
```

### Key Terms You Should Know

| Term | Meaning |
|------|---------|
| **Host Machine** | Your physical computer (the real one) |
| **Guest Machine** | The virtual machine (the fake one running inside) |
| **Host OS** | The operating system on your physical machine (e.g., Windows 11) |
| **Guest OS** | The operating system inside the VM (e.g., Ubuntu) |
| **Hypervisor** | Software that creates and manages VMs (VirtualBox, VMware, Hyper-V) |
| **VM (Virtual Machine)** | A software-based computer running inside your real computer |
| **Snapshot** | A "save point" — capture the VM's state and restore it later |
| **ISO file** | A disk image file used to install an operating system (like a virtual DVD) |

### Types of Hypervisors

```
Type 1 — "Bare Metal" Hypervisor
    └── Runs directly on hardware (no host OS needed)
    └── Examples: VMware ESXi, Microsoft Hyper-V (server), KVM
    └── Used in: Data centers, cloud providers, production servers
    └── More efficient — no host OS overhead

Type 2 — "Hosted" Hypervisor
    └── Runs on top of a host OS (Windows, macOS, Linux)
    └── Examples: VirtualBox, VMware Workstation, VMware Fusion
    └── Used for: Learning, development, testing
    └── Less efficient — but easy to use on your desktop/laptop
```

For learning DevOps, you'll use **Type 2** hypervisors (VirtualBox or VMware).

## 📦 Option 1: VirtualBox (Free & Open Source)

### What is VirtualBox?

VirtualBox is a **free, open-source** Type 2 hypervisor made by **Oracle**. It runs on Windows, macOS, and Linux. It's the most popular choice for beginners because it's free and easy to use.

### Step-by-Step: Install VirtualBox

#### Step 1: Download VirtualBox

1. Go to: **https://www.virtualbox.org/wiki/Downloads**
2. Click on the download for your operating system (Windows, macOS, or Linux)
3. Also download the **Extension Pack** (for extra features like USB support, RDP)

#### Step 2: Install VirtualBox

**On Windows:**
1. Run the downloaded `.exe` file
2. Click Next → Next → Next → Install
3. It will install virtual network adapters (click Yes/Allow)
4. Click Finish

**On macOS:**
1. Open the downloaded `.dmg` file
2. Double-click the VirtualBox installer
3. Follow the prompts
4. You may need to allow the kernel extensions in System Preferences → Security & Privacy

**On Linux (Ubuntu):**
```bash
# Method 1: From Ubuntu repos (may not be latest)
sudo apt update
sudo apt install virtualbox -y

# Method 2: From Oracle's repo (latest version)
# Add Oracle's GPG key
wget -q https://www.virtualbox.org/download/oracle_vbox_2016.asc -O- | sudo apt-key add -

# Add repo
sudo add-apt-repository "deb [arch=amd64] https://download.virtualbox.org/virtualbox/debian $(lsb_release -cs) contrib"

# Install
sudo apt update
sudo apt install virtualbox-7.0 -y
```

#### Step 3: Install the Extension Pack

1. Open VirtualBox
2. Go to File → Tools → Extension Pack Manager (or Preferences → Extensions)
3. Click the "+" icon and select the downloaded Extension Pack
4. Click Install → Agree to the license

### Step-by-Step: Create an Ubuntu VM in VirtualBox

#### Step 1: Download Ubuntu ISO

1. Go to: **https://ubuntu.com/download/server**
2. Download the latest LTS version (e.g., Ubuntu 24.04 LTS)
3. You'll get an `.iso` file (~2GB)

#### Step 2: Create a New VM

1. Open VirtualBox → Click **"New"**
2. Fill in:
   - **Name:** `ubuntu-server` (or any name you want)
   - **Machine Folder:** Leave default or choose a folder
   - **Type:** Linux
   - **Version:** Ubuntu (64-bit)
   - **ISO Image:** Select the Ubuntu ISO you downloaded

3. Click **Next**

#### Step 3: Configure Resources

Give your VM some resources from your host machine:

| Resource | Recommended Minimum | Good for Learning |
|----------|--------------------|--------------------|
| **RAM** | 1 GB | 2-4 GB |
| **CPU** | 1 core | 2 cores |
| **Disk** | 10 GB | 25 GB |

**⚠️ Don't allocate more than 50% of your host's resources to VMs**, or your host will slow down!

4. **Memory:** Set to 2048 MB (2 GB) or more
5. **Processors:** Set to 2
6. Click **Next**

#### Step 4: Create Virtual Hard Disk

7. Select **"Create a Virtual Hard Disk Now"**
8. Set size to **25 GB**
9. Leave file type as **VDI (VirtualBox Disk Image)**
10. Select **"Dynamically Allocated"** (disk file grows as needed, doesn't use 25 GB immediately)
11. Click **Next** → **Finish**

#### Step 5: Install Ubuntu

12. Select your new VM → Click **Start**
13. The VM will boot from the ISO
14. Follow the Ubuntu installer:
    - Choose language
    - Choose keyboard layout
    - Choose "Ubuntu Server" (minimized or full)
    - Configure network (DHCP is fine for learning)
    - Configure storage (use entire disk)
    - Set up your username and password
    - Optionally install OpenSSH server (**DO THIS** — you'll need SSH)
    - Wait for installation to complete
    - Reboot

15. **Remove the ISO:** After install, go to VM Settings → Storage → Remove the ISO from the virtual CD drive

16. Start the VM → Log in with your username and password

**🎉 Congratulations! You now have a Linux server running on your machine!**

### VirtualBox Networking Modes

This is important for DevOps — you'll need your VM to connect to the internet and to your host machine.

| Network Mode | VM → Internet | Host → VM | VM → VM | Best For |
|-------------|:-------------:|:---------:|:-------:|----------|
| **NAT** | ✅ | ❌ (need port forwarding) | ❌ | Basic internet access |
| **Bridged Adapter** | ✅ | ✅ | ✅ | VM acts like a real machine on your network |
| **Host-Only Adapter** | ❌ | ✅ | ✅ | Isolated lab environment |
| **NAT Network** | ✅ | ❌ (need port forwarding) | ✅ | Multiple VMs with internet |
| **Internal Network** | ❌ | ❌ | ✅ | Completely isolated network |

**Recommended for learning:**
- Use **NAT** for internet access (default)
- Add a **Host-Only Adapter** as a second adapter so you can SSH from your host

#### Setting Up SSH Access to Your VM:

**Option A — Using NAT + Port Forwarding:**
1. VM Settings → Network → Adapter 1 (NAT) → Advanced → Port Forwarding
2. Add a rule:

| Name | Protocol | Host IP | Host Port | Guest IP | Guest Port |
|------|----------|---------|-----------|----------|------------|
| SSH | TCP | 127.0.0.1 | 2222 | (empty) | 22 |

3. Now SSH from your host:
```bash
ssh -p 2222 your_username@127.0.0.1
```

**Option B — Using Bridged Adapter:**
1. VM Settings → Network → Adapter 1 → Change to "Bridged Adapter"
2. Select your host's network adapter
3. Start the VM → Find its IP:
```bash
ip addr show
# Look for the IP address (usually 192.168.x.x)
```
4. SSH from your host:
```bash
ssh your_username@192.168.x.x
```

### Useful VirtualBox Features

**Snapshots — Your Safety Net:**
```
Take a snapshot before making risky changes.
If something breaks, restore the snapshot and you're back to normal!

How:
1. Machine → Take Snapshot
2. Give it a name like "Fresh Install" or "Before Docker Setup"
3. To restore: Machine → Snapshots → Right-click → Restore
```

**Clone — Duplicate a VM:**
```
Right-click a VM → Clone
Creates an exact copy — useful for creating multiple test servers
```

**Headless Mode — Run VM Without a Window:**
```bash
# Start VM in headless mode (no GUI window, runs in background)
VBoxManage startvm "ubuntu-server" --type headless

# Stop a VM
VBoxManage controlvm "ubuntu-server" poweroff

# List all VMs
VBoxManage list vms

# List running VMs
VBoxManage list runningvms
```

## 🖥️ Option 2: VMware (Professional Grade)

### What is VMware?

VMware is the **industry leader** in virtualization. They make:

| Product | Type | Cost | OS |
|---------|------|------|-----|
| **VMware Workstation Pro** | Type 2 (Desktop) | Free (since 2024) | Windows, Linux |
| **VMware Fusion** | Type 2 (Desktop) | Free (since 2024) | macOS |
| **VMware ESXi / vSphere** | Type 1 (Bare Metal) | Enterprise (paid) | Bare Metal |

**Good news:** As of 2024, VMware Workstation Pro and Fusion are **free for personal use**!

### VirtualBox vs VMware — Which Should You Choose?

| Feature | VirtualBox | VMware Workstation |
|---------|------------|-------------------|
| **Cost** | Free (open source) | Free (personal use) |
| **Performance** | Good | Better (slightly faster) |
| **Ease of Use** | Easy | Easy |
| **Snapshots** | ✅ | ✅ |
| **3D Graphics** | Basic | Better |
| **USB Support** | Needs Extension Pack | Built-in |
| **Enterprise Use** | Not common | Industry standard |
| **Networking** | Good | More advanced options |
| **Drag & Drop** | ✅ | ✅ |

**My recommendation:**
- **Beginner?** → VirtualBox (simpler, open source)
- **Want better performance?** → VMware Workstation Pro
- **On macOS?** → VMware Fusion or UTM (for Apple Silicon)

### Step-by-Step: Install VMware Workstation Pro

1. Go to: **https://www.vmware.com/products/workstation-pro.html**
2. Download for your OS
3. Run the installer → Next → Next → Install
4. Choose "Free for personal use" when asked for license

### Creating a VM in VMware

The process is very similar to VirtualBox:

1. Open VMware → **Create a New Virtual Machine**
2. Choose **"Typical"** configuration
3. Select **"Installer disc image file (ISO)"** → Browse to your Ubuntu ISO
4. VMware will auto-detect "Ubuntu 64-bit" → Click Next
5. Enter your **name, username, and password** (VMware can auto-install!)
6. Choose a location for the VM files
7. Set disk size → **25 GB** → Store as a single file
8. Review settings → Click **Finish**
9. VMware will automatically install Ubuntu for you (Easy Install feature)

**💡 VMware's "Easy Install" is a nice feature — it automates the entire OS installation!**

## 🪟 Option 3: WSL2 — Windows Subsystem for Linux (Recommended for Windows Users)

### What is WSL2?

WSL2 (Windows Subsystem for Linux version 2) is a feature built into **Windows 10/11** that lets you run a **real Linux kernel** directly inside Windows. No VM software needed, no dual booting, no ISO files.

It's the **fastest way** to get a Linux environment on Windows.

### WSL1 vs WSL2

```
WSL1 (2016):
    └── Translated Linux system calls to Windows system calls
    └── No real Linux kernel
    └── Slow file I/O, limited compatibility
    └── Some Linux programs didn't work

WSL2 (2019):
    └── Runs a REAL Linux kernel in a lightweight VM
    └── Full system call compatibility
    └── MUCH faster file I/O
    └── Docker can run natively on WSL2
    └── Almost everything works!
```

**Always use WSL2.** WSL1 is outdated.

### How WSL2 Works Under the Hood

```
┌───────────────────────────────────────────────┐
│              Windows 11 / 10                   │
│                                               │
│   ┌─────────────────────────────────────┐     │
│   │        WSL2 (Lightweight VM)         │     │
│   │  ┌──────────┐  ┌──────────┐         │     │
│   │  │  Ubuntu   │  │  Debian  │         │     │
│   │  │  22.04    │  │  12      │         │     │
│   │  └──────────┘  └──────────┘         │     │
│   │                                     │     │
│   │      Real Linux Kernel 5.15+        │     │
│   └─────────────────────────────────────┘     │
│                                               │
│   ┌─────────────────────────────────────┐     │
│   │    Windows Apps (VS Code, Browser)   │     │
│   └─────────────────────────────────────┘     │
│                                               │
│          Windows NT Kernel                     │
│          Hardware                              │
└───────────────────────────────────────────────┘
```

### Step-by-Step: Install WSL2

#### Prerequisites
- **Windows 10** version 2004+ (Build 19041+) or **Windows 11**
- **Virtualization** enabled in BIOS (usually enabled by default)

#### Step 1: Open PowerShell as Administrator

```powershell
# Right-click Start Menu → Terminal (Admin) or PowerShell (Admin)
```

#### Step 2: Install WSL2 with One Command

```powershell
wsl --install
```

That's it! This single command:
- Enables the WSL feature
- Enables the Virtual Machine Platform
- Downloads and installs the Linux kernel
- Sets WSL2 as the default
- Installs **Ubuntu** as the default distro

**Restart your computer when prompted.**

#### Step 3: Set Up Ubuntu

After restart, Ubuntu will open automatically (or open it from the Start Menu):

```
Installing, this may take a few minutes...
Please create a default UNIX user account.
Enter new UNIX username: ritesh
New password: ********
Retype new password: ********

Welcome to Ubuntu 22.04.3 LTS (GNU/Linux 5.15.133.1-microsoft-standard-WSL2 x86_64)

ritesh@DESKTOP-ABC123:~$
```

**🎉 You now have Linux running inside Windows!**

#### Step 4: Verify WSL2

```powershell
# In PowerShell, check the version
wsl --list --verbose
# or
wsl -l -v

# Output:
#   NAME      STATE           VERSION
# * Ubuntu    Running         2        ← Make sure VERSION is 2
```

If the version shows 1, upgrade it:
```powershell
wsl --set-version Ubuntu 2
```

### Install Different Distros on WSL2

You're not limited to Ubuntu! You can install multiple distros side by side:

```powershell
# See all available distros
wsl --list --online

# Output:
# NAME                   FRIENDLY NAME
# Ubuntu                 Ubuntu
# Debian                 Debian GNU/Linux
# kali-linux             Kali Linux Rolling
# Ubuntu-22.04           Ubuntu 22.04 LTS
# Ubuntu-24.04           Ubuntu 24.04 LTS
# OracleLinux_9_1        Oracle Linux 9.1
# openSUSE-Leap-15.6     openSUSE Leap 15.6
# SLES-15-SP6            SUSE Linux Enterprise Server 15 SP6

# Install a specific distro
wsl --install -d Debian
wsl --install -d kali-linux
wsl --install -d Ubuntu-24.04
```

### Essential WSL2 Commands

```powershell
# --- PowerShell Commands (run from Windows) ---

# List installed distros
wsl --list --verbose

# Start a specific distro
wsl -d Ubuntu-24.04

# Shut down all WSL instances
wsl --shutdown

# Shut down a specific distro
wsl --terminate Ubuntu

# Set default distro
wsl --set-default Ubuntu-24.04

# Check WSL status
wsl --status

# Update WSL
wsl --update

# Unregister (delete) a distro
wsl --unregister Ubuntu
# ⚠️ WARNING: This deletes ALL data in that distro!

# Export a distro (backup)
wsl --export Ubuntu D:\backups\ubuntu-backup.tar

# Import a distro (restore)
wsl --import Ubuntu-Restored D:\wsl\ubuntu-restored D:\backups\ubuntu-backup.tar
```

### Accessing Files Between Windows and WSL2

This is one of the best features — you can access files both ways:

```bash
# From INSIDE WSL2 — Access Windows files:
cd /mnt/c/Users/YourName/Desktop
ls
# Your Windows C: drive is mounted at /mnt/c/
# D: drive is at /mnt/d/
# E: drive is at /mnt/e/

# From INSIDE WSL2 — Open Windows Explorer in current directory:
explorer.exe .

# From INSIDE WSL2 — Open a file in VS Code:
code myfile.py
```

```powershell
# From WINDOWS — Access WSL2 files:
# Open File Explorer and go to:
# \\wsl$\Ubuntu\home\your_username\

# Or in the address bar type:
# \\wsl$\
# You'll see all your WSL distros listed as network drives
```

**⚠️ Performance Tip:** Keep your project files **inside the WSL2 filesystem** (`/home/your_username/`), NOT on the Windows filesystem (`/mnt/c/`). Cross-filesystem access is slower.

### WSL2 + VS Code (Perfect Combo)

VS Code has amazing WSL2 integration:

1. Install the **"WSL" extension** in VS Code (by Microsoft)
2. From inside WSL2, navigate to your project and run:
```bash
code .
```
3. VS Code will open with full Linux support — terminal, file access, extensions, everything!

You'll see `WSL: Ubuntu` in the bottom-left corner of VS Code, indicating you're connected to WSL2.

### WSL2 + Docker (Essential for DevOps)

Docker Desktop on Windows uses WSL2 as its backend. This means Docker containers run natively on the Linux kernel inside WSL2:

1. Install **Docker Desktop** for Windows
2. In Docker Desktop settings → General → Check **"Use the WSL 2 based engine"**
3. Go to Resources → WSL Integration → Enable for your distro (e.g., Ubuntu)
4. Now you can run Docker commands from inside WSL2:
```bash
docker --version
docker run hello-world
```

This is **much faster** than the old Hyper-V backend.

## 🔧 Post-Installation — First Things to Do

No matter which method you used (VM, WSL2, or cloud), here are the **first things** you should do on a fresh Linux install:

### 1. Update the System
```bash
# Ubuntu/Debian
sudo apt update && sudo apt upgrade -y

# RHEL/CentOS/Rocky
sudo dnf update -y
```

### 2. Install Essential Tools
```bash
# Ubuntu/Debian
sudo apt install -y \
    curl \
    wget \
    git \
    vim \
    nano \
    htop \
    net-tools \
    tree \
    unzip \
    zip \
    software-properties-common

# RHEL/CentOS/Rocky
sudo dnf install -y \
    curl \
    wget \
    git \
    vim \
    nano \
    htop \
    net-tools \
    tree \
    unzip \
    zip
```

### 3. Set Up SSH (if not already installed)
```bash
# Install SSH server
sudo apt install openssh-server -y   # Ubuntu/Debian
sudo dnf install openssh-server -y   # RHEL/CentOS

# Start and enable SSH
sudo systemctl start sshd
sudo systemctl enable sshd

# Check SSH status
sudo systemctl status sshd
```

### 4. Check System Info
```bash
# Operating system info
cat /etc/os-release

# Kernel version
uname -r

# Hostname
hostname

# IP address
ip addr show
# or
hostname -I

# Available disk space
df -h

# Available memory
free -h

# CPU info
lscpu
```

## 📊 Quick Comparison: VirtualBox vs VMware vs WSL2

| Feature | VirtualBox | VMware | WSL2 |
|---------|-----------|--------|------|
| **Cost** | Free | Free (personal) | Free (built into Windows) |
| **OS Support** | Win, Mac, Linux | Win, Mac, Linux | Windows only |
| **Setup Time** | 15-30 min | 15-30 min | 2-5 min |
| **Performance** | Good | Better | Best (near native) |
| **Full GUI** | ✅ | ✅ | Possible but not ideal |
| **Snapshots** | ✅ | ✅ | Export/Import |
| **Multiple Distros** | ✅ | ✅ | ✅ |
| **Docker Support** | Inside VM | Inside VM | Native (Docker Desktop) |
| **Network Isolation** | Full control | Full control | Shared with Windows |
| **Resource Usage** | High | High | Low |
| **Good for Servers** | ✅ | ✅ | ✅ (CLI only) |
| **Good for GUI Desktop** | ✅ | ✅ | Limited |
| **Best for** | Full VM experience | Professional use | Quick CLI access |
