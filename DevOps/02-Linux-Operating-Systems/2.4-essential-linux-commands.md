Imagine you're a chef. Before you cook fancy dishes, you need to know how to **chop, stir, boil, and fry**. These basic commands are exactly that — the fundamental skills you need before doing anything advanced in Linux.

As a DevOps engineer, you'll SSH into servers, troubleshoot issues, read logs, move files, search for errors — **all from the terminal**. No fancy GUI. Just you and the command line.

```
Your Daily Life as a DevOps Engineer:

  SSH into a server
       ↓
  Navigate to the right directory (cd, ls)
       ↓
  Look at log files (cat, less, tail)
       ↓
  Search for errors (grep, find)

       ↓
```

## 📂 Navigation & Listing Commands

### 1. `pwd` — Print Working Directory (Where Am I?)

This tells you your **current location** in the file system. Think of it like GPS for your terminal.

```bash
$ pwd
/home/ritesh
```

That's it. It just tells you where you are right now.

**When to use:** When you're lost and don't know which directory you're in (happens more often than you'd think! 😄).

---

### 2. `ls` — List Directory Contents (What's Here?)

`ls` shows you all the files and folders in the current directory (or any directory you specify).

#### Basic Usage:

```bash
# List files in current directory
$ ls
Documents  Downloads  Music  Pictures  Videos

# List files in a specific directory
$ ls /var/log
syslog  auth.log  kern.log  dmesg
```

#### The Flags That Make `ls` Powerful:

```bash
# -l = Long format (detailed view — permissions, owner, size, date)
$ ls -l
total 20
drwxr-xr-x 2 ritesh ritesh 4096 May 27 10:00 Documents
-rw-r--r-- 1 ritesh ritesh  220 May 27 09:30 notes.txt
-rwxr-xr-x 1 ritesh ritesh  512 May 27 08:15 script.sh

# Breaking down what each column means:
# drwxr-xr-x  → File type + permissions (d = directory, - = file)
# 2           → Number of hard links
# ritesh      → Owner
# ritesh      → Group
# 4096        → Size in bytes
# May 27 10:00 → Last modified date
# Documents   → Name
```

```bash
# -a = Show ALL files (including hidden files starting with .)
$ ls -a
.  ..  .bashrc  .profile  .ssh  Documents  Downloads

# Hidden files in Linux start with a dot (.)
# .bashrc, .profile, .ssh — these are all hidden but VERY important
```

```bash
# -la = Combine both (most commonly used combo in DevOps)
$ ls -la
total 36
drwxr-xr-x 5 ritesh ritesh 4096 May 27 10:00 .
drwxr-xr-x 3 root   root   4096 May 20 08:00 ..
-rw-r--r-- 1 ritesh ritesh  220 May 20 08:00 .bashrc
drwx------ 2 ritesh ritesh 4096 May 25 12:00 .ssh
drwxr-xr-x 2 ritesh ritesh 4096 May 27 10:00 Documents
```

```bash
# -lh = Human-readable sizes (KB, MB, GB instead of bytes)
$ ls -lh
-rw-r--r-- 1 ritesh ritesh 4.5K May 27 09:30 notes.txt
-rw-r--r-- 1 ritesh ritesh 2.1M May 27 08:15 backup.tar.gz
-rw-r--r-- 1 ritesh ritesh 1.3G May 26 22:00 database-dump.sql

# Much easier to read than raw bytes!
```

```bash
# -lt = Sort by time (newest first)
$ ls -lt
-rw-r--r-- 1 ritesh ritesh 4096 May 27 10:00 newest-file.txt
-rw-r--r-- 1 ritesh ritesh 2048 May 26 08:00 older-file.txt
-rw-r--r-- 1 ritesh ritesh 1024 May 25 06:00 oldest-file.txt

# -ltr = Sort by time reversed (oldest first)
$ ls -ltr
```

```bash
# -R = Recursive (show everything inside subdirectories too)
$ ls -R
.:
Documents  scripts

./Documents:
notes.txt  todo.md

./scripts:
deploy.sh  backup.sh
```

#### Real-World DevOps Usage:

```bash
# Check what's in the Nginx config directory
$ ls -la /etc/nginx/
$ ls -la /etc/nginx/sites-enabled/

# Check log files sorted by time
$ ls -lhtr /var/log/

# See how big log files are getting
$ ls -lhS /var/log/    # -S sorts by size (biggest first)
```

### 3. `cd` — Change Directory (Move Around)

`cd` lets you move from one directory to another. This is like walking through folders.

```bash
# Go to a specific directory
$ cd /var/log
$ pwd
/var/log

# Go to your home directory (3 ways to do it)
$ cd ~          # ~ means "home directory"
$ cd $HOME      # $HOME is a variable that points to home
$ cd            # Just cd with nothing also goes home

# Go UP one directory (parent directory)
$ cd ..
# If you were in /var/log, now you're in /var

# Go UP two directories
$ cd ../..
# If you were in /var/log, now you're in /

# Go to the PREVIOUS directory (super handy!)
$ cd -
# This toggles between your current and previous directory
# Example:
$ cd /var/log
$ cd /etc/nginx
$ cd -          # Takes you back to /var/log
$ cd -          # Takes you back to /etc/nginx
```

#### Pro Tips:

```bash
# Use Tab completion to avoid typing full paths
$ cd /etc/ng[TAB]    # Auto-completes to /etc/nginx/

# Use absolute paths (start from /) when you know exactly where to go
$ cd /home/ritesh/projects/my-app

# Use relative paths (from current location) for nearby directories
$ cd ../scripts      # Go up one, then into scripts/
```

## 📄 File Operations — The Big 4: cp, mv, rm, mkdir

### 4. `cp` — Copy Files and Directories

`cp` creates a **duplicate** of a file or directory.

```bash
# Copy a file
$ cp source.txt destination.txt
# Now you have TWO files: source.txt and destination.txt

# Copy a file to a different directory
$ cp notes.txt /home/ritesh/backup/
# notes.txt still exists in original location + a copy in backup/

# Copy a file with a new name to a different directory
$ cp notes.txt /home/ritesh/backup/notes-backup.txt
```

```bash
# Copy a DIRECTORY (must use -r for recursive)
$ cp -r my-project/ my-project-backup/
# -r means "recursive" — copy the folder AND everything inside it
# Without -r, copying a directory will FAIL

# Copy with verbose output (see what's being copied)
$ cp -rv my-project/ my-project-backup/
'my-project/index.html' -> 'my-project-backup/index.html'
'my-project/style.css' -> 'my-project-backup/style.css'
'my-project/app.js' -> 'my-project-backup/app.js'
```

```bash
# Preserve permissions and timestamps while copying
$ cp -p config.yml /backup/config.yml
# -p keeps the original file's permissions, ownership, and timestamps

# The ultimate safe copy command
$ cp -rp /etc/nginx/ /backup/nginx-backup/
# -r = recursive (copy everything inside)
# -p = preserve permissions and timestamps
```

#### Real-World DevOps Usage:

```bash
# Backup a config before editing
$ cp /etc/nginx/nginx.conf /etc/nginx/nginx.conf.bak

# Backup entire config directory before making changes
$ cp -rp /etc/nginx/ /etc/nginx-backup-$(date +%Y%m%d)/

# The $(date +%Y%m%d) part adds today's date to the backup folder name
# Result: /etc/nginx-backup-20260527/
```

---

### 5. `mv` — Move or Rename Files

`mv` does two things: **moves** files to a different location OR **renames** files. Same command, different use.

```bash
# RENAME a file (source and destination are in same directory)
$ mv old-name.txt new-name.txt
# old-name.txt is GONE. It's now called new-name.txt

# MOVE a file to a different directory
$ mv report.txt /home/ritesh/documents/
# report.txt is no longer here — it's now in documents/

# MOVE and RENAME at the same time
$ mv report.txt /home/ritesh/documents/final-report.txt
```

```bash
# Move a DIRECTORY (no -r needed! Unlike cp)
$ mv my-project/ /home/ritesh/archive/
# Entire directory moved. No -r flag required.

# Move multiple files at once
$ mv file1.txt file2.txt file3.txt /destination/folder/
```

```bash
# Safety flag: -i (interactive — ask before overwriting)
$ mv -i important.txt /backup/
mv: overwrite '/backup/important.txt'? y

# Without -i, mv will silently overwrite existing files!
# ALWAYS use -i when moving important files.
```

#### Real-World DevOps Usage:

```bash
# Rename a log file for rotation
$ mv /var/log/app.log /var/log/app.log.$(date +%Y%m%d)

# Move old deployments to archive
$ mv /deployments/v1.2.3/ /archive/deployments/

# Rename a config file to disable it
$ mv /etc/nginx/sites-enabled/old-site.conf /etc/nginx/sites-enabled/old-site.conf.disabled
```

### 6. `rm` — Remove (Delete) Files and Directories

`rm` **permanently deletes** files. There is **NO recycle bin** in Linux. Once it's gone, it's gone. Be careful!

```bash
# Delete a single file
$ rm unwanted-file.txt
# Gone. Forever. No undo.

# Delete with confirmation (ALWAYS use this for important stuff)
$ rm -i important-file.txt
rm: remove regular file 'important-file.txt'? y
```

```bash
# Delete an EMPTY directory
$ rmdir empty-folder/
# Only works if the folder is completely empty

# Delete a directory AND everything inside it
$ rm -r my-folder/
# -r = recursive (delete folder + all contents)
# ⚠️ Be VERY careful with this!

# Force delete (no confirmation, no errors for missing files)
$ rm -f file-that-might-not-exist.txt
# -f = force. Won't ask, won't complain if file doesn't exist

# The DANGEROUS combo (use with extreme caution)
$ rm -rf my-folder/
# -r = recursive
# -f = force
# Deletes everything inside my-folder without asking
```

#### ⚠️ THE MOST DANGEROUS COMMAND IN LINUX:

```bash
# ❌ NEVER EVER DO THIS:
$ rm -rf /
# This deletes EVERYTHING on your entire system. EVERYTHING.
# Your OS, your data, your configs — ALL GONE.

# ❌ Also NEVER do this:
$ rm -rf /*
# Same effect. Deletes all contents of root directory.

# ❌ Be careful with variables:
$ rm -rf $FOLDER/
# If $FOLDER is empty/undefined, this becomes: rm -rf /
# ALWAYS validate variables before using them in rm commands!
```

#### Safe Deletion Practices:

```bash
# Always use -i for interactive confirmation
$ rm -ri /var/log/old-logs/

# Use ls first to see what you're about to delete
$ ls /var/log/old-logs/        # See what's there
$ rm -r /var/log/old-logs/     # Then delete

# Use echo to test your command first
$ echo rm -rf /var/log/old-*   # This just PRINTS the command, doesn't run it
rm -rf /var/log/old-2024 /var/log/old-2025
# Looks right? Then run it for real.
```

```bash
# Create a single directory
```

## 📖 File Viewing Commands

### 8. `cat` — Display File Contents

`cat` dumps the **entire file content** to your screen. Good for small files.

```bash
# Display file contents
$ cat /etc/hostname
my-server

# Display with line numbers
$ cat -n script.sh
     1  #!/bin/bash
     2  echo "Hello World"
     3  
     4  # This is a comment
     5  echo "Done"

# Combine multiple files
$ cat file1.txt file2.txt > combined.txt
# This merges file1 and file2 into combined.txt
```

---

### 9. `less` — View Files Page by Page

For **large files** (like log files), `cat` dumps everything at once (useless for a 10GB log file!). `less` lets you **scroll through** the file.

```bash
$ less /var/log/syslog
```

**Navigation inside `less`:**

| Key | Action |
|-----|--------|
| `Space` or `f` | Next page (forward) |
| `b` | Previous page (backward) |
| `g` | Go to beginning of file |
| `G` | Go to end of file |
| `/pattern` | Search forward for "pattern" |
| `?pattern` | Search backward for "pattern" |
| `n` | Next search result |
| `N` | Previous search result |
| `q` | Quit less |

```bash
# Real-world: Check the last part of a huge log file
$ less +G /var/log/syslog
# +G starts at the END of the file (most recent logs)
```

---

### 10. `head` and `tail` — View Beginning or End of Files

```bash
# head = Show first 10 lines (default)
$ head /var/log/syslog

# Show first 20 lines
$ head -n 20 /var/log/syslog
$ head -20 /var/log/syslog    # Short form

# tail = Show last 10 lines (default)
$ tail /var/log/syslog

# Show last 50 lines
$ tail -n 50 /var/log/syslog
```

#### The MOST Important Flag: `tail -f` (Follow)

This is a **DevOps superstar** command. It watches a file in real-time and shows new lines as they're added.

```bash
# Watch a log file in real-time
$ tail -f /var/log/syslog
# Now your terminal is "live" — every new log entry appears instantly
# Press Ctrl+C to stop

# Watch multiple log files at once
$ tail -f /var/log/syslog /var/log/auth.log

# Watch and filter in real-time (combine with grep!)
$ tail -f /var/log/nginx/error.log | grep "500"
# Only shows lines containing "500" (server errors) in real-time
```

**When to use `tail -f`:** When you deploy something and want to watch the logs to see if it works. This is something you'll do **every single day** as a DevOps engineer.

## 🔍 Search Commands — Finding Stuff

### 11. `find` — Search for Files and Directories

`find` searches for files based on name, type, size, time, permissions — basically anything.

#### Basic Syntax:

```bash
find [where to look] [what to look for]
```

#### Find by Name:

```bash
# Find a file by exact name
$ find / -name "nginx.conf"
/etc/nginx/nginx.conf

# Find by name (case-insensitive)
$ find / -iname "readme.md"
/home/ritesh/projects/README.md
/home/ritesh/projects/readme.md

# Find using wildcards
$ find /var/log -name "*.log"
/var/log/syslog.log
/var/log/auth.log
/var/log/nginx/access.log
/var/log/nginx/error.log

# Find all YAML files in a project
$ find /projects/my-app -name "*.yml" -o -name "*.yaml"
# -o means OR
```

#### Find by Type:

```bash
# Find only files (not directories)
$ find /etc -type f -name "*.conf"

# Find only directories
$ find /home -type d -name "node_modules"

# Types:
# f = regular file
# d = directory
# l = symbolic link
```

#### Find by Size:

```bash
# Find files larger than 100MB
$ find / -type f -size +100M
/var/log/huge-log.log
/backups/database.dump

# Find files smaller than 1KB
$ find /tmp -type f -size -1k

# Size units:
# c = bytes
# k = kilobytes
# M = megabytes
# G = gigabytes
```

#### Find by Time:

```bash
# Find files modified in the last 7 days
$ find /var/log -type f -mtime -7

# Find files modified MORE than 30 days ago
$ find /tmp -type f -mtime +30

# Find files modified in the last 60 minutes
$ find /var/log -type f -mmin -60
```

#### Find + Execute Actions:

```bash
# Find and delete old temp files (older than 30 days)
$ find /tmp -type f -mtime +30 -delete

# Find and execute a command on each result
$ find /var/log -name "*.log" -exec ls -lh {} \;
# {} is replaced with each found file
# \; ends the -exec command

# Find large log files and show their sizes
$ find /var/log -type f -size +50M -exec du -h {} \;
512M    /var/log/syslog.1
1.1G    /var/log/application.log
```

#### Real-World DevOps Usage:

```bash
# Find all Dockerfiles in a project
$ find /projects -name "Dockerfile"

# Find all config files changed in the last 24 hours
$ find /etc -type f -mtime -1

# Find and clean up old backup files
$ find /backups -name "*.tar.gz" -mtime +90 -delete

# Find files with wrong permissions (security audit)
$ find /var/www -type f -perm 777
# Files with 777 permissions are a security risk!

# Find empty directories
$ find /home -type d -empty
```

### 12. `grep` — Search Inside Files (The KING of Search)

If `find` searches for **files**, `grep` searches for **text inside files**. This is probably the most-used command in DevOps troubleshooting.

> **grep** = **G**lobal **R**egular **E**xpression **P**rint

#### Basic Usage:

```bash
# Search for a word in a file
$ grep "error" /var/log/syslog
May 27 10:15:22 server1 app[1234]: ERROR: Connection timeout
May 27 10:16:05 server1 app[1234]: ERROR: Database unreachable

# Case-insensitive search
$ grep -i "error" /var/log/syslog
# Finds: Error, ERROR, error, ErRoR — all of them
```

#### Essential Flags:

```bash
# -r = Recursive (search in all files in a directory)
$ grep -r "password" /etc/
# Searches EVERY file inside /etc/ for the word "password"

# -n = Show line numbers
$ grep -n "error" app.log
42:ERROR: Connection failed
108:ERROR: Timeout reached
256:ERROR: Disk full

# -c = Count matches (don't show them, just count)
$ grep -c "error" app.log
3

# -l = Show only file names (not the matching lines)
$ grep -rl "TODO" /projects/my-app/
/projects/my-app/src/auth.js
/projects/my-app/src/database.js

# -v = Invert match (show lines that DON'T match)
$ grep -v "debug" app.log
# Shows all lines EXCEPT those containing "debug"
# Great for filtering out noise

# -w = Match whole word only
$ grep -w "error" app.log
# Matches "error" but NOT "errors" or "error_code"

# -A, -B, -C = Show context Around the match
$ grep -A 3 "error" app.log   # Show 3 lines AFTER each match
$ grep -B 2 "error" app.log   # Show 2 lines BEFORE each match
$ grep -C 2 "error" app.log   # Show 2 lines BEFORE and AFTER (Context)
```

#### Using grep with Regex (Regular Expressions):

```bash
# Match lines starting with "Error"
$ grep "^Error" app.log
# ^ means "start of line"

# Match lines ending with "failed"
$ grep "failed$" app.log
# $ means "end of line"

# Match IP addresses (basic pattern)
$ grep -E "[0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3}" access.log
# -E enables Extended regex

# Match either "error" or "warning"
$ grep -E "error|warning" app.log
# | means OR

# Match any line with a port number (like :8080, :3000)
$ grep -E ":[0-9]{2,5}" config.yml
```

#### The Pipe `|` — grep's Best Friend:

```bash
# Chain commands together using | (pipe)
$ cat /etc/passwd | grep "ritesh"
ritesh:x:1000:1000::/home/ritesh:/bin/bash

# Find running processes containing "nginx"
$ ps aux | grep "nginx"
root      1234  0.0  0.1  nginx: master process
www-data  1235  0.0  0.2  nginx: worker process

# Count how many times an error appeared
$ cat app.log | grep "error" | wc -l
42

# Chain multiple greps (AND logic)
$ cat app.log | grep "error" | grep "database" | grep -v "debug"
# Find lines with "error" AND "database" but NOT "debug"
```

#### Real-World DevOps Usage:

```bash
# Find which process is using port 8080
$ netstat -tlnp | grep ":8080"
$ ss -tlnp | grep ":8080"

# Check if a service is running
$ systemctl status nginx | grep "Active"
Active: active (running) since Mon 2026-05-27 08:00:00

# Search for a specific error in all log files
$ grep -r "OutOfMemoryError" /var/log/

# Find all environment variables containing "PATH"
$ env | grep PATH

# Check SSH config for specific settings
$ grep -n "PermitRootLogin" /etc/ssh/sshd_config
57:PermitRootLogin no
```

## ✂️ Text Processing Commands — The Power Tools

These commands are what separate a regular Linux user from a **power user**. They let you slice, dice, filter, and transform text data.

### 13. `cut` — Cut Out Sections of Each Line

`cut` extracts specific columns or fields from text.

```bash
# The /etc/passwd file has fields separated by colons (:)
$ cat /etc/passwd
root:x:0:0:root:/root:/bin/bash
ritesh:x:1000:1000:Ritesh:/home/ritesh:/bin/bash

# Extract only usernames (field 1)
$ cut -d':' -f1 /etc/passwd
root
ritesh

# -d':' = delimiter is colon
# -f1   = field number 1

# Extract username and home directory (fields 1 and 6)
$ cut -d':' -f1,6 /etc/passwd
root:/root
ritesh:/home/ritesh

# Extract fields 1 through 3
$ cut -d':' -f1-3 /etc/passwd
root:x:0
ritesh:x:1000
```

```bash
# Cut by character position
$ echo "Hello World" | cut -c1-5
Hello

# Extract the first 10 characters of each line in a file
$ cut -c1-10 /var/log/syslog
```

#### Real-World DevOps Usage:

```bash
# Extract IP addresses from access logs (assuming space-separated)
$ cut -d' ' -f1 /var/log/nginx/access.log
192.168.1.10
10.0.0.5
172.16.0.20

# Get just the usernames from a list
$ cut -d':' -f1 /etc/passwd | sort
```

### 14. `sort` — Sort Lines of Text

```bash
# Sort alphabetically (default)
$ sort names.txt
Alice
Bob
Charlie
Dave

# Sort in reverse order
$ sort -r names.txt
Dave
Charlie
Bob
Alice

# Sort numerically
$ sort -n numbers.txt
1
5
10
25
100
# Without -n, it would sort as strings: 1, 10, 100, 25, 5 (wrong!)

# Sort by a specific column
$ sort -t',' -k2 data.csv
# -t',' = delimiter is comma
# -k2   = sort by 2nd column

# Sort and remove duplicates
$ sort -u names.txt
# -u = unique (remove duplicate lines)
```

#### Real-World DevOps Usage:

```bash
# Sort processes by memory usage
$ ps aux --sort=-%mem | head -10
# Top 10 processes using the most memory

# Sort files by size
$ du -h /var/log/* | sort -h
# -h = human-readable sort (understands K, M, G)

# Sort IP addresses in access log by frequency
$ cut -d' ' -f1 /var/log/nginx/access.log | sort | uniq -c | sort -rn | head -10
# This is a CLASSIC DevOps one-liner! (We'll break it down below)
```

---

### 15. `uniq` — Report or Remove Duplicate Lines

**Important:** `uniq` only removes **adjacent** duplicates. You almost always need to `sort` first!

```bash
# Remove adjacent duplicate lines
$ sort names.txt | uniq
Alice
Bob
Charlie

# Count occurrences of each line
$ sort names.txt | uniq -c
      3 Alice
      1 Bob
      5 Charlie
      2 Dave

# Show ONLY duplicate lines
$ sort names.txt | uniq -d
Alice
Charlie
Dave

# Show ONLY unique lines (lines that appear exactly once)
$ sort names.txt | uniq -u
Bob
```

#### The Classic "Top 10" One-Liner:

```bash
# Find the top 10 IP addresses hitting your server
$ cut -d' ' -f1 /var/log/nginx/access.log | sort | uniq -c | sort -rn | head -10

# Let's break this down step by step:
# cut -d' ' -f1     → Extract just the IP address (first field)
# sort               → Sort IPs alphabetically (so duplicates are adjacent)
# uniq -c            → Count how many times each IP appears
# sort -rn           → Sort by count, reverse, numerically (highest first)
# head -10           → Show only the top 10

# Output:
    1523 192.168.1.10
     987 10.0.0.45
     654 172.16.0.100
     432 192.168.1.25
     ...
```

### 16. `awk` — Pattern Scanning and Processing Language

`awk` is a **mini programming language** for text processing. It's incredibly powerful but we'll keep it simple here.

#### The Big Idea:

`awk` reads text **line by line**, splits each line into **fields** (columns), and lets you work with those fields.

```
By default, fields are separated by spaces/tabs:

Line:    "Hello World DevOps"
Field 1: Hello     → $1
Field 2: World     → $2
Field 3: DevOps    → $3
Entire line:       → $0
```

#### Basic Usage:

```bash
# Print the entire line (same as cat)
$ awk '{print $0}' file.txt

# Print only the first column
$ awk '{print $1}' file.txt

# Print the first and third columns
$ awk '{print $1, $3}' file.txt
```

#### Custom Delimiter:

```bash
# Use colon as delimiter (like /etc/passwd)
$ awk -F':' '{print $1}' /etc/passwd
root
ritesh

# -F':' sets the Field separator to colon

# Print username and shell
$ awk -F':' '{print $1, $7}' /etc/passwd
root /bin/bash
ritesh /bin/bash
nobody /usr/sbin/nologin
```

#### Conditions and Filters:

```bash
# Print lines where the 3rd field is greater than 1000
$ awk -F':' '$3 > 1000 {print $1, $3}' /etc/passwd
ritesh 1000
nobody 65534

# Print lines containing "error"
$ awk '/error/ {print}' app.log

# Print lines where disk usage is above 80%
$ df -h | awk '$5 > 80 {print $1, $5}'
# $5 is the "Use%" column in df output
```

#### Built-in Variables:

```bash
# NR = Number of Record (current line number)
$ awk '{print NR, $0}' file.txt
1 first line
2 second line
3 third line

# NF = Number of Fields (how many columns in this line)
$ awk '{print NF, $0}' file.txt
3 Hello World DevOps
2 foo bar

# Print the LAST field of each line
$ awk '{print $NF}' file.txt
# $NF = the value of the last field
```

#### BEGIN and END Blocks:

```bash
# Print a header, process data, then print a footer
$ awk 'BEGIN {print "=== User List ==="} {print $1} END {print "=== Done ==="}' /etc/passwd
=== User List ===
root
ritesh
...
=== Done ===

# Calculate the SUM of a column
$ awk '{sum += $1} END {print "Total:", sum}' numbers.txt
Total: 150
```

#### Real-World DevOps Usage:

```bash
# Get memory usage from free command
$ free -m | awk '/Mem/ {print "Used:", $3, "MB / Total:", $2, "MB"}'
Used: 4562 MB / Total: 16384 MB

# Calculate average response time from logs
$ awk '{sum += $NF; count++} END {print "Average:", sum/count, "ms"}' response-times.log

# Print failed SSH login attempts with IP
$ awk '/Failed password/ {print $11}' /var/log/auth.log | sort | uniq -c | sort -rn

# Extract specific data from a CSV
$ awk -F',' '{print $1, $3}' data.csv

# Find processes using more than 1GB of memory
$ ps aux | awk '$6 > 1048576 {print $11, $6/1024, "MB"}'
```

### 17. `sed` — Stream Editor (Find and Replace on Steroids)

`sed` lets you **transform text** — find and replace, delete lines, insert lines — all without opening a text editor.

#### Basic Find and Replace:

```bash
# Replace first occurrence of "old" with "new" on each line
$ sed 's/old/new/' file.txt

# Replace ALL occurrences on each line (global)
$ sed 's/old/new/g' file.txt
# The 'g' at the end means "global" — replace all, not just the first

# Case-insensitive replace
$ sed 's/error/WARNING/gi' file.txt
# i = case-insensitive
```

#### The Key Difference: Display vs. Modify

```bash
# By default, sed just PRINTS the result — doesn't change the file
$ sed 's/old/new/g' file.txt
# Output goes to screen. file.txt is unchanged.

# To actually MODIFY the file, use -i (in-place)
$ sed -i 's/old/new/g' file.txt
# Now file.txt is actually changed!

# Create a backup before modifying (SAFE approach)
$ sed -i.bak 's/old/new/g' file.txt
# Saves original as file.txt.bak, then modifies file.txt
```

#### Delete Lines:

```bash
# Delete line 5
$ sed '5d' file.txt

# Delete lines 5 through 10
$ sed '5,10d' file.txt

# Delete the LAST line
$ sed '$d' file.txt

# Delete lines containing "debug"
$ sed '/debug/d' file.txt

# Delete empty lines
$ sed '/^$/d' file.txt
# ^$ = line that starts and immediately ends (empty line)
```

#### Insert and Append Lines:

```bash
# Insert a line BEFORE line 3
$ sed '3i\This is a new line' file.txt

# Append a line AFTER line 3
$ sed '3a\This is a new line' file.txt
```

#### Print Specific Lines:

```bash
# Print only line 5
$ sed -n '5p' file.txt
# -n = suppress normal output
# p  = print matched lines

# Print lines 10 through 20
$ sed -n '10,20p' file.txt

# Print lines containing "error"
$ sed -n '/error/p' file.txt
```

#### Real-World DevOps Usage:

```bash
# Change the port in a config file
$ sed -i 's/listen 80;/listen 8080;/g' /etc/nginx/nginx.conf

# Replace a placeholder in a template with an actual value
$ sed -i "s/{{DB_HOST}}/$DB_HOST/g" config.yml
# This is how you do simple templating!

# Comment out a line (add # at the beginning)
$ sed -i 's/^PermitRootLogin yes/# PermitRootLogin yes/' /etc/ssh/sshd_config

# Uncomment a line (remove # from the beginning)
$ sed -i 's/^# PermitRootLogin no/PermitRootLogin no/' /etc/ssh/sshd_config

# Remove all comment lines and empty lines from a config
$ sed '/^#/d; /^$/d' nginx.conf
# Shows the "clean" config without comments or blanks

# Add a line after a pattern match
$ sed -i '/\[database\]/a host=localhost' config.ini
# Adds "host=localhost" after the line containing "[database]"
```

## 🔗 Piping & Command Chaining — Putting It All Together

The real power comes when you **combine commands** using pipes (`|`) and chaining operators.

### The Pipe `|`

The pipe takes the **output** of one command and feeds it as **input** to the next command.

```bash
# The idea:
command1 | command2 | command3

# Output of command1 → becomes input of command2
# Output of command2 → becomes input of command3
```

### Chaining Operators:

```bash
# ; → Run next command regardless (success or failure)
$ mkdir logs ; cd logs

# && → Run next command ONLY if previous succeeded
$ mkdir logs && cd logs
# If mkdir fails, cd won't run (this is safer!)

# || → Run next command ONLY if previous failed
$ cd /nonexistent || echo "Directory not found!"
```

### Real-World Power Combos:

```bash
# 1. Find the top 10 biggest files on the system
$ find / -type f -exec du -h {} \; 2>/dev/null | sort -rh | head -10

# 2. Count unique HTTP status codes in access logs
$ awk '{print $9}' /var/log/nginx/access.log | sort | uniq -c | sort -rn
  15234 200
   3456 404
    789 500
    234 301

# 3. Find which users are currently logged in and how many
$ who | awk '{print $1}' | sort | uniq -c | sort -rn

# 4. Real-time log monitoring with filtering
$ tail -f /var/log/syslog | grep --line-buffered "error" | awk '{print $1, $2, $3, $NF}'

# 5. Quick system health check
$ echo "=== Disk ===" && df -h | grep '/dev/' && echo "=== Memory ===" && free -h && echo "=== Load ===" && uptime

# 6. Find all failed SSH login attempts and the IPs
$ grep "Failed password" /var/log/auth.log | awk '{print $(NF-3)}' | sort | uniq -c | sort -rn | head -10
```
