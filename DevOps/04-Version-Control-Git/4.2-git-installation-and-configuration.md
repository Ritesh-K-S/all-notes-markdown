## 🤔 Why Does Configuration Matter?

Every Git commit records **who** made the change. If you don't configure your name and email, Git won't know who you are.

```
Example Git log entry:

commit a1b2c3d4e5f6g7h8
Author: Ritesh Singh <ritesh@example.com>    ← This comes from your config!
Date:   Mon Jan 15 10:30:00 2025 +0530

    Added user authentication feature
```

Without proper configuration, your commits might show up as "unknown" — and your team won't know who did what. So let's set everything up properly.

## 📥 Installing Git

### On Linux (Ubuntu/Debian)

```bash
# Update package list first
sudo apt update

# Install Git
sudo apt install git -y

# Verify installation
git --version
```

**Output:**
```
git version 2.43.0
```

### On Linux (CentOS/RHEL/Fedora)

```bash
# For CentOS/RHEL
sudo yum install git -y

# For Fedora
sudo dnf install git -y

# Verify
git --version
```

### On macOS

**Option 1: Using Homebrew (Recommended)**
```bash
# Install Homebrew first (if you don't have it)
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Install Git
brew install git

# Verify
git --version
```

**Option 2: Xcode Command Line Tools**
```bash
# This installs Git along with other dev tools
xcode-select --install
```

### On Windows

**Option 1: Download from Official Website**
1. Go to [https://git-scm.com/download/win](https://git-scm.com/download/win)
2. Download the installer
3. Run the installer — **use the default settings** (they're good for most people)
4. This installs **Git Bash** (a terminal emulator) along with Git

**Option 2: Using Winget (Windows Package Manager)**
```powershell
winget install --id Git.Git -e --source winget
```

**Option 3: Using Chocolatey**
```powershell
choco install git -y
```

### On WSL2 (Windows Subsystem for Linux)

If you're using WSL2 (which many DevOps engineers do on Windows):

```bash
# Inside your WSL2 terminal (Ubuntu)
sudo apt update && sudo apt install git -y
```

### Verify Installation (All Platforms)

```bash
git --version
```

If you see a version number, you're good to go! 🎉

## ⚙️ Git Configuration — The Three Levels

Git configuration works at **three levels**. This is important to understand:

```
┌────────────────────────────────────────────────────────────────────┐
│                                                                    │
│  Level 1: SYSTEM   (/etc/gitconfig)                               │
│  └── Applies to ALL users on the computer                         │
│  └── Command: git config --system                                 │
│  └── Rarely used                                                  │
│                                                                    │
│  Level 2: GLOBAL   (~/.gitconfig  or  ~/.config/git/config)       │
│  └── Applies to YOUR user account (all your repositories)         │
│  └── Command: git config --global                                 │
│  └── ⭐ Most commonly used — set up ONCE                         │
│                                                                    │
│  Level 3: LOCAL    (.git/config inside each repository)            │
│  └── Applies to ONLY that specific repository                     │
│  └── Command: git config --local (or just git config)             │
│  └── Used when you need different settings for a specific repo    │
│                                                                    │
└────────────────────────────────────────────────────────────────────┘

Priority:  LOCAL > GLOBAL > SYSTEM
(Local settings override Global, Global overrides System)
```

### Why Three Levels?

Imagine this scenario:
- Your **personal** GitHub email is `ritesh@gmail.com`
- Your **work** GitLab email is `ritesh@company.com`

You can set:
- **Global** config: `ritesh@gmail.com` (used for all personal projects)
- **Local** config (in work repo): `ritesh@company.com` (used only for that repo)

This way, the right email is used in the right place!

## 🛠️ Essential Configuration — Do This First!

### 1. Set Your Name

```bash
git config --global user.name "Your Full Name"
```

**Example:**
```bash
git config --global user.name "Ritesh Singh"
```

### 2. Set Your Email

```bash
git config --global user.email "your.email@example.com"
```

**Example:**
```bash
git config --global user.email "ritesh@example.com"
```

> **Important:** Use the **same email** that you use on GitHub/GitLab. This is how platforms match your commits to your profile.

### 3. Set Your Default Editor

When Git needs you to type something (like a commit message), it opens a text editor. By default, it's usually **Vim** — and if you've never used Vim, you'll be stuck 😅

```bash
# Set VS Code as default editor
git config --global core.editor "code --wait"

# Set Nano (simple terminal editor)
git config --global core.editor "nano"

# Set Vim (for Vim lovers)
git config --global core.editor "vim"

# Set Notepad++ on Windows
git config --global core.editor "'C:/Program Files/Notepad++/notepad++.exe' -multiInst -notabbar -nosession -noPlugin"
```

The `--wait` flag for VS Code tells Git to **wait** until you close the file in VS Code before continuing.

### 4. Set Default Branch Name

By default, Git creates a branch called `master` when you initialize a new repo. The industry has been moving toward `main` as the default:

```bash
git config --global init.defaultBranch main
```

Now when you do `git init`, the default branch will be called `main` instead of `master`.

### 5. Set Line Ending Handling (Important for Cross-Platform Teams!)

Windows uses `CRLF` (Carriage Return + Line Feed) for line endings, while Linux/macOS use `LF` (Line Feed only). This can cause messy diffs. Git can handle this automatically:

```bash
# On Windows:
git config --global core.autocrlf true
# Converts LF → CRLF when checking out, CRLF → LF when committing

# On Linux/macOS:
git config --global core.autocrlf input
# Converts CRLF → LF when committing, no conversion when checking out
```

**Why this matters:**
```
Without autocrlf:
  → Alice (Mac) commits a file with LF line endings
  → Bob (Windows) opens it — Git shows EVERY line as changed (because CRLF ≠ LF)
  → Massive fake diffs, confusion everywhere

With autocrlf:
  → Git automatically handles the conversion
  → No fake diffs. Everyone is happy.
```

## 🎨 Optional But Useful Configuration

### Enable Colored Output

Makes Git output much more readable:

```bash
git config --global color.ui auto
```

This is usually enabled by default in modern Git, but it's good to make sure.

### Set Up Aliases (Shortcuts)

Git commands can be long. Aliases let you create shortcuts:

```bash
# Instead of typing 'git status', just type 'git st'
git config --global alias.st status

# Instead of 'git checkout', type 'git co'
git config --global alias.co checkout

# Instead of 'git branch', type 'git br'
git config --global alias.br branch

# Instead of 'git commit', type 'git ci'
git config --global alias.ci commit

# A beautiful one-line log
git config --global alias.lg "log --oneline --graph --all --decorate"

# Show last commit
git config --global alias.last "log -1 HEAD"
```

**Using aliases:**
```bash
# Before aliases:
git log --oneline --graph --all --decorate

# After aliases:
git lg

# Same output, much less typing!
```

### Set Default Pull Behavior

When you do `git pull`, Git can either **merge** or **rebase**. Set your preference:

```bash
# Rebase on pull (cleaner history — recommended for most teams)
git config --global pull.rebase true

# Merge on pull (default behavior)
git config --global pull.rebase false

# Only pull if it can fast-forward (safest option)
git config --global pull.ff only
```

Don't worry if you don't understand rebase vs merge yet — we'll cover that in a later section. For now, this is a good default:

```bash
git config --global pull.rebase false
```

### Set Credential Helper (Avoid Typing Password Every Time)

```bash
# On macOS (stores in Keychain):
git config --global credential.helper osxkeychain

# On Windows (stores in Windows Credential Manager):
git config --global credential.helper manager

# On Linux (cache for 1 hour = 3600 seconds):
git config --global credential.helper 'cache --timeout=3600'

# On Linux (store permanently — less secure):
git config --global credential.helper store
```

> **Better approach:** Use **SSH keys** instead of HTTPS + password. We'll cover SSH key setup later in this chapter.

## 📋 View Your Configuration

### See All Configuration

```bash
git config --list
```

**Example output:**
```
user.name=Ritesh Singh
user.email=ritesh@example.com
core.editor=code --wait
init.defaultbranch=main
core.autocrlf=true
color.ui=auto
alias.st=status
alias.co=checkout
alias.br=branch
alias.lg=log --oneline --graph --all --decorate
```

### See Where Each Config is Set

```bash
git config --list --show-origin
```

This shows you **which file** each setting comes from:

```
file:C:/Users/ritesh/.gitconfig   user.name=Ritesh Singh
file:C:/Users/ritesh/.gitconfig   user.email=ritesh@example.com
file:.git/config                  remote.origin.url=https://github.com/ritesh/my-repo.git
```

### Check a Specific Setting

```bash
git config user.name
# Output: Ritesh Singh

git config user.email
# Output: ritesh@example.com
```

## 📄 The Configuration File

All global config is stored in a plain text file: `~/.gitconfig` (or `C:\Users\<username>\.gitconfig` on Windows).

You can also edit this file directly instead of using commands:

```bash
# Open the global config in your editor
git config --global --edit
```

**What the file looks like:**

```ini
# ~/.gitconfig

[user]
    name = Ritesh Singh
    email = ritesh@example.com

[core]
    editor = code --wait
    autocrlf = true

[init]
    defaultBranch = main

[color]
    ui = auto

[alias]
    st = status
    co = checkout
    br = branch
    ci = commit
    lg = log --oneline --graph --all --decorate
    last = log -1 HEAD

[pull]
    rebase = false

[credential]
    helper = manager
```

It's just a simple **INI-style** file. You can edit it with any text editor.

## 🔧 Troubleshooting Common Issues

### "git is not recognized as a command"

**Windows:** Git wasn't added to your PATH during installation.
- Fix: Reinstall Git and make sure to check **"Add Git to PATH"** during setup
- Or manually add `C:\Program Files\Git\bin` to your system PATH

**Linux/macOS:** Git isn't installed.
```bash
# Check if Git exists
which git

# If nothing shows up, install it
sudo apt install git    # Ubuntu/Debian
brew install git        # macOS
```

### "Please tell me who you are" Error

```
*** Please tell me who you are.

Run
  git config --global user.email "you@example.com"
  git config --global user.name "Your Name"
```

This means you haven't configured your name and email. Just run those two commands!

### "LF will be replaced by CRLF" Warning

```
warning: LF will be replaced by CRLF in file.txt.
```

This is Git's line ending conversion at work. It's normal on Windows. If it bothers you, you've already set `core.autocrlf` above.
