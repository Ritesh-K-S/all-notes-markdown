## 1. What is PowerShell?

**PowerShell** is a **command-line shell + scripting language** built by Microsoft. Think of it as the "Bash of the Windows world" — but on steroids.

Here's the key difference that makes PowerShell special:

> **Bash** works with **text**. You pipe plain text from one command to another.  
> **PowerShell** works with **objects**. You pipe rich data objects with properties and methods.

Let me explain with an analogy:

> Imagine you're managing a warehouse.
>
> **Bash** is like getting a **handwritten list** of items: `"Apple, 5kg, Red"`. If you want just the color, you have to count commas and cut the text. Messy.
>
> **PowerShell** is like getting a **structured package** with labels: `Name=Apple, Weight=5kg, Color=Red`. Want the color? Just say `$item.Color`. Clean and easy.

### Two Versions of PowerShell

| Version | Name | Platform | Notes |
|---------|------|----------|-------|
| **Windows PowerShell** | `powershell.exe` | Windows only | Built into Windows (version 5.1). Uses .NET Framework. |
| **PowerShell (Core)** | `pwsh.exe` | Windows, Linux, macOS | Open-source, cross-platform (version 7+). Uses .NET Core. |

> **For DevOps:** Use **PowerShell 7+** (`pwsh`) whenever possible — it's cross-platform and actively developed.

## 2. PowerShell vs Bash — What's the Difference?

| Feature | Bash | PowerShell |
|---------|------|------------|
| **Data type** | Text (strings) | Objects (.NET objects) |
| **Pipeline** | Passes text between commands | Passes objects between commands |
| **Syntax** | `command -flag argument` | `Verb-Noun -Parameter Value` |
| **Platform** | Linux/macOS (native), Windows (via WSL) | Windows (native), Linux/macOS (via pwsh) |
| **Scripting** | `.sh` files | `.ps1` files |
| **Config management** | Ansible, Chef (common) | DSC (Desired State Configuration) |
| **Cloud** | AWS CLI, gcloud | Azure PowerShell, AWS Tools for PowerShell |
| **Learning curve** | Simpler for basic tasks | Steeper initially, but more powerful |

### The Same Task — Bash vs PowerShell

**Task:** Find processes using more than 100MB of memory.

**Bash:**
```bash
ps aux | awk '$6 > 102400 {print $11, $6/1024 "MB"}'
```
You're cutting text by column position. If the output format changes, your script breaks.

**PowerShell:**
```powershell
Get-Process | Where-Object { $_.WorkingSet64 -gt 100MB } | Select-Object Name, @{N='MemoryMB';E={[math]::Round($_.WorkingSet64/1MB,2)}}
```
You're working with **actual process objects**. No text parsing needed!

## 4. PowerShell Basics — Your First Commands

### Cmdlets — The Building Blocks

PowerShell commands are called **cmdlets** (pronounced "command-lets"). They follow a strict **Verb-Noun** naming pattern:

```
Verb-Noun
```

Examples:

| Cmdlet | What it Does |
|--------|-------------|
| `Get-Process` | Get running processes |
| `Get-Service` | Get Windows services |
| `Get-ChildItem` | List files/folders (like `ls` or `dir`) |
| `Set-Location` | Change directory (like `cd`) |
| `New-Item` | Create a file or folder |
| `Remove-Item` | Delete a file or folder |
| `Copy-Item` | Copy a file or folder |
| `Move-Item` | Move/rename a file or folder |
| `Get-Content` | Read file contents (like `cat`) |
| `Set-Content` | Write content to a file |
| `Write-Output` | Print to screen (like `echo`) |
| `Get-Help` | Get help for any command |

### Common Verbs

| Verb | Purpose |
|------|---------|
| `Get` | Retrieve / Read |
| `Set` | Change / Update |
| `New` | Create |
| `Remove` | Delete |
| `Start` | Begin |
| `Stop` | End |
| `Restart` | Stop + Start |
| `Test` | Check / Validate |
| `Invoke` | Execute / Run |
| `Import` / `Export` | Bring in / Send out |

### Let's Try Some Commands

```powershell
# Print a message
Write-Output "Hello, DevOps World!"

# Get today's date
Get-Date

# List files in current directory
Get-ChildItem

# List only .txt files
Get-ChildItem -Filter *.txt

# Change directory
Set-Location C:\Users

# Get your current directory
Get-Location

# Clear the screen
Clear-Host
```

### Aliases — Shortcuts

PowerShell has built-in **aliases** so you can use familiar commands:

| Alias | Full Cmdlet | Bash Equivalent |
|-------|------------|----------------|
| `ls` | `Get-ChildItem` | `ls` |
| `dir` | `Get-ChildItem` | `ls` |
| `cd` | `Set-Location` | `cd` |
| `pwd` | `Get-Location` | `pwd` |
| `cat` | `Get-Content` | `cat` |
| `cp` | `Copy-Item` | `cp` |
| `mv` | `Move-Item` | `mv` |
| `rm` | `Remove-Item` | `rm` |
| `echo` | `Write-Output` | `echo` |
| `cls` | `Clear-Host` | `clear` |
| `man` | `Get-Help` | `man` |
| `kill` | `Stop-Process` | `kill` |

```powershell
# These are all the same:
Get-ChildItem

# See all aliases

# Find the alias for a cmdlet
Get-Alias -Definition Get-ChildItem
```

### Getting Help

PowerShell has **excellent** built-in help:

```powershell
# Get help for any command
Get-Help Get-Process

# Detailed help with examples
Get-Help Get-Process -Detailed

# Show only examples
Get-Help Get-Process -Examples

# Online help (opens browser)
Get-Help Get-Process -Online

# Update help files (run as admin, one-time setup)
Update-Help -Force
```

### Discovering Commands

```powershell
# Find commands related to a topic
Get-Command *process*
# Output: Get-Process, Start-Process, Stop-Process, Wait-Process...

# Find commands with a specific verb
Get-Command -Verb Get

# Find commands with a specific noun
Get-Command -Noun Service
```

## 5. The Pipeline — PowerShell's Superpower

The **pipeline** (`|`) passes the **output of one command as input to the next**. But unlike Bash (which passes text), PowerShell passes **objects**.

### Basic Pipeline

```powershell
# Get all processes, then sort by memory usage (descending)
Get-Process | Sort-Object WorkingSet64 -Descending

# Get all processes, sort by memory, show top 5
Get-Process | Sort-Object WorkingSet64 -Descending | Select-Object -First 5

# Get services, filter only running ones
Get-Service | Where-Object { $_.Status -eq "Running" }
```

### Understanding `$_` — The Current Object

In the pipeline, `$_` represents the **current object** being processed (like a "this" variable).

```powershell
# Get processes where CPU time is greater than 10 seconds
Get-Process | Where-Object { $_.CPU -gt 10 }

# Get files larger than 1MB
Get-ChildItem | Where-Object { $_.Length -gt 1MB }
```

Think of it like this:

```
Get-Process gives you a list of process objects.
Each object flows through the pipeline ONE BY ONE.
$_ = the current process object being examined.
$_.CPU = the CPU property of the current process.
```

### Pipeline Example — Step by Step

```powershell
# Task: Find the top 5 largest files in C:\Windows\Temp

Get-ChildItem C:\Windows\Temp -File -Recurse -ErrorAction SilentlyContinue |
    Sort-Object Length -Descending |
    Select-Object -First 5 |
    Format-Table Name, @{N='SizeMB';E={[math]::Round($_.Length/1MB,2)}}, LastWriteTime
```

Let's break it down:

| Step | What It Does |
|------|-------------|
| `Get-ChildItem ... -File -Recurse` | Get all files (not folders) recursively |
| `Sort-Object Length -Descending` | Sort by file size, biggest first |
| `Select-Object -First 5` | Take only the top 5 |
| `Format-Table ...` | Display as a nice table |

Output might look like:

```
Name                    SizeMB  LastWriteTime
----                    ------  -------------
big_dump.log            245.78  1/15/2025 3:00 AM
temp_data.dat            89.34  1/14/2025 11:00 PM
cache_file.tmp           56.12  1/15/2025 1:00 AM
setup_log.txt            23.45  1/13/2025 9:00 AM
output.csv               12.67  1/15/2025 2:00 AM
```

## 6. Variables

Variables in PowerShell start with `$`. No need to declare types — PowerShell figures it out automatically.

```powershell
# Assign a value
$name = "DevOps Engineer"
$age = 25
$isActive = $true
$salary = 95000.50

# Print variables
Write-Output $name          # DevOps Engineer
Write-Output "I am $name"   # I am DevOps Engineer (interpolation!)

# Check the type of a variable
$name.GetType().Name        # String
$age.GetType().Name         # Int32
$isActive.GetType().Name    # Boolean
```

### Special Variables

| Variable | Meaning |
|----------|---------|
| `$_` | Current object in pipeline |
| `$PSVersionTable` | PowerShell version info |
| `$HOME` | User's home directory |
| `$PWD` | Current directory |
| `$null` | Null (nothing) |
| `$true` / `$false` | Boolean values |
| `$Error` | Array of recent errors |
| `$LASTEXITCODE` | Exit code of last external command |
| `$env:PATH` | Environment variable (PATH) |
| `$args` | Arguments passed to a script |

### Environment Variables

```powershell
# Read an environment variable
$env:USERNAME         # Current user
$env:COMPUTERNAME     # Computer name
$env:PATH             # System PATH

# Set an environment variable (current session only)
$env:MY_VAR = "hello"

# Set a permanent environment variable
[Environment]::SetEnvironmentVariable("MY_VAR", "hello", "User")

# List all environment variables
Get-ChildItem Env:
```

## 7. Data Types

PowerShell is built on **.NET**, so it has access to all .NET data types. But you usually don't need to worry about types — PowerShell handles it.

| Type | Example | Notes |
|------|---------|-------|
| `[string]` | `"hello"` | Text |
| `[int]` | `42` | 32-bit integer |
| `[long]` | `42L` | 64-bit integer |
| `[double]` | `3.14` | Decimal number |
| `[bool]` | `$true`, `$false` | Boolean |
| `[datetime]` | `Get-Date` | Date and time |
| `[array]` | `@(1,2,3)` | Array |
| `[hashtable]` | `@{key="value"}` | Dictionary / Map |

### Type Casting

```powershell
# Convert string to integer
[int]"42"           # 42

# Convert integer to string
[string]42          # "42"

# Convert string to datetime
[datetime]"2025-01-15"   # Wednesday, January 15, 2025 12:00:00 AM

# Check if something is a certain type
"hello" -is [string]     # True
42 -is [int]             # True
42 -is [string]          # False
```

## 8. Strings — Quotes, Interpolation & Here-Strings

### Double Quotes vs Single Quotes

```powershell
$name = "World"

# Double quotes — variables and expressions are EXPANDED
"Hello, $name!"           # Hello, World!

# Single quotes — everything is LITERAL (no expansion)
'Hello, $name!'           # Hello, $name!
```

**Rule of thumb:**
- Use **double quotes** when you want variable interpolation
- Use **single quotes** when you want the literal text

### Subexpressions in Strings

For anything more complex than a simple variable, use `$()`:

```powershell
$processes = (Get-Process).Count
"There are $($processes) processes running."
# Output: There are 284 processes running.

"Today is $(Get-Date -Format 'yyyy-MM-dd')"
# Output: Today is 2025-01-15
```

### String Operations

```powershell
$str = "Hello, World!"

$str.Length              # 13
$str.ToUpper()           # HELLO, WORLD!
$str.ToLower()           # hello, world!
$str.Contains("World")   # True
$str.Replace("World", "DevOps")  # Hello, DevOps!
$str.Split(",")          # @("Hello", " World!")
$str.Trim()              # Remove leading/trailing whitespace
$str.Substring(0, 5)     # Hello
$str.StartsWith("Hello") # True
$str.EndsWith("!")        # True
```

### Here-Strings (Multi-Line Strings)

```powershell
# Double-quoted here-string (with interpolation)
$name = "Nginx"
$config = @"
server {
    listen 80;
    server_name $name;
    location / {
        proxy_pass http://localhost:3000;
    }
}
"@

Write-Output $config

# Single-quoted here-string (literal)
$template = @'
No variables expanded here.
$name stays as $name.
Great for templates and regex patterns.
'@
```

> **Important:** The closing `"@` or `'@` **must** be at the very beginning of the line — no spaces before it!

## 9. Arrays & ArrayLists

### Arrays

```powershell
# Create an array
$servers = @("web-01", "web-02", "db-01", "db-02")

# Access elements (zero-indexed)
$servers[0]          # web-01
$servers[1]          # web-02
$servers[-1]         # db-02 (last element)

# Array length
$servers.Count       # 4
$servers.Length       # 4

# Slice
$servers[0..2]       # web-01, web-02, db-01

# Check if array contains an element
$servers -contains "db-01"    # True

# Add element (creates a NEW array — arrays are fixed-size!)
$servers += "cache-01"

# Loop through array
foreach ($server in $servers) {
    Write-Output "Checking $server..."
}
```

### ArrayList (Dynamic — Better for Adding/Removing)

Regular arrays are **fixed-size** in PowerShell. Every time you use `+=`, PowerShell creates a **new array** and copies everything — slow for large datasets!

Use `ArrayList` instead:

```powershell
# Create an ArrayList
$servers = [System.Collections.ArrayList]@()

# Add elements
$servers.Add("web-01") | Out-Null    # | Out-Null suppresses the index output
$servers.Add("web-02") | Out-Null
$servers.Add("db-01") | Out-Null

# Remove an element
$servers.Remove("web-02")

# Count
$servers.Count    # 2

# Check
$servers.Contains("db-01")    # True
```

### Array of Objects

```powershell
# Create an array of custom objects (very useful in DevOps!)
$inventory = @(
    [PSCustomObject]@{ Name="web-01"; IP="10.0.1.10"; Role="WebServer" }
    [PSCustomObject]@{ Name="db-01";  IP="10.0.2.10"; Role="Database" }
    [PSCustomObject]@{ Name="app-01"; IP="10.0.3.10"; Role="AppServer" }
)

# Access properties
$inventory[0].Name    # web-01
$inventory[0].IP      # 10.0.1.10

# Filter
$inventory | Where-Object { $_.Role -eq "Database" }

# Display as table
$inventory | Format-Table -AutoSize
```

## 10. Hashtables (Dictionaries)

A **hashtable** is a collection of **key-value pairs** (like a Python dictionary or a JSON object).

```powershell
# Create a hashtable
$server = @{
    Name   = "web-01"
    IP     = "10.0.1.10"
    OS     = "Windows Server 2022"
    CPU    = 4
    RAM_GB = 16
}

# Access values
$server["Name"]      # web-01
$server.Name         # web-01 (dot notation also works!)

# Add a new key
$server["Environment"] = "Production"
$server.Status = "Running"

# Remove a key
$server.Remove("CPU")

# Check if key exists
$server.ContainsKey("IP")      # True
$server.ContainsValue("web-01") # True

# Get all keys and values
$server.Keys
$server.Values

# Iterate
foreach ($key in $server.Keys) {
    Write-Output "$key = $($server[$key])"
}
```

### Ordered Hashtable

Regular hashtables don't preserve order. Use `[ordered]` if order matters:

```powershell
$config = [ordered]@{
    Step1 = "Install dependencies"
    Step2 = "Run tests"
    Step3 = "Build artifact"
    Step4 = "Deploy to staging"
    Step5 = "Deploy to production"
}

$config
# Output is in order: Step1, Step2, Step3, Step4, Step5
```

## 11. Operators

### Comparison Operators

PowerShell uses **text-based operators** instead of symbols (because `>` and `<` are used for redirection):

| Operator | Meaning | Example |
|----------|---------|---------|
| `-eq` | Equal | `5 -eq 5` → True |
| `-ne` | Not equal | `5 -ne 3` → True |
| `-gt` | Greater than | `5 -gt 3` → True |
| `-ge` | Greater or equal | `5 -ge 5` → True |
| `-lt` | Less than | `3 -lt 5` → True |
| `-le` | Less or equal | `3 -le 3` → True |
| `-like` | Wildcard match | `"hello" -like "hel*"` → True |
| `-notlike` | Not wildcard match | `"hello" -notlike "xyz*"` → True |
| `-match` | Regex match | `"hello123" -match "\d+"` → True |
| `-notmatch` | Not regex match | `"hello" -notmatch "\d+"` → True |
| `-contains` | Array contains | `@(1,2,3) -contains 2` → True |
| `-notcontains` | Array doesn't contain | `@(1,2,3) -notcontains 5` → True |
| `-in` | Value in array | `2 -in @(1,2,3)` → True |
| `-notin` | Value not in array | `5 -notin @(1,2,3)` → True |

### Logical Operators

| Operator | Meaning |
|----------|---------|
| `-and` | Logical AND |
| `-or` | Logical OR |
| `-not` or `!` | Logical NOT |

```powershell
$age = 25
$role = "Engineer"

if ($age -ge 18 -and $role -eq "Engineer") {
    Write-Output "You qualify!"
}
```

### Arithmetic Operators

```powershell
5 + 3     # 8
10 - 4    # 6
3 * 7     # 21
20 / 4    # 5
20 % 3    # 2 (modulus / remainder)

# PowerShell understands size units!
1KB       # 1024
1MB       # 1048576
1GB       # 1073741824
1TB       # 1099511627776

# Useful for size comparisons
$fileSize = 5MB
if ($fileSize -gt 1MB) { Write-Output "Large file!" }
```

## 12. Conditional Statements (if/else, switch)

### if / elseif / else

```powershell
$diskUsage = 85

if ($diskUsage -gt 90) {
    Write-Output "CRITICAL: Disk usage is above 90%!"
} elseif ($diskUsage -gt 75) {
    Write-Output "WARNING: Disk usage is above 75%"
} else {
    Write-Output "OK: Disk usage is normal"
}
# Output: WARNING: Disk usage is above 75%
```

### Real DevOps Example

```powershell
# Check if a service is running, restart if stopped
$service = Get-Service -Name "W3SVC" -ErrorAction SilentlyContinue

if ($null -eq $service) {
    Write-Output "Service not found!"
} elseif ($service.Status -eq "Running") {
    Write-Output "IIS is running. All good!"
} else {
    Write-Output "IIS is stopped. Restarting..."
    Start-Service -Name "W3SVC"
    Write-Output "IIS restarted successfully."
}
```

### switch Statement

`switch` is cleaner than multiple `if/elseif` blocks:

```powershell
$environment = "staging"

switch ($environment) {
    "dev"        { Write-Output "Deploying to Development" }
    "staging"    { Write-Output "Deploying to Staging" }
    "production" { Write-Output "Deploying to Production — be careful!" }
    default      { Write-Output "Unknown environment: $environment" }
}
# Output: Deploying to Staging
```

### switch with Regex

```powershell
$logLine = "2025-01-15 ERROR: Database connection failed"

switch -Regex ($logLine) {
    "ERROR"    { Write-Output "Found an error!" }
    "WARN"     { Write-Output "Found a warning!" }
    "INFO"     { Write-Output "Informational message" }
    "CRITICAL" { Write-Output "Critical issue!" }
}
# Output: Found an error!
```

## 13. Loops (for, foreach, while, do-while)

### foreach — Most Common Loop

```powershell
$servers = @("web-01", "web-02", "db-01", "db-02")

foreach ($server in $servers) {
    Write-Output "Pinging $server..."
    Test-Connection -ComputerName $server -Count 1 -Quiet
}
```

### ForEach-Object — Pipeline Version

```powershell
# Same thing but using the pipeline
@("web-01", "web-02", "db-01") | ForEach-Object {
    Write-Output "Checking $_..."
}
```

### for Loop

```powershell
# Classic for loop
for ($i = 1; $i -le 5; $i++) {
    Write-Output "Iteration $i"
}
# Output:
# Iteration 1
# Iteration 2
# Iteration 3
# Iteration 4
# Iteration 5
```

### while Loop

```powershell
# Wait for a service to start (with timeout)
$attempts = 0
$maxAttempts = 10

while ($attempts -lt $maxAttempts) {
    $service = Get-Service -Name "MyApp" -ErrorAction SilentlyContinue
    if ($service.Status -eq "Running") {
        Write-Output "Service is running!"
        break
    }
    Write-Output "Waiting... attempt $($attempts + 1)/$maxAttempts"
    Start-Sleep -Seconds 5
    $attempts++
}

if ($attempts -eq $maxAttempts) {
    Write-Output "Service failed to start after $maxAttempts attempts!"
}
```

### do-while Loop

```powershell
# Runs at least once
$count = 1
do {
    Write-Output "Count: $count"
    $count++
} while ($count -le 3)
# Output:
# Count: 1
# Count: 2
# Count: 3
```

### do-until Loop (PowerShell exclusive)

```powershell
# Loop UNTIL a condition is true (opposite of while)
$count = 1
do {
    Write-Output "Count: $count"
    $count++
} until ($count -gt 3)
```

## 14. Functions

### Basic Function

```powershell
function Say-Hello {
    Write-Output "Hello, DevOps World!"
}

# Call it
Say-Hello
# Output: Hello, DevOps World!
```

### Function with Parameters

```powershell
function Get-Greeting {
    param (
        [string]$Name,
        [string]$Role = "Engineer"   # Default value
    )

    Write-Output "Hello, $Name! You are a $Role."
}

# Call with parameters
Get-Greeting -Name "Alice" -Role "SRE"
# Output: Hello, Alice! You are a SRE.

Get-Greeting -Name "Bob"
# Output: Hello, Bob! You are a Engineer.
```

### Function with Mandatory Parameters & Validation

```powershell
function Deploy-Application {
    param (
        [Parameter(Mandatory=$true)]
        [string]$AppName,

        [Parameter(Mandatory=$true)]
        [ValidateSet("dev", "staging", "production")]
        [string]$Environment,

        [int]$Replicas = 1
    )

    Write-Output "Deploying $AppName to $Environment with $Replicas replica(s)..."
}

# Usage

# Output: Deploying MyWebApp to staging with 3 replica(s)...

# If you pass an invalid environment:

# ERROR: Cannot validate argument — "invalid" is not in the set "dev", "staging", "production"
```

## 15. Error Handling (try/catch/finally)

### Basic try/catch

```powershell
try {
    # Attempt something that might fail
    Get-Content -Path "C:\nonexistent\file.txt" -ErrorAction Stop
}
catch {
    Write-Output "Error occurred: $($_.Exception.Message)"
}
finally {
    # This ALWAYS runs, whether error happened or not
    Write-Output "Cleanup complete."
}
```

### ErrorAction Parameter

Every cmdlet has an `-ErrorAction` parameter:

| Value | Behavior |
|-------|----------|
| `Continue` | Show error, keep going (default) |
| `Stop` | Throw a terminating error (needed for `catch` to work) |
| `SilentlyContinue` | Suppress error, keep going |
| `Inquire` | Ask the user what to do |

```powershell
# Without -ErrorAction Stop, catch won't trigger for non-terminating errors
try {
    Get-Service -Name "NonExistentService" -ErrorAction Stop
}
catch {
    Write-Output "Service not found: $($_.Exception.Message)"
}
```

### Real DevOps Error Handling

```powershell
function Restart-ServiceSafely {
    param (
        [Parameter(Mandatory)]
        [string]$ServiceName
    )

    try {
        $service = Get-Service -Name $ServiceName -ErrorAction Stop

        if ($service.Status -eq "Running") {
            Write-Output "Stopping $ServiceName..."
            Stop-Service -Name $ServiceName -ErrorAction Stop

            Write-Output "Starting $ServiceName..."
            Start-Service -Name $ServiceName -ErrorAction Stop

            Write-Output "$ServiceName restarted successfully!"
        }
        else {
            Write-Output "$ServiceName is not running. Starting it..."
            Start-Service -Name $ServiceName -ErrorAction Stop
            Write-Output "$ServiceName started!"
        }
    }
    catch [Microsoft.PowerShell.Commands.ServiceCommandException] {
        Write-Output "Service error: $($_.Exception.Message)"
    }
    catch {
        Write-Output "Unexpected error: $($_.Exception.Message)"
        Write-Output "Error type: $($_.Exception.GetType().FullName)"
    }
}
```

## 21. Writing & Running Scripts (.ps1 files)

### Creating a Script

Create a file with the `.ps1` extension:

```powershell
# health-check.ps1

param (
    [string[]]$Servers = @("localhost"),
    [string]$OutputFile = "health-report.csv"
)

$results = foreach ($server in $Servers) {
    try {
        $os = Get-CimInstance -ClassName Win32_OperatingSystem -ComputerName $server -ErrorAction Stop
        $cpu = Get-CimInstance -ClassName Win32_Processor -ComputerName $server -ErrorAction Stop
        $disk = Get-CimInstance -ClassName Win32_LogicalDisk -ComputerName $server -Filter "DeviceID='C:'" -ErrorAction Stop

        [PSCustomObject]@{
            Server       = $server
            Status       = "Online"
            OS           = $os.Caption
            Uptime       = (New-TimeSpan -Start $os.LastBootUpTime).ToString("dd\.hh\:mm\:ss")
            CPUUsage     = "$([math]::Round($cpu.LoadPercentage, 2))%"
            TotalRAM_GB  = [math]::Round($os.TotalVisibleMemorySize / 1MB, 2)
            FreeRAM_GB   = [math]::Round($os.FreePhysicalMemory / 1MB, 2)
            DiskTotal_GB = [math]::Round($disk.Size / 1GB, 2)
            DiskFree_GB  = [math]::Round($disk.FreeSpace / 1GB, 2)
        }
    }
    catch {
        [PSCustomObject]@{
            Server       = $server
            Status       = "Offline / Error"
            OS           = "N/A"
            Uptime       = "N/A"
            CPUUsage     = "N/A"
            TotalRAM_GB  = "N/A"
            FreeRAM_GB   = "N/A"
            DiskTotal_GB = "N/A"
            DiskFree_GB  = "N/A"
        }
    }
}

# Display on screen
$results | Format-Table -AutoSize

# Export to CSV
$results | Export-Csv -Path $OutputFile -NoTypeInformation
Write-Output "Report saved to $OutputFile"
```

Windows restricts running scripts by default. You may need to change the **Execution Policy**:

## 22. PowerShell in DevOps — Real-World Use Cases

### Use Case 1: Automated Deployment Script

```powershell
# deploy.ps1 — Simple deployment script

param (
    [Parameter(Mandatory)]
    [ValidateSet("dev", "staging", "production")]
    [string]$Environment,

    [Parameter(Mandatory)]
    [string]$Version
)

$deployPath = "C:\WebApps\MyApp\$Environment"
$artifactUrl = "https://artifacts.example.com/myapp/v$Version/myapp.zip"
$backupPath = "C:\Backups\MyApp\$Environment\$(Get-Date -Format 'yyyyMMdd-HHmmss')"

Write-Output "=== Deploying MyApp v$Version to $Environment ==="

# Step 1: Backup current version
Write-Output "[1/4] Creating backup..."

    Copy-Item -Path $deployPath -Destination $backupPath -Recurse
    Write-Output "  Backup created at: $backupPath"
} else {
    Write-Output "  No existing deployment to backup."
}

# Step 2: Download artifact
Write-Output "[2/4] Downloading artifact..."
$zipFile = "$env:TEMP\myapp-$Version.zip"
Invoke-WebRequest -Uri $artifactUrl -OutFile $zipFile
Write-Output "  Downloaded to: $zipFile"

# Step 3: Extract
Write-Output "[3/4] Extracting..."
if (Test-Path $deployPath) { Remove-Item -Path $deployPath -Recurse -Force }
Expand-Archive -Path $zipFile -DestinationPath $deployPath
Write-Output "  Extracted to: $deployPath"

# Step 4: Restart service
Write-Output "[4/4] Restarting application pool..."
Restart-Service -Name "W3SVC" -Force
Write-Output "  Service restarted."

Write-Output "=== Deployment complete! ==="
```

## 24. Best Practices

### 1. Use Proper Cmdlet Names in Scripts

```powershell
# ❌ Don't use aliases in scripts (hard to read, might not exist on all systems)
ls | ? { $_.Length -gt 1MB } | % { $_.Name }

# ✅ Use full cmdlet names
Get-ChildItem | Where-Object { $_.Length -gt 1MB } | ForEach-Object { $_.Name }
```

### 2. Always Use `-ErrorAction` in Automation

```powershell
# ❌ Error silently ignored — script continues with bad data
$service = Get-Service -Name "FakeService"

# ✅ Handle errors explicitly
try {
    $service = Get-Service -Name "FakeService" -ErrorAction Stop
} catch {
    Write-Output "Service not found!"
    exit 1
}
```

### 3. Use `[CmdletBinding()]` for Advanced Functions

```powershell
function Deploy-App {
    [CmdletBinding()]
    param (
        [Parameter(Mandatory)]
        [string]$AppName
    )

    Write-Verbose "Starting deployment of $AppName..."
    # ... deployment logic ...
    Write-Verbose "Deployment complete."
}

# Now you can use -Verbose
Deploy-App -AppName "MyApp" -Verbose
```

### 4. Use Approved Verbs

```powershell
# See all approved verbs
Get-Verb

# Common approved verbs:
# Get, Set, New, Remove, Start, Stop, Restart, Test,
# Invoke, Import, Export, Copy, Move, Enable, Disable
```

### 5. Output Objects, Not Strings

```powershell
# ❌ Returns a string — hard to use in pipeline
function Get-ServerInfo {
    return "Server: web-01, IP: 10.0.1.10, Status: Online"
}

# ✅ Returns an object — pipeline-friendly
function Get-ServerInfo {
    [PSCustomObject]@{
        Server = "web-01"
        IP     = "10.0.1.10"
        Status = "Online"
    }
}

# Now you can do:
Get-ServerInfo | Where-Object { $_.Status -eq "Online" }
Get-ServerInfo | Export-Csv -Path "servers.csv"
Get-ServerInfo | ConvertTo-Json
```

### 6. Comment Your Scripts

```powershell
<#
.SYNOPSIS
    Deploys an application to the specified environment.

.DESCRIPTION
    This script downloads the application artifact, creates a backup
    of the current version, extracts the new version, and restarts
    the service.

.PARAMETER Environment
    Target environment: dev, staging, or production.

.PARAMETER Version
    Version number to deploy (e.g., "1.2.3").

.EXAMPLE
    .\deploy.ps1 -Environment staging -Version 1.2.3

.NOTES
    Author: DevOps Team
    Date:   2025-01-15
#>
```

This special comment format enables `Get-Help .\deploy.ps1` to show proper help!

```powershell
Get-Process | Sort-Object CPU -Descending | Select-Object -First 5
Get-Service | Where-Object { $_.Status -eq "Stopped" -and $_.StartType -eq "Automatic" }
Restart-Service -Name "W3SVC" -Force

# Remote management
Invoke-Command -ComputerName "Server01" -ScriptBlock { Get-Service }
Enter-PSSession -ComputerName "Server01"

# API calls
Invoke-RestMethod -Uri "https://api.example.com/health" -Method GET
Invoke-WebRequest -Uri "https://example.com/file.zip" -OutFile "file.zip"

# Data export
Get-Process | Export-Csv -Path "processes.csv" -NoTypeInformation
@{name="test"} | ConvertTo-Json | Set-Content "config.json"
```

## Key Takeaways

1. **PowerShell passes objects through the pipeline** — this is its killer feature over Bash/CMD.
2. **`Verb-Noun` naming** makes commands discoverable — use `Get-Command *keyword*` to find what you need.
3. **`Get-Help`** and **`Get-Member`** are your best friends — use them constantly.
4. **PowerShell 7 is cross-platform** — runs on Windows, Linux, and macOS.
5. **Always use full cmdlet names in scripts** — avoid aliases for readability and portability.
6. **Return objects from functions** — never return formatted strings.
7. **Use `try/catch` with `-ErrorAction Stop`** in automation scripts.
8. **PowerShell Remoting** lets you manage hundreds of servers from one machine.
9. **Modules extend PowerShell** — install `Az` for Azure, `AWS.Tools` for AWS, `Pester` for testing.
10. **Practice on real tasks** — automate something you do manually every day.

> 💡 **Pro tip:** PowerShell's real power shows when you combine the pipeline, objects, and remoting. Instead of SSHing into 50 servers one by one, you run a single `Invoke-Command` against all 50 in parallel. That's the DevOps way.
