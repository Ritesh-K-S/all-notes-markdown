## 🤔 What is an API and Why Should You Care?

Think of an API (Application Programming Interface) like a **waiter in a restaurant**:

- You (the script) want something (data, an action)
- The kitchen (the server/service) has what you need
- The waiter (the API) takes your **request** to the kitchen and brings back the **response**

You don't go into the kitchen yourself. You just talk to the waiter.

```
┌──────────────┐          Request           ┌──────────────┐
│              │  ────────────────────────→  │              │
│  Your Python │       (HTTP Request)        │   Server     │
│  Script      │                             │   (AWS,      │
│              │  ←────────────────────────  │    GitHub,   │
│              │          Response            │    K8s...)   │
└──────────────┘       (JSON Data)           └──────────────┘
```

## 📡 Part 1 — REST APIs (The Foundation)

### What is a REST API?

REST (Representational State Transfer) is the **most common type of API** on the internet. It uses standard HTTP methods:

| HTTP Method | What It Does | Example |
|-------------|-------------|---------|
| `GET` | **Read/Fetch** data | Get list of servers |
| `POST` | **Create** something new | Create a new server |
| `PUT` | **Update/Replace** something entirely | Replace server config |
| `PATCH` | **Partially update** something | Change server's name only |
| `DELETE` | **Delete** something | Delete a server |

Think of it like CRUD operations:
- **C**reate → POST
- **R**ead → GET
- **U**pdate → PUT/PATCH
- **D**elete → DELETE

### HTTP Status Codes You Must Know

When the API responds, it sends back a **status code** to tell you what happened:

| Code | Meaning | What To Do |
|------|---------|------------|
| `200` | ✅ OK — Success | Everything worked |
| `201` | ✅ Created | New resource was created |
| `204` | ✅ No Content | Success, but nothing to return |
| `400` | ❌ Bad Request | You sent wrong data — fix your request |
| `401` | ❌ Unauthorized | Wrong or missing API key/token |
| `403` | ❌ Forbidden | You don't have permission |
| `404` | ❌ Not Found | The resource doesn't exist |
| `429` | ❌ Too Many Requests | You're hitting the API too fast — slow down |
| `500` | ❌ Internal Server Error | Server messed up, not your fault |
| `503` | ❌ Service Unavailable | Server is overloaded or down |

```python
import requests

# Make a GET request to fetch a random joke
response = requests.get("https://official-joke-api.appspot.com/random_joke")
```

**Output:**

**Let's break this down line by line:**

| Line | What It Does |
|------|-------------|

| `response.status_code` | The HTTP status code (200 = success) |
| `response.json()` | Convert the JSON response to a Python dictionary |

### GET Request with Parameters

```python
params = {
```

### POST Request — Creating Something

POST requests **send data to create something new**:

```python
import requests

# Example: Create a new webhook (using httpbin for testing)
url = "https://httpbin.org/post"

# The data we want to send
payload = {
    "name": "my-webhook",
    "url": "https://myapp.com/webhook",
    "events": ["push", "pull_request"],
    "active": True
}

# Send POST request with JSON data
response = requests.post(url, json=payload)

print(f"Status Code: {response.status_code}")
print(f"Response: {response.json()['json']}")
```

**Key difference from GET:**
- `requests.get()` → retrieves data
- `requests.post(url, json=payload)` → sends data in the request body
- The `json=payload` parameter automatically:
  - Converts your Python dict to JSON
  - Sets the `Content-Type: application/json` header

| Header | What It's For |

| `Content-Type: application/json` | Tell API you're sending JSON |
| `Accept: application/json` | Tell API you want JSON back |
| `X-API-Key: <key>` | API key auth (some services) |

### Handling Errors Like a Pro

In DevOps, your scripts run **unattended** (cron jobs, CI/CD pipelines). You MUST handle errors properly:

```python
import requests
import sys

def make_api_call(url, method="GET", data=None, headers=None):
    """
    Make an API call with proper error handling.
    Returns the response data or exits with error.
    """
    try:
        if method == "GET":
            response = requests.get(url, headers=headers, timeout=30)
        elif method == "POST":
            response = requests.post(url, json=data, headers=headers, timeout=30)
        elif method == "PUT":
            response = requests.put(url, json=data, headers=headers, timeout=30)
        elif method == "DELETE":
            response = requests.delete(url, headers=headers, timeout=30)
        
        # Raise an exception for 4xx and 5xx status codes
        response.raise_for_status()
        
        # Return the JSON data
        if response.text:
            return response.json()
        return None
        
    except requests.exceptions.Timeout:
        print(f"ERROR: Request to {url} timed out after 30 seconds")
        sys.exit(1)
        
    except requests.exceptions.ConnectionError:
        print(f"ERROR: Could not connect to {url}. Check your network.")
        sys.exit(1)
        
    except requests.exceptions.HTTPError as e:
        print(f"ERROR: HTTP {response.status_code} — {e}")
        if response.status_code == 401:
            print("→ Check your API token/credentials")
        elif response.status_code == 403:
            print("→ You don't have permission for this action")
        elif response.status_code == 404:
            print("→ Resource not found — check the URL")
        elif response.status_code == 429:
            print("→ Rate limited — wait and try again")
        sys.exit(1)
        
    except requests.exceptions.RequestException as e:
        print(f"ERROR: Unexpected error — {e}")
        sys.exit(1)


# Usage
data = make_api_call("https://api.github.com/repos/torvalds/linux")
print(f"Linux repo has {data['stargazers_count']} stars")
```

**Let me explain the error handling pattern:**

| Exception | When It Happens |
|-----------|----------------|
| `Timeout` | Server took too long to respond |
| `ConnectionError` | Can't reach the server (DNS, network, firewall) |
| `HTTPError` | Server responded with 4xx or 5xx error |
| `RequestException` | Catch-all for anything else |

**The `timeout=30` parameter** is super important — without it, your script could **hang forever** waiting for a response. Always set a timeout!

**The `response.raise_for_status()`** method checks if the status code indicates an error (4xx or 5xx) and raises an `HTTPError` if it does.

### Retry Logic (Because Networks Fail)

In real DevOps, you **always** want retry logic. Networks are unreliable:

```python
import requests
import time

def api_call_with_retry(url, max_retries=3, backoff_factor=2, **kwargs):
    """
    Make an API call with automatic retries and exponential backoff.
    
    backoff_factor=2 means:
      - 1st retry waits 2 seconds
      - 2nd retry waits 4 seconds
      - 3rd retry waits 8 seconds
    """
    for attempt in range(max_retries + 1):
        try:
            response = requests.get(url, timeout=30, **kwargs)
            response.raise_for_status()
            return response.json()
            
        except (requests.exceptions.Timeout, 
                requests.exceptions.ConnectionError,
                requests.exceptions.HTTPError) as e:
            
            if attempt == max_retries:
                print(f"FAILED after {max_retries + 1} attempts: {e}")
                raise
            
            wait_time = backoff_factor ** (attempt + 1)
            print(f"Attempt {attempt + 1} failed: {e}")
            print(f"Retrying in {wait_time} seconds...")
            time.sleep(wait_time)


# Usage
data = api_call_with_retry("https://api.github.com/repos/torvalds/linux")
print(f"Stars: {data['stargazers_count']}")
```

**Exponential Backoff Explained:**

```
Attempt 1 fails → Wait 2 seconds  → Retry
Attempt 2 fails → Wait 4 seconds  → Retry  
Attempt 3 fails → Wait 8 seconds  → Retry
Attempt 4 fails → GIVE UP and raise error
```

Why? Because if the server is overloaded, hammering it with retries makes things **worse**. You wait longer each time to give the server a chance to recover.

### Sessions — Reusing Connections

If you're making **multiple API calls to the same server**, use a Session. It's faster because it **reuses the TCP connection**:

```python
import requests

# Without session (opens and closes connection for each request)
for i in range(10):
    requests.get("https://api.github.com/repos/torvalds/linux")  # Slow!

# With session (reuses connection) — MUCH faster
session = requests.Session()
session.headers.update({
    "Authorization": "Bearer your-token",
    "Accept": "application/json"
})

for i in range(10):
    response = session.get("https://api.github.com/repos/torvalds/linux")  # Fast!

# Always close the session when done
session.close()

# Or even better — use a context manager
with requests.Session() as session:
    session.headers.update({"Authorization": "Bearer your-token"})
    response = session.get("https://api.github.com/repos/torvalds/linux")
    print(response.json()["stargazers_count"])
```

**Why Sessions are better:**
1. **Reuses TCP connections** → faster
2. **Persists headers** → you don't repeat auth headers every time
3. **Persists cookies** → useful for cookie-based auth
4. **Cleaner code** → configure once, use many times

## ☁️ Part 3 — `boto3` (AWS SDK for Python)

### What is boto3?

`boto3` is the **official AWS SDK for Python**. It lets you manage **any AWS service** from Python code:

- Create/delete EC2 instances
- Upload files to S3
- Manage IAM users
- Work with DynamoDB, Lambda, CloudFormation, etc.

Think of it as **the AWS Console, but in Python code**.

### Installing boto3

```bash
pip install boto3
```

### Setting Up AWS Credentials

Before using boto3, you need AWS credentials. There are several ways:

**Method 1: AWS CLI (Recommended for local development)**
```bash
# Install AWS CLI first, then:
aws configure
```
It will ask for:
```
AWS Access Key ID: AKIA...............
AWS Secret Access Key: wJalr................
Default region name: us-east-1
Default output format: json
```

This saves credentials in `~/.aws/credentials` and config in `~/.aws/config`.

**Method 2: Environment Variables (Recommended for CI/CD)**
```bash
export AWS_ACCESS_KEY_ID="AKIA..............."
export AWS_SECRET_ACCESS_KEY="wJalr..............."
export AWS_DEFAULT_REGION="us-east-1"
```

**Method 3: IAM Roles (Recommended for EC2/ECS/Lambda)**
If your script runs on AWS (EC2, ECS, Lambda), attach an **IAM Role** to the service. boto3 picks it up automatically — no credentials needed in code!

> ⚠️ **NEVER hardcode credentials in your Python code.** That's a security disaster. Always use one of the methods above.

### boto3 Basics — Clients vs Resources

boto3 has two ways to interact with AWS:

```python
import boto3

# 1. CLIENT — Low-level, maps 1:1 to AWS API calls
#    Returns raw dictionaries
client = boto3.client('s3')

# 2. RESOURCE — High-level, object-oriented, easier to use
#    Returns Python objects with methods
resource = boto3.resource('s3')
```

| Feature | Client | Resource |
|---------|--------|----------|
| Level | Low-level | High-level |
| Returns | Dictionaries | Python objects |
| Covers all AWS APIs | ✅ Yes | ❌ Only some services |
| Easier to use | ❌ More verbose | ✅ Cleaner code |
| Best for | Full API access | Common operations |

**Rule of thumb:** Use **Resource** when available, fall back to **Client** for advanced stuff.

### Practical Example: Automated S3 Cleanup Script

Here's a real-world script you'd actually use in DevOps — **delete files older than 30 days from S3**:

```python
#!/usr/bin/env python3
"""
S3 Cleanup Script
Deletes objects older than a specified number of days from an S3 bucket.
Usage: python3 s3_cleanup.py --bucket my-logs-bucket --days 30 --prefix logs/
"""

import boto3
import argparse
from datetime import datetime, timezone

def cleanup_old_s3_objects(bucket_name, days, prefix=""):
    """Delete S3 objects older than specified days."""
    
    s3 = boto3.resource('s3')
    bucket = s3.Bucket(bucket_name)
    
    cutoff_date = datetime.now(timezone.utc) - timedelta(days=days)
    
    deleted_count = 0
    total_size = 0
    
    print(f"🔍 Scanning s3://{bucket_name}/{prefix}")
    print(f"📅 Deleting files older than {days} days (before {cutoff_date.strftime('%Y-%m-%d')})")
    print("-" * 60)
    
    for obj in bucket.objects.filter(Prefix=prefix):
        if obj.last_modified < cutoff_date:
            size_mb = obj.size / (1024 * 1024)
            print(f"  🗑️  Deleting: {obj.key} ({size_mb:.2f} MB) — {obj.last_modified.strftime('%Y-%m-%d')}")
            obj.delete()
            deleted_count += 1
            total_size += obj.size
    
    total_size_gb = total_size / (1024 ** 3)
    print("-" * 60)
    print(f"✅ Done! Deleted {deleted_count} files, freed {total_size_gb:.2f} GB")


if __name__ == "__main__":
    from datetime import timedelta
    
    parser = argparse.ArgumentParser(description="Clean up old S3 objects")
    parser.add_argument("--bucket", required=True, help="S3 bucket name")
```

### Create a Deployment

```python
from kubernetes import client, config

config.load_kube_config()
apps_v1 = client.AppsV1Api()

deployment = client.V1Deployment(
    metadata=client.V1ObjectMeta(name="nginx-deployment"),
    spec=client.V1DeploymentSpec(
        replicas=3,
        selector=client.V1LabelSelector(
            match_labels={"app": "nginx"}
        ),
        template=client.V1PodTemplateSpec(
            metadata=client.V1ObjectMeta(
                labels={"app": "nginx"}
            ),
            spec=client.V1PodSpec(
                containers=[
                    client.V1Container(
                        name="nginx",
                        image="nginx:1.25",
                        ports=[client.V1ContainerPort(container_port=80)],
                        resources=client.V1ResourceRequirements(
                            requests={"cpu": "100m", "memory": "128Mi"},
                            limits={"cpu": "250m", "memory": "256Mi"}
                        )
                    )
                ]
            )
        )
    )
)

# Create the deployment
result = apps_v1.create_namespaced_deployment(
    namespace="default",
    body=deployment
)

print(f"✅ Deployment '{result.metadata.name}' created with {result.spec.replicas} replicas!")
```

### Practical Example: Health Check Script

Here's a real-world script that checks the health of all pods in a namespace:

```python
#!/usr/bin/env python3
"""
Kubernetes Pod Health Checker
Checks all pods in a namespace and reports unhealthy ones.
"""

from kubernetes import client, config
import sys

def check_pod_health(namespace="default"):
    """Check health of all pods in a namespace."""
    
    config.load_kube_config()
    v1 = client.CoreV1Api()
    
    pods = v1.list_namespaced_pod(namespace)
    
    healthy = 0
    unhealthy = 0
    issues = []
    
    for pod in pods.items:
        pod_name = pod.metadata.name
        phase = pod.status.phase
        
        # Check if pod is running
        if phase != "Running":
            unhealthy += 1
            issues.append(f"❌ {pod_name} — Status: {phase}")
            continue
        
        # Check container statuses
        if pod.status.container_statuses:
            for cs in pod.status.container_statuses:
                if not cs.ready:
                    unhealthy += 1
                    issues.append(f"❌ {pod_name} — Container '{cs.name}' not ready")
                    break
                if cs.restart_count > 5:
                    issues.append(f"⚠️  {pod_name} — Container '{cs.name}' has {cs.restart_count} restarts!")
            else:
                healthy += 1
        else:
            unhealthy += 1
            issues.append(f"❌ {pod_name} — No container status available")
    
    # Print report
    total = healthy + unhealthy
    print(f"\n{'='*60}")
    print(f"  Pod Health Report — Namespace: {namespace}")
    print(f"{'='*60}")
    print(f"  Total Pods:    {total}")
    print(f"  Healthy:       {healthy} ✅")
    print(f"  Unhealthy:     {unhealthy} ❌")
    print(f"{'='*60}")
    
    if issues:
        print("\n📋 Issues Found:")
        for issue in issues:
            print(f"  {issue}")
    else:
        print("\n✅ All pods are healthy!")
    
    # Exit with error code if there are unhealthy pods
    if unhealthy > 0:
        sys.exit(1)


if __name__ == "__main__":
    namespace = sys.argv[1] if len(sys.argv) > 1 else "default"
    check_pod_health(namespace)
```

**Run it:**
```bash
python3 k8s_health_check.py default
python3 k8s_health_check.py monitoring
```

## 📬 Part 6 — Working with the Slack API (Sending Alerts)

Sending notifications to Slack is extremely common in DevOps — deployment alerts, monitoring notifications, incident alerts, etc.

### Using Slack Webhooks (Simplest Method)

```python
import requests
import json

# Get this URL from: Slack App → Incoming Webhooks → Add New Webhook
SLACK_WEBHOOK_URL = "https://hooks.slack.com/services/T00000000/B00000000/XXXXXXXXXXXXXXXXXXXXXXXX"

def send_slack_message(text, channel=None):
    """Send a simple text message to Slack."""
    payload = {"text": text}
    if channel:
        payload["channel"] = channel
    
    response = requests.post(SLACK_WEBHOOK_URL, json=payload)
    
    if response.status_code == 200:
        print("✅ Message sent to Slack!")
    else:
        print(f"❌ Failed to send: {response.status_code} — {response.text}")


# Simple message
send_slack_message("🚀 Deployment to production completed successfully!")

# Alert message
send_slack_message("🔥 ALERT: CPU usage on web-server-1 exceeds 90%!")
```

### Rich Formatted Messages (Blocks)

```python

                    }
```

```python
import os

API_TOKEN = os.environ.get("GITHUB_TOKEN")
```

## 📝 Summary — Quick Reference

| Library | What It Does | Install |
|---------|-------------|---------|
| `requests` | Make HTTP API calls (GET, POST, PUT, DELETE) | `pip install requests` |
| `boto3` | Interact with AWS services (S3, EC2, IAM, etc.) | `pip install boto3` |
| `kubernetes` | Interact with Kubernetes clusters | `pip install kubernetes` |

### When to Use What?

```
Need to talk to a REST API?          → requests
Need to manage AWS resources?        → boto3
Need to manage Kubernetes resources? → kubernetes client
Need to send Slack notifications?    → requests + Slack webhooks
Need to interact with GitHub/GitLab? → requests + their API
```

### Key Rules:

1. **Always set timeouts** — `requests.get(url, timeout=30)`
2. **Always handle errors** — try/except around API calls
3. **Always use retry logic** — networks fail, deal with it
4. **Never hardcode credentials** — use environment variables
5. **Always check status codes** — `response.raise_for_status()`
6. **Use sessions** for multiple calls to the same API
