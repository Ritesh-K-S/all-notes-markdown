## 🤔 Why Should a DevOps Engineer Learn Go?

Let's start with a mind-blowing fact. Look at what's written in Go:

| Tool | What It Is | Written In |
|------|-----------|------------|
| **Docker** | Container runtime | Go |
| **Kubernetes** | Container orchestration | Go |
| **Terraform** | Infrastructure as Code | Go |
| **Prometheus** | Monitoring & alerting | Go |
| **Grafana** | Dashboards & visualization | Go |
| **Helm** | Kubernetes package manager | Go |
| **Istio** | Service mesh | Go |
| **Consul** | Service discovery | Go |
| **Vault** | Secrets management | Go |
| **etcd** | Distributed key-value store | Go |
| **Traefik** | Reverse proxy | Go |
| **CockroachDB** | Distributed database | Go |

**See the pattern?** If you want to work in DevOps, you're literally surrounded by Go.

### Go vs Python — When to Use What?

| Feature | Python | Go |
|---------|--------|-----|
| **Speed** | Slow (interpreted) | Blazing fast (compiled) |
| **Binary** | Needs Python installed | Single binary, no dependencies |
| **Concurrency** | Threading (limited by GIL) | Goroutines (built-in, lightweight) |
| **Typing** | Dynamic (errors at runtime) | Static (errors at compile time) |
| **Learning Curve** | Easy | Moderate |
| **Best For** | Scripts, automation, APIs | CLI tools, microservices, system tools |
| **Cross-compilation** | Needs packaging (PyInstaller) | Built-in: `GOOS=linux go build` |

**The simple rule:**

```
Quick automation scripts, API calls, data processing → Use Python
CLI tools, performance-critical tools, system utilities → Use Go
Kubernetes operators, custom controllers → Use Go
```

## 🛠️ Part 1 — Installing Go

### On Linux (Ubuntu/Debian)

```bash
# Download the latest Go (check https://go.dev/dl/ for latest version)
wget https://go.dev/dl/go1.22.0.linux-amd64.tar.gz

# Remove any previous installation
sudo rm -rf /usr/local/go

# Extract to /usr/local
sudo tar -C /usr/local -xzf go1.22.0.linux-amd64.tar.gz

# Add Go to your PATH (add to ~/.bashrc or ~/.zshrc)
echo 'export PATH=$PATH:/usr/local/go/bin' >> ~/.bashrc
echo 'export GOPATH=$HOME/go' >> ~/.bashrc
echo 'export PATH=$PATH:$GOPATH/bin' >> ~/.bashrc
source ~/.bashrc

# Verify installation
go version
```

**Output:**
```
go version go1.22.0 linux/amd64
```

### On macOS

```bash
brew install go
```

### On Windows

Download the installer from [go.dev/dl](https://go.dev/dl/) and run it. Or use:
```powershell
winget install GoLang.Go
```

### Understanding GOPATH

```
$HOME/go/              ← GOPATH (your Go workspace)
├── bin/               ← Compiled binaries go here
├── pkg/               ← Package cache
└── src/               ← Your source code (older convention)
```

Modern Go uses **modules** instead of GOPATH, so you can put projects anywhere. But `$GOPATH/bin` is still where installed tools go.

## 📝 Part 2 — Go Basics (Everything You Need to Start)

### Your First Go Program

Create a file called `main.go`:

```go
package main

import "fmt"

func main() {
    fmt.Println("Hello, DevOps Engineer! 🚀")
}
```

**Run it:**
```bash
go run main.go
```

**Output:**
```
Hello, DevOps Engineer! 🚀
```

**Let's break it down:**

| Part | What It Means |
|------|-------------|
| `package main` | Every Go file belongs to a package. `main` is special — it's the entry point. |
| `import "fmt"` | Import the `fmt` package for printing (like Python's `print`). |
| `func main()` | The `main` function is where your program starts (like Python's `if __name__ == "__main__"`). |
| `fmt.Println()` | Print a line to the console. |

**Build it into a binary:**
```bash
go build -o hello main.go
./hello
```

Now you have a **single binary** — no Go installation needed to run it! You can copy this binary to any Linux machine and it just works.

### Variables & Data Types

```go
package main

import "fmt"

func main() {
    // ═══════════════════════════════════════
    // METHOD 1: Explicit type declaration
    // ═══════════════════════════════════════
    var name string = "Ritesh"
    var age int = 25
    var isDevOps bool = true
    var salary float64 = 75000.50

    fmt.Println(name, age, isDevOps, salary)

    // ═══════════════════════════════════════
    // METHOD 2: Short declaration (most common)
    // Go figures out the type automatically
    // ═══════════════════════════════════════
    city := "Bangalore"      // string
    experience := 3          // int
    rating := 4.8            // float64

    fmt.Println(city, experience, rating)

    // ═══════════════════════════════════════
    // CONSTANTS (values that never change)
    // ═══════════════════════════════════════
    const maxRetries = 3
    const appName = "my-devops-tool"

    fmt.Println(appName, "max retries:", maxRetries)
}
```

**Key difference from Python:** In Go, you must **declare the type** of every variable (or let Go infer it with `:=`). If you declare a variable but never use it, Go will give you a **compile error**. Go is strict!

**Common types in Go:**

| Type | Example | Python Equivalent |
|------|---------|------------------|
| `string` | `"hello"` | `str` |
| `int` | `42` | `int` |
| `float64` | `3.14` | `float` |
| `bool` | `true`, `false` | `True`, `False` |
| `[]string` | `["a", "b"]` | `list` |
| `map[string]int` | `{"a": 1}` | `dict` |
| `byte` | `'A'` | `bytes` |

### Strings

```go
package main

import (
    "fmt"
    "strings"
)

func main() {
    name := "DevOps Engineer"

    // String operations (similar to Python)

    version := "v2.3.1"
    env := "production"
    fmt.Printf("Deploying %s to %s\n", version, env)
    // Output: Deploying v2.3.1 to production

    // Sprintf returns a string (doesn't print)
    msg := fmt.Sprintf("Server %s is %s", "web-01", "healthy")
    fmt.Println(msg)
}
```

### If/Else & Loops

```go
package main

import "fmt"

func main() {
    // ═══════════════════════════════════════
    // IF/ELSE (no parentheses needed!)
    // ═══════════════════════════════════════
    cpuUsage := 85

    if cpuUsage >= 90 {
        fmt.Println("🔥 CRITICAL: CPU is extremely high!")
    } else if cpuUsage >= 75 {
        fmt.Println("⚠️  WARNING: CPU is getting high")
    } else {
        fmt.Println("✅ CPU is normal")
    }

    // Go has a cool feature: you can declare a variable in the if statement
    if status := checkHealth(); status == "healthy" {
        fmt.Println("Server is healthy!")
    }

    // ═══════════════════════════════════════
    // FOR LOOP (Go only has 'for' — no while!)
    // ═══════════════════════════════════════

    // Traditional for loop
    for i := 0; i < 5; i++ {
        fmt.Printf("Attempt %d\n", i+1)
    }

    // While-style loop (for without init and post)
    count := 0
    for count < 3 {
        fmt.Printf("Count: %d\n", count)
        count++
    }

    // Infinite loop (useful for servers/watchers)
    // for {
    //     // do something forever
    //     // use 'break' to exit
    // }

    // Range loop (iterate over slices, maps, strings)
    servers := []string{"web-01", "web-02", "web-03"}
    for index, server := range servers {
        fmt.Printf("  %d: %s\n", index, server)
    }

    // If you don't need the index, use _
    for _, server := range servers {
        fmt.Printf("  Checking: %s\n", server)
    }
}

func checkHealth() string {
    return "healthy"
}
```

**Key differences from Python:**
- Go uses `{` and `}` instead of indentation
- No parentheses around conditions: `if x > 5 {` not `if (x > 5) {`
- Go only has `for` — no `while` or `do-while`
- Use `_` to ignore a value you don't need (like Python)

### Functions

```go
package main

import (
    "fmt"
    "errors"
)

// ═══════════════════════════════════════
// BASIC FUNCTION
// ═══════════════════════════════════════
func greet(name string) string {
    return fmt.Sprintf("Hello, %s!", name)
}

// ═══════════════════════════════════════
// MULTIPLE RETURN VALUES (Go's superpower!)
// ═══════════════════════════════════════
// In Go, functions commonly return (result, error)
// This is THE Go pattern — you'll see it everywhere
func divide(a, b float64) (float64, error) {
    if b == 0 {
        return 0, errors.New("cannot divide by zero")
    }
    return a / b, nil  // nil means "no error"
}

// ═══════════════════════════════════════
// FUNCTION WITH MULTIPLE PARAMETERS OF SAME TYPE
// ═══════════════════════════════════════
func add(a, b int) int {
    return a + b
}

// ═══════════════════════════════════════
// VARIADIC FUNCTION (accepts any number of arguments)
// ═══════════════════════════════════════
func sum(numbers ...int) int {
    total := 0
    for _, n := range numbers {
        total += n
    }
    return total
}

func main() {
    // Call basic function
    msg := greet("Ritesh")
    fmt.Println(msg) // Hello, Ritesh!

    // Call function with multiple returns
    result, err := divide(10, 3)
    if err != nil {
        fmt.Println("Error:", err)
    } else {
        fmt.Printf("10 / 3 = %.2f\n", result) // 10 / 3 = 3.33
    }

    // Handle error case
    _, err = divide(10, 0)
    if err != nil {
        fmt.Println("Error:", err) // Error: cannot divide by zero
    }

    // Variadic function
    fmt.Println(sum(1, 2, 3, 4, 5)) // 15
}
```

**The (result, error) Pattern — THE Most Important Go Pattern:**

```go
// This pattern is EVERYWHERE in Go:
result, err := someFunction()
if err != nil {
    // Handle the error
    fmt.Println("Error:", err)
    return
}
// Use result safely
```

In Python, you use `try/except`. In Go, you **check errors explicitly**. It might seem verbose, but it forces you to **think about errors at every step** — which is exactly what you want in DevOps!

### Arrays, Slices & Maps

```go
package main

import "fmt"

func main() {
    // ═══════════════════════════════════════
    // SLICES (dynamic arrays — used 99% of the time)
    // ═══════════════════════════════════════
    
    // Create a slice
    servers := []string{"web-01", "web-02", "web-03"}
    fmt.Println("Servers:", servers)        // [web-01 web-02 web-03]
    fmt.Println("Count:", len(servers))     // 3
    fmt.Println("First:", servers[0])       // web-01
    fmt.Println("Last:", servers[len(servers)-1]) // web-03

    // Add elements (append)
    servers = append(servers, "web-04")
    servers = append(servers, "web-05", "web-06")
    fmt.Println("After append:", servers)

    // Slicing (like Python)
    firstTwo := servers[0:2]  // [web-01 web-02]
    fmt.Println("First two:", firstTwo)

    // ═══════════════════════════════════════
    // MAPS (like Python dictionaries)
    // ═══════════════════════════════════════
    
    // Create a map
    serverStatus := map[string]string{
        "web-01": "running",
        "web-02": "running",
        "web-03": "stopped",
    }

    // Access a value
    fmt.Println("web-01 status:", serverStatus["web-01"]) // running

    // Add/update a value
    serverStatus["web-04"] = "running"

    // Check if a key exists
    status, exists := serverStatus["web-05"]
    if exists {
        fmt.Println("web-05:", status)
    } else {
        fmt.Println("web-05 not found!")
    }

    // Delete a key
    delete(serverStatus, "web-03")

    // Loop over a map
    for server, status := range serverStatus {
        fmt.Printf("  %s → %s\n", server, status)
    }

    // ═══════════════════════════════════════
    // MAP OF SLICES (very common in DevOps)
    // ═══════════════════════════════════════
    environments := map[string][]string{
        "production": {"prod-web-01", "prod-web-02", "prod-web-03"},
        "staging":    {"stg-web-01"},
        "dev":        {"dev-web-01"},
    }

    for env, servers := range environments {
        fmt.Printf("  %s: %d servers → %v\n", env, len(servers), servers)
    }
}
```

### Structs (Like Python Classes, but simpler)

```go
package main

import "fmt"

// ═══════════════════════════════════════
// DEFINE A STRUCT (like a Python class)
// ═══════════════════════════════════════
type Server struct {
    Name       string
    IP         string
    Port       int
    IsHealthy  bool
    Tags       map[string]string
}

// ═══════════════════════════════════════
// METHOD ON A STRUCT (like a Python class method)
// ═══════════════════════════════════════
func (s Server) GetAddress() string {
    return fmt.Sprintf("%s:%d", s.IP, s.Port)
}

func (s *Server) SetHealthy(healthy bool) {
    s.IsHealthy = healthy  // Using pointer receiver to modify the struct
}

func main() {
    // Create a server
    web1 := Server{
        Name:      "web-01",
        IP:        "192.168.1.10",
        Port:      8080,
        IsHealthy: true,
        Tags: map[string]string{
            "environment": "production",
            "team":        "backend",
        },
    }

    fmt.Printf("Server: %s at %s\n", web1.Name, web1.GetAddress())
    fmt.Printf("Healthy: %v\n", web1.IsHealthy)
    fmt.Printf("Environment: %s\n", web1.Tags["environment"])

    // Modify using pointer method
    web1.SetHealthy(false)
    fmt.Printf("After update — Healthy: %v\n", web1.IsHealthy)

    // Create a slice of servers
    servers := []Server{
        {Name: "web-01", IP: "192.168.1.10", Port: 8080, IsHealthy: true},
        {Name: "web-02", IP: "192.168.1.11", Port: 8080, IsHealthy: true},
        {Name: "web-03", IP: "192.168.1.12", Port: 8080, IsHealthy: false},
    }

    for _, s := range servers {
        status := "✅"
        if !s.IsHealthy {
            status = "❌"
        }
        fmt.Printf("  %s %s (%s)\n", status, s.Name, s.GetAddress())
    }
}
```

**Pointer receivers vs Value receivers:**

| Type | Syntax | When To Use |
|------|--------|-------------|
| Value receiver | `func (s Server)` | When you only READ data |
| Pointer receiver | `func (s *Server)` | When you MODIFY data |

Think of `*Server` as "I'm working with the **actual** server, not a copy."

### Error Handling in Go

```go
package main

import (
    "errors"
    "fmt"
    "os"
    "strconv"
)

// ═══════════════════════════════════════
// CUSTOM ERROR TYPES
// ═══════════════════════════════════════
type ServerError struct {
    ServerName string
    Message    string
}

func (e *ServerError) Error() string {
    return fmt.Sprintf("server %s: %s", e.ServerName, e.Message)
}

// ═══════════════════════════════════════
// FUNCTION THAT RETURNS AN ERROR
// ═══════════════════════════════════════
func connectToServer(host string, port int) error {
    if host == "" {
        return errors.New("host cannot be empty")
    }
    if port <= 0 || port > 65535 {
        return fmt.Errorf("invalid port: %d (must be 1-65535)", port)
    }
    // Simulate connection failure
    if host == "bad-server" {
        return &ServerError{
            ServerName: host,
            Message:    "connection refused",
        }
    }
    return nil // No error = success
}

func main() {
    // ═══════════════════════════════════════
    // BASIC ERROR HANDLING
    // ═══════════════════════════════════════
    err := connectToServer("web-01", 8080)
    if err != nil {
        fmt.Println("Error:", err)
        os.Exit(1)
    }
    fmt.Println("Connected to web-01 ✅")

    // ═══════════════════════════════════════
    // ERROR WITH TYPE CHECKING
    // ═══════════════════════════════════════
    err = connectToServer("bad-server", 8080)
    if err != nil {
        var serverErr *ServerError
        if errors.As(err, &serverErr) {
            fmt.Printf("Server '%s' error: %s\n", serverErr.ServerName, serverErr.Message)
        } else {
            fmt.Println("Generic error:", err)
        }
    }

    // ═══════════════════════════════════════
    // WRAPPING ERRORS (adding context)
    // ═══════════════════════════════════════
    _, err = strconv.Atoi("not-a-number")
    if err != nil {
        wrappedErr := fmt.Errorf("failed to parse config value: %w", err)
        fmt.Println(wrappedErr)
        // Output: failed to parse config value: strconv.Atoi: parsing "not-a-number": invalid syntax
    }
}
```

### Goroutines & Concurrency (Go's Killer Feature)

This is **why** Go is perfect for DevOps tools. Goroutines let you do many things at the same time:

```go
package main

import (
    "fmt"
    "net/http"
    "sync"
    "time"
)

// ═══════════════════════════════════════
// CHECK MULTIPLE SERVERS CONCURRENTLY
// ═══════════════════════════════════════
func checkServer(url string, wg *sync.WaitGroup, results chan<- string) {
    defer wg.Done() // Tell WaitGroup this goroutine is done when function exits

    start := time.Now()
    
    resp, err := http.Get(url)
    duration := time.Since(start)
    
    if err != nil {
        results <- fmt.Sprintf("❌ %s — DOWN (%v)", url, err)
        return
    }
    defer resp.Body.Close()
    
    results <- fmt.Sprintf("✅ %s — %d (%v)", url, resp.StatusCode, duration.Round(time.Millisecond))
}

func main() {
    urls := []string{
        "https://google.com",
        "https://github.com",
        "https://stackoverflow.com",
        "https://go.dev",
        "https://kubernetes.io",
        "https://this-does-not-exist-12345.com",
    }

    var wg sync.WaitGroup
    results := make(chan string, len(urls))

    fmt.Println("🔍 Checking servers concurrently...")
    start := time.Now()

    // Launch all checks at the same time (goroutines)
    for _, url := range urls {
        wg.Add(1)
        go checkServer(url, &wg, results)  // 'go' keyword starts a goroutine
    }

    // Wait for all goroutines to finish
    go func() {
        wg.Wait()
        close(results)
    }()

    // Print results as they come in
    for result := range results {
        fmt.Println("  ", result)
    }

    fmt.Printf("\n⏱️  Total time: %v\n", time.Since(start).Round(time.Millisecond))
    fmt.Println("(Checking 6 servers sequentially would take ~18 seconds)")
}
```

**Output:**
```
🔍 Checking servers concurrently...
  ✅ https://go.dev — 200 (142ms)
  ✅ https://google.com — 200 (189ms)
  ✅ https://github.com — 200 (234ms)
  ✅ https://stackoverflow.com — 200 (312ms)
  ✅ https://kubernetes.io — 200 (287ms)
  ❌ https://this-does-not-exist-12345.com — DOWN (no such host)

⏱️  Total time: 315ms
(Checking 6 servers sequentially would take ~18 seconds)
```

**Let's break down the concurrency pieces:**

| Concept | What It Is | Analogy |
|---------|-----------|---------|
| `go func()` | Start a **goroutine** (lightweight thread) | "Go do this task in the background" |
| `sync.WaitGroup` | Counter to track running goroutines | "Tell me when all workers are done" |
| `chan` (channel) | A pipe to send data between goroutines | "Pass results through this tube" |
| `wg.Add(1)` | "One more worker is starting" | +1 to the counter |
| `wg.Done()` | "I'm finished" | -1 from the counter |
| `wg.Wait()` | "Block until counter is 0" | Wait for all workers |

**Why this matters for DevOps:**
- Checking 100 servers? Do it in **parallel**, not one by one
- Downloading files from 10 URLs? Start all downloads at once
- Running health checks? Check everything simultaneously

## 🔨 Part 3 — Building CLI Tools with Go

### Why Go for CLI Tools?

1. **Single binary** — no runtime needed, just copy and run
2. **Cross-compilation** — build for Linux, Mac, Windows from one machine
3. **Fast startup** — no interpreter warmup
4. **Flag parsing** — built into the standard library

### A Simple CLI Tool: Server Pinger

```go
package main

import (
    "flag"
    "fmt"
    "net/http"
    "os"
    "sync"
    "time"
)

func main() {
    // ═══════════════════════════════════════
    // COMMAND-LINE FLAGS (like argparse in Python)
    // ═══════════════════════════════════════
    timeout := flag.Int("timeout", 5, "Request timeout in seconds")
    concurrent := flag.Int("concurrent", 10, "Max concurrent checks")
    verbose := flag.Bool("verbose", false, "Show detailed output")
    flag.Parse()

    // flag.Args() returns non-flag arguments
    urls := flag.Args()
    if len(urls) == 0 {
        fmt.Println("Usage: pinger [options] <url1> <url2> ...")
        fmt.Println("Options:")
        flag.PrintDefaults()
        os.Exit(1)
    }

    client := &http.Client{
        Timeout: time.Duration(*timeout) * time.Second,
    }

    var wg sync.WaitGroup
    semaphore := make(chan struct{}, *concurrent) // Limit concurrency

    fmt.Printf("Checking %d URLs (timeout: %ds, concurrency: %d)\n\n", len(urls), *timeout, *concurrent)

    for _, url := range urls {
        wg.Add(1)
        go func(url string) {
            defer wg.Done()
            semaphore <- struct{}{}        // Acquire semaphore slot
            defer func() { <-semaphore }() // Release when done

            start := time.Now()
            resp, err := client.Get(url)
            duration := time.Since(start)

            if err != nil {
                fmt.Printf("❌ %-40s DOWN  (%v)\n", url, err)
                return
            }
            defer resp.Body.Close()

            if *verbose {
                fmt.Printf("✅ %-40s %d  %v  (Content-Length: %d)\n",
                    url, resp.StatusCode, duration.Round(time.Millisecond), resp.ContentLength)
            } else {
                fmt.Printf("✅ %-40s %d  %v\n",
                    url, resp.StatusCode, duration.Round(time.Millisecond))
            }
        }(url)
    }

    wg.Wait()
}
```

**Build and use it:**
```bash
# Build the binary
go build -o pinger main.go

# Use it!
./pinger https://google.com https://github.com https://kubernetes.io

# With options
./pinger --timeout 10 --concurrent 5 --verbose https://google.com https://github.com
```

`cobra` is the library used by **kubectl, docker, helm, gh** and almost every major Go CLI tool. Let's build a real DevOps CLI tool:


**`cmd/health.go`** — Health check subcommand:
```go
package cmd

import (
    "fmt"
    "net/http"
    "sync"
    "time"

    "github.com/spf13/cobra"
)

var (
    healthTimeout    int
    healthNamespace  string
)

var healthCmd = &cobra.Command{
    Use:   "health [urls...]",
    Short: "Check health of services",
    Long:  "Check the health of one or more services by hitting their health endpoints.",
    Args:  cobra.MinimumNArgs(1),
    Run:   runHealthCheck,
}

func init() {
    // Add flags to the health command
    healthCmd.Flags().IntVarP(&healthTimeout, "timeout", "t", 5, "Request timeout in seconds")
    healthCmd.Flags().StringVarP(&healthNamespace, "namespace", "n", "default", "Kubernetes namespace")
    
    // Register the command under root
    rootCmd.AddCommand(healthCmd)
}

func runHealthCheck(cmd *cobra.Command, args []string) {
    client := &http.Client{
        Timeout: time.Duration(healthTimeout) * time.Second,
    }

    var wg sync.WaitGroup
    
    fmt.Println("🏥 Health Check Results:")
    fmt.Println("─────────────────────────────────────────────────────")

    for _, url := range args {
        wg.Add(1)
        go func(url string) {
            defer wg.Done()
            
            start := time.Now()
            resp, err := client.Get(url)
            duration := time.Since(start)
            
            if err != nil {
                fmt.Printf("  ❌ %-40s FAIL  (%v)\n", url, err)
                return
            }
            defer resp.Body.Close()
            
            status := "PASS"
            icon := "✅"
            if resp.StatusCode >= 400 {
                status = "FAIL"
                icon = "❌"
            }
            
            fmt.Printf("  %s %-40s %s  %d  %v\n", icon, url, status, resp.StatusCode, duration.Round(time.Millisecond))
        }(url)
    }

    wg.Wait()
    fmt.Println("─────────────────────────────────────────────────────")
}
```

## 📦 Part 4 — Cross-Compilation (Build for Any OS)

One of Go's superpowers — build for **any OS and architecture** from your machine:

```bash
# Build for Linux (most common — your servers run Linux)
GOOS=linux GOARCH=amd64 go build -o devops-cli-linux .

# Build for macOS
GOOS=darwin GOARCH=amd64 go build -o devops-cli-mac .

# Build for macOS (Apple Silicon / M1/M2/M3)
GOOS=darwin GOARCH=arm64 go build -o devops-cli-mac-arm .

# Build for Windows
GOOS=windows GOARCH=amd64 go build -o devops-cli.exe .

# Build for ARM (Raspberry Pi, AWS Graviton)
GOOS=linux GOARCH=arm64 go build -o devops-cli-arm .
```

**That's it!** No Docker, no virtual machines, no cross-compilation toolchain. Just set two environment variables and build. The output is a **single binary** — copy it to the target machine and run it.

```bash
# Copy to a remote server and run
scp devops-cli-linux user@server:/usr/local/bin/devops-cli
ssh user@server "devops-cli health https://localhost:8080/health"
```

## 📂 Part 5 — Working with Files & JSON in Go

Since DevOps is all about config files, logs, and JSON data:

```go
package main

import (
    "encoding/json"
    "fmt"
    "os"
)

// Define config structure
type AppConfig struct {
    AppName     string   `json:"app_name"`
    Version     string   `json:"version"`
    Port        int      `json:"port"`
    Debug       bool     `json:"debug"`
    AllowedIPs  []string `json:"allowed_ips"`
    Database    DBConfig `json:"database"`
}

type DBConfig struct {
    Host     string `json:"host"`
    Port     int    `json:"port"`
    Name     string `json:"name"`
    User     string `json:"user"`
}

func main() {
    // ═══════════════════════════════════════
    // READ JSON CONFIG FILE
    // ═══════════════════════════════════════
    data, err := os.ReadFile("config.json")
    if err != nil {
        fmt.Printf("Error reading config: %v\n", err)
        os.Exit(1)
    }

    var config AppConfig
    err = json.Unmarshal(data, &config)
    if err != nil {
        fmt.Printf("Error parsing JSON: %v\n", err)
        os.Exit(1)
    }

    fmt.Printf("App: %s v%s (port %d)\n", config.AppName, config.Version, config.Port)
    fmt.Printf("Database: %s@%s:%d/%s\n", config.Database.User, config.Database.Host, config.Database.Port, config.Database.Name)

    // ═══════════════════════════════════════
    // MODIFY AND WRITE JSON
    // ═══════════════════════════════════════
    config.Version = "v2.4.0"
    config.Debug = false

    output, err := json.MarshalIndent(config, "", "  ")
    if err != nil {
        fmt.Printf("Error creating JSON: %v\n", err)
        os.Exit(1)
    }

    err = os.WriteFile("config_updated.json", output, 0644)
    if err != nil {
        fmt.Printf("Error writing file: %v\n", err)
        os.Exit(1)
    }

    fmt.Println("✅ Config updated and saved!")

    // ═══════════════════════════════════════
    // READ A TEXT FILE (like a log file)
    // ═══════════════════════════════════════
    content, err := os.ReadFile("/var/log/syslog")
    if err != nil {
        fmt.Println("Error:", err)
        return
    }
    fmt.Printf("Log file size: %d bytes\n", len(content))
    
    // For large files, read line by line:
    // file, _ := os.Open("large_log.txt")
    // scanner := bufio.NewScanner(file)
    // for scanner.Scan() {
    //     line := scanner.Text()
    //     // process each line
    // }
}
```

**The `config.json` file this reads:**
```json
{
    "app_name": "my-service",
    "version": "v2.3.1",
    "port": 8080,
    "debug": true,
    "allowed_ips": ["10.0.0.0/8", "192.168.0.0/16"],
```

## 🧪 Part 6 — Running External Commands from Go

DevOps tools often need to run shell commands (like `kubectl`, `docker`, `terraform`):

```go
package main

import (
    "fmt"
    "os"
    "os/exec"
    "strings"
)

func main() {
    // ═══════════════════════════════════════
    // RUN A SIMPLE COMMAND
    // ═══════════════════════════════════════
    out, err := exec.Command("hostname").Output()
    if err != nil {
        fmt.Println("Error:", err)
        os.Exit(1)
    }
    fmt.Printf("Hostname: %s", out)

    // ═══════════════════════════════════════
    // COMMAND WITH ARGUMENTS
    // ═══════════════════════════════════════
    out, err = exec.Command("df", "-h", "/").Output()
    if err != nil {
        fmt.Println("Error:", err)
        os.Exit(1)
    }
    fmt.Printf("Disk Usage:\n%s\n", out)

    // ═══════════════════════════════════════
    // RUN KUBECTL
    // ═══════════════════════════════════════
    out, err = exec.Command("kubectl", "get", "pods", "-n", "default", "-o", "json").Output()
    if err != nil {
        fmt.Printf("kubectl error: %v\n", err)
    } else {
        fmt.Printf("Pods JSON (first 200 chars): %.200s...\n", out)
    }

    // ═══════════════════════════════════════
    // COMMAND WITH COMBINED OUTPUT (stdout + stderr)
    // ═══════════════════════════════════════
    cmd := exec.Command("docker", "ps", "--format", "{{.Names}}\t{{.Status}}")
    output, err := cmd.CombinedOutput()
    if err != nil {
        fmt.Printf("Docker error: %v\nOutput: %s\n", err, output)
    } else {
        fmt.Println("Running containers:")
        for _, line := range strings.Split(string(output), "\n") {
            if line != "" {
                fmt.Printf("  %s\n", line)
            }
        }
    }

    // ═══════════════════════════════════════
    // PIPE COMMANDS (like 'ps aux | grep nginx')
    // ═══════════════════════════════════════
    psCmd := exec.Command("ps", "aux")
    grepCmd := exec.Command("grep", "nginx")
    
    // Connect ps output to grep input
    grepCmd.Stdin, _ = psCmd.StdoutPipe()
    grepCmd.Stdout = os.Stdout
    
    grepCmd.Start()
    psCmd.Run()
    grepCmd.Wait()
}
```

## 🌐 Part 7 — Making HTTP Requests in Go

Building API clients is a core DevOps task:

```go
package main

import (
    "encoding/json"
    "fmt"
    "io"
    "net/http"
    "time"
)

// GitHub Repository structure
type GitHubRepo struct {
    FullName    string `json:"full_name"`
    Description string `json:"description"`
    Stars       int    `json:"stargazers_count"`
    Forks       int    `json:"forks_count"`
    Language    string `json:"language"`
}

func main() {
    // Create an HTTP client with timeout
    client := &http.Client{
        Timeout: 10 * time.Second,
    }

    // ═══════════════════════════════════════
    // GET REQUEST
    // ═══════════════════════════════════════
    resp, err := client.Get("https://api.github.com/repos/kubernetes/kubernetes")
    if err != nil {
        fmt.Printf("Error: %v\n", err)
        return
    }
    defer resp.Body.Close()

    // Read the response body
    body, err := io.ReadAll(resp.Body)
    if err != nil {
        fmt.Printf("Error reading body: %v\n", err)
        return
    }

    // Parse JSON into struct
    var repo GitHubRepo
    err = json.Unmarshal(body, &repo)
    if err != nil {
        fmt.Printf("Error parsing JSON: %v\n", err)
        return
    }

    fmt.Printf("📦 %s\n", repo.FullName)
    fmt.Printf("📝 %s\n", repo.Description)
    fmt.Printf("⭐ Stars: %d\n", repo.Stars)
    fmt.Printf("🍴 Forks: %d\n", repo.Forks)
    fmt.Printf("💻 Language: %s\n", repo.Language)
}
```

## 🧠 Summary — What You Learned

1. **Why Go for DevOps** — Docker, K8s, Terraform are all Go. It compiles to single binaries, has built-in concurrency, and cross-compiles easily.

2. **Go Basics** — Variables, types, strings, if/else, loops, functions, slices, maps, structs.

3. **Error Handling** — The `(result, error)` pattern and `if err != nil {}` is the core of Go.

4. **Goroutines** — Lightweight concurrent execution with `go func()`, `sync.WaitGroup`, and channels.

5. **CLI Tools** — Using `flag` for simple CLIs and `cobra` for production-grade CLIs (like kubectl).

6. **Cross-Compilation** — Build for any OS with `GOOS` and `GOARCH`.

7. **File & JSON Handling** — Reading configs, parsing JSON, writing files.

8. **Running Commands** — Using `exec.Command()` to run shell commands.

### When to Use Go vs Python

```
Write Go when:
  ✅ Building CLI tools that will be distributed
  ✅ Need high performance (millions of operations)
  ✅ Need concurrency (checking hundreds of servers)
  ✅ Building microservices
  ✅ Building Kubernetes operators/controllers
  ✅ Want single binary deployment

Write Python when:
  ✅ Quick automation scripts
  ✅ Working with APIs (boto3, requests)
  ✅ Data processing and analysis
  ✅ Prototyping and experimentation
  ✅ CI/CD pipeline scripts
  ✅ Config management scripts
```
