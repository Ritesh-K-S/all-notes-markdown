## 1. Why Build Custom Tools with Go?

You might be wondering: *"I can already write scripts in Bash and Python — why would I need Go?"*

Here's the deal — Bash and Python are perfect for quick automations and glue code. But when you need to build **production-grade tools** that are fast, reliable, and easy to distribute — Go is king.

### Look at what's built in Go:

```
Tool                    What It Does
─────────────────       ─────────────────────────────────
Docker                  Container runtime
Kubernetes              Container orchestration
Terraform               Infrastructure as Code
Prometheus              Monitoring & alerting
Grafana (backend)       Dashboard & observability
Helm                    Kubernetes package manager
etcd                    Distributed key-value store
Istio                   Service mesh
Vault                   Secrets management
Consul                  Service discovery
Argo CD                 GitOps for Kubernetes
```

That's basically the **entire DevOps ecosystem** — built in Go. Not Python. Not Java. Not Rust. **Go.**

### Why Go Dominates DevOps Tooling

| Feature | Why It Matters for DevOps |
|---------|--------------------------|
| **Single binary output** | No dependencies, no runtime needed. Just copy the binary and run. |
| **Cross-compilation** | Build for Linux, Mac, Windows, ARM from your laptop. |
| **Fast startup** | Milliseconds to start (unlike JVM-based tools). |
| **Built-in concurrency** | Goroutines make it easy to handle many tasks at once. |
| **Strong standard library** | HTTP servers, JSON parsing, file handling — all built in. |
| **Static typing** | Catch bugs at compile time, not at 3 AM in production. |
| **Great Kubernetes support** | Official Go client libraries for Kubernetes API. |

### The Simple Way to Think About It

```
Bash   → Quick scripts, small automations (< 100 lines)
Python → Medium automations, API scripts, data processing
Go     → Production-grade tools, CLI utilities, operators, exporters

Think of it like transportation:
Bash   = Walking (fine for short distances)
Python = Bicycle (great for medium trips)
Go     = Car (fast, reliable, goes anywhere)
```

## 2. Recap — Go Basics You Need

Before we start building, let's make sure you're comfortable with the Go concepts we'll use heavily. (If you've already done section 5.6, this will be a quick refresher.)

### Go Project Structure

```
my-tool/
├── go.mod           # Module definition (like package.json)
├── go.sum           # Dependency checksums (like package-lock.json)
├── main.go          # Entry point
├── cmd/             # CLI commands
│   ├── root.go
│   ├── version.go
│   └── deploy.go
├── pkg/             # Reusable packages
│   ├── health/
│   │   └── checker.go
│   └── config/
│       └── loader.go
└── internal/        # Internal packages (not importable by others)
    └── utils/
        └── helpers.go
```

### Key Concepts We'll Use

```go
// 1. Structs — Go's way of defining data structures (like classes in other languages)
type Server struct {
    Name    string
    Address string
    Port    int
    Healthy bool
}

// 2. Interfaces — Define what methods a type must have
type HealthChecker interface {
    Check() (bool, error)
}

// 3. Error handling — Go doesn't have exceptions, it returns errors explicitly
result, err := doSomething()
if err != nil {
    // handle the error
    log.Fatalf("Something went wrong: %v", err)
}

// 4. Goroutines — Lightweight concurrency (start a function in parallel)
go func() {
    checkHealth(server)
}()

// 5. Channels — Communication between goroutines
results := make(chan bool, 10)
go func() {
    results <- true  // send a value into the channel
}()
value := <-results   // receive a value from the channel
```

## 3. Building CLI Tools with Cobra

**Cobra** is THE library for building CLI tools in Go. Almost every popular Go CLI tool uses it — `kubectl`, `helm`, `hugo`, `gh` (GitHub CLI), and many more.

### What Cobra Gives You

```
Feature                     Example
──────────                  ──────────────────────
Commands                    mytool deploy, mytool status
Subcommands                 mytool cluster create
Flags                       mytool deploy --env production --dry-run
Auto-generated help         mytool --help, mytool deploy --help
Shell completions           Tab completion in bash/zsh/fish
Man page generation         Auto-generates man pages
```

### Installing Cobra

```bash
# Initialize a new Go module
mkdir mytool && cd mytool
go mod init github.com/yourusername/mytool

# Add Cobra as a dependency
go get github.com/spf13/cobra@latest

# (Optional) Install the Cobra CLI generator
go install github.com/spf13/cobra-cli@latest
```

### Cobra Structure — How It Works

```
Every Cobra app has 3 key concepts:

1. Commands   → The actions (deploy, status, rollback)
2. Args       → The targets (deploy <app-name>)
3. Flags      → The options (deploy --env staging)

mytool deploy my-app --env staging --replicas 3
│       │       │         │              │
│       │       │         │              └── Flag: replicas=3
│       │       │         └── Flag: env=staging
│       │       └── Arg: my-app
│       └── Command: deploy
└── Root command: mytool
```

## 4. Building a Real CLI Tool — Step by Step

Let's build a real DevOps CLI tool called `srvcheck` — a tool that checks the health of multiple servers concurrently.

### Step 1: Initialize the Project

```bash
mkdir srvcheck && cd srvcheck
go mod init github.com/yourusername/srvcheck
go get github.com/spf13/cobra@latest
```

### Step 2: Create the Root Command

```go
// File: cmd/root.go
package cmd

import (
    "fmt"
    "os"

    "github.com/spf13/cobra"
)

// rootCmd is the base command — what happens when you type just "srvcheck"
var rootCmd = &cobra.Command{
    Use:   "srvcheck",                              // How to use this command
    Short: "A DevOps server health check tool",      // One-line description
    Long: `srvcheck is a fast, concurrent server health checker
built for DevOps engineers. It checks HTTP endpoints,
TCP ports, and DNS resolution across multiple servers.`, // Detailed description
}

// Execute runs the root command — called from main.go
func Execute() {
    if err := rootCmd.Execute(); err != nil {
        fmt.Fprintf(os.Stderr, "Error: %s\n", err)
        os.Exit(1)
    }
}
```

**What's happening here?**
- We define a `rootCmd` using `cobra.Command`. This is the base of our tool.
- `Use` tells Cobra the command name.
- `Short` is the one-liner shown in `--help`.
- `Long` is the detailed description.
- `Execute()` is the entry point that Cobra calls to run the whole command tree.

### Step 3: Create the Main Entry Point

```go
// File: main.go
package main

import "github.com/yourusername/srvcheck/cmd"

func main() {
    cmd.Execute()
}
```

**That's it!** The `main.go` file is tiny. All the real logic lives in the `cmd/` package. This is the standard Cobra pattern.

### Step 4: Create the "check" Command

```go
// File: cmd/check.go
package cmd

import (
    "fmt"
    "net/http"
    "sync"
    "time"

    "github.com/spf13/cobra"
)

// Flags — these hold the values the user passes via --timeout, --verbose, etc.
var (
    timeout int
    verbose bool
)

// checkCmd represents the "srvcheck check" command
var checkCmd = &cobra.Command{
    Use:   "check [urls...]",                           // Usage pattern
    Short: "Check health of one or more HTTP endpoints", // One-liner
    Long: `Check performs HTTP GET requests to the specified URLs 
and reports whether each endpoint is healthy (returns 2xx status).

Example:
  srvcheck check https://google.com https://github.com
  srvcheck check https://myapi.com/health --timeout 10`,
    Args: cobra.MinimumNArgs(1), // Require at least 1 URL
    Run: func(cmd *cobra.Command, args []string) {
        // args contains the URLs passed by the user
        checkEndpoints(args)
    },
}

// init() is called automatically when the package loads
// We use it to register this command and its flags
func init() {
    // Add "check" as a subcommand of the root command
    rootCmd.AddCommand(checkCmd)

    // Define flags for the check command
    // --timeout with default of 5 seconds
    checkCmd.Flags().IntVarP(&timeout, "timeout", "t", 5, "Timeout in seconds for each check")
    // --verbose flag (boolean, defaults to false)
    checkCmd.Flags().BoolVarP(&verbose, "verbose", "v", false, "Show detailed output")
}

// checkEndpoints performs concurrent health checks on all provided URLs
func checkEndpoints(urls []string) {
    // WaitGroup keeps track of how many goroutines are still running
    var wg sync.WaitGroup

    // Create an HTTP client with our timeout
    client := &http.Client{
        Timeout: time.Duration(timeout) * time.Second,
    }

    fmt.Printf("🔍 Checking %d endpoint(s) with %ds timeout...\n\n", len(urls), timeout)

    // Loop through each URL and check it concurrently
    for _, url := range urls {
        wg.Add(1) // Tell WaitGroup: "one more goroutine is starting"

        // Launch a goroutine for each URL (concurrent check)
        go func(u string) {
            defer wg.Done() // Tell WaitGroup: "this goroutine is done"

            start := time.Now()           // Record start time
            resp, err := client.Get(u)     // Make the HTTP GET request
            duration := time.Since(start)  // Calculate how long it took

            if err != nil {
                // Request failed (timeout, DNS error, connection refused, etc.)
                fmt.Printf("  ❌ FAIL  %s\n", u)
                if verbose {
                    fmt.Printf("          Error: %v\n", err)
                }
                return
            }
            defer resp.Body.Close() // Always close the response body

            // Check if the status code is in the 2xx range (success)
            if resp.StatusCode >= 200 && resp.StatusCode < 300 {
                fmt.Printf("  ✅ OK    %s [%d] (%s)\n", u, resp.StatusCode, duration.Round(time.Millisecond))
            } else {
                fmt.Printf("  ⚠️  WARN  %s [%d] (%s)\n", u, resp.StatusCode, duration.Round(time.Millisecond))
            }
        }(url)
    }

    wg.Wait() // Wait for ALL goroutines to finish
    fmt.Println("\n✅ Done!")
}
```

**Let's break down the important parts:**

- **`sync.WaitGroup`**: This is how Go tracks concurrent work. We call `wg.Add(1)` before starting a goroutine, and `wg.Done()` when it finishes. `wg.Wait()` blocks until all goroutines are done.
- **`go func(u string) { ... }(url)`**: This launches the function as a goroutine (runs in parallel). The `(url)` at the end passes the URL as an argument to avoid closure issues.
- **`defer resp.Body.Close()`**: In Go, you MUST close HTTP response bodies. `defer` means "do this when the function returns."

## 5. Working with REST APIs in Go

DevOps tools constantly interact with APIs — GitHub, AWS, Kubernetes, monitoring systems, etc. Go makes this very clean.

### Making GET Requests

```go
package main

import (
    "encoding/json"
    "fmt"
    "io"
    "log"
    "net/http"
)

// Define a struct that matches the JSON response
type GitHubUser struct {
    Login     string `json:"login"`       // `json:"login"` maps JSON field to struct field
    Name      string `json:"name"`
    Company   string `json:"company"`
    PublicRepos int  `json:"public_repos"`
}

func main() {
    // Make a GET request to the GitHub API
    resp, err := http.Get("https://api.github.com/users/torvalds")
    if err != nil {
        log.Fatalf("Failed to fetch: %v", err)
    }
    defer resp.Body.Close() // Always close the body!

    // Read the response body
    body, err := io.ReadAll(resp.Body)
    if err != nil {
        log.Fatalf("Failed to read body: %v", err)
    }

    // Parse JSON into our struct
    var user GitHubUser
    err = json.Unmarshal(body, &user)  // &user means "put the result into user"
    if err != nil {
        log.Fatalf("Failed to parse JSON: %v", err)
    }

    // Use the data!
    fmt.Printf("User: %s (%s)\n", user.Login, user.Name)
    fmt.Printf("Company: %s\n", user.Company)
    fmt.Printf("Public repos: %d\n", user.PublicRepos)
}
```

**Key concepts:**
- **`json:"field_name"`** — These are called struct tags. They tell Go how to map JSON fields to struct fields.
- **`json.Unmarshal`** — Converts a JSON byte array into a Go struct.
- **`defer resp.Body.Close()`** — Always close the response body to prevent memory leaks.

### Making POST Requests with Headers

```go
package main

import (
    "bytes"
    "encoding/json"
    "fmt"
    "io"
    "log"
    "net/http"
)

type DeployRequest struct {
    AppName     string `json:"app_name"`
    Environment string `json:"environment"`
    Version     string `json:"version"`
}

type DeployResponse struct {
    Status    string `json:"status"`
    DeployID  string `json:"deploy_id"`
    Message   string `json:"message"`
}

func triggerDeploy(apiURL, token string) (*DeployResponse, error) {
    // Create the request body
    reqBody := DeployRequest{
        AppName:     "my-web-app",
        Environment: "staging",
        Version:     "v2.1.0",
    }

    // Convert struct to JSON bytes
    jsonBody, err := json.Marshal(reqBody)
    if err != nil {
        return nil, fmt.Errorf("failed to marshal request: %w", err)
    }

    // Create an HTTP request (not just a simple Get/Post — we need to set headers)
    req, err := http.NewRequest("POST", apiURL+"/api/deploy", bytes.NewBuffer(jsonBody))
    if err != nil {
        return nil, fmt.Errorf("failed to create request: %w", err)
    }

    // Set headers
    req.Header.Set("Content-Type", "application/json")
    req.Header.Set("Authorization", "Bearer "+token) // Auth token

    // Send the request
    client := &http.Client{}
    resp, err := client.Do(req)
    if err != nil {
        return nil, fmt.Errorf("request failed: %w", err)
    }
    defer resp.Body.Close()

    // Read response
    body, err := io.ReadAll(resp.Body)
    if err != nil {
        return nil, fmt.Errorf("failed to read response: %w", err)
    }

    // Check status code
    if resp.StatusCode != http.StatusOK {
        return nil, fmt.Errorf("deploy failed with status %d: %s", resp.StatusCode, string(body))
    }

    // Parse response
    var deployResp DeployResponse
    err = json.Unmarshal(body, &deployResp)
    if err != nil {
        return nil, fmt.Errorf("failed to parse response: %w", err)
    }

    return &deployResp, nil
}

func main() {
    resp, err := triggerDeploy("https://deploy.example.com", "my-secret-token")
    if err != nil {
        log.Fatalf("Deploy failed: %v", err)
    }
    fmt.Printf("Deploy %s: %s (ID: %s)\n", resp.Status, resp.Message, resp.DeployID)
}
```

**What's `%w` in error formatting?** It's Go's way of wrapping errors. When you use `fmt.Errorf("something: %w", err)`, the original error is preserved inside, so callers can "unwrap" it later to check the root cause.

## 6. Building a Health Check Tool

Let's build a more complete health check tool that reads a config file and checks multiple endpoints concurrently with proper error handling and output formatting.

```go
// File: main.go
package main

import (
    "encoding/json"
    "fmt"
    "net/http"
    "os"
    "sync"
    "time"
)

// Endpoint represents a single service to check
type Endpoint struct {
    Name    string `json:"name"`
    URL     string `json:"url"`
    Method  string `json:"method"`   // GET, POST, etc.
    Timeout int    `json:"timeout"`  // in seconds
}

// Config holds all the endpoints to check
type Config struct {
    Endpoints []Endpoint `json:"endpoints"`
}

// Result holds the outcome of a health check
type Result struct {
    Name       string
    URL        string
    Healthy    bool
    StatusCode int
    Duration   time.Duration
    Error      string
}

// checkEndpoint performs a health check on a single endpoint
func checkEndpoint(ep Endpoint) Result {
    // Create HTTP client with a timeout for this specific endpoint
    client := &http.Client{
        Timeout: time.Duration(ep.Timeout) * time.Second,
    }

    // Default method to GET if not specified
    method := ep.Method
    if method == "" {
        method = "GET"
    }

    start := time.Now()

    // Create and send the request
    req, err := http.NewRequest(method, ep.URL, nil)
    if err != nil {
        return Result{
            Name:  ep.Name,
            URL:   ep.URL,
            Error: fmt.Sprintf("bad request: %v", err),
        }
    }

    resp, err := client.Do(req)
    duration := time.Since(start)

    if err != nil {
        return Result{
            Name:     ep.Name,
            URL:      ep.URL,
            Duration: duration,
            Error:    err.Error(),
        }
    }
    defer resp.Body.Close()

    return Result{
        Name:       ep.Name,
        URL:        ep.URL,
        Healthy:    resp.StatusCode >= 200 && resp.StatusCode < 300,
        StatusCode: resp.StatusCode,
        Duration:   duration,
    }
}

func main() {
    // Read config file
    configFile := "endpoints.json"
    if len(os.Args) > 1 {
        configFile = os.Args[1]
    }

    data, err := os.ReadFile(configFile)
    if err != nil {
        fmt.Fprintf(os.Stderr, "Error reading config: %v\n", err)
        os.Exit(1)
    }

    var config Config
    if err := json.Unmarshal(data, &config); err != nil {
        fmt.Fprintf(os.Stderr, "Error parsing config: %v\n", err)
        os.Exit(1)
    }

    fmt.Printf("🏥 Health Check — %d endpoints\n", len(config.Endpoints))
    fmt.Println("─────────────────────────────────────────────────────")

    // Run all checks concurrently
    var wg sync.WaitGroup
    results := make([]Result, len(config.Endpoints))

    for i, ep := range config.Endpoints {
        wg.Add(1)
        go func(index int, endpoint Endpoint) {
            defer wg.Done()
            results[index] = checkEndpoint(endpoint)
        }(i, ep)
    }

    wg.Wait()

    // Print results
    healthy := 0
    for _, r := range results {
        if r.Healthy {
            healthy++
            fmt.Printf("  ✅ %-20s %s [%d] (%s)\n",
                r.Name, r.URL, r.StatusCode,
                r.Duration.Round(time.Millisecond))
        } else if r.Error != "" {
            fmt.Printf("  ❌ %-20s %s — %s\n", r.Name, r.URL, r.Error)
        } else {
            fmt.Printf("  ⚠️  %-20s %s [%d] (%s)\n",
                r.Name, r.URL, r.StatusCode,
                r.Duration.Round(time.Millisecond))
        }
    }

    fmt.Println("─────────────────────────────────────────────────────")
    fmt.Printf("📊 Results: %d/%d healthy\n", healthy, len(results))

    if healthy < len(results) {
        os.Exit(1) // Exit with error code so CI/CD pipelines catch failures
    }
}
```

**And the config file:**

```json
// File: endpoints.json
{
  "endpoints": [
    {
      "name": "Google",
      "url": "https://google.com",
      "method": "GET",
      "timeout": 5
    },
    {
      "name": "GitHub API",
      "url": "https://api.github.com",
      "method": "GET",
      "timeout": 10
    },
    {
      "name": "My App Health",
      "url": "https://myapp.example.com/health",
      "method": "GET",
      "timeout": 3
    }
  ]
}
```

**Why `os.Exit(1)` at the end?** In DevOps, tools are often used in CI/CD pipelines. An exit code of `0` means success, and any non-zero exit code means failure. So if any health check fails, we exit with `1` to signal to the pipeline: "Something is wrong, don't continue."

## 7. Working with Kubernetes API in Go

The official Kubernetes Go client (`client-go`) lets you interact with your cluster programmatically — the same way `kubectl` does.

### Installing the Client

```bash
go get k8s.io/client-go@latest
go get k8s.io/apimachinery@latest
```

### Listing Pods in a Namespace

```go
package main

import (
    "context"
    "fmt"
    "log"
    "path/filepath"

    metav1 "k8s.io/apimachinery/pkg/apis/meta/v1"
    "k8s.io/client-go/kubernetes"
    "k8s.io/client-go/tools/clientcmd"
    "k8s.io/client-go/util/homedir"
)

func main() {
    // Build the kubeconfig path (usually ~/.kube/config)
    home := homedir.HomeDir()
    kubeconfig := filepath.Join(home, ".kube", "config")

    // Create a Kubernetes config from the kubeconfig file
    config, err := clientcmd.BuildConfigFromFlags("", kubeconfig)
    if err != nil {
        log.Fatalf("Failed to build kubeconfig: %v", err)
    }

    // Create the Kubernetes clientset (this is our "connection" to the cluster)
    clientset, err := kubernetes.NewForConfig(config)
    if err != nil {
        log.Fatalf("Failed to create client: %v", err)
    }

    // List all pods in the "default" namespace
    pods, err := clientset.CoreV1().Pods("default").List(
        context.TODO(),           // Context for cancellation
        metav1.ListOptions{},     // No filters — get everything
    )
    if err != nil {
        log.Fatalf("Failed to list pods: %v", err)
    }

    // Print each pod's name and status
    fmt.Printf("Found %d pods in 'default' namespace:\n\n", len(pods.Items))
    for _, pod := range pods.Items {
        fmt.Printf("  📦 %-40s Status: %s\n", pod.Name, pod.Status.Phase)
    }
}
```

**What's happening here:**
- **`clientcmd.BuildConfigFromFlags`** — Reads your `~/.kube/config` file (the same one `kubectl` uses) and creates a config object.
- **`kubernetes.NewForConfig`** — Creates a clientset (a collection of API clients for different Kubernetes resources).
- **`clientset.CoreV1().Pods("default").List(...)`** — This is like running `kubectl get pods -n default`. It talks to the Kubernetes API server and gets back a list of pods.
- **`context.TODO()`** — A placeholder context. In production, you'd use a proper context with cancellation/timeout.

### Creating a Deployment Programmatically

```go
package main

import (
    "context"
    "fmt"
    "log"
    "path/filepath"

    appsv1 "k8s.io/api/apps/v1"
    corev1 "k8s.io/api/core/v1"
    metav1 "k8s.io/apimachinery/pkg/apis/meta/v1"
    "k8s.io/client-go/kubernetes"
    "k8s.io/client-go/tools/clientcmd"
    "k8s.io/client-go/util/homedir"
)

func int32Ptr(i int32) *int32 { return &i }

func main() {
    home := homedir.HomeDir()
    kubeconfig := filepath.Join(home, ".kube", "config")

    config, err := clientcmd.BuildConfigFromFlags("", kubeconfig)
    if err != nil {
        log.Fatalf("Failed to build config: %v", err)
    }

    clientset, err := kubernetes.NewForConfig(config)
    if err != nil {
        log.Fatalf("Failed to create client: %v", err)
    }

    // Define the deployment — this is equivalent to writing a YAML manifest
    deployment := &appsv1.Deployment{
        ObjectMeta: metav1.ObjectMeta{
            Name:      "nginx-deploy",
            Namespace: "default",
            Labels: map[string]string{
                "app":        "nginx",
                "managed-by": "go-tool",
            },
        },
        Spec: appsv1.DeploymentSpec{
            Replicas: int32Ptr(3), // Go requires a pointer to int32 for this field
            Selector: &metav1.LabelSelector{
                MatchLabels: map[string]string{"app": "nginx"},
            },
            Template: corev1.PodTemplateSpec{
                ObjectMeta: metav1.ObjectMeta{
                    Labels: map[string]string{"app": "nginx"},
                },
                Spec: corev1.PodSpec{
                    Containers: []corev1.Container{
                        {
                            Name:  "nginx",
                            Image: "nginx:1.25",
                            Ports: []corev1.ContainerPort{
                                {ContainerPort: 80},
                            },
                        },
                    },
                },
            },
        },
    }

    // Create the deployment in the cluster
    result, err := clientset.AppsV1().Deployments("default").Create(
        context.TODO(),
        deployment,
        metav1.CreateOptions{},
    )
    if err != nil {
        log.Fatalf("Failed to create deployment: %v", err)
    }

    fmt.Printf("✅ Created deployment: %s\n", result.Name)
}
```

**Why `int32Ptr(3)` instead of just `3`?** Kubernetes API types use pointers for optional fields. A `nil` pointer means "not specified" (use the default), while `*int32` with a value means "use this value." This is a common Go pattern for optional values.

## 9. How Operators Work — The Control Loop

Every operator runs a **reconciliation loop** (also called a control loop). This is the heart of how Kubernetes works.

```
The Reconciliation Loop — How ALL Controllers Work:

    ┌──────────────────────────────────────────┐
    │                                          │
    │   1. OBSERVE                             │
    │      What is the CURRENT state?          │
    │      (read from Kubernetes API)          │
    │                                          │
    │              │                           │
    │              ▼                           │
    │                                          │
    │   2. COMPARE                             │
    │      Current state == Desired state?     │
    │      (compare with the CR spec)          │
    │                                          │
    │              │                           │
    │         ┌────┴────┐                      │
    │         │         │                      │
    │        YES        NO                     │
    │         │         │                      │
    │         ▼         ▼                      │
    │                                          │
    │   (do nothing)  3. ACT                   │
    │                    Make changes to move   │
    │                    current → desired      │
    │                                          │
    │              │                           │
    │              ▼                           │
    │                                          │
    │   4. Loop back to step 1                 │
    │      (keep watching continuously)        │
    │                                          │
    └──────────────────────────────────────────┘
```

### Example: A Simple "Website" Operator

```
You create a custom resource:

apiVersion: mytools.example.com/v1
kind: Website
metadata:
  name: my-blog
spec:
  image: nginx:1.25
  replicas: 3
  domain: blog.example.com

The Operator sees this and:
1. Creates a Deployment with 3 nginx replicas
2. Creates a Service to expose it
3. Creates an Ingress with the domain name
4. Watches for changes — if you update replicas to 5, it scales up
5. If someone deletes the Deployment, it recreates it!
```

## 10. Building a Simple Kubernetes Operator

Let's build a basic operator from scratch (without frameworks) to understand the fundamentals.

```go
// File: main.go
// A simple operator that watches for ConfigMaps with a special label
// and creates a corresponding Secret from the ConfigMap data

package main

import (
    "context"
    "fmt"
    "log"
    "path/filepath"
    "time"

    corev1 "k8s.io/api/core/v1"
    metav1 "k8s.io/apimachinery/pkg/apis/meta/v1"
    "k8s.io/apimachinery/pkg/watch"
    "k8s.io/client-go/kubernetes"
    "k8s.io/client-go/tools/clientcmd"
    "k8s.io/client-go/util/homedir"
)

func main() {
    // Connect to the Kubernetes cluster
    home := homedir.HomeDir()
    kubeconfig := filepath.Join(home, ".kube", "config")

    config, err := clientcmd.BuildConfigFromFlags("", kubeconfig)
    if err != nil {
        log.Fatalf("Failed to build config: %v", err)
    }

    clientset, err := kubernetes.NewForConfig(config)
    if err != nil {
        log.Fatalf("Failed to create client: %v", err)
    }

    fmt.Println("🚀 Operator started — watching for ConfigMaps with label 'auto-secret=true'")

    // Start watching (this is the "observe" step)
    for {
        runWatcher(clientset)
        // If the watcher disconnects, wait and reconnect
        log.Println("Watcher disconnected. Reconnecting in 5 seconds...")
        time.Sleep(5 * time.Second)
    }
}

func runWatcher(clientset *kubernetes.Clientset) {
    // Watch ConfigMaps in ALL namespaces that have our special label
    watcher, err := clientset.CoreV1().ConfigMaps("").Watch(
        context.TODO(),
        metav1.ListOptions{
            LabelSelector: "auto-secret=true", // Only watch ConfigMaps with this label
        },
    )
    if err != nil {
        log.Printf("Failed to create watcher: %v", err)
        return
    }
    defer watcher.Stop()

    // Process events from the watcher
    for event := range watcher.ResultChan() {
        cm, ok := event.Object.(*corev1.ConfigMap)
        if !ok {
            continue
        }

        switch event.Type {
        case watch.Added, watch.Modified:
            // A ConfigMap was created or updated — create/update the Secret
            fmt.Printf("📝 ConfigMap %s/%s changed — syncing Secret...\n", cm.Namespace, cm.Name)
            if err := reconcileSecret(clientset, cm); err != nil {
                log.Printf("Failed to reconcile: %v", err)
            }

        case watch.Deleted:
            // A ConfigMap was deleted — delete the corresponding Secret
            fmt.Printf("🗑️  ConfigMap %s/%s deleted — cleaning up Secret...\n", cm.Namespace, cm.Name)
            secretName := cm.Name + "-auto-secret"
            err := clientset.CoreV1().Secrets(cm.Namespace).Delete(
                context.TODO(), secretName, metav1.DeleteOptions{},
            )
            if err != nil {
                log.Printf("Failed to delete secret: %v", err)
            }
        }
    }
}

// reconcileSecret creates or updates a Secret based on a ConfigMap
func reconcileSecret(clientset *kubernetes.Clientset, cm *corev1.ConfigMap) error {
    secretName := cm.Name + "-auto-secret"

    // Convert ConfigMap data (string) to Secret data ([]byte)
    secretData := make(map[string][]byte)
    for key, value := range cm.Data {
        secretData[key] = []byte(value)
    }

    // Build the Secret object
    secret := &corev1.Secret{
        ObjectMeta: metav1.ObjectMeta{
            Name:      secretName,
            Namespace: cm.Namespace,
            Labels: map[string]string{
                "managed-by":    "my-operator",
                "source-config": cm.Name,
            },
        },
        Data: secretData,
        Type: corev1.SecretTypeOpaque,
    }

    // Try to get the existing Secret
    existing, err := clientset.CoreV1().Secrets(cm.Namespace).Get(
        context.TODO(), secretName, metav1.GetOptions{},
    )

    if err != nil {
        // Secret doesn't exist — create it
        _, err = clientset.CoreV1().Secrets(cm.Namespace).Create(
            context.TODO(), secret, metav1.CreateOptions{},
        )
        if err != nil {
            return fmt.Errorf("failed to create secret: %w", err)
        }
        fmt.Printf("  ✅ Created Secret: %s/%s\n", cm.Namespace, secretName)
    } else {
        // Secret exists — update it
        existing.Data = secretData
        _, err = clientset.CoreV1().Secrets(cm.Namespace).Update(
            context.TODO(), existing, metav1.UpdateOptions{},
        )
        if err != nil {
            return fmt.Errorf("failed to update secret: %w", err)
        }
        fmt.Printf("  🔄 Updated Secret: %s/%s\n", cm.Namespace, secretName)
    }

    return nil
}
```

**How this operator works:**

1. It **watches** for ConfigMaps that have the label `auto-secret=true`.
2. When one is created or modified, it creates/updates a corresponding Secret with the same data.
3. When a labeled ConfigMap is deleted, it cleans up the Secret.
4. If the watch connection drops, it reconnects.

This is a simplified example — real operators use frameworks like Kubebuilder for proper error handling, caching, and leader election.

### Step 3: Define the Custom Resource

```go
// File: api/v1/website_types.go
package v1

import (
    metav1 "k8s.io/apimachinery/pkg/apis/meta/v1"
)

// WebsiteSpec defines the DESIRED state of a Website
// This is what the user writes in their YAML
type WebsiteSpec struct {
    // Image is the container image for the website (e.g., "nginx:1.25")
    Image string `json:"image"`

    // Replicas is the number of pod replicas to run
    // +kubebuilder:validation:Minimum=1
    // +kubebuilder:validation:Maximum=10
    // +kubebuilder:default=1
    Replicas int32 `json:"replicas,omitempty"`

    // Port is the container port to expose
    // +kubebuilder:default=80
    Port int32 `json:"port,omitempty"`
}

// WebsiteStatus defines the OBSERVED state of a Website
// This is updated by the controller to report what's actually happening
type WebsiteStatus struct {
    // AvailableReplicas is the number of pods that are ready
    AvailableReplicas int32 `json:"availableReplicas,omitempty"`

    // Conditions represent the current state
    Conditions []metav1.Condition `json:"conditions,omitempty"`
}

// +kubebuilder:object:root=true
// +kubebuilder:subresource:status
// +kubebuilder:printcolumn:name="Image",type=string,JSONPath=`.spec.image`
// +kubebuilder:printcolumn:name="Replicas",type=integer,JSONPath=`.spec.replicas`
// +kubebuilder:printcolumn:name="Available",type=integer,JSONPath=`.status.availableReplicas`

// Website is the Schema for the websites API
type Website struct {
    metav1.TypeMeta   `json:",inline"`
    metav1.ObjectMeta `json:"metadata,omitempty"`

    Spec   WebsiteSpec   `json:"spec,omitempty"`
    Status WebsiteStatus `json:"status,omitempty"`
}

// +kubebuilder:object:root=true

// WebsiteList contains a list of Websites
type WebsiteList struct {
    metav1.TypeMeta `json:",inline"`
    metav1.ListMeta `json:"metadata,omitempty"`
    Items           []Website `json:"items"`
}

func init() {
    SchemeBuilder.Register(&Website{}, &WebsiteList{})
}
```

**What are those `+kubebuilder:` comments?**
They're special markers that Kubebuilder uses to generate CRD manifests:
- `+kubebuilder:validation:Minimum=1` — Enforces minimum value in the CRD schema
- `+kubebuilder:default=1` — Sets a default value
- `+kubebuilder:subresource:status` — Enables the `/status` subresource
- `+kubebuilder:printcolumn` — Adds columns to `kubectl get websites` output

### Step 4: Write the Controller (Reconciliation Logic)

```go
// File: internal/controller/website_controller.go
package controller

import (
    "context"
    "fmt"

    appsv1 "k8s.io/api/apps/v1"
    corev1 "k8s.io/api/core/v1"
    "k8s.io/apimachinery/pkg/api/errors"
    metav1 "k8s.io/apimachinery/pkg/apis/meta/v1"
    "k8s.io/apimachinery/pkg/runtime"
    "k8s.io/apimachinery/pkg/types"
    ctrl "sigs.k8s.io/controller-runtime"
    "sigs.k8s.io/controller-runtime/pkg/log"

    webappv1 "github.com/yourusername/website-operator/api/v1"
)

// WebsiteReconciler reconciles a Website object
type WebsiteReconciler struct {
    client.Client
    Scheme *runtime.Scheme
}

// Reconcile is the main logic — called every time something changes
// This is where the magic happens!
func (r *WebsiteReconciler) Reconcile(ctx context.Context, req ctrl.Request) (ctrl.Result, error) {
    logger := log.FromContext(ctx)

    // Step 1: Fetch the Website resource
    var website webappv1.Website
    if err := r.Get(ctx, req.NamespacedName, &website); err != nil {
        if errors.IsNotFound(err) {
            // Website was deleted — nothing to do (Kubernetes GC handles cleanup)
            logger.Info("Website resource not found — probably deleted")
            return ctrl.Result{}, nil
        }
        return ctrl.Result{}, err
    }

    logger.Info("Reconciling Website", "name", website.Name)

    // Step 2: Create or update the Deployment
    if err := r.reconcileDeployment(ctx, &website); err != nil {
        return ctrl.Result{}, err
    }

    // Step 3: Create or update the Service
    if err := r.reconcileService(ctx, &website); err != nil {
        return ctrl.Result{}, err
    }

    // Step 4: Update the status
    if err := r.updateStatus(ctx, &website); err != nil {
        return ctrl.Result{}, err
    }

    logger.Info("Successfully reconciled Website", "name", website.Name)
    return ctrl.Result{}, nil
}

// reconcileDeployment ensures a Deployment exists for the Website
func (r *WebsiteReconciler) reconcileDeployment(ctx context.Context, website *webappv1.Website) error {
    deployName := website.Name + "-deployment"

    // Check if Deployment already exists
    var existingDeploy appsv1.Deployment
    err := r.Get(ctx, types.NamespacedName{
        Name:      deployName,
        Namespace: website.Namespace,
    }, &existingDeploy)

    replicas := website.Spec.Replicas
    if replicas == 0 {
        replicas = 1
    }

    deploy := &appsv1.Deployment{
        ObjectMeta: metav1.ObjectMeta{
            Name:      deployName,
            Namespace: website.Namespace,
            Labels:    map[string]string{"app": website.Name, "managed-by": "website-operator"},
        },
        Spec: appsv1.DeploymentSpec{
            Replicas: &replicas,
            Selector: &metav1.LabelSelector{
                MatchLabels: map[string]string{"app": website.Name},
            },
            Template: corev1.PodTemplateSpec{
                ObjectMeta: metav1.ObjectMeta{
                    Labels: map[string]string{"app": website.Name},
                },
                Spec: corev1.PodSpec{
                    Containers: []corev1.Container{
                        {
                            Name:  "web",
                            Image: website.Spec.Image,
                            Ports: []corev1.ContainerPort{
                                {ContainerPort: website.Spec.Port},
                            },
                        },
                    },
                },
            },
        },
    }

    // Set the Website as the owner of this Deployment
    // This means: if the Website is deleted, the Deployment is automatically cleaned up
    ctrl.SetControllerReference(website, deploy, r.Scheme)

    if errors.IsNotFound(err) {
        // Create the Deployment
        return r.Create(ctx, deploy)
    } else if err != nil {
        return err
    }

    // Update the existing Deployment
    existingDeploy.Spec = deploy.Spec
    return r.Update(ctx, &existingDeploy)
}

// reconcileService ensures a Service exists for the Website
func (r *WebsiteReconciler) reconcileService(ctx context.Context, website *webappv1.Website) error {
    svcName := website.Name + "-service"

    var existingSvc corev1.Service
    err := r.Get(ctx, types.NamespacedName{
        Name:      svcName,
        Namespace: website.Namespace,
    }, &existingSvc)

    svc := &corev1.Service{
        ObjectMeta: metav1.ObjectMeta{
            Name:      svcName,
            Namespace: website.Namespace,
            Labels:    map[string]string{"app": website.Name, "managed-by": "website-operator"},
        },
        Spec: corev1.ServiceSpec{
            Selector: map[string]string{"app": website.Name},
            Ports: []corev1.ServicePort{
                {
                    Port:       80,
                    TargetPort: intstr.FromInt(int(website.Spec.Port)),
                },
            },
            Type: corev1.ServiceTypeClusterIP,
        },
    }

    ctrl.SetControllerReference(website, svc, r.Scheme)

    if errors.IsNotFound(err) {
        return r.Create(ctx, svc)
    } else if err != nil {
        return err
    }

    return nil // Service doesn't need frequent updates
}

// updateStatus updates the Website's status with current information
func (r *WebsiteReconciler) updateStatus(ctx context.Context, website *webappv1.Website) error {
    deployName := website.Name + "-deployment"

    var deploy appsv1.Deployment
    if err := r.Get(ctx, types.NamespacedName{
        Name:      deployName,
        Namespace: website.Namespace,
    }, &deploy); err != nil {
        return err
    }

    website.Status.AvailableReplicas = deploy.Status.AvailableReplicas
    return r.Status().Update(ctx, website)
}

// SetupWithManager registers this controller with the manager
func (r *WebsiteReconciler) SetupWithManager(mgr ctrl.Manager) error {
    return ctrl.NewControllerManagedBy(mgr).
        For(&webappv1.Website{}).         // Watch Website resources
        Owns(&appsv1.Deployment{}).       // Also watch Deployments owned by Website
        Owns(&corev1.Service{}).          // Also watch Services owned by Website
        Complete(r)
}
```

## 14. Building a Custom Terraform Provider in Go

Terraform providers are plugins (written in Go) that tell Terraform how to manage a specific service or resource.

### When Would You Build One?

```
Build a custom provider when:
 • Your company has an internal API that Terraform doesn't support
 • You want to manage a niche service via Terraform
 • You need to extend an existing provider with custom resources
```

### Basic Provider Structure

```go
// File: main.go
package main

import (
    "github.com/hashicorp/terraform-plugin-sdk/v2/helper/schema"
    "github.com/hashicorp/terraform-plugin-sdk/v2/plugin"
)

func main() {
    plugin.Serve(&plugin.ServeOpts{
        ProviderFunc: func() *schema.Provider {
            return &schema.Provider{
                // Define resources this provider manages
                ResourcesMap: map[string]*schema.Resource{
                    "myservice_server": resourceServer(),
                },
                // Define data sources this provider can read
                DataSourcesMap: map[string]*schema.Resource{
                    "myservice_server": dataSourceServer(),
                },
            }
        },
    })
}

// resourceServer defines a "myservice_server" resource
func resourceServer() *schema.Resource {
    return &schema.Resource{
        Create: resourceServerCreate,
        Read:   resourceServerRead,
        Update: resourceServerUpdate,
        Delete: resourceServerDelete,

        Schema: map[string]*schema.Schema{
            "name": {
                Type:        schema.TypeString,
                Required:    true,
                Description: "The name of the server",
            },
            "size": {
                Type:        schema.TypeString,
                Required:    true,
                Description: "Server size (small, medium, large)",
            },
            "ip_address": {
                Type:        schema.TypeString,
                Computed:    true, // Set by the provider, not the user
                Description: "The assigned IP address",
            },
        },
    }
}

func resourceServerCreate(d *schema.ResourceData, m interface{}) error {
    name := d.Get("name").(string)
    size := d.Get("size").(string)

    // Call your internal API to create the server
    // server, err := myapi.CreateServer(name, size)

    // Set the resource ID (Terraform uses this to track the resource)
    d.SetId("server-123")
    d.Set("ip_address", "10.0.1.5")

    return nil
}

func resourceServerRead(d *schema.ResourceData, m interface{}) error   { return nil }
func resourceServerUpdate(d *schema.ResourceData, m interface{}) error { return nil }
func resourceServerDelete(d *schema.ResourceData, m interface{}) error { return nil }

func dataSourceServer() *schema.Resource { return &schema.Resource{} }
```

**How Terraform uses this:** When someone writes `resource "myservice_server" "web" { name = "web-1" size = "medium" }` in their `.tf` file, Terraform calls your provider's `Create` function. When they run `terraform destroy`, it calls `Delete`.

## 15. Building a Prometheus Exporter in Go

A **Prometheus Exporter** is a small HTTP server that exposes metrics in a format Prometheus can scrape.

### When to Build One

```
Build an exporter when:
 • Your application doesn't expose Prometheus metrics natively
 • You want to monitor an internal service or hardware
 • You need custom business metrics (orders per minute, active users, etc.)
```

### Simple Exporter Example

```go
// File: main.go
package main

import (
    "log"
    "math/rand"
    "net/http"
    "time"

    "github.com/prometheus/client_golang/prometheus"
    "github.com/prometheus/client_golang/prometheus/promhttp"
)

// Define metrics — these will be exposed at /metrics
var (
    // Counter — only goes up (e.g., total requests)
    httpRequestsTotal = prometheus.NewCounterVec(
        prometheus.CounterOpts{
            Name: "myapp_http_requests_total",
            Help: "Total number of HTTP requests",
        },
        []string{"method", "endpoint", "status"}, // Labels
    )

    // Gauge — can go up or down (e.g., current temperature, active connections)
    activeConnections = prometheus.NewGauge(
        prometheus.GaugeOpts{
            Name: "myapp_active_connections",
            Help: "Number of currently active connections",
        },
    )

    // Histogram — tracks the distribution of values (e.g., request duration)
    requestDuration = prometheus.NewHistogramVec(
        prometheus.HistogramOpts{
            Name:    "myapp_request_duration_seconds",
            Help:    "Time spent processing requests",
            Buckets: prometheus.DefBuckets, // Default buckets: .005, .01, .025, .05, .1, .25, .5, 1, 2.5, 5, 10
        },
        []string{"endpoint"},
    )
)

func init() {
    // Register all metrics with Prometheus
    prometheus.MustRegister(httpRequestsTotal)
    prometheus.MustRegister(activeConnections)
    prometheus.MustRegister(requestDuration)
}

func main() {
    // Simulate some metrics in a background goroutine
    go func() {
        for {
            // Simulate random request counts
            httpRequestsTotal.WithLabelValues("GET", "/api/users", "200").Inc()

            // Simulate active connections going up and down
            activeConnections.Set(float64(rand.Intn(100)))

            // Simulate request duration
            requestDuration.WithLabelValues("/api/users").Observe(
                rand.Float64() * 2, // Random duration between 0-2 seconds
            )

            time.Sleep(1 * time.Second)
        }
    }()

    // Expose metrics at /metrics endpoint
    http.Handle("/metrics", promhttp.Handler())

    log.Println("🔥 Exporter listening on :9090/metrics")
    log.Fatal(http.ListenAndServe(":9090", nil))
}
```

**How this works:**
1. We define three types of metrics: Counter, Gauge, and Histogram.
2. We register them with Prometheus in `init()`.
3. A background goroutine simulates updating these metrics.
4. We expose them at `/metrics` using `promhttp.Handler()`.
5. Prometheus scrapes this endpoint every few seconds.

**Running and testing:**

```bash
go run main.go

# In another terminal:
curl http://localhost:9090/metrics

# You'll see output like:
# myapp_http_requests_total{endpoint="/api/users",method="GET",status="200"} 42
# myapp_active_connections 67
# myapp_request_duration_seconds_bucket{endpoint="/api/users",le="0.5"} 12
```

## 16. Cross-Compiling Go Binaries

One of Go's superpowers is **cross-compilation** — build a binary for any OS/architecture from your machine.

```bash
# Build for Linux (most common for DevOps — servers run Linux)
GOOS=linux GOARCH=amd64 go build -o mytool-linux-amd64 .

# Build for Linux ARM (Raspberry Pi, AWS Graviton)
GOOS=linux GOARCH=arm64 go build -o mytool-linux-arm64 .

# Build for macOS (Intel)
GOOS=darwin GOARCH=amd64 go build -o mytool-darwin-amd64 .

# Build for macOS (Apple Silicon - M1/M2/M3)
GOOS=darwin GOARCH=arm64 go build -o mytool-darwin-arm64 .

# Build for Windows
GOOS=windows GOARCH=amd64 go build -o mytool-windows-amd64.exe .

# Build ALL platforms at once (a common Makefile target)
platforms=("linux/amd64" "linux/arm64" "darwin/amd64" "darwin/arm64" "windows/amd64")
for platform in "${platforms[@]}"; do
    GOOS=${platform%/*} GOARCH=${platform#*/} \
    go build -o "mytool-${platform%/*}-${platform#*/}" .
done
```

### Build with Version Information

```bash
# Inject version, build date, and git commit at build time
go build -ldflags "-X main.version=v1.2.3 \
                   -X main.buildDate=$(date -u +%Y-%m-%dT%H:%M:%SZ) \
                   -X main.gitCommit=$(git rev-parse --short HEAD)" \
         -o mytool .

# Now when you run: mytool version
# Output: mytool v1.2.3
#         Build date: 2025-01-15T10:30:00Z
#         Git commit: abc1234
```

**What's `-ldflags`?** It stands for "linker flags." The `-X` flag sets the value of a Go variable at build time. So we can embed version info without hardcoding it in the source code.

## 18. Testing Your Go DevOps Tools

Testing is crucial for DevOps tools — you don't want a buggy deployment tool!

### Unit Testing

```go
// File: health_test.go
package main

import (
    "net/http"
    "net/http/httptest"
    "testing"
)

func TestCheckEndpoint_Healthy(t *testing.T) {
    // Create a fake HTTP server that returns 200
    server := httptest.NewServer(http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        w.WriteHeader(http.StatusOK)
        w.Write([]byte(`{"status": "healthy"}`))
    }))
    defer server.Close() // Clean up after the test

    // Test our checkEndpoint function against the fake server
    result := checkEndpoint(Endpoint{
        Name:    "Test Server",
        URL:     server.URL,
        Method:  "GET",
        Timeout: 5,
    })

    // Assert the results
    if !result.Healthy {
        t.Errorf("Expected healthy=true, got false")
    }
    if result.StatusCode != 200 {
        t.Errorf("Expected status 200, got %d", result.StatusCode)
    }
}

func TestCheckEndpoint_Unhealthy(t *testing.T) {
    // Create a fake server that returns 500
    server := httptest.NewServer(http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        w.WriteHeader(http.StatusInternalServerError)
    }))
    defer server.Close()

    result := checkEndpoint(Endpoint{
        Name:    "Bad Server",
        URL:     server.URL,
        Method:  "GET",
        Timeout: 5,
    })

    if result.Healthy {
        t.Errorf("Expected healthy=false, got true")
    }
    if result.StatusCode != 500 {
        t.Errorf("Expected status 500, got %d", result.StatusCode)
    }
}

func TestCheckEndpoint_Timeout(t *testing.T) {
    // Create a server that never responds (simulates timeout)
    server := httptest.NewServer(http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        select {} // Block forever
    }))
    defer server.Close()

    result := checkEndpoint(Endpoint{
        Name:    "Slow Server",
        URL:     server.URL,
        Method:  "GET",
        Timeout: 1, // 1 second timeout
    })

    if result.Healthy {
        t.Errorf("Expected healthy=false for timed-out server")
    }
    if result.Error == "" {
        t.Errorf("Expected an error message for timeout")
    }
}
```

```bash
# Run all tests
go test ./...

# Run tests with verbose output
go test -v ./...

# Run tests with coverage
go test -cover ./...

# Generate a coverage report
go test -coverprofile=coverage.out ./...
go tool cover -html=coverage.out  # Opens in browser
```

**Key concepts:**
- **`httptest.NewServer`** — Creates a fake HTTP server for testing. No real network calls needed!
- **`t.Errorf`** — Reports a test failure but continues running the rest of the test.
- **`defer server.Close()`** — Always clean up test servers.

### Table-Driven Tests (Go Best Practice)

```go
func TestCheckEndpoint_TableDriven(t *testing.T) {
    // Define test cases in a table
    tests := []struct {
        name           string
        serverStatus   int
        expectHealthy  bool
        expectStatus   int
    }{
        {"healthy server",   200, true,  200},
        {"redirect",         301, false, 301},
        {"not found",        404, false, 404},
        {"server error",     500, false, 500},
        {"service unavail",  503, false, 503},
    }

    for _, tc := range tests {
        t.Run(tc.name, func(t *testing.T) {
            server := httptest.NewServer(http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
                w.WriteHeader(tc.serverStatus)
            }))
            defer server.Close()

            result := checkEndpoint(Endpoint{
                Name: tc.name, URL: server.URL, Method: "GET", Timeout: 5,
            })

            if result.Healthy != tc.expectHealthy {
                t.Errorf("healthy: got %v, want %v", result.Healthy, tc.expectHealthy)
            }
            if result.StatusCode != tc.expectStatus {
                t.Errorf("status: got %d, want %d", result.StatusCode, tc.expectStatus)
            }
        })
    }
}
```
