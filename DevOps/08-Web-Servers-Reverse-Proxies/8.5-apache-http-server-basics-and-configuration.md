## 1. What is Apache HTTP Server?

Apache HTTP Server (often just called **Apache** or **httpd**) is the **world's most widely used web server** since 1996. It's developed by the **Apache Software Foundation** and is completely free and open source.

**Quick facts:**
- First released in **1995** (older than Google!)
- Written in **C**
- Powers roughly **30-35%** of all websites (as of 2024)
- Name comes from "**A Patchy Server**" — it was a collection of patches for the NCSA HTTPd server
- Default on most Linux distributions
- Runs on Linux, Windows, macOS

**Think of Apache like this:** If Nginx is a Formula 1 car (fast, streamlined, purpose-built), Apache is a heavy-duty truck (versatile, customizable, can carry anything, been around forever).

## 2. Apache vs Nginx — When to Use Which?

Before we dive deep, let's understand when you'd pick Apache over Nginx:

| Feature | Apache | Nginx |
|---------|--------|-------|
| **Architecture** | Process/Thread-based | Event-driven, async |
| **Static content** | Good | Excellent (faster) |
| **Dynamic content** | Excellent (built-in via modules like mod_php) | Needs external process (PHP-FPM, etc.) |
| **Configuration** | `.htaccess` (per-directory, no restart needed) | Central config only (requires reload) |
| **Flexibility** | Extremely flexible (60+ modules) | More focused, less modular |
| **Memory usage** | Higher (especially under load) | Lower |
| **Concurrent connections** | Handles thousands | Handles tens of thousands |
| **WordPress/PHP** | Native support (mod_php) | Needs PHP-FPM |
| **Learning curve** | Moderate | Moderate |
| **Config style** | XML-like directives | C-like blocks |

**When to use Apache:**
- Running PHP applications (WordPress, Laravel, Drupal)
- Need `.htaccess` per-directory configs (shared hosting)
- Need dynamic module loading
- Working with legacy applications

**When to use Nginx:**
- High-traffic static file serving
- Reverse proxy / load balancer
- Microservices architecture
- Maximum performance with minimal resources

**In real-world DevOps:** Many setups use **Nginx in front of Apache** — Nginx handles static files and SSL, and proxies dynamic requests to Apache.

## 3. Apache Architecture — How It Handles Requests

Apache uses **Multi-Processing Modules (MPMs)** to handle incoming requests. Think of MPMs as different strategies for answering phone calls at a call center:

### MPM Prefork (The Old Way)

```
Client 1 ──→ [Process 1] ──→ Handle request
Client 2 ──→ [Process 2] ──→ Handle request
Client 3 ──→ [Process 3] ──→ Handle request
Client 4 ──→ [Process 4] ──→ Handle request
```

- **One process per connection**
- Each process handles ONE request at a time
- Safe for non-thread-safe libraries (like older PHP)
- **High memory usage** — each process uses 10-50MB
- 100 connections = 100 processes = 1-5 GB of RAM!

**Analogy:** Each caller gets their own dedicated person. Expensive but safe.

### MPM Worker (The Better Way)

```
Client 1 ──→ [Process 1] ──→ Thread 1 ──→ Handle request
Client 2 ──→            ──→ Thread 2 ──→ Handle request
Client 3 ──→ [Process 2] ──→ Thread 1 ──→ Handle request
Client 4 ──→            ──→ Thread 2 ──→ Handle request
```

- **Multiple threads per process**
- Each process spawns multiple threads
- **Lower memory usage** — threads share memory
- Requires thread-safe applications

**Analogy:** Each person handles multiple calls using a switchboard. More efficient.

### MPM Event (The Modern Way — Default since Apache 2.4)

```
Client 1 ──→ [Listener Thread] ──→ Routes to available thread
Client 2 ──→                   ──→ Keep-alive connections handled separately
Client 3 ──→ [Worker Threads]  ──→ Handle actual request processing
Client 4 ──→                   ──→ Idle connections don't waste threads
```

- **Event-driven** (similar concept to Nginx)
- Dedicates threads only for active requests
- Keep-alive connections don't tie up threads
- **Best performance** of all MPMs
- Default in modern Apache installations

**Analogy:** A smart receptionist routes calls — only connects people when there's actually something to discuss.

**Check your current MPM:**
```bash
# On Ubuntu/Debian
apachectl -V | grep MPM

# Or
apache2ctl -M | grep mpm

# Output: mpm_event_module (shared)
```

## 4. Installation

### Ubuntu / Debian

```bash
# Update package list
sudo apt update

# Install Apache
sudo apt install apache2 -y

# Start Apache
sudo systemctl start apache2

# Enable on boot
sudo systemctl enable apache2

# Check status
sudo systemctl status apache2

# Verify it's running
curl http://localhost
# You should see "Apache2 Default Page"
```

### CentOS / RHEL / Rocky Linux

```bash
# Install Apache (called httpd on RHEL-based systems)
sudo dnf install httpd -y

# Start and enable
sudo systemctl start httpd
sudo systemctl enable httpd

# Open firewall
sudo firewall-cmd --permanent --add-service=http
sudo firewall-cmd --permanent --add-service=https
sudo firewall-cmd --reload

# Verify
curl http://localhost
```

### Verify Installation

```bash
# Check Apache version
apache2 -v       # Debian/Ubuntu
httpd -v          # CentOS/RHEL

# Output:
# Server version: Apache/2.4.58 (Ubuntu)
# Server built:   2024-01-15T12:34:56

# Test configuration syntax
apache2ctl configtest   # Debian/Ubuntu
httpd -t                 # CentOS/RHEL
# Output: Syntax OK
```

## 5. File & Directory Structure

### Debian/Ubuntu Layout

```
/etc/apache2/                    ← Main config directory
├── apache2.conf                 ← Main config file
├── envvars                      ← Environment variables (user, group, etc.)
├── ports.conf                   ← Which ports Apache listens on
├── magic                        ← MIME type detection rules
├── conf-available/              ← Available configurations
│   ├── charset.conf
│   ├── security.conf
│   └── ...
├── conf-enabled/                ← Symlinks to enabled configurations
│   └── ...
├── mods-available/              ← Available modules
│   ├── rewrite.load
│   ├── ssl.load
│   ├── proxy.load
│   └── ...
├── mods-enabled/                ← Symlinks to enabled modules
│   └── ...
├── sites-available/             ← Available virtual hosts
│   ├── 000-default.conf         ← Default HTTP virtual host
│   └── default-ssl.conf         ← Default HTTPS virtual host
└── sites-enabled/               ← Symlinks to enabled virtual hosts
    └── 000-default.conf → ../sites-available/000-default.conf

/var/www/html/                   ← Default document root (web files go here)
/var/log/apache2/                ← Log files
├── access.log                   ← Who accessed what
└── error.log                    ← Errors and warnings
```

### CentOS/RHEL Layout

```
/etc/httpd/                      ← Main config directory
├── conf/
│   └── httpd.conf               ← Main config file (everything in one file)
├── conf.d/                      ← Additional configs (virtual hosts go here)
│   ├── ssl.conf
│   └── welcome.conf
├── conf.modules.d/              ← Module configurations
│   ├── 00-base.conf
│   └── 00-ssl.conf
└── modules/                     ← Compiled modules (.so files)

/var/www/html/                   ← Default document root
/var/log/httpd/                  ← Log files
├── access_log
└── error_log
```

**Key difference:** Debian/Ubuntu uses the `*-available` / `*-enabled` pattern with symlinks. CentOS/RHEL puts everything in `conf.d/` directory.

## 6. Core Configuration — httpd.conf / apache2.conf

Let's understand the main configuration file piece by piece:

```apache
# ==========================================
# GLOBAL SETTINGS
# ==========================================

# Where Apache is installed
ServerRoot "/etc/apache2"

# Who Apache runs as (security — never run as root!)
# On Debian these are set in /etc/apache2/envvars
User www-data
Group www-data

# Server's hostname (used in redirects and error messages)
ServerName example.com

# Admin email (shown on error pages)
ServerAdmin admin@example.com

# Hide Apache version in HTTP headers (security)
ServerTokens Prod
ServerSignature Off

# Process ID file
PidFile ${APACHE_PID_FILE}

# Request timeout (seconds)
Timeout 300

# Keep connections alive (reuse TCP connection for multiple requests)
KeepAlive On
MaxKeepAliveRequests 100
KeepAliveTimeout 5

# ==========================================
# MPM EVENT SETTINGS
# ==========================================
<IfModule mpm_event_module>
    StartServers             2      # Number of child processes at startup
    MinSpareThreads         25      # Minimum idle threads
    MaxSpareThreads         75      # Maximum idle threads
    ThreadLimit             64      # Max threads per process (hard limit)
    ThreadsPerChild         25      # Threads each child process creates
    MaxRequestWorkers      150      # Total max simultaneous connections
    MaxConnectionsPerChild   0      # Requests before process is recycled (0 = never)
</IfModule>

# ==========================================
# DIRECTORY PERMISSIONS
# ==========================================

# Default: Deny access to everything
<Directory />
    Options None
    AllowOverride None
    Require all denied
</Directory>

# Allow access to the document root
<Directory /var/www/html>
    Options -Indexes +FollowSymLinks    # No directory listing, allow symlinks
    AllowOverride All                    # Allow .htaccess files
    Require all granted                  # Allow everyone
</Directory>

# ==========================================
# MODULES
# ==========================================

# Load modules (Debian uses mods-enabled/ symlinks instead)
LoadModule rewrite_module modules/mod_rewrite.so
LoadModule ssl_module modules/mod_ssl.so
LoadModule proxy_module modules/mod_proxy.so

# ==========================================
# INCLUDE OTHER CONFIGS
# ==========================================
IncludeOptional conf-enabled/*.conf
IncludeOptional sites-enabled/*.conf
```

**Understanding the `<Directory>` directive:**

The `<Directory>` directive controls what Apache allows for a specific directory:

```apache
<Directory /var/www/html>
    # Options — what features are enabled
    Options -Indexes       # Don't show file listing if no index.html exists
    Options +FollowSymLinks # Follow symbolic links
    Options -ExecCGI       # Don't allow CGI scripts
    Options +MultiViews    # Allow content negotiation
    Options None           # Disable everything

    # AllowOverride — what .htaccess can change
    AllowOverride None     # .htaccess is completely ignored
    AllowOverride All      # .htaccess can override anything
    AllowOverride AuthConfig # .htaccess can only set authentication

    # Require — who can access
    Require all granted    # Everyone can access
    Require all denied     # No one can access
    Require ip 192.168.1.0/24    # Only this network
    Require valid-user     # Only authenticated users
</Directory>
```

**The `Options` flags explained:**

| Option | What It Does | Should You Enable It? |
|--------|-------------|----------------------|
| `Indexes` | Shows directory listing if no index file exists | ❌ No (security risk!) |
| `FollowSymLinks` | Allows following symbolic links | ✅ Yes (usually needed) |
| `SymLinksIfOwnerMatch` | Follow symlinks only if same owner | ✅ Safer alternative |
| `ExecCGI` | Allow CGI script execution | ⚠️ Only if needed |
| `MultiViews` | Content negotiation (auto-pick language, etc.) | ⚠️ Only if needed |
| `None` | Disable all options | ✅ Most secure |
| `All` | Enable everything | ❌ Too permissive |

## 7. Virtual Hosts — Serving Multiple Sites

Virtual hosts let one Apache server serve **multiple websites**. It's like having multiple apartments in one building — each has its own address (domain), but they share the same physical server.

### Name-Based Virtual Hosts (Most Common)

Multiple domains on the same IP, differentiated by the `Host` header:

```apache
# /etc/apache2/sites-available/example.com.conf

<VirtualHost *:80>
    # Which domain this virtual host responds to
    ServerName example.com
    ServerAlias www.example.com    # Also respond to www.

    # Admin email
    ServerAdmin admin@example.com

    # Where the website files are
    DocumentRoot /var/www/example.com/public

    # Directory permissions
    <Directory /var/www/example.com/public>
        Options -Indexes +FollowSymLinks
        AllowOverride All
        Require all granted
    </Directory>

    # Logging (separate logs per site!)
    ErrorLog ${APACHE_LOG_DIR}/example.com-error.log
    CustomLog ${APACHE_LOG_DIR}/example.com-access.log combined
</VirtualHost>
```

```apache
# /etc/apache2/sites-available/blog.example.com.conf

<VirtualHost *:80>
    ServerName blog.example.com

    DocumentRoot /var/www/blog.example.com/public

    <Directory /var/www/blog.example.com/public>
        Options -Indexes +FollowSymLinks
        AllowOverride All
        Require all granted
    </Directory>

    ErrorLog ${APACHE_LOG_DIR}/blog-error.log
    CustomLog ${APACHE_LOG_DIR}/blog-access.log combined
</VirtualHost>
```

**Enable and test:**
```bash
# Create the document root directories
sudo mkdir -p /var/www/example.com/public
sudo mkdir -p /var/www/blog.example.com/public

# Create a simple test page
echo "<h1>Welcome to example.com</h1>" | sudo tee /var/www/example.com/public/index.html
echo "<h1>Welcome to blog.example.com</h1>" | sudo tee /var/www/blog.example.com/public/index.html

# Set permissions
sudo chown -R www-data:www-data /var/www/

# Enable the sites (Debian/Ubuntu)
sudo a2ensite example.com.conf
sudo a2ensite blog.example.com.conf

# Disable the default site (optional)
sudo a2dissite 000-default.conf

# Test config
sudo apache2ctl configtest
# Output: Syntax OK

# Reload Apache
sudo systemctl reload apache2
```

### IP-Based Virtual Hosts

Different websites on different IP addresses:

```apache
# Site 1 on IP 192.168.1.10
<VirtualHost 192.168.1.10:80>
    ServerName site1.com
    DocumentRoot /var/www/site1
</VirtualHost>

# Site 2 on IP 192.168.1.11
<VirtualHost 192.168.1.11:80>
    ServerName site2.com
    DocumentRoot /var/www/site2
</VirtualHost>
```

### Port-Based Virtual Hosts

Different sites on different ports:

```apache
# First, tell Apache to listen on both ports
# In ports.conf:
Listen 80
Listen 8080

# Site 1 on port 80
<VirtualHost *:80>
    ServerName example.com
    DocumentRoot /var/www/main
</VirtualHost>

# Site 2 on port 8080
<VirtualHost *:8080>
    ServerName example.com
    DocumentRoot /var/www/dev
</VirtualHost>
```

### How Apache Decides Which Virtual Host to Use

```
1. Client connects to IP:Port
2. Apache finds all VirtualHosts matching that IP:Port
3. Apache checks the "Host" header from the client
4. Apache matches it to ServerName / ServerAlias
5. If no match → uses the FIRST VirtualHost defined (default)
```

> 💡 **Tip:** Always define a default virtual host first to catch unmatched requests.

## 8. Apache Modules — Extending Functionality

Apache's power comes from its **modular architecture**. You can load and unload features as needed, like plugins.

### Managing Modules (Debian/Ubuntu)

```bash
# List all available modules
apache2ctl -M           # Currently loaded modules
ls /etc/apache2/mods-available/   # All available modules

# Enable a module
sudo a2enmod rewrite     # Enable mod_rewrite
sudo a2enmod ssl         # Enable SSL
sudo a2enmod proxy       # Enable proxy
sudo a2enmod proxy_http  # Enable HTTP proxy
sudo a2enmod headers     # Enable custom headers
sudo a2enmod expires     # Enable browser caching

# Disable a module
sudo a2dismod autoindex  # Disable directory listing module

# Always reload after changing modules
sudo systemctl reload apache2
```

### Managing Modules (CentOS/RHEL)

```bash
# Modules are configured in /etc/httpd/conf.modules.d/
# Edit the files directly to comment/uncomment LoadModule lines

# List loaded modules
httpd -M
```

### Essential Modules for DevOps

| Module | What It Does | Enable With |
|--------|-------------|-------------|
| `mod_rewrite` | URL rewriting (pretty URLs, redirects) | `a2enmod rewrite` |
| `mod_ssl` | HTTPS/TLS support | `a2enmod ssl` |
| `mod_proxy` | Reverse proxy functionality | `a2enmod proxy` |
| `mod_proxy_http` | HTTP proxy support | `a2enmod proxy_http` |
| `mod_proxy_balancer` | Load balancing | `a2enmod proxy_balancer` |
| `mod_headers` | Manipulate HTTP headers | `a2enmod headers` |
| `mod_expires` | Browser cache control | `a2enmod expires` |
| `mod_deflate` | Gzip compression | `a2enmod deflate` |
| `mod_security` | Web Application Firewall (WAF) | Install separately |
| `mod_evasive` | DDoS/brute-force protection | Install separately |
| `mod_php` | Run PHP directly in Apache | `a2enmod php8.2` |
| `mod_wsgi` | Run Python WSGI apps | `a2enmod wsgi` |
| `mod_auth_basic` | HTTP Basic authentication | Usually loaded by default |
| `mod_status` | Server status page | `a2enmod status` |

## 9. .htaccess — Directory-Level Configuration

### What is .htaccess?

`.htaccess` (HyperText Access) is a **per-directory configuration file**. It lets you change Apache behavior for a specific directory WITHOUT editing the main config and WITHOUT restarting Apache.

**Think of it like this:**
- Main config (`apache2.conf`) = Building rules that apply to all apartments
- `.htaccess` = House rules for YOUR specific apartment

**Where to put it:** Place `.htaccess` in any directory you want to control. It affects that directory and all subdirectories.

```
/var/www/html/
├── .htaccess           ← Affects everything in /var/www/html/
├── index.html
├── about/
│   ├── .htaccess       ← Overrides parent .htaccess for /about/ and below
│   └── index.html
└── admin/
    ├── .htaccess       ← Separate rules for /admin/
    └── dashboard.html
```

### Enabling .htaccess

By default, `.htaccess` might be disabled. You need to enable it:

```apache
# In your virtual host or main config
<Directory /var/www/html>
    AllowOverride All     # Allow .htaccess to override everything
    Require all granted
</Directory>
```

`AllowOverride` options:
| Value | What .htaccess Can Control |
|-------|---------------------------|
| `None` | .htaccess is completely ignored |
| `All` | Everything |
| `AuthConfig` | Authentication settings only |
| `FileInfo` | File type, headers, rewrite rules |
| `Indexes` | Directory index settings |
| `Limit` | Access control |

> ⚠️ **Performance Warning:** Apache checks for `.htaccess` files in **every** directory in the path for **every** request. If a user requests `/var/www/html/images/photo.jpg`, Apache checks:
> - `/var/.htaccess`
> - `/var/www/.htaccess`
> - `/var/www/html/.htaccess`
> - `/var/www/html/images/.htaccess`
>
> That's 4 file reads per request! In high-performance environments, put rules in the main config instead and set `AllowOverride None`.

### Common .htaccess Examples

```apache
# ==========================================
# REDIRECTS
# ==========================================

# Redirect one page to another
Redirect 301 /old-page.html /new-page.html

# Redirect entire site to new domain
Redirect 301 / https://newdomain.com/

# Redirect HTTP to HTTPS
RewriteEngine On
RewriteCond %{HTTPS} off
RewriteRule ^(.*)$ https://%{HTTP_HOST}%{REQUEST_URI} [L,R=301]

# Redirect www to non-www
RewriteEngine On
RewriteCond %{HTTP_HOST} ^www\.(.+)$ [NC]
RewriteRule ^(.*)$ https://%1/$1 [R=301,L]

# ==========================================
# CUSTOM ERROR PAGES
# ==========================================

ErrorDocument 404 /errors/404.html
ErrorDocument 403 /errors/403.html
ErrorDocument 500 /errors/500.html

# ==========================================
# SECURITY
# ==========================================

# Block access to hidden files (. files)
<FilesMatch "^\.">
    Require all denied
</FilesMatch>

# Block access to sensitive files
<FilesMatch "\.(env|log|ini|sh|sql|bak|config)$">
    Require all denied
</FilesMatch>

# Prevent directory listing
Options -Indexes

# ==========================================
# CACHING (Browser)
# ==========================================

<IfModule mod_expires.c>
    ExpiresActive On
    ExpiresByType image/jpeg "access plus 1 month"
    ExpiresByType image/png "access plus 1 month"
    ExpiresByType text/css "access plus 1 week"
    ExpiresByType application/javascript "access plus 1 week"
    ExpiresByType text/html "access plus 0 seconds"
</IfModule>

# ==========================================
# COMPRESSION
# ==========================================

<IfModule mod_deflate.c>
    AddOutputFilterByType DEFLATE text/html text/plain text/css
    AddOutputFilterByType DEFLATE application/javascript application/json
    AddOutputFilterByType DEFLATE text/xml application/xml
</IfModule>

# ==========================================
# SECURITY HEADERS
# ==========================================

<IfModule mod_headers.c>
    Header always set X-Frame-Options "SAMEORIGIN"
    Header always set X-Content-Type-Options "nosniff"
    Header always set X-XSS-Protection "1; mode=block"
    Header always set Referrer-Policy "strict-origin-when-cross-origin"
    Header always set Strict-Transport-Security "max-age=31536000; includeSubDomains"
</IfModule>
```

## 10. URL Rewriting with mod_rewrite

`mod_rewrite` is one of Apache's most powerful modules. It lets you **transform URLs** using regular expressions.

### Enable mod_rewrite

```bash
sudo a2enmod rewrite
sudo systemctl reload apache2
```

### Basic Syntax

```apache
RewriteEngine On                    # Turn on the rewrite engine
RewriteCond %{VARIABLE} pattern     # Condition (optional, can have multiple)
RewriteRule pattern replacement [flags]  # The actual rule
```

### RewriteRule Explained

```apache
# Syntax: RewriteRule <URL-pattern> <replacement> [flags]

# Example: /products/123 → /product.php?id=123
RewriteRule ^products/([0-9]+)$ /product.php?id=$1 [L]
```

**Breaking it down:**

| Part | Meaning |
|------|---------|
| `^products/` | URL starts with "products/" |
| `([0-9]+)` | Capture one or more digits (stored as `$1`) |
| `$` | End of URL |
| `/product.php?id=$1` | Rewrite to this (internally, user doesn't see it) |
| `[L]` | Last rule — stop processing if this matches |

### RewriteRule Flags

| Flag | Name | What It Does |
|------|------|-------------|
| `[L]` | Last | Stop processing further rules |
| `[R=301]` | Redirect | Send a 301 redirect to the browser (URL changes) |
| `[R=302]` | Redirect | Send a 302 temporary redirect |
| `[NC]` | No Case | Case-insensitive matching |
| `[QSA]` | Query String Append | Keep existing query parameters |
| `[F]` | Forbidden | Return 403 Forbidden |
| `[G]` | Gone | Return 410 Gone |
| `[PT]` | Pass Through | Pass result to next handler |
| `[P]` | Proxy | Proxy the request (requires mod_proxy) |

### RewriteCond Variables

```apache
# Common conditions
RewriteCond %{HTTP_HOST}        # The requested hostname
RewriteCond %{HTTPS}            # "on" or "off"
RewriteCond %{REQUEST_URI}      # The requested URI
RewriteCond %{REQUEST_METHOD}   # GET, POST, etc.
RewriteCond %{QUERY_STRING}     # Query string parameters
RewriteCond %{HTTP_USER_AGENT}  # Browser user agent
RewriteCond %{REMOTE_ADDR}      # Client IP address
RewriteCond %{REQUEST_FILENAME} # Full filesystem path
```

### Practical Examples

```apache
RewriteEngine On

# ==========================================
# 1. Pretty URLs (used by most frameworks)
# ==========================================
# Send everything to index.php (Laravel, WordPress, etc.)
# /about → index.php  (but /images/logo.png stays as-is)
RewriteCond %{REQUEST_FILENAME} !-f     # If the file doesn't exist
RewriteCond %{REQUEST_FILENAME} !-d     # If the directory doesn't exist
RewriteRule ^(.*)$ /index.php [L,QSA]

# ==========================================
# 2. Force HTTPS
# ==========================================
RewriteCond %{HTTPS} off
RewriteRule ^(.*)$ https://%{HTTP_HOST}%{REQUEST_URI} [R=301,L]

# ==========================================
# 3. Remove .html extension
# ==========================================
# /about.html → /about
RewriteCond %{REQUEST_FILENAME} !-d
RewriteCond %{REQUEST_FILENAME}.html -f
RewriteRule ^(.*)$ $1.html [L]

# ==========================================
# 4. Force trailing slash
# ==========================================
RewriteCond %{REQUEST_FILENAME} !-f
RewriteCond %{REQUEST_URI} !(.*)/$
RewriteRule ^(.*)$ $1/ [L,R=301]

# ==========================================
# 5. Block bad bots
# ==========================================
RewriteCond %{HTTP_USER_AGENT} (SemrushBot|AhrefsBot|MJ12bot) [NC]
RewriteRule .* - [F,L]

# ==========================================
# 6. Maintenance mode
# ==========================================
# Show maintenance page to everyone except your IP
RewriteCond %{REMOTE_ADDR} !^203\.0\.113\.50$
RewriteCond %{REQUEST_URI} !^/maintenance\.html$
RewriteRule ^(.*)$ /maintenance.html [R=302,L]

# ==========================================
# 7. API versioning
# ==========================================
# /api/v1/users → /api/users?version=1
RewriteRule ^api/v([0-9]+)/(.*)$ /api/$2?version=$1 [L,QSA]
```

## 11. Apache as a Reverse Proxy

Apache can act as a reverse proxy, forwarding requests to backend servers (Node.js, Python, Java, etc.)

### Enable Required Modules

```bash
sudo a2enmod proxy
sudo a2enmod proxy_http
sudo a2enmod proxy_balancer    # For load balancing
sudo a2enmod lbmethod_byrequests  # Round-robin load balancing
sudo a2enmod headers
sudo systemctl reload apache2
```

### Simple Reverse Proxy

```apache
<VirtualHost *:80>
    ServerName app.example.com

    # Forward all requests to Node.js app on port 3000
    ProxyPreserveHost On
    ProxyPass / http://127.0.0.1:3000/
    ProxyPassReverse / http://127.0.0.1:3000/

    # Pass original client info to backend
    RequestHeader set X-Forwarded-For "%{REMOTE_ADDR}s"
    RequestHeader set X-Forwarded-Proto "%{REQUEST_SCHEME}s"

    ErrorLog ${APACHE_LOG_DIR}/app-error.log
    CustomLog ${APACHE_LOG_DIR}/app-access.log combined
</VirtualHost>
```

**What these directives do:**

| Directive | What It Does |
|-----------|-------------|
| `ProxyPreserveHost On` | Sends the original `Host` header to the backend (not `127.0.0.1`) |
| `ProxyPass /` | Forward requests from `/` to the backend |
| `ProxyPassReverse /` | Rewrite `Location` headers in backend responses (so redirects work) |

### Path-Based Proxy (Multiple Backends)

```apache
<VirtualHost *:80>
    ServerName example.com

    # API requests → Node.js (port 3000)
    ProxyPass /api/ http://127.0.0.1:3000/api/
    ProxyPassReverse /api/ http://127.0.0.1:3000/api/

    # Admin panel → Python Django (port 8000)
    ProxyPass /admin/ http://127.0.0.1:8000/admin/
    ProxyPassReverse /admin/ http://127.0.0.1:8000/admin/

    # Everything else → static files
    DocumentRoot /var/www/example.com/public

    # IMPORTANT: ProxyPass rules are matched in order
    # Put more specific paths first!
</VirtualHost>
```

### Load Balancing with mod_proxy_balancer

```apache
<VirtualHost *:80>
    ServerName app.example.com

    # Define a cluster of backend servers
    <Proxy "balancer://mycluster">
        BalancerMember http://127.0.0.1:3001 loadfactor=1
        BalancerMember http://127.0.0.1:3002 loadfactor=1
        BalancerMember http://127.0.0.1:3003 loadfactor=2   # Gets 2x traffic

        # Load balancing method
        ProxySet lbmethod=byrequests    # Round-robin (default)
        # ProxySet lbmethod=bytraffic   # By data volume
        # ProxySet lbmethod=bybusyness  # Least connections
        # ProxySet lbmethod=heartbeat   # Based on health checks
    </Proxy>

    ProxyPreserveHost On
    ProxyPass / balancer://mycluster/
    ProxyPassReverse / balancer://mycluster/
</VirtualHost>
```

### WebSocket Proxy

```apache
# Enable additional modules
# sudo a2enmod proxy_wstunnel

<VirtualHost *:80>
    ServerName app.example.com

    # Regular HTTP proxy
    ProxyPass / http://127.0.0.1:3000/
    ProxyPassReverse / http://127.0.0.1:3000/

    # WebSocket proxy (must come before the regular ProxyPass)
    RewriteEngine On
    RewriteCond %{HTTP:Upgrade} websocket [NC]
    RewriteCond %{HTTP:Connection} upgrade [NC]
    RewriteRule ^/?(.*) ws://127.0.0.1:3000/$1 [P,L]
</VirtualHost>
```

## 14. Logging

### Access Log Formats

```apache
# Combined format (most common — includes referer and user agent)
LogFormat "%h %l %u %t \"%r\" %>s %b \"%{Referer}i\" \"%{User-Agent}i\"" combined

# Common format (simpler)
LogFormat "%h %l %u %t \"%r\" %>s %b" common

# Custom format with response time
LogFormat "%h %l %u %t \"%r\" %>s %b %D \"%{Referer}i\" \"%{User-Agent}i\"" detailed
```

**Log format variables:**

| Variable | Meaning | Example |
|----------|---------|---------|
| `%h` | Client IP | `192.168.1.100` |
| `%l` | Identity (usually `-`) | `-` |
| `%u` | Authenticated user | `admin` or `-` |
| `%t` | Timestamp | `[29/May/2026:14:30:00 +0000]` |
| `%r` | Request line | `GET /page HTTP/1.1` |
| `%>s` | Final status code | `200` |
| `%b` | Response size (bytes) | `1234` |
| `%D` | Response time (microseconds) | `123456` |
| `%T` | Response time (seconds) | `0` |
| `%{Referer}i` | Referer header | `https://google.com` |
| `%{User-Agent}i` | Browser info | `Mozilla/5.0...` |

### Conditional Logging

```apache
# Don't log health check requests
SetEnvIf Request_URI "^/health" dontlog
CustomLog /var/log/apache2/access.log combined env=!dontlog

# Don't log static file requests
SetEnvIf Request_URI "\.(css|js|png|jpg|gif|ico)$" static
CustomLog /var/log/apache2/access.log combined env=!static

# Separate log for errors vs everything else
ErrorLog /var/log/apache2/error.log
LogLevel warn
```

### Log Rotation

Apache log files can grow huge. Rotate them automatically:

```bash
# View existing log rotation config
cat /etc/logrotate.d/apache2
```

```
# /etc/logrotate.d/apache2
/var/log/apache2/*.log {
    daily              # Rotate daily
    missingok          # Don't error if log is missing
    rotate 14          # Keep 14 days of logs
    compress           # Gzip old logs
    delaycompress      # Don't compress the most recent rotation
    notifempty         # Don't rotate empty files
    create 640 root adm   # Create new log with these permissions
    sharedscripts
    postrotate
        # Gracefully reload Apache to open new log files
        if invoke-rc.d apache2 status > /dev/null 2>&1; then
            invoke-rc.d apache2 reload > /dev/null
        fi
    endscript
}
```

## 15. Performance Tuning

### MPM Event Tuning

```apache
<IfModule mpm_event_module>
    # For a server with 4GB RAM:
    StartServers             4      # Start 4 child processes
    MinSpareThreads         75      # Keep at least 75 idle threads
    MaxSpareThreads        250      # Don't keep more than 250 idle threads
    ThreadsPerChild         25      # Each process has 25 threads
    MaxRequestWorkers      400      # Max 400 simultaneous connections
    MaxConnectionsPerChild 10000    # Recycle process after 10000 requests
</IfModule>
```

**How to calculate `MaxRequestWorkers`:**

```
MaxRequestWorkers = (Total RAM - RAM for OS and other apps) / RAM per Apache process

Example:
- 4GB total RAM
- 1GB for OS and apps
- Each Apache process uses ~50MB
- MaxRequestWorkers = 3000MB / 50MB = 60 processes
- With 25 threads each = 1500 concurrent connections
```

### Enable Compression

```apache
<IfModule mod_deflate.c>
    # Compress these types
    AddOutputFilterByType DEFLATE text/html
    AddOutputFilterByType DEFLATE text/plain
    AddOutputFilterByType DEFLATE text/css
    AddOutputFilterByType DEFLATE application/javascript
    AddOutputFilterByType DEFLATE application/json
    AddOutputFilterByType DEFLATE text/xml
    AddOutputFilterByType DEFLATE application/xml
    AddOutputFilterByType DEFLATE image/svg+xml

    # Don't compress images and already-compressed files
    SetEnvIfNoCase Request_URI \.(?:gif|jpe?g|png|webp|zip|gz|bz2)$ no-gzip

    # Minimum file size to compress (don't compress tiny files)
    DeflateCompressionLevel 6
</IfModule>
```

### Enable Browser Caching

```apache
<IfModule mod_expires.c>
    ExpiresActive On

    # Default: 1 hour
    ExpiresDefault "access plus 1 hour"

    # HTML: don't cache
    ExpiresByType text/html "access plus 0 seconds"

    # CSS & JavaScript: 1 week
    ExpiresByType text/css "access plus 1 week"
    ExpiresByType application/javascript "access plus 1 week"

    # Images: 1 month
    ExpiresByType image/jpeg "access plus 1 month"
    ExpiresByType image/png "access plus 1 month"
    ExpiresByType image/gif "access plus 1 month"
    ExpiresByType image/svg+xml "access plus 1 month"
    ExpiresByType image/webp "access plus 1 month"

    # Fonts: 1 year
    ExpiresByType font/woff2 "access plus 1 year"
    ExpiresByType font/woff "access plus 1 year"
    ExpiresByType application/font-woff2 "access plus 1 year"
</IfModule>
```

## 16. Security Hardening

```apache
# ==========================================
# HIDE INFORMATION
# ==========================================

# Don't show Apache version
ServerTokens Prod          # Shows only "Apache" instead of "Apache/2.4.58 (Ubuntu)"
ServerSignature Off        # Remove version from error pages

# Remove the Server header entirely (requires mod_headers)
Header always unset Server

# ==========================================
# DISABLE DANGEROUS FEATURES
# ==========================================

# Disable directory listing globally
<Directory />
    Options -Indexes
</Directory>

# Disable TRACE method (used in cross-site tracing attacks)
TraceEnable Off

# ==========================================
# BLOCK SENSITIVE FILES
# ==========================================

# Block .git, .env, .htaccess viewing, etc.
<FilesMatch "^\.">
    Require all denied
</FilesMatch>

# Block backup and config files
<FilesMatch "\.(bak|config|sql|fla|psd|ini|log|sh|inc|swp|dist|env)$">
    Require all denied
</FilesMatch>

# ==========================================
# PREVENT CLICKJACKING
# ==========================================
Header always set X-Frame-Options "SAMEORIGIN"

# ==========================================
# PREVENT MIME SNIFFING
# ==========================================
Header always set X-Content-Type-Options "nosniff"

# ==========================================
# REQUEST SIZE LIMITS
# ==========================================

# Limit request body size (prevents large file upload attacks)
LimitRequestBody 10485760    # 10MB max

# Limit request headers
LimitRequestFields 50        # Max number of headers
LimitRequestFieldSize 8190   # Max header size
LimitRequestLine 8190        # Max URL length

# ==========================================
# TIMEOUT SETTINGS
# ==========================================
Timeout 60                   # Connection timeout
RequestReadTimeout header=20-40,MinRate=500 body=20,MinRate=500
```

## 18. Troubleshooting Common Issues

| Problem | Likely Cause | Solution |
|---------|-------------|----------|
| "Permission denied" errors | Wrong file ownership | `sudo chown -R www-data:www-data /var/www/` |
| `.htaccess` not working | `AllowOverride None` | Change to `AllowOverride All` and reload |
| `mod_rewrite` not working | Module not enabled | `sudo a2enmod rewrite && sudo systemctl reload apache2` |
| Port 80 already in use | Another service (Nginx?) on port 80 | `sudo lsof -i :80` to find it, stop the other service |
| "Could not bind to address" | Apache already running or port taken | `sudo systemctl stop apache2` first, check for duplicates |
| Virtual host not responding | Site not enabled | `sudo a2ensite yoursite.conf && sudo systemctl reload apache2` |
| SSL certificate errors | Wrong path or permissions | Check paths in config, ensure files are readable by `www-data` |
| 403 Forbidden | Wrong `Require` directive or file permissions | Check `<Directory>` settings and file permissions |
| Blank page for PHP | `mod_php` not installed or not enabled | `sudo apt install libapache2-mod-php && sudo a2enmod php8.2` |
| "AH00558: Could not reliably determine the server's fully qualified domain name" | Missing `ServerName` | Add `ServerName localhost` to `apache2.conf` |

**Debug checklist:**
```bash
# 1. Check syntax
sudo apache2ctl configtest

# 2. Check error log
sudo tail -20 /var/log/apache2/error.log

# 3. Check permissions
ls -la /var/www/html/
namei -l /var/www/html/index.html

# 4. Check what Apache sees
sudo apache2ctl -S    # Virtual host mapping
sudo apache2ctl -M    # Loaded modules

# 5. Test the request
curl -v http://localhost/
```
