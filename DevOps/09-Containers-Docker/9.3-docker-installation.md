## 🤔 Before You Install — Understanding the Options

Docker wasn't designed for Windows or Mac. It relies on **Linux kernel features** (namespaces, cgroups). So how does it run on other platforms?

| Platform | How Docker Runs |
|----------|----------------|
| **Linux** | **Natively** — Docker talks directly to the Linux kernel. Best performance. |
| **Mac** | Via **Docker Desktop** — runs a lightweight Linux VM (using Apple's Virtualization framework or HyperKit) |
| **Windows (WSL2)** | Via **Docker Desktop + WSL2** — WSL2 provides a real Linux kernel. Recommended for Windows. |
| **Windows (Hyper-V)** | Via **Docker Desktop + Hyper-V** — runs a Linux VM using Hyper-V. Older approach. |
| **Windows Server** | Natively for **Windows containers**, or via WSL2/Hyper-V for Linux containers |

### Docker Desktop vs Docker Engine:

| Feature | Docker Engine | Docker Desktop |
|---------|--------------|----------------|
| **What** | The core Docker runtime (daemon + CLI) | A full GUI application with Docker Engine bundled |
| **Platform** | Linux only | Mac, Windows, Linux |
| **GUI** | No (CLI only) | Yes (dashboard, Kubernetes, extensions) |
| **Cost** | Free (open-source) | Free for personal/small business, **Paid** for large enterprises (250+ employees or $10M+ revenue) |
| **Includes** | dockerd, docker CLI, containerd | Docker Engine + Docker Compose + Kubernetes + GUI + Extensions |

**Bottom line:**
- On **Linux servers** → Install Docker Engine (free, CLI-only)
- On **Mac/Windows** (development) → Install Docker Desktop
- On **Windows** → Always use the **WSL2 backend** (it's faster and better)
- On **Windows** the Docker Desktop application must start its backend before the Docker daemon is available. Simply having Docker Desktop installed is not enough; Docker Desktop initializes the WSL2 VM and engine.

## 🐧 Option 1: Install Docker on Linux (Ubuntu/Debian)

This is the most common server setup. We'll cover Ubuntu (which also applies to Debian) since it's the most popular.

### Step 1: Remove Old Docker Versions (if any)

Old versions might be installed as `docker`, `docker.io`, or `docker-engine`. Let's remove them:

```bash
# Remove old versions
sudo apt-get remove docker docker-engine docker.io containerd runc

# It's OK if apt-get says none of these are installed
```

### Step 2: Update System Packages

```bash
# Update the package index
sudo apt-get update

# Install prerequisite packages
sudo apt-get install -y \
    ca-certificates \
    curl \
    gnupg \
    lsb-release
```

**What are these packages?**
| Package | Why We Need It |
|---------|---------------|
| `ca-certificates` | For HTTPS connections to Docker's repository |
| `curl` | To download Docker's GPG key |
| `gnupg` | To verify the GPG key (ensures packages are genuine) |
| `lsb-release` | To detect your Ubuntu/Debian version automatically |

### Step 3: Add Docker's Official GPG Key

```bash
# Create the keyrings directory
sudo install -m 0755 -d /etc/apt/keyrings

# Download Docker's official GPG key
sudo curl -fsSL https://download.docker.com/linux/ubuntu/gpg -o /etc/apt/keyrings/docker.asc

# Make it readable
sudo chmod a+r /etc/apt/keyrings/docker.asc
```

**What's a GPG key?**
- It's a cryptographic key used to **verify** that the Docker packages you download are actually from Docker and haven't been tampered with
- Think of it like a seal of authenticity

### Step 4: Add Docker's Repository

```bash
# Add the Docker repository to apt sources
echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.asc] https://download.docker.com/linux/ubuntu \
  $(. /etc/os-release && echo "${VERSION_CODENAME}") stable" | \
  sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
```

**What this does:**
- Adds Docker's package repository to your system
- `$(dpkg --print-architecture)` detects your CPU architecture (amd64, arm64, etc.)
- `$(. /etc/os-release && echo "${VERSION_CODENAME}")` detects your Ubuntu version (jammy, noble, etc.)
- `signed-by=...` tells apt to verify packages using Docker's GPG key

### Step 5: Install Docker Engine

```bash
# Update package index (now includes Docker's repo)
sudo apt-get update

# Install Docker Engine, CLI, containerd, and plugins
sudo apt-get install -y \
    docker-ce \
    docker-ce-cli \
    containerd.io \
    docker-buildx-plugin \
    docker-compose-plugin
```

**What each package does:**

| Package | Purpose |
|---------|---------|
| `docker-ce` | Docker Engine (Community Edition) — the daemon |
| `docker-ce-cli` | Docker CLI — the command-line tool |
| `containerd.io` | containerd — the container runtime |
| `docker-buildx-plugin` | BuildKit-based builder (multi-platform builds) |
| `docker-compose-plugin` | Docker Compose v2 (run multi-container apps) |

### Step 6: Verify the Installation

```bash
# Check Docker version
docker --version
# Output: Docker version 27.x.x, build xxxxxxx

# Check all component versions
docker version
# Shows Client AND Server versions

# Run the hello-world test container
sudo docker run hello-world
```

The `hello-world` output should look like:

```
Hello from Docker!
This message shows that your installation appears to be working correctly.

To generate this message, Docker took the following steps:
 1. The Docker client contacted the Docker daemon.
 2. The Docker daemon pulled the "hello-world" image from the Docker Hub.
 3. The Docker daemon created a new container from that image which runs the
    executable that produces the output you are currently reading.
 4. The Docker daemon streamed that output to the Docker client, which sent it
    to your terminal.
```

If you see this, Docker is working! 🎉

### Step 7: Run Docker Without `sudo` (Optional but Recommended)

By default, you need `sudo` for every Docker command. Let's fix that:

```bash
# Create the docker group (may already exist)
sudo groupadd docker

# Add your user to the docker group
sudo usermod -aG docker $USER

# Apply the new group membership
# OPTION 1: Log out and log back in
# OPTION 2: Use this command (for current session)
newgrp docker

# Test without sudo
docker run hello-world
# Should work without sudo now! ✅
```

> ⚠️ **Security Note:** Adding a user to the `docker` group gives them root-equivalent access. Only do this for trusted users. (We explained why in section 9.2)

### Step 8: Configure Docker to Start on Boot

```bash
# Enable Docker to start automatically on boot
sudo systemctl enable docker.service
sudo systemctl enable containerd.service

# Check Docker service status
sudo systemctl status docker
```

### Step 9: Verify Everything is Working

```bash
# Check Docker info (shows system-wide info)
docker info

# Key things to look for in the output:
#  Server Version: 27.x.x
#  Storage Driver: overlay2
#  Cgroup Driver: systemd
#  Kernel Version: 5.15.x
#  Operating System: Ubuntu 22.04.x LTS
#  Total Memory: 16 GiB
#  CPUs: 4
```

## 🐧 Option 1b: Install Docker on Linux (CentOS/RHEL/Fedora)

### For CentOS/RHEL:

```bash
# Step 1: Remove old versions
sudo yum remove docker \
    docker-client \
    docker-client-latest \
    docker-common \
    docker-latest \
    docker-latest-logrotate \
    docker-logrotate \
    docker-engine

# Step 2: Install prerequisites
sudo yum install -y yum-utils

# Step 3: Add Docker repository
sudo yum-config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo

# Step 4: Install Docker
sudo yum install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

# Step 5: Start Docker
sudo systemctl start docker
sudo systemctl enable docker

# Step 6: Verify
sudo docker run hello-world

# Step 7: Add user to docker group
sudo usermod -aG docker $USER
```

### For Fedora:

```bash
# Step 1: Remove old versions
sudo dnf remove docker \
    docker-client \
    docker-client-latest \
    docker-common \
    docker-latest \
    docker-latest-logrotate \
    docker-logrotate \
    docker-selinux \
    docker-engine-selinux \
    docker-engine

# Step 2: Install prerequisites
sudo dnf -y install dnf-plugins-core

# Step 3: Add Docker repository
sudo dnf-3 config-manager --add-repo https://download.docker.com/linux/fedora/docker-ce.repo

# Step 4: Install Docker
sudo dnf install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

# Step 5: Start Docker
sudo systemctl start docker
sudo systemctl enable docker

# Step 6: Verify
sudo docker run hello-world
```

## 🍎 Option 2: Install Docker on macOS

On Mac, you use **Docker Desktop** — a native Mac application that includes everything you need.

### System Requirements:

| Chip | Requirements |
|------|-------------|
| **Apple Silicon (M1/M2/M3/M4)** | macOS 12 (Monterey) or later, 4 GB RAM minimum |
| **Intel** | macOS 12 (Monterey) or later, 4 GB RAM minimum |

### Installation Steps:

#### Method 1: Download from Website (Recommended)

```
1. Go to https://docs.docker.com/desktop/install/mac-install/

2. Download the correct version:
   - Apple Silicon (M1/M2/M3/M4): "Docker Desktop for Mac with Apple Silicon"
   - Intel: "Docker Desktop for Mac with Intel chip"

3. Open the downloaded .dmg file

4. Drag Docker.app to your Applications folder

5. Open Docker from Applications
   - It will ask for your password (needs privileged helper)
   - Accept the license agreement

6. Wait for Docker to start (whale icon in menu bar)
   - Green = running ✅
   - Yellow = starting 🟡
   - Red = error 🔴
```

#### Method 2: Install via Homebrew

```bash
# Install Docker Desktop using Homebrew
brew install --cask docker

# Open Docker Desktop
open /Applications/Docker.app

# Wait for it to start, then verify
docker --version
docker run hello-world
```

### Post-Installation Configuration:

Click the Docker icon in the menu bar → **Settings/Preferences**:

| Setting | Recommendation |
|---------|---------------|
| **General → Start Docker Desktop when you log in** | Enable if you use Docker daily |
| **Resources → CPUs** | Allocate at least 2 CPUs (more if building heavy images) |
| **Resources → Memory** | At least 4 GB (8 GB recommended for Kubernetes) |
| **Resources → Disk image size** | 60-100 GB for most use cases |
| **Docker Engine** | Edit daemon.json here if needed |
| **Kubernetes** | Enable if you want a local Kubernetes cluster |

### Verify Installation:

```bash
# Check Docker version
docker --version

# Check Docker Compose version
docker compose version

# Run test container
docker run hello-world

# Run an interactive Ubuntu container
docker run -it ubuntu bash
# You're now inside an Ubuntu container on your Mac! 🎉
# Type 'exit' to leave
```

## 🪟 Option 3: Install Docker on Windows (with WSL2) — Recommended

This is the **recommended approach** for Windows users. WSL2 (Windows Subsystem for Linux 2) provides a real Linux kernel, which gives Docker native Linux performance.

### What is WSL2?

WSL2 is a feature of Windows that lets you run a **real Linux kernel** inside Windows. It's not a VM in the traditional sense — it's deeply integrated into Windows.

```
┌──────────────────────────────────────┐
│           WINDOWS                    │
│                                      │
│  ┌────────────────────────────────┐  │
│  │  WSL2 (Real Linux Kernel)     │  │
│  │                                │  │
│  │  ┌──────────────────────────┐  │  │
│  │  │  Docker Engine           │  │  │
│  │  │  (runs natively here!)   │  │  │
│  │  │                          │  │  │
│  │  │  ┌──────┐ ┌──────┐     │  │  │
│  │  │  │ C1   │ │ C2   │     │  │  │
│  │  │  └──────┘ └──────┘     │  │  │
│  │  └──────────────────────────┘  │  │
│  └────────────────────────────────┘  │
│                                      │
│  Docker CLI works from:              │
│  - Windows Terminal (PowerShell/CMD) │
│  - WSL2 Ubuntu terminal              │
│  - VS Code terminal                  │
└──────────────────────────────────────┘
```

### System Requirements:

| Requirement | Details |
|-------------|---------|
| **OS** | Windows 10 (version 2004, Build 19041 or later) or Windows 11 |
| **Architecture** | 64-bit (x64 or ARM64) |
| **RAM** | 4 GB minimum (8 GB+ recommended) |
| **Virtualization** | Must be enabled in BIOS (usually on by default) |
| **WSL2** | Must be installed (we'll do this below) |

### Step 1: Enable WSL2

Open **PowerShell as Administrator** and run:

```powershell
# Install WSL with Ubuntu (default distro)
wsl --install

# This command does several things:
# 1. Enables the "Virtual Machine Platform" feature
# 2. Enables the "Windows Subsystem for Linux" feature
# 3. Downloads and installs the Linux kernel update
# 4. Sets WSL2 as the default version
# 5. Downloads and installs Ubuntu

# RESTART YOUR COMPUTER after this!
```

After restart, Ubuntu will launch automatically and ask you to create a username and password.

```powershell
# Verify WSL2 is installed and running
wsl --list --verbose

# Output should show:
#   NAME      STATE           VERSION
# * Ubuntu    Running         2

# Make sure VERSION is 2 (not 1)
# If it shows VERSION 1, run:
wsl --set-version Ubuntu 2
```

### Step 2: Install Docker Desktop

```
1. Download Docker Desktop for Windows from:
   https://docs.docker.com/desktop/install/windows-install/

2. Run the installer (Docker Desktop Installer.exe)

3. During installation, make sure these options are checked:
   ✅ "Use WSL 2 instead of Hyper-V" (IMPORTANT!)
   ✅ "Add shortcut to desktop"

4. Click "Ok" and wait for installation to complete

5. Click "Close and restart" (another restart needed)

6. After restart, Docker Desktop will start
   - Accept the license agreement
   - Skip/Complete the tutorial
```

### Step 3: Configure Docker Desktop for WSL2

```
1. Open Docker Desktop
2. Go to Settings (gear icon)
3. Navigate to "General"
   ✅ "Use the WSL 2 based engine" (should already be checked)

4. Navigate to "Resources" → "WSL Integration"
   ✅ Enable integration with your Ubuntu distro
   - Toggle ON for "Ubuntu" (or whatever distro you installed)
   - This allows you to use the docker command from inside WSL2

5. Click "Apply & Restart"
```

### Step 4: Verify Installation

**From PowerShell or Command Prompt (Windows side):**

```powershell
# Check Docker version
docker --version

# Check Docker Compose version
docker compose version

# Run test container
docker run hello-world
```

**From WSL2 Ubuntu terminal:**

```bash
# Open WSL2 Ubuntu
wsl

# Check Docker version
docker --version

# Run test container
docker run hello-world

# Run an interactive container
docker run -it ubuntu bash
# You're inside a container, inside WSL2, inside Windows! 🤯
```

### Step 5: Verify WSL2 Integration is Working

```bash
# From WSL2 Ubuntu terminal:

# Check Docker is using WSL2 backend
docker info | grep -i "operating system"
# Should show something like: Operating System: Docker Desktop

# Check containers run correctly
docker run -d --name test-nginx -p 8080:80 nginx

# Test from Windows browser: http://localhost:8080
# You should see the Nginx welcome page! 🎉

# Clean up
docker rm -f test-nginx
```

## 🪟 Option 3b: Install Docker Inside WSL2 (Without Docker Desktop)

If you don't want to use Docker Desktop (maybe to avoid licensing costs), you can install Docker Engine directly inside WSL2.

### When to Use This Approach:

| Docker Desktop | Docker in WSL2 (no Desktop) |
|---------------|----------------------------|
| GUI dashboard | CLI only |
| Kubernetes included | Need to install separately |
| Auto-starts with Windows | Need to start manually |
| License required for large companies | Always free |
| Easier for beginners | More control, more work |

### Installation Steps:

```bash
# Open your WSL2 Ubuntu terminal

# Step 1: Remove any old Docker packages
sudo apt-get remove docker docker-engine docker.io containerd runc

# Step 2: Install prerequisites
sudo apt-get update
sudo apt-get install -y \
    ca-certificates \
    curl \
    gnupg \
    lsb-release

# Step 3: Add Docker's GPG key
sudo install -m 0755 -d /etc/apt/keyrings
sudo curl -fsSL https://download.docker.com/linux/ubuntu/gpg -o /etc/apt/keyrings/docker.asc
sudo chmod a+r /etc/apt/keyrings/docker.asc

# Step 4: Add Docker repository
echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.asc] https://download.docker.com/linux/ubuntu \
  $(. /etc/os-release && echo "${VERSION_CODENAME}") stable" | \
  sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# Step 5: Install Docker
sudo apt-get update
sudo apt-get install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

# Step 6: Start Docker service
# Note: WSL2 doesn't use systemd by default (newer versions do)
# If systemd is available:
sudo systemctl start docker
sudo systemctl enable docker

# If systemd is NOT available:
sudo service docker start

# Step 7: Add your user to docker group
sudo usermod -aG docker $USER
newgrp docker

# Step 8: Verify
docker run hello-world
```

### Enable systemd in WSL2 (if not already enabled):

Edit or create `/etc/wsl.conf`:

```bash
sudo nano /etc/wsl.conf
```

Add:

```ini
[boot]
systemd=true
```

Then restart WSL2 from PowerShell:

```powershell
wsl --shutdown
wsl
```

Now Docker will auto-start with WSL2.

### Auto-Start Docker When WSL2 Launches:

If you're not using systemd, add this to your `~/.bashrc` or `~/.profile`:

```bash
# Add to ~/.bashrc
if service docker status 2>&1 | grep -q "is not running"; then
    wsl.exe -d "${WSL_DISTRO_NAME}" -u root -e /usr/sbin/service docker start > /dev/null 2>&1
fi
```

## 🏢 Option 4: Install Docker on a Linux Server (Cloud/Production)

For production servers (AWS EC2, Azure VM, GCP Compute Engine), the installation is the same as Linux, but here's a **quick one-liner script** that Docker provides:

### Convenience Script (Quick Install):

```bash
# Download and run Docker's convenience script
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh

# This script:
# - Detects your Linux distro
# - Adds Docker's repository
# - Installs Docker Engine, CLI, containerd
# - Does everything automatically

# Post-install:
sudo usermod -aG docker $USER
sudo systemctl enable docker
```

> ⚠️ **Warning:** The convenience script is great for dev/test environments but for production:
> - Always review scripts before running them with `sudo`
> - Use the manual installation method for production servers
> - Pin Docker to a specific version for stability

### Pin Docker to a Specific Version (Production):

```bash
# List available versions
apt-cache madison docker-ce

# Install a specific version
VERSION_STRING=5:27.0.3-1~ubuntu.22.04~jammy
sudo apt-get install -y \
    docker-ce=$VERSION_STRING \
    docker-ce-cli=$VERSION_STRING \
    containerd.io

# Hold the version to prevent auto-updates
sudo apt-mark hold docker-ce docker-ce-cli containerd.io

# To unhold later:
sudo apt-mark unhold docker-ce docker-ce-cli containerd.io
```

## ⚙️ Post-Installation Configuration (All Platforms)

After installing Docker on any platform, here are important configurations.

### 1. Configure Daemon Settings

Create or edit `/etc/docker/daemon.json` (Linux) or configure via Docker Desktop Settings (Mac/Windows):

```json
{
  "storage-driver": "overlay2",
  "log-driver": "json-file",
  "log-opts": {
    "max-size": "10m",
    "max-file": "3"
  },
  "default-address-pools": [
    {
      "base": "172.17.0.0/16",
      "size": 24
    }
  ],
  "live-restore": true,
  "userland-proxy": false,
  "features": {
    "buildkit": true
  }
}
```

**Important settings explained:**

| Setting | Why It Matters |
|---------|---------------|
| `log-opts.max-size` + `max-file` | **Critical for production!** Without this, container logs grow forever and fill your disk |
| `live-restore: true` | Containers keep running when Docker daemon is restarted (great for daemon updates) |
| `buildkit: true` | Use BuildKit for faster, better builds (default in newer versions) |
| `userland-proxy: false` | Better performance for port forwarding (uses iptables instead of a proxy process) |

After editing, restart Docker:

```bash
sudo systemctl restart docker
```

### 2. Configure Log Rotation (Very Important!)

If you don't configure log rotation, container logs can fill up your entire disk:

```bash
# Check how much space Docker logs are using
sudo du -sh /var/lib/docker/containers/*/*-json.log

# You might see something like:
# 2.1G /var/lib/docker/containers/abc123.../abc123...-json.log
# 😱 That's a 2GB log file from one container!
```

The `daemon.json` config above handles this globally, but you can also set it per-container:

```bash
# Limit logs for a specific container
docker run -d \
    --log-opt max-size=10m \
    --log-opt max-file=3 \
    nginx
```

### 3. Set Up Docker Compose

Docker Compose v2 comes as a Docker plugin (already installed if you followed the steps above):

```bash
# Check Compose version
docker compose version
# Output: Docker Compose version v2.x.x

# Note: It's "docker compose" (with a space), NOT "docker-compose" (with a hyphen)
# The old "docker-compose" was a separate Python tool (v1) — it's deprecated
```

### 4. Test Your Complete Setup

Run this full test to make sure everything works:

```bash
# Test 1: Basic container
docker run hello-world
echo "✅ Basic container works"

# Test 2: Interactive container
docker run -it --rm ubuntu cat /etc/os-release
echo "✅ Interactive container works"

# Test 3: Port mapping
docker run -d --name test-web -p 8080:80 nginx
sleep 2
curl -s http://localhost:8080 | head -5
docker rm -f test-web
echo "✅ Port mapping works"

# Test 4: Volume mounting
docker run --rm -v $(pwd):/data ubuntu ls /data
echo "✅ Volume mounting works"

# Test 5: Docker Compose
cat > /tmp/test-compose.yml << 'EOF'
services:
  web:
    image: nginx
    ports:
      - "8080:80"
EOF
docker compose -f /tmp/test-compose.yml up -d
sleep 2
curl -s http://localhost:8080 | head -5
docker compose -f /tmp/test-compose.yml down
rm /tmp/test-compose.yml
echo "✅ Docker Compose works"

# Test 6: Build an image
cat > /tmp/Dockerfile << 'EOF'
FROM alpine
RUN echo "Hello from my custom image!"
CMD ["echo", "It works!"]
EOF
docker build -t test-image /tmp/ -f /tmp/Dockerfile
docker run --rm test-image
docker rmi test-image
rm /tmp/Dockerfile
echo "✅ Image building works"

echo ""
echo "🎉 All tests passed! Docker is fully operational."
```

## 🔧 Troubleshooting Common Installation Issues

### Issue 1: "permission denied" When Running Docker

```bash
# Error: Got permission denied while trying to connect to the Docker daemon socket

# Solution: Add your user to the docker group
sudo usermod -aG docker $USER

# Then log out and log back in, or run:
newgrp docker
```

### Issue 2: Docker Daemon Won't Start

```bash
# Check daemon status
sudo systemctl status docker

# Check daemon logs
sudo journalctl -u docker.service -n 50

# Common causes:
# 1. Invalid daemon.json — check JSON syntax
sudo cat /etc/docker/daemon.json | python3 -m json.tool

# 2. Port conflict — another process using the same port
sudo ss -tlnp | grep 2376

# 3. Storage issues — disk full
df -h /var/lib/docker
```

### Issue 3: Can't Pull Images (Network Issues)

```bash
# Error: Error response from daemon: Get https://registry-1.docker.io/v2/...

# Solution 1: Check DNS
nslookup registry-1.docker.io

# Solution 2: Check if proxy is needed
# Add proxy config to /etc/docker/daemon.json or systemd override

# Create the systemd override directory
sudo mkdir -p /etc/systemd/system/docker.service.d

# Create proxy config
sudo tee /etc/systemd/system/docker.service.d/http-proxy.conf << 'EOF'
[Service]
Environment="HTTP_PROXY=http://proxy.example.com:8080"
Environment="HTTPS_PROXY=http://proxy.example.com:8080"
Environment="NO_PROXY=localhost,127.0.0.1"
EOF

# Reload and restart
sudo systemctl daemon-reload
sudo systemctl restart docker
```

### Issue 4: WSL2 — Docker Command Not Found

```bash
# Make sure Docker Desktop's WSL2 integration is enabled:
# Docker Desktop → Settings → Resources → WSL Integration
# Toggle ON for your distro

# Or if using Docker Engine in WSL2 directly:
sudo service docker start
```

### Issue 5: Slow Performance on Mac/Windows

```
# Docker Desktop → Settings → Resources
# Increase CPUs: At least 2 (4 recommended)
# Increase Memory: At least 4 GB (8 GB recommended)
# Increase Disk: At least 60 GB

# On Mac with file sharing issues:
# Use :cached or :delegated volume mount options
docker run -v $(pwd):/app:cached myimage
```

### Issue 6: "Cannot connect to the Docker daemon"

```bash
# Error: Cannot connect to the Docker daemon at unix:///var/run/docker.sock

# Is Docker running?
sudo systemctl start docker

# Is the socket accessible?
ls -la /var/run/docker.sock
# Should be: srw-rw---- 1 root docker

# Is your user in the docker group?
groups $USER
# Should include: docker
```

## 🗑️ Uninstalling Docker

### On Linux (Ubuntu/Debian):

```bash
# Remove Docker packages
sudo apt-get purge -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

# Remove all images, containers, volumes, and config
sudo rm -rf /var/lib/docker
sudo rm -rf /var/lib/containerd
sudo rm -rf /etc/docker

# Remove the Docker group
sudo groupdel docker
```

### On macOS:

```bash
# From Applications, drag Docker to Trash
# Or via Homebrew:
brew uninstall --cask docker

# Clean up remaining files
rm -rf ~/Library/Group\ Containers/group.com.docker
rm -rf ~/Library/Containers/com.docker.docker
rm -rf ~/.docker
```

### On Windows:

```
1. Settings → Apps → Installed Apps → Docker Desktop → Uninstall
2. (Optional) Uninstall WSL2: wsl --unregister Ubuntu
```

## 🎯 Interview Questions You Should Be Able to Answer

1. **How do you install Docker on Linux?**
   → Add Docker's GPG key and repository, then `apt-get install docker-ce docker-ce-cli containerd.io`. Run `docker run hello-world` to verify.

2. **What's the difference between Docker Engine and Docker Desktop?**
   → Docker Engine is the core runtime (CLI + daemon), Linux-only, free. Docker Desktop is a GUI application for Mac/Windows/Linux that includes Docker Engine plus extras (Compose, Kubernetes, GUI). Paid for large enterprises.

3. **How does Docker run on macOS/Windows if it needs a Linux kernel?**
   → Docker Desktop runs a lightweight Linux VM. On Windows with WSL2, it uses WSL2's real Linux kernel. On Mac, it uses Apple's Virtualization framework.

4. **Why should Windows users prefer WSL2 over Hyper-V for Docker?**
   → WSL2 provides a real Linux kernel with better performance, lower memory usage, and faster startup times compared to the Hyper-V backend.

5. **What is the Docker socket and why is it a security concern?**
   → `/var/run/docker.sock` — anyone with access to this socket can control Docker and effectively has root access to the host. Only add trusted users to the `docker` group.

6. **How do you configure Docker daemon settings?**
   → Edit `/etc/docker/daemon.json` and restart the Docker service. Common settings include storage driver, log rotation, DNS, and registry mirrors.

7. **What's the most important production config people forget?**
   → Log rotation! Without `max-size` and `max-file` in the log options, container logs grow infinitely and can fill up the disk.
