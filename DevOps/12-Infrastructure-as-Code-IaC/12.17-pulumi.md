## 1. What is Pulumi?

**Pulumi** is an Infrastructure as Code (IaC) tool that lets you define your cloud infrastructure using **real programming languages** you already know — like **TypeScript**, **Python**, **Go**, **C#**, **Java**, and **YAML**.

### The Big Idea

Instead of learning a special language (like Terraform's HCL), Pulumi says:

> "Hey, you already know Python/TypeScript/Go? Great! Just use that to define your infrastructure."

### Simple Analogy

Think of it this way:
- **Terraform/HCL** = Writing instructions in a special form (like filling out a government form — specific format, limited expressiveness)
- **Pulumi** = Writing instructions in plain English (or any language you speak) — you can use loops, conditions, classes, functions, EVERYTHING

### Key Facts

| Feature | Details |
|---------|---------|
| **Created By** | Pulumi Corporation (2017) |
| **License** | Apache 2.0 (open-source) |
| **Languages** | TypeScript, Python, Go, C#, Java, YAML |
| **Cloud Support** | AWS, Azure, GCP, Kubernetes, 150+ providers |
| **State** | Pulumi Cloud (default), S3, Azure Blob, local files |
| **CLI Tool** | `pulumi` |

## 2. Why Pulumi? — The Problem It Solves

### The Limitations of HCL (Terraform's Language)

HCL is great, but it has limitations:

```hcl
# Terraform HCL — Creating 3 S3 buckets with different configs
# This gets awkward quickly...

variable "buckets" {
  type = map(object({
    versioning = bool
    acl        = string
  }))
  default = {
    "logs"    = { versioning = true, acl = "private" }
    "assets"  = { versioning = false, acl = "public-read" }
    "backups" = { versioning = true, acl = "private" }
  }
}

resource "aws_s3_bucket" "buckets" {
  for_each = var.buckets
  bucket   = "mycompany-${each.key}"
}

# Need separate resource blocks for each setting...
# Gets messy with complex logic
```

### The Pulumi Way (Python)

```python
# Pulumi Python — Same thing, but with real programming power
import pulumi_aws as aws

buckets_config = {
    "logs":    {"versioning": True,  "acl": "private"},
    "assets":  {"versioning": False, "acl": "public-read"},
    "backups": {"versioning": True,  "acl": "private"},
}

for name, config in buckets_config.items():
    bucket = aws.s3.Bucket(
        f"mycompany-{name}",
        bucket=f"mycompany-{name}",
        versioning=aws.s3.BucketVersioningArgs(
            enabled=config["versioning"]
        ),
    )
    
    # You can use if/else, loops, functions — anything Python can do!
    if config["versioning"]:
        pulumi.log.info(f"Bucket {name} has versioning enabled")
```

**See the difference?** With Pulumi, you get:
- ✅ Real loops (for, while)
- ✅ Real conditionals (if/else/elif)
- ✅ Functions and classes
- ✅ Error handling (try/except)
- ✅ Package managers (pip, npm)
- ✅ IDE support (autocomplete, type checking, refactoring)
- ✅ Unit testing with familiar frameworks

## 3. Pulumi vs Terraform — Honest Comparison

| Feature | Terraform | Pulumi |
|---------|-----------|--------|
| **Language** | HCL (domain-specific) | Python, TypeScript, Go, C#, Java, YAML |
| **Learning Curve** | Learn HCL | Use languages you know |
| **IDE Support** | Basic (HCL plugins) | Full (IntelliSense, type checking, debugging) |
| **Testing** | Terratest (Go only) | Native testing in any supported language |
| **Abstraction** | Modules (HCL) | Classes, functions, packages, inheritance |
| **State** | terraform.tfstate | Pulumi Cloud, S3, local |
| **Plan** | `terraform plan` | `pulumi preview` |
| **Apply** | `terraform apply` | `pulumi up` |
| **Destroy** | `terraform destroy` | `pulumi destroy` |
| **Loops** | `count`, `for_each` (limited) | Real `for` loops |
| **Conditions** | Ternary only | Full `if/else/elif` |
| **Community** | Very large | Growing rapidly |
| **Providers** | 4000+ | 150+ (can use Terraform providers via Bridge) |
| **Drift Detection** | `terraform plan` | `pulumi refresh` |
| **Pricing** | Open-source CLI (BSL), paid Cloud | Open-source CLI (Apache 2.0), paid Cloud |

### When Terraform Wins

- Larger community and more learning resources
- More battle-tested in production
- HCL is simpler for people who don't code
- More providers available natively

### When Pulumi Wins

- Teams with strong programming backgrounds
- Complex infrastructure with lots of logic
- Need for unit testing infrastructure
- Projects that mix application code and infrastructure
- When you want IDE features like autocomplete and debugging

## 4. How Pulumi Works — Architecture

### The Flow

```
┌──────────────────┐
│  Your Code       │  ← You write Python/TypeScript/Go
│  (index.ts,      │
│   __main__.py)   │
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│  Pulumi CLI       │  ← The `pulumi` command
│  (pulumi up)      │
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│  Language Runtime │  ← Runs your code (Node.js, Python, Go runtime)
│  (SDK)            │
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│  Pulumi Engine    │  ← Figures out what to create/update/delete
│  (Deployment      │
│   Engine)         │
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│  Cloud Providers  │  ← Actually talks to AWS/Azure/GCP APIs
│  (AWS, Azure,     │
│   GCP, K8s)       │
└──────────────────┘
```

### How It Works Step by Step

1. **You write code** in Python, TypeScript, etc. that describes your infrastructure.
2. **You run `pulumi up`** from the terminal.
3. **Pulumi runs your code** using the appropriate language runtime (Node.js for TypeScript, Python for Python, etc.).
4. **Your code registers resources** with the Pulumi engine (e.g., "I want an S3 bucket").
5. **The engine compares** desired state (your code) vs current state (state file).
6. **The engine creates a plan** — what to add, update, or delete.
7. **You approve** the plan.
8. **The engine calls cloud APIs** to make changes.
9. **State is updated** to reflect the new reality.

## 6. Your First Pulumi Project (TypeScript)

### Step 1: Create a New Project


Pulumi will ask some questions:

```

```

### What Gets Created?

```
my-first-pulumi/
├── Pulumi.yaml          # Project metadata
├── Pulumi.dev.yaml      # Stack-specific config
├── index.ts             # Your infrastructure code
├── package.json         # Node.js dependencies
└── tsconfig.json        # TypeScript config
```

### Step 2: Write Your Infrastructure Code

Edit `index.ts`:

```typescript
// index.ts
import * as pulumi from "@pulumi/pulumi";
import * as aws from "@pulumi/aws";

// Create an S3 bucket
const bucket = new aws.s3.Bucket("my-bucket", {
    website: {
        indexDocument: "index.html",
    },
});

// Upload an HTML file to the bucket
const indexHtml = new aws.s3.BucketObject("index.html", {
    bucket: bucket.id,          // Reference the bucket we just created
    content: "<h1>Hello from Pulumi! 🚀</h1>",
    contentType: "text/html",
});

// Create a bucket policy to allow public read access
const bucketPolicy = new aws.s3.BucketPolicy("bucket-policy", {
    bucket: bucket.id,
    policy: bucket.arn.apply(arn => JSON.stringify({
        Version: "2012-10-17",
        Statement: [{
            Effect: "Allow",
            Principal: "*",
            Action: ["s3:GetObject"],
            Resource: [`${arn}/*`],
        }],
    })),
});

// Export the bucket URL so we can see it
export const bucketUrl = pulumi.interpolate`http://${bucket.websiteEndpoint}`;
export const bucketName = bucket.id;
```

**Let's break this down:**

- `import * as aws from "@pulumi/aws"` — Import the AWS SDK for Pulumi.
- `new aws.s3.Bucket(...)` — Create an S3 bucket. Just like calling a constructor in any OOP language!
- `bucket.id` — Get the ID of the bucket we created. Pulumi handles the dependency automatically.
- `bucket.arn.apply(arn => ...)` — The `.apply()` method transforms an output value. Since the ARN isn't known until the resource is created, we use `.apply()` to work with it.
- `export const bucketUrl = ...` — Exports are like Terraform's `output` blocks.

### Step 1: Create the Project

```bash
mkdir pulumi-python && cd pulumi-python

```

This creates:

```
pulumi-python/
├── Pulumi.yaml
├── Pulumi.dev.yaml
├── __main__.py          # Your infrastructure code (Python!)
├── requirements.txt     # Python dependencies
└── venv/                # Virtual environment
```

### Step 2: Write Your Infrastructure

Edit `__main__.py`:

```python
"""A simple AWS infrastructure with Pulumi."""

import pulumi
import pulumi_aws as aws

# -------------------------------------------
# Configuration
# -------------------------------------------
config = pulumi.Config()
environment = config.get("environment") or "dev"
instance_type = config.get("instanceType") or "t3.micro"

# -------------------------------------------
# VPC
# -------------------------------------------
vpc = aws.ec2.Vpc(
    "main-vpc",
    cidr_block="10.0.0.0/16",
    enable_dns_hostnames=True,
    enable_dns_support=True,
    tags={
        "Name": f"pulumi-{environment}-vpc",
        "Environment": environment,
        "ManagedBy": "Pulumi",
    },
)

# -------------------------------------------
# Public Subnet
# -------------------------------------------
public_subnet = aws.ec2.Subnet(
    "public-subnet",
    vpc_id=vpc.id,
    cidr_block="10.0.1.0/24",
    availability_zone="us-east-1a",
    map_public_ip_on_launch=True,
    tags={"Name": f"pulumi-{environment}-public"},
)

# -------------------------------------------
# Internet Gateway
# -------------------------------------------
igw = aws.ec2.InternetGateway(
    "internet-gateway",
    vpc_id=vpc.id,
    tags={"Name": f"pulumi-{environment}-igw"},
)

# -------------------------------------------
# Route Table
# -------------------------------------------
route_table = aws.ec2.RouteTable(
    "public-route-table",
    vpc_id=vpc.id,
    routes=[
        aws.ec2.RouteTableRouteArgs(
            cidr_block="0.0.0.0/0",
            gateway_id=igw.id,
        ),
    ],
    tags={"Name": f"pulumi-{environment}-public-rt"},
)

aws.ec2.RouteTableAssociation(
    "public-rt-association",
    subnet_id=public_subnet.id,
    route_table_id=route_table.id,
)

# -------------------------------------------
# Security Group
# -------------------------------------------
web_sg = aws.ec2.SecurityGroup(
    "web-security-group",
    vpc_id=vpc.id,
    description="Allow HTTP and SSH",
    ingress=[
        aws.ec2.SecurityGroupIngressArgs(
            protocol="tcp",
            from_port=80,
            to_port=80,
            cidr_blocks=["0.0.0.0/0"],
            description="HTTP",
        ),
        aws.ec2.SecurityGroupIngressArgs(
            protocol="tcp",
            from_port=22,
            to_port=22,
            cidr_blocks=["0.0.0.0/0"],  # ⚠️ Restrict in production!
            description="SSH",
        ),
    ],
    egress=[
        aws.ec2.SecurityGroupEgressArgs(
            protocol="-1",
            from_port=0,
            to_port=0,
            cidr_blocks=["0.0.0.0/0"],
        ),
    ],
    tags={"Name": f"pulumi-{environment}-web-sg"},
)

# -------------------------------------------
# EC2 Instance
# -------------------------------------------
# Get the latest Amazon Linux 2023 AMI
ami = aws.ec2.get_ami(
    most_recent=True,
    owners=["amazon"],
    filters=[
        aws.ec2.GetAmiFilterArgs(
            name="name",
            values=["al2023-ami-*-x86_64"],
        ),
    ],
)

user_data = """#!/bin/bash
yum update -y
yum install -y httpd
systemctl start httpd
systemctl enable httpd
echo '<h1>Hello from Pulumi! 🚀</h1>' > /var/www/html/index.html
"""

web_server = aws.ec2.Instance(
    "web-server",
    instance_type=instance_type,
    ami=ami.id,
    subnet_id=public_subnet.id,
    vpc_security_group_ids=[web_sg.id],
    user_data=user_data,
    tags={
        "Name": f"pulumi-{environment}-web-server",
        "Environment": environment,
    },
)

# -------------------------------------------
# Outputs
# -------------------------------------------
pulumi.export("vpc_id", vpc.id)
pulumi.export("public_ip", web_server.public_ip)
pulumi.export("website_url", pulumi.Output.concat("http://", web_server.public_ip))
```

**What's cool about this Python code?**

- It's just **regular Python**! No new syntax to learn.
- You get **autocomplete** in your IDE for every resource.
- You can use **f-strings**, **lists**, **dictionaries** — everything.
- Error messages are standard Python errors — easy to debug.

## 8. Core Concepts Explained

### 8.1 — Projects

A **project** is a directory with a `Pulumi.yaml` file. It's like a Terraform root module.

```yaml
# Pulumi.yaml
name: my-project
runtime: python          # or nodejs, go, dotnet, java
description: My awesome infrastructure
```

### 8.2 — Stacks

A **stack** is an **instance** of your project — like Terraform workspaces. You typically have one stack per environment:

```bash
# Create stacks
pulumi stack init dev
pulumi stack init staging
pulumi stack init prod

# Switch between stacks
pulumi stack select dev

# List all stacks
pulumi stack ls
```

```
NAME      LAST UPDATE     RESOURCE COUNT
dev*      2 minutes ago   5
staging   1 hour ago      5
prod      1 day ago       5
```

### 8.3 — Resources

Resources are the cloud things you want to create. Every resource is a **class instance**:

```python
# Python
import pulumi_aws as aws

# Creating a resource = creating an object
bucket = aws.s3.Bucket("my-bucket", bucket="my-unique-bucket-name")
```

```typescript
// TypeScript
import * as aws from "@pulumi/aws";

// Same thing in TypeScript
const bucket = new aws.s3.Bucket("my-bucket", { bucket: "my-unique-bucket-name" });
```

**Resource names have two parts:**
1. **Logical name** (first argument) — How Pulumi tracks it internally: `"my-bucket"`
2. **Physical name** (property) — The actual name in the cloud: `bucket="my-unique-bucket-name"`

### 8.4 — Inputs and Outputs

This is the trickiest concept in Pulumi. Let me explain it simply:

- **Inputs** = Values you give TO a resource (like arguments to a function).
- **Outputs** = Values you get FROM a resource AFTER it's created (like return values).

The catch: **Outputs aren't known until the resource is actually created!**

```python
import pulumi
import pulumi_aws as aws

# Create a bucket
bucket = aws.s3.Bucket("my-bucket")

# bucket.id is an OUTPUT — we don't know the actual value yet!
# We can't just do: print(bucket.id)  ← This won't work as expected

# Instead, use .apply() to work with outputs:
bucket.id.apply(lambda id: print(f"Bucket ID: {id}"))

# Or use pulumi.export() to display it:
pulumi.export("bucket_id", bucket.id)

# Pass outputs as inputs to other resources:
# Pulumi handles the dependency automatically!
aws.s3.BucketObject(
    "my-file",
    bucket=bucket.id,      # ← Output used as Input — Pulumi resolves it
    content="Hello!",
)
```

**Think of Outputs as Promises/Futures** — they're placeholders that will be filled in later.

### 8.5 — Stack References (Cross-Stack References)

You can share outputs between stacks:

```python
# In the "networking" stack:
pulumi.export("vpc_id", vpc.id)

# In the "application" stack:
network_stack = pulumi.StackReference("myorg/networking/dev")
vpc_id = network_stack.get_output("vpc_id")

# Use it!
instance = aws.ec2.Instance(
    "web",
    subnet_id=subnet_id,
    vpc_security_group_ids=[sg_id],
    # ...
)
```

```python
config = pulumi.Config()

count = config.get_int("instanceCount") or 1

```

```python

    "staging": "t3.small",
    "prod": "t3.large",
```

## 10. Pulumi Configuration and Secrets

### Setting Configuration

```bash
# Plain text config
pulumi config set dbName myapp_db

# Secret config (encrypted!)
pulumi config set --secret dbPassword SuperSecret123

# View config
pulumi config
```

Output:

```
KEY             VALUE
aws:region      us-east-1
dbName          myapp_db
dbPassword      [secret]           # ← Encrypted!
```

### Accessing Secrets in Code

```python
import pulumi

config = pulumi.Config()

# Regular config
db_name = config.require("dbName")

# Secret config — Pulumi keeps it encrypted
db_password = config.require_secret("dbPassword")

# db_password is an Output<string> — it's encrypted in state
```

### How Secrets Work

- Secrets are **encrypted** in the state file.
- By default, Pulumi Cloud manages the encryption key.
- You can use your own key provider (AWS KMS, Azure KeyVault, etc.):

```bash
# Use AWS KMS for encryption
pulumi stack init prod --secrets-provider="awskms://alias/pulumi-secrets?region=us-east-1"
```

## 12. Component Resources — Building Reusable Components

This is where Pulumi really shines! You can create **reusable infrastructure components** using classes — just like in application development.

### Example: A Reusable "Static Website" Component

```python
"""A reusable component for static websites on AWS."""

import pulumi
import pulumi_aws as aws
from pulumi import ComponentResource, ResourceOptions


class StaticWebsite(ComponentResource):
    """Creates an S3 static website with all the necessary configuration."""

    def __init__(self, name: str, domain: str, index_content: str,
                 opts: ResourceOptions = None):
        # Initialize the component
        super().__init__("custom:app:StaticWebsite", name, None, opts)

        # Create the S3 bucket
        self.bucket = aws.s3.Bucket(
            f"{name}-bucket",
            bucket=domain,
            website=aws.s3.BucketWebsiteArgs(
                index_document="index.html",
                error_document="error.html",
            ),
            tags={"Component": "StaticWebsite"},
            opts=ResourceOptions(parent=self),
        )

        # Upload index.html
        self.index = aws.s3.BucketObject(
            f"{name}-index",
            bucket=self.bucket.id,
            key="index.html",
            content=index_content,
            content_type="text/html",
            opts=ResourceOptions(parent=self),
        )

        # Set up the bucket policy for public access
        self.policy = aws.s3.BucketPolicy(
            f"{name}-policy",
            bucket=self.bucket.id,
            policy=self.bucket.arn.apply(
                lambda arn: pulumi.Output.from_input({
                    "Version": "2012-10-17",
                    "Statement": [{
                        "Effect": "Allow",
                        "Principal": "*",
                        "Action": ["s3:GetObject"],
                        "Resource": [f"{arn}/*"],
                    }],
                })
            ).apply(lambda p: pulumi.Output.json_dumps(p)),
            opts=ResourceOptions(parent=self),
        )

        # Register outputs
        self.url = self.bucket.website_endpoint
        self.register_outputs({
            "url": self.url,
            "bucket_name": self.bucket.id,
        })


# ---------- USAGE ----------

# Create multiple websites with ONE line each!
blog = StaticWebsite("blog", "blog.example.com", "<h1>My Blog</h1>")
docs = StaticWebsite("docs", "docs.example.com", "<h1>Documentation</h1>")
landing = StaticWebsite("landing", "www.example.com", "<h1>Welcome!</h1>")

# Export URLs
pulumi.export("blog_url", blog.url)
pulumi.export("docs_url", docs.url)
pulumi.export("landing_url", landing.url)
```

**Why is this amazing?**

- You defined a complex pattern (S3 website) **once**.
- Now you can create **any number of websites** with a single line.
- The component handles all the internal resources.
- It's a regular Python class — you can **inherit**, **extend**, **test** it!

## 13. Real-World Example — AWS Infrastructure with Pulumi (Python)

### A Complete 3-Tier Application Setup

```python
"""
Complete 3-tier AWS infrastructure:
- VPC with public and private subnets
- Application Load Balancer
- EC2 instances in an Auto Scaling Group
- RDS database in private subnet
"""

import pulumi
import pulumi_aws as aws

# -------------------------------------------
# Configuration
# -------------------------------------------
config = pulumi.Config()
stack = pulumi.get_stack()

env_config = {
    "dev":  {"instance_type": "t3.micro",  "db_instance": "db.t3.micro",  "min_size": 1, "max_size": 2},
    "prod": {"instance_type": "t3.medium", "db_instance": "db.t3.medium", "min_size": 2, "max_size": 10},
}

settings = env_config.get(stack, env_config["dev"])

# -------------------------------------------
# VPC with Multiple Subnets
# -------------------------------------------
azs = ["us-east-1a", "us-east-1b"]

vpc = aws.ec2.Vpc(
    "main-vpc",
    cidr_block="10.0.0.0/16",
    enable_dns_hostnames=True,
    tags={"Name": f"{stack}-vpc"},
)

igw = aws.ec2.InternetGateway(
    "igw",
    vpc_id=vpc.id,
    tags={"Name": f"{stack}-igw"},
)

# Create public and private subnets in each AZ
public_subnets = []
private_subnets = []

for i, az in enumerate(azs):
    # Public subnet
    public = aws.ec2.Subnet(
        f"public-{i}",
        vpc_id=vpc.id,
        cidr_block=f"10.0.{i * 2}.0/24",
        availability_zone=az,
        map_public_ip_on_launch=True,
        tags={"Name": f"{stack}-public-{az}"},
    )
    public_subnets.append(public)

    # Private subnet
    private = aws.ec2.Subnet(
        f"private-{i}",
        vpc_id=vpc.id,
        cidr_block=f"10.0.{i * 2 + 1}.0/24",
        availability_zone=az,
        tags={"Name": f"{stack}-private-{az}"},
    )
    private_subnets.append(private)

# Public route table
public_rt = aws.ec2.RouteTable(
    "public-rt",
    vpc_id=vpc.id,
    routes=[aws.ec2.RouteTableRouteArgs(
        cidr_block="0.0.0.0/0",
        gateway_id=igw.id,
    )],
)

for i, subnet in enumerate(public_subnets):
    aws.ec2.RouteTableAssociation(
        f"public-rta-{i}",
        subnet_id=subnet.id,
        route_table_id=public_rt.id,
    )

# -------------------------------------------
# Security Groups
# -------------------------------------------
alb_sg = aws.ec2.SecurityGroup(
    "alb-sg",
    vpc_id=vpc.id,
    description="Allow HTTP/HTTPS to ALB",
    ingress=[
        aws.ec2.SecurityGroupIngressArgs(
            protocol="tcp", from_port=80, to_port=80,
            cidr_blocks=["0.0.0.0/0"],
        ),
        aws.ec2.SecurityGroupIngressArgs(
            protocol="tcp", from_port=443, to_port=443,
            cidr_blocks=["0.0.0.0/0"],
        ),
    ],
    egress=[aws.ec2.SecurityGroupEgressArgs(
        protocol="-1", from_port=0, to_port=0,
        cidr_blocks=["0.0.0.0/0"],
    )],
)

app_sg = aws.ec2.SecurityGroup(
    "app-sg",
    vpc_id=vpc.id,
    description="Allow traffic from ALB only",
    ingress=[aws.ec2.SecurityGroupIngressArgs(
        protocol="tcp", from_port=80, to_port=80,
        security_groups=[alb_sg.id],   # Only from ALB!
    )],
    egress=[aws.ec2.SecurityGroupEgressArgs(
        protocol="-1", from_port=0, to_port=0,
        cidr_blocks=["0.0.0.0/0"],
    )],
)

# -------------------------------------------
# Application Load Balancer
# -------------------------------------------
alb = aws.lb.LoadBalancer(
    "app-alb",
    internal=False,
    load_balancer_type="application",
    security_groups=[alb_sg.id],
    subnets=[s.id for s in public_subnets],
    tags={"Name": f"{stack}-alb"},
)

target_group = aws.lb.TargetGroup(
    "app-tg",
    port=80,
    protocol="HTTP",
    vpc_id=vpc.id,
    health_check=aws.lb.TargetGroupHealthCheckArgs(
        path="/",
        healthy_threshold=2,
        unhealthy_threshold=3,
    ),
)

aws.lb.Listener(
    "http-listener",
    load_balancer_arn=alb.arn,
    port=80,
    default_actions=[aws.lb.ListenerDefaultActionArgs(
        type="forward",
        target_group_arn=target_group.arn,
    )],
)

# -------------------------------------------
# Outputs
# -------------------------------------------
pulumi.export("vpc_id", vpc.id)
pulumi.export("alb_dns", alb.dns_name)
pulumi.export("website_url", pulumi.Output.concat("http://", alb.dns_name))
```

**Notice how natural this feels!** Loops, f-strings, lists, dictionaries — it's just Python, but it creates real cloud infrastructure.

```typescript
// index.ts — Deploy an app to Kubernetes with Pulumi
import * as pulumi from "@pulumi/pulumi";
import * as k8s from "@pulumi/kubernetes";

const config = new pulumi.Config();
const appName = config.get("appName") || "my-app";
const replicas = config.getNumber("replicas") || 3;
const image = config.get("image") || "nginx:latest";

// -------------------------------------------
// Namespace
// -------------------------------------------
const namespace = new k8s.core.v1.Namespace("app-namespace", {
    metadata: { name: appName },
});

// -------------------------------------------
// Deployment
// -------------------------------------------
const deployment = new k8s.apps.v1.Deployment("app-deployment", {
    metadata: {
        namespace: namespace.metadata.name,
        name: appName,
        labels: { app: appName },
    },
    spec: {
        replicas: replicas,
        selector: {
            matchLabels: { app: appName },
        },
        template: {
            metadata: {
                labels: { app: appName },
            },
            spec: {
                containers: [{
                    name: appName,
                    image: image,
                    ports: [{ containerPort: 80 }],
                    resources: {
                        requests: { cpu: "100m", memory: "128Mi" },
                        limits: { cpu: "250m", memory: "256Mi" },
                    },
                    livenessProbe: {
                        httpGet: { path: "/", port: 80 },
                        initialDelaySeconds: 10,
                        periodSeconds: 30,
                    },
                }],
            },
        },
    },
});

// -------------------------------------------
// Service (LoadBalancer)
// -------------------------------------------
const service = new k8s.core.v1.Service("app-service", {
    metadata: {
        namespace: namespace.metadata.name,
        name: appName,
        labels: { app: appName },
    },
    spec: {
        type: "LoadBalancer",
        selector: { app: appName },
        ports: [{
            port: 80,
            targetPort: 80,
        }],
    },
});

// -------------------------------------------
// Outputs
// -------------------------------------------
export const namespaceName = namespace.metadata.name;
export const deploymentName = deployment.metadata.name;
export const serviceEndpoint = service.status.loadBalancer.ingress[0].hostname;
```

## 16. Testing Your Infrastructure

This is one of Pulumi's **biggest advantages** — you can unit test your infrastructure using standard testing frameworks!

### Unit Tests (Python — pytest)

```python
# test_infra.py
import pulumi
import pytest


class MockedMixin:
    """Mock Pulumi for testing."""

    resources = {}

    @staticmethod
    def set_mocks():
        pulumi.runtime.set_mocks(
            MockedMixin.MyMocks(),
            preview=False,
        )

    class MyMocks(pulumi.runtime.Mocks):
        def new_resource(self, args):
            # Return a fake ID and the inputs as state
            return [f"{args.name}_id", args.inputs]

        def call(self, args):
            return {}


# Set up mocks before importing our infra code
MockedMixin.set_mocks()

# Now import infrastructure
import __main__ as infra


@pulumi.runtime.test
def test_vpc_has_dns_support():
    """VPC should have DNS support enabled."""
    def check_dns(args):
        enable_dns_support = args[0]
        assert enable_dns_support is True, "VPC must have DNS support enabled"

    return pulumi.Output.all(
        infra.vpc.enable_dns_support
    ).apply(check_dns)


@pulumi.runtime.test
def test_security_group_no_open_ssh():
    """Security group should NOT allow SSH from 0.0.0.0/0."""
    def check_ingress(args):
        ingress = args[0]
        for rule in ingress:
            if rule.get("from_port") == 22:
                assert "0.0.0.0/0" not in rule.get("cidr_blocks", []), \
                    "SSH should not be open to the world!"

    return pulumi.Output.all(
        infra.web_sg.ingress
    ).apply(check_ingress)
```

```python
# policy/__main__.py
from pulumi_policy import (
    EnforcementLevel,
    PolicyPack,

def no_public_s3(args, report_violation):
    """S3 buckets must not be publicly accessible."""
    if args.resource_type == "aws:s3/bucket:Bucket":
        acl = args.props.get("acl")
        if acl and "public" in acl:

PolicyPack("aws-best-practices", [

        name="no-public-s3",
        description="Prohibits public S3 buckets",
        validate=no_public_s3,
        enforcement_level=EnforcementLevel.MANDATORY,
    ),
])
```

## 17. Pulumi Automation API

The **Automation API** lets you run Pulumi **programmatically** — without the CLI. This is perfect for building custom platforms.

```python
"""
Run Pulumi programmatically — no CLI needed!
Great for building internal platforms or self-service portals.
"""

import pulumi
from pulumi import automation as auto
import pulumi_aws as aws


def create_s3_website(content: str):
    """A Pulumi program that creates an S3 static website."""
    
    bucket = aws.s3.Bucket(
        "website-bucket",
        website=aws.s3.BucketWebsiteArgs(
            index_document="index.html",
        ),
    )

    aws.s3.BucketObject(
        "index.html",
        bucket=bucket.id,
        content=content,
        content_type="text/html",
    )

    pulumi.export("url", bucket.website_endpoint)


# --- Use the Automation API ---

# Create or select a stack
stack = auto.create_or_select_stack(
    stack_name="dev",
    project_name="auto-website",
    program=lambda: create_s3_website("<h1>Built with Automation API!</h1>"),
)

# Set config
stack.set_config("aws:region", auto.ConfigValue(value="us-east-1"))

# Run preview
preview_result = stack.preview(on_output=print)
print(f"Changes: {preview_result.change_summary}")

# Deploy!
up_result = stack.up(on_output=print)
print(f"Website URL: {up_result.outputs['url'].value}")

# To destroy:
# stack.destroy(on_output=print)
```

**Use cases for Automation API:**
- Building **self-service portals** ("Click here to create a new environment")
- **Integration testing** — spin up/tear down infrastructure in tests
- **Multi-tenant platforms** — create isolated infrastructure per customer
- **Custom CLI tools** — wrap Pulumi in your own tooling

## 19. Common Patterns and Best Practices

### 1. Use Config for Environment-Specific Values

```python
# DON'T hardcode
instance_type = "t3.micro"   # ❌

# DO use config
config = pulumi.Config()
instance_type = config.get("instanceType") or "t3.micro"   # ✅
```

### 2. Use Component Resources for Reusability

```python
# DON'T repeat yourself
bucket1 = aws.s3.Bucket("bucket1", ...)   # ❌ Copy-paste
bucket2 = aws.s3.Bucket("bucket2", ...)   # ❌ Copy-paste

# DO create components
site1 = StaticWebsite("site1", ...)   # ✅ Reusable
site2 = StaticWebsite("site2", ...)   # ✅ Reusable
```

### 3. Tag Everything

```python
# Create a helper function for consistent tagging
def common_tags(name: str) -> dict:
    return {
        "Name": name,
        "Environment": pulumi.get_stack(),
        "ManagedBy": "Pulumi",
        "Project": pulumi.get_project(),
    }

vpc = aws.ec2.Vpc("vpc", tags=common_tags("main-vpc"))
```

### 4. Use Stack References for Multi-Stack

```python
# Stack 1: Networking (outputs VPC ID)
# Stack 2: Application (imports VPC ID)
network = pulumi.StackReference("myorg/networking/dev")
vpc_id = network.get_output("vpc_id")
```

### 5. Protect Critical Resources

```python
# Prevent accidental deletion of databases
db = aws.rds.Instance(
    "prod-db",
    # ...
    opts=pulumi.ResourceOptions(protect=True),   # Can't be deleted!
)
```
