## 🤔 What Problem Does Lambda Solve?

Let's say you have a simple task: when a user uploads a profile picture, you want to resize it into a thumbnail.

### The Old Way (With Servers):

```
1. Spin up an EC2 instance (or a whole server)
2. Install the OS, runtime, dependencies
3. Write the image-resizing code and deploy it
4. Set up auto-scaling in case 1000 users upload at the same time
5. Pay for the server 24/7 — even when NO ONE is uploading anything
6. Patch the server, update the OS, monitor for crashes...

💸 You're paying for the server even at 3 AM when nobody is using it!
```

### The Lambda Way:

```
1. Write the image-resizing code (just a function)
2. Upload it to Lambda
3. Configure it to trigger when an image is uploaded to S3
4. Done! ✅

- No servers to manage
- Scales automatically (1 request or 10,000 — Lambda handles it)
- Pay only when your code runs (per millisecond!)
- Zero cost when idle
```

---

## ✅ What is AWS Lambda?

AWS Lambda is a **serverless compute service** that:

| Feature | What It Means |
|---|---|
| **Serverless** | No servers to provision, manage, or patch — ever |
| **Event-driven** | Your code runs in response to events (HTTP request, S3 upload, SQS message, etc.) |
| **Auto-scaling** | Handles 1 request or 1 million — scales automatically and instantly |
| **Pay-per-use** | You only pay for the time your code runs (billed per millisecond) |
| **Stateless** | Each invocation is independent — no state is shared between runs |

### The Simple Version:

```
Lambda = "Hey AWS, here's my function. Run it when THIS event happens. I don't care about servers."
```

```
│  └───┬───┘  └──┬───┘  └──┬───┘  └───┬───┘  └──────┬───────┘ │
│      │         │         │          │              │          │
│      └────────┬┴─────────┴──────────┴──────────────┘          │
│               │                                                │
│               ▼                                                │
│  ┌─────────────────────────────────────────────────────┐      │
│  │              AWS Lambda Service                      │      │
│  │                                                      │      │
│  │  ┌──────────┐  ┌──────────┐  ┌──────────┐          │      │
│  │  │ Container│  │ Container│  │ Container│  ...      │      │
│  │  │ (Your    │  │ (Your    │  │ (Your    │          │      │
│  │  │  Code)   │  │  Code)   │  │  Code)   │          │      │
│  │  └──────────┘  └──────────┘  └──────────┘          │      │
│  │                                                      │      │
│  │  Auto-scales: 1 container per concurrent request    │      │
│  └─────────────────────────────────────────────────────┘      │
│               │                                                │
│               ▼                                                │
│  ┌─────────────────────────────────────────────────────┐      │
│  │           Downstream Services                        │      │
│  │  DynamoDB, S3, RDS, SNS, SQS, Step Functions...     │      │
│  └─────────────────────────────────────────────────────┘      │
│                                                                │
└───────────────────────────────────────────────────────────────┘
```

### How Lambda Executes Your Code:

```
Event arrives (e.g., HTTP request)
        ↓
Lambda checks: Is there a warm container available?
        ↓
    ┌─── YES ───→ Reuse the container (fast! ~1ms overhead)
    │
    └─── NO ────→ Cold Start:
                    1. Download your code
                    2. Create a new container (micro-VM)
                    3. Initialize the runtime (Node.js, Python, etc.)
                    4. Run your handler function
                    (This adds 100ms - 2s of latency)
        ↓
Your function runs and returns a response
        ↓
Container stays warm for ~15 minutes
(reused for the next request)
```

## 🛠️ Creating Your First Lambda Function

### Method 1: AWS Console (Quick Start)

1. Go to **AWS Console → Lambda → Create Function**
2. Choose **Author from scratch**
3. Give it a name: `my-first-lambda`
4. Choose Runtime: `Python 3.12`
5. Click **Create Function**

### Method 2: Writing Lambda Functions (Code)

#### Python Lambda Function:

```python
# lambda_function.py

import json

def lambda_handler(event, context):
    """
    This is the entry point for Lambda.
    
    Parameters:
    - event: The incoming data (HTTP request body, S3 event, SQS message, etc.)
    - context: Runtime info (function name, memory, time remaining, etc.)
    
    Returns:
    - A response (for API Gateway, this must be in a specific format)
    """
    
    # Get the name from the event (e.g., from query string or body)
    name = event.get("name", "World")
    
    # Your business logic here
    message = f"Hello, {name}! Welcome to Lambda!"
    
    # Return a response (API Gateway format)
    return {
        "statusCode": 200,
        "headers": {
            "Content-Type": "application/json"
        },
        "body": json.dumps({
            "message": message
        })
    }
```

**Let's understand each part:**

```python
def lambda_handler(event, context):
#   ^^^^^^^^^^^^^^ — This is the handler function. Lambda calls THIS function.
#                    You configure it as: filename.function_name
#                    e.g., lambda_function.lambda_handler

#                  event — Contains the trigger data:
#                    - API Gateway: HTTP method, path, headers, body, query params
#                    - S3: bucket name, object key, event type
#                    - SQS: message body, message attributes

#                  context — Runtime info:
#                    - context.function_name → "my-first-lambda"
#                    - context.memory_limit_in_mb → 128
#                    - context.get_remaining_time_in_millis() → time left before timeout
```

#### Node.js Lambda Function:

```javascript
// index.mjs (ES Module)

export const handler = async (event, context) => {
    console.log("Received event:", JSON.stringify(event));
    
    const name = event.queryStringParameters?.name || "World";
    
    return {
        statusCode: 200,
        headers: {
            "Content-Type": "application/json"
        },
        body: JSON.stringify({
            message: `Hello, ${name}! Welcome to Lambda!`
        })
    };
};
```

#### Go Lambda Function:

```go
// main.go
package main

import (
    "context"
    "fmt"
    "github.com/aws/aws-lambda-go/lambda"
)

type Request struct {
    Name string `json:"name"`
}

type Response struct {
    StatusCode int               `json:"statusCode"`
    Body       string            `json:"body"`
}

func handler(ctx context.Context, req Request) (Response, error) {
    name := req.Name
    if name == "" {
        name = "World"
    }
    
    return Response{
        StatusCode: 200,
        Body:       fmt.Sprintf(`{"message": "Hello, %s!"}`, name),
    }, nil
}

func main() {
    lambda.Start(handler)
}
```

### Method 1: AWS CLI (Zip Upload)

```bash
# Step 1: Write your code
mkdir my-lambda && cd my-lambda
cat > lambda_function.py << 'EOF'
import json

def lambda_handler(event, context):
    return {
        "statusCode": 200,
        "body": json.dumps({"message": "Hello from Lambda!"})
    }

# Step 2: Zip it
zip function.zip lambda_function.py

# Step 3: Create the Lambda function
aws lambda create-function \
  --function-name my-first-lambda \
  --runtime python3.12 \
  --role arn:aws:iam::123456789012:role/lambda-execution-role \
  --handler lambda_function.lambda_handler \
  --zip-file fileb://function.zip \
  --timeout 30 \
  --memory-size 256

# Step 4: Test it
aws lambda invoke \
  --function-name my-first-lambda \
  --payload '{"name": "Ritesh"}' \
  --cli-binary-format raw-in-base64-out \
  response.json

cat response.json
# {"statusCode": 200, "body": "{\"message\": \"Hello, Ritesh! Welcome to Lambda!\"}"}
```

## 🔗 Lambda Event Sources — What Can Trigger Lambda?

Lambda can be triggered by **tons of AWS services**:

```
┌──────────────────────────────────────────────────────────┐
│              Lambda Event Sources                         │
├──────────────────────────────────────────────────────────┤
│                                                           │
│  Synchronous (wait for response):                        │
│  ├── API Gateway (HTTP requests)                         │
│  ├── ALB (Application Load Balancer)                     │
│  ├── CloudFront (Lambda@Edge)                            │
│  └── Cognito (user authentication)                       │
│                                                           │
│  Asynchronous (fire and forget):                         │
│  ├── S3 (object created/deleted)                         │
│  ├── SNS (notifications)                                 │
│  ├── CloudWatch Events / EventBridge (scheduled/cron)    │
│  ├── CodeCommit, CodePipeline                            │
│  └── IoT, SES (email)                                    │
│                                                           │
│  Stream-based (poll for records):                        │
│  ├── SQS (message queue)                                 │
│  ├── DynamoDB Streams (database changes)                 │
│  ├── Kinesis (data streams)                              │
│  └── MSK (Apache Kafka)                                  │
│                                                           │
└──────────────────────────────────────────────────────────┘
```

### Example: S3 Trigger (Auto-resize uploaded images)

```python
# lambda_function.py — Triggered when an image is uploaded to S3
import boto3
from PIL import Image
import io

s3 = boto3.client('s3')

def lambda_handler(event, context):
    # Get the bucket name and file key from the S3 event
    bucket = event['Records'][0]['s3']['bucket']['name']
    key = event['Records'][0]['s3']['object']['key']
    
    print(f"New file uploaded: s3://{bucket}/{key}")
    
    # Download the image from S3
    response = s3.get_object(Bucket=bucket, Key=key)
    image_content = response['Body'].read()
    
    # Resize the image
    image = Image.open(io.BytesIO(image_content))
    image.thumbnail((200, 200))  # Resize to 200x200 thumbnail
    
    # Save to a buffer
    buffer = io.BytesIO()
    image.save(buffer, 'JPEG')
    buffer.seek(0)
    
    # Upload the thumbnail to a different folder
    thumbnail_key = f"thumbnails/{key}"
    s3.put_object(
        Bucket=bucket,
        Key=thumbnail_key,
        Body=buffer,
        ContentType='image/jpeg'
    )
    
    return {
        "statusCode": 200,
        "body": f"Thumbnail created: {thumbnail_key}"
    }
```

### Example: Scheduled Cron Job (Run every day at midnight)

```python
# lambda_function.py — Runs on a schedule
import boto3

def lambda_handler(event, context):
    """
    This Lambda runs every day at midnight UTC.
    It cleans up old files from an S3 bucket.
    """
    s3 = boto3.client('s3')
    bucket = 'my-temp-bucket'
    
    # List and delete files older than 7 days
    # (simplified example)
    response = s3.list_objects_v2(Bucket=bucket, Prefix='temp/')
    
    if 'Contents' in response:
        for obj in response['Contents']:
            print(f"Deleting: {obj['Key']}")
            s3.delete_object(Bucket=bucket, Key=obj['Key'])
    
    return {"deleted": len(response.get('Contents', []))}
```

### Example: SQS Trigger (Process messages from a queue)

```python
# lambda_function.py — Processes SQS messages
import json

def lambda_handler(event, context):
    """
    Lambda receives a batch of SQS messages.
    Each record in event['Records'] is one message.
    """
    for record in event['Records']:

        process_order(body)
    
    return {"processed": len(event['Records'])}

def process_order(order):
    # Your business logic here
    print(f"Order {order['order_id']} processed successfully!")
```

## 🌐 API Gateway — Putting an HTTP Endpoint in Front of Lambda

### What is API Gateway?

API Gateway is a **fully managed service** that lets you create HTTP APIs:

```
Without API Gateway:
  Lambda has no URL → How does anyone call it?

With API Gateway:
  https://abc123.execute-api.us-east-1.amazonaws.com/prod/users
       └── This URL routes to your Lambda function!
```

### Types of API Gateway:

| Type | Best For | Price |
|---|---|---|
| **HTTP API** | Simple APIs, Lambda proxy | Cheaper ($1.00/million requests) |
| **REST API** | Full-featured, API keys, caching | More expensive ($3.50/million) |
| **WebSocket API** | Real-time, chat, live updates | Per message pricing |

> **Recommendation:** Use **HTTP API** unless you need REST API specific features like caching, request validation, or API keys.

### Creating an API Gateway + Lambda (AWS CLI)

```bash
# Step 1: Create an HTTP API
aws apigatewayv2 create-api \
  --name my-http-api \
  --protocol-type HTTP

# Step 2: Create an integration with Lambda
aws apigatewayv2 create-integration \
  --api-id <api-id> \
  --integration-type AWS_PROXY \
  --integration-uri arn:aws:lambda:us-east-1:123456789012:function:my-first-lambda \
  --payload-format-version 2.0

# Step 3: Create a route
aws apigatewayv2 create-route \
  --api-id <api-id> \
  --route-key "GET /hello"

# Step 4: Create a stage and deploy
aws apigatewayv2 create-stage \
  --api-id <api-id> \
  --stage-name prod \
  --auto-deploy

# Now you can call: https://<api-id>.execute-api.us-east-1.amazonaws.com/prod/hello
```

```python
# src/app.py — The Lambda functions

import json
import boto3

dynamodb = boto3.resource('dynamodb')
table = dynamodb.Table('Users')

def get_users(event, context):
    """GET /users — List all users"""
    result = table.scan()
    return {
        "statusCode": 200,
        "body": json.dumps(result['Items'])
    }

def create_user(event, context):
    """POST /users — Create a new user"""
    body = json.loads(event['body'])
    table.put_item(Item={
        'id': body['id'],
        'name': body['name'],
        'email': body['email']
    })
    return {
        "statusCode": 201,
        "body": json.dumps({"message": "User created!"})
    }

def get_user(event, context):
    """GET /users/{id} — Get a specific user"""
    user_id = event['pathParameters']['id']
    result = table.get_item(Key={'id': user_id})
    
    if 'Item' not in result:
        return {
            "statusCode": 404,
            "body": json.dumps({"error": "User not found"})
        }
    
    return {
        "statusCode": 200,
        "body": json.dumps(result['Item'])
    }
```

## 🧊 Cold Starts — The Biggest Lambda Challenge

### What is a Cold Start?

```
Request #1 (Cold Start):
  ┌─────────────────────────────────────────────────────────┐
  │ Download Code │ Init Runtime │ Init Your Code │ Run │    │
  │    ~100ms     │   ~200ms     │    ~500ms      │ 50ms│    │
  └─────────────────────────────────────────────────────────┘
  Total: ~850ms ❄️

Request #2 (Warm — same container reused):
  ┌──────────┐
  │   Run    │
  │   50ms   │
  └──────────┘
  Total: ~50ms 🔥
```

### Cold Start Times by Runtime:

| Runtime | Typical Cold Start | Notes |
|---|---|---|
| Python | 100-300ms | Very fast cold starts |
| Node.js | 100-300ms | Very fast cold starts |
| Go | 50-100ms | Fastest cold starts (compiled binary) |
| Java | 1-5 seconds | Slowest! JVM startup is heavy |
| .NET | 500ms-2s | Moderate |

### How to Reduce Cold Starts:

```
1. Use smaller deployment packages (less code to download)
2. Use Python, Node.js, or Go (faster than Java/.NET)
3. Keep functions warm with Provisioned Concurrency
4. Minimize imports — only import what you need
5. Initialize SDK clients OUTSIDE the handler (reused across invocations)
```

**Good pattern — Initialize outside the handler:**

```python
import json
import boto3

# ✅ These run ONCE during cold start and are REUSED for warm invocations
dynamodb = boto3.resource('dynamodb')
table = dynamodb.Table('Users')

def lambda_handler(event, context):
    # ✅ This runs EVERY invocation
    result = table.get_item(Key={'id': event['id']})
    return {
        "statusCode": 200,
        "body": json.dumps(result['Item'])
    }
```

### Provisioned Concurrency (Pre-warm Lambda)

```bash
# Keep 5 Lambda instances always warm (no cold starts ever!)
aws lambda put-provisioned-concurrency-config \
  --function-name my-function \
  --qualifier prod \
  --provisioned-concurrent-executions 5

# ⚠️ You pay for provisioned concurrency even when idle!
# Use this only for latency-critical applications.
```

## ⚙️ Lambda Configuration & Limits

### Key Configuration Options:

| Setting | Default | Max | Description |
|---|---|---|---|
| **Memory** | 128 MB | 10,240 MB (10 GB) | More memory = more CPU = faster execution |
| **Timeout** | 3 seconds | 900 seconds (15 min) | Max time a function can run |
| **Ephemeral storage** | 512 MB | 10,240 MB | /tmp directory space |
| **Package size** | - | 50 MB (zip) / 250 MB (unzipped) | Deployment package limit |
| **Container image** | - | 10 GB | If using container images |
| **Concurrency** | 1000 | Adjustable (request limit increase) | Concurrent executions per account |
| **Environment vars** | - | 4 KB total | Key-value pairs |

### Lambda Layers (Share Code Between Functions)

```
Problem: You have 10 Lambda functions that all use the same libraries
         (e.g., boto3, requests, pandas)
         
Without layers: Each function has its own copy → 10x the storage → slow deployments

With layers: Put shared libraries in a Layer → all functions share it
```

```bash
# Create a Lambda Layer (example: Python packages)

# Step 1: Create the directory structure
mkdir -p python/lib/python3.12/site-packages

# Step 2: Install packages into it
pip install requests -t python/lib/python3.12/site-packages/

# Step 3: Zip it
zip -r layer.zip python/

# Step 4: Publish the layer
aws lambda publish-layer-version \
  --layer-name my-shared-libs \
  --zip-file fileb://layer.zip \
  --compatible-runtimes python3.12

# Step 5: Attach the layer to your function
aws lambda update-function-configuration \
  --function-name my-function \
  --layers arn:aws:lambda:us-east-1:123456789012:layer:my-shared-libs:1
```

## 🔐 Lambda Security Best Practices

```
1. ✅ Give Lambda the MINIMUM IAM permissions needed (least privilege)
     Bad:  "Effect": "Allow", "Action": "*", "Resource": "*"
     Good: "Effect": "Allow", "Action": "s3:GetObject", "Resource": "arn:aws:s3:::my-bucket/*"

2. ✅ Use environment variables for config (not hardcoded values)
3. ✅ Store secrets in AWS Secrets Manager or SSM Parameter Store
4. ✅ Enable VPC access only if Lambda needs to reach private resources
5. ✅ Use Lambda resource policies to control who can invoke it
6. ✅ Enable AWS X-Ray for tracing and debugging
7. ✅ Set appropriate timeout (don't use 15 minutes for a 2-second function)
```

## 💰 Lambda Pricing

```
Lambda Pricing (as of 2024):
├── Requests: First 1 million requests FREE, then $0.20 per million
│
├── Duration: Based on memory allocated
│   ├── 128 MB: $0.0000000021 per ms (~$0.0000021 per second)
│   ├── 256 MB: $0.0000000042 per ms
│   ├── 512 MB: $0.0000000083 per ms
│   └── 1024 MB: $0.0000000167 per ms
│
└── Free Tier (forever): 1 million requests + 400,000 GB-seconds per month

Example Calculation:
  - 10 million requests/month
  - 256 MB memory
  - 200ms average duration each
  
  Requests: (10M - 1M free) × $0.20/M = $1.80
  Duration: 10M × 0.2s × 0.25GB = 500,000 GB-seconds
            (500K - 400K free) × $0.0000166667 = $1.67
  
  Total: $3.47/month 🎉 (Try doing that with EC2!)
```

## 🆚 Lambda vs EC2 vs ECS vs Fargate — When to Use What?

| Criteria | Lambda | EC2 | ECS/Fargate |
|---|---|---|---|
| **Best for** | Short tasks, events, APIs | Long-running apps, custom OS | Containerized apps |
| **Max runtime** | 15 minutes | Unlimited | Unlimited |
| **Scaling** | Instant, automatic | Manual or ASG | Manual or auto |
| **Pricing** | Per-request | Per-hour | Per-hour/per-second |
| **Idle cost** | $0 | Still paying | Still paying |
| **Cold start** | Yes (100ms - 5s) | No | No |
| **Use case** | APIs, event processing, cron | Databases, ML training | Microservices, web apps |

### Decision Flowchart:

```
Does your task complete in < 15 minutes?
    ├── YES → Is it triggered by events? → Lambda ✅
    └── NO  → Is it containerized?
                ├── YES → Do you want to manage servers?
                │           ├── YES → ECS on EC2
                │           └── NO  → ECS/EKS on Fargate
                └── NO  → EC2
```

## 🎯 Key Takeaways

| Concept | Remember This |
|---|---|
| Lambda | Serverless functions — no servers, pay-per-use, auto-scales |
| Handler | The function Lambda calls (e.g., `lambda_function.lambda_handler`) |
| Event | The data that triggered the function (HTTP request, S3 event, etc.) |
| Cold Start | First invocation is slower (~100ms-5s), subsequent ones are fast |
| API Gateway | Puts an HTTP URL in front of Lambda — creates REST/HTTP APIs |
| Layers | Share common code/libraries across multiple functions |
| SAM | Best tool for building serverless apps (template + CLI) |
| Timeout | Max 15 minutes — Lambda is NOT for long-running tasks |
| Pricing | Per-request + per-ms of execution — practically free for small workloads |
