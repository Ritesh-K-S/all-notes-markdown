# 14.34 — Cloud Comparison — AWS vs Azure vs GCP

> **One Line Answer:** AWS, Azure, and GCP are the **Big Three** cloud providers — AWS leads in market share and service breadth, Azure excels in enterprise/Microsoft integration, and GCP shines in data analytics, AI/ML, and Kubernetes — choosing the right one depends on your **use case, existing ecosystem, and team expertise.**

---

## 🤔 Why Do We Even Need to Compare?

Imagine you want to build a house. You have **three construction companies** to choose from:

| Company | Known For |
|---------|-----------|
| **AWS** | The biggest and oldest — has EVERY tool imaginable, but can feel overwhelming |
| **Azure** | Best friends with Microsoft — if you already use Windows, Office 365, Active Directory, this feels like home |
| **GCP** | The brainy one — amazing for data, AI, and Kubernetes (built by the company that invented containers at scale!) |

All three can build your house. But depending on **what kind of house you want** and **what tools you already own**, one might be a better fit than the others.

Let's break it all down — **service by service, feature by feature, strength by weakness.**

This is where it gets really useful. Let's compare the equivalent services across all three clouds — organized by category.

### 1. Compute Services

> *"I need a server to run my application."*

| What You Need | AWS | Azure | GCP |
|--------------|-----|-------|-----|
| Virtual Machines | **EC2** (Elastic Compute Cloud) | **Virtual Machines** | **Compute Engine** |
| Auto-Scaling VMs | **Auto Scaling Groups** | **VM Scale Sets (VMSS)** | **Managed Instance Groups (MIGs)** |
| Containers (Managed) | **ECS** (Elastic Container Service) | **Azure Container Instances (ACI)** | **Cloud Run** |
| Kubernetes (Managed) | **EKS** (Elastic Kubernetes Service) | **AKS** (Azure Kubernetes Service) | **GKE** (Google Kubernetes Engine) |
| Serverless Functions | **Lambda** | **Azure Functions** | **Cloud Functions** |
| Serverless Containers | **Fargate** | **Container Apps** | **Cloud Run** |
| Bare Metal | **EC2 Bare Metal** | **Dedicated Hosts** | **Bare Metal Solution** |
| Spot/Preemptible VMs | **Spot Instances** | **Spot VMs** | **Spot VMs (formerly Preemptible)** |

#### 💡 Which is Best for Compute?

```
AWS EC2    → Most instance types, most flexibility, largest ecosystem
Azure VMs  → Best if you're running Windows workloads or .NET apps
GCP GKE    → Best managed Kubernetes service (Google literally invented Kubernetes!)
```

**Real Talk:**
- If you need **Kubernetes**, GKE is widely considered the best managed K8s service. It's built by the people who created Kubernetes (the Borg team at Google).
- If you need **raw VM flexibility** with hundreds of instance types, AWS EC2 is king.
- If you're a **Windows/.NET shop**, Azure VMs feel the most natural.

### 2. Networking

> *"I need to connect my services, set up a private network, and load balance traffic."*

| What You Need | AWS | Azure | GCP |
|--------------|-----|-------|-----|
| Virtual Private Network | **VPC** (Virtual Private Cloud) | **VNet** (Virtual Network) | **VPC** (Virtual Private Cloud) |
| Subnets | **Subnets** (per AZ) | **Subnets** (per VNet) | **Subnets** (regional) |
| Load Balancer (L4) | **NLB** (Network Load Balancer) | **Azure Load Balancer** | **Network Load Balancer** |
| Load Balancer (L7) | **ALB** (Application Load Balancer) | **Application Gateway** | **HTTP(S) Load Balancer** |
| DNS Service | **Route 53** | **Azure DNS** | **Cloud DNS** |
| CDN | **CloudFront** | **Azure CDN / Front Door** | **Cloud CDN** |
| VPN Gateway | **VPN Gateway** | **VPN Gateway** | **Cloud VPN** |
| Direct/Dedicated Connection | **Direct Connect** | **ExpressRoute** | **Cloud Interconnect** |
| Firewall | **Security Groups + NACLs** | **NSG + Azure Firewall** | **Firewall Rules + Cloud Armor** |
| Web Application Firewall | **AWS WAF** | **Azure WAF** | **Cloud Armor** |
| Service Mesh | **App Mesh** | **Open Service Mesh** | **Traffic Director / Anthos Service Mesh** |

#### 💡 Key Networking Differences

```
AWS   → VPC subnets are per Availability Zone (AZ-scoped)
Azure → VNet subnets span across all AZs in a region  
GCP   → VPC is GLOBAL by default! Subnets are regional.

This is a HUGE difference!

GCP's global VPC means:
  - You create ONE VPC and it spans ALL regions worldwide
  - No need for VPC peering between regions
  - Simplifies multi-region architectures significantly

AWS/Azure:
  - VPCs/VNets are regional
  - Need VPC Peering or Transit Gateway to connect across regions
```

> **🎯 Winner for Networking:** GCP's global VPC model is the most elegant. AWS has the most mature networking features. Azure integrates well with on-prem via ExpressRoute.

### 3. Storage

> *"I need to store files, objects, and data."*

| What You Need | AWS | Azure | GCP |
|--------------|-----|-------|-----|
| Object Storage | **S3** (Simple Storage Service) | **Blob Storage** | **Cloud Storage** |
| Block Storage (Disk) | **EBS** (Elastic Block Store) | **Managed Disks** | **Persistent Disks** |

| Archive/Cold Storage | **S3 Glacier** | **Blob Archive Tier** | **Archive Storage** |
| Storage Gateway | **Storage Gateway** | **StorSimple** | **Transfer Service** |

#### 💡 Storage Comparison

```
AWS S3  → The gold standard. Most feature-rich object storage.
           99.999999999% (11 9's) durability.
           Industry standard — many tools integrate with S3 natively.

Azure Blob → Excellent with tiering (Hot, Cool, Archive).
              Great integration with Azure AD for access control.

GCP Cloud Storage → Simple, fast, unified API for all storage classes.
                     Multi-regional buckets by default.
                     Simpler pricing model.
```

**Quick pricing note (approximate, always verify current pricing):**
- All three offer similar durability (11 9's)
- GCP often has slightly lower egress costs
- AWS S3 has the most storage classes (Standard, IA, One Zone-IA, Intelligent-Tiering, Glacier, Glacier Deep Archive)
- Azure has Hot, Cool, Cold, and Archive tiers

### 5. Identity & Access Management (IAM)

> *"Who can access what in my cloud?"*

| Feature | AWS | Azure | GCP |
|---------|-----|-------|-----|
| Identity Service | **IAM** | **Azure AD (now Entra ID)** | **Cloud IAM** |
| Users & Groups | IAM Users, Groups | Azure AD Users, Groups | Google Workspace / Cloud Identity |
| Roles | IAM Roles (attached to resources) | RBAC Roles (built-in & custom) | IAM Roles (predefined & custom) |
| Policies | JSON-based IAM Policies | Azure Role Assignments | IAM Policy Bindings |
| SSO / Federation | **IAM Identity Center (SSO)** | **Azure AD SSO** | **Cloud Identity / Workforce Identity** |
| MFA | IAM MFA | Azure AD MFA | 2-Step Verification |
| Service Accounts | IAM Roles for Services | Managed Identity | Service Accounts |

#### 💡 IAM Differences

```
AWS IAM
  ├── Policies are JSON documents
  ├── Very granular (resource-level permissions)
  ├── Can be complex — policy evaluation logic is tricky
  └── IAM is GLOBAL (not per region)

Azure AD (Entra ID)
  ├── Full-blown identity provider (not just cloud IAM)
  ├── Integrates with on-prem Active Directory
  ├── Best for enterprises already using Microsoft 365
  ├── RBAC is cleaner — roles assigned at scope (subscription/resource group/resource)
  └── Managed Identity = no credentials to manage!

GCP IAM
  ├── Simplest model among the three
  ├── member → role → resource binding
  ├── Predefined roles are well-designed
  ├── Organization → Folder → Project hierarchy is clean
  └── Service accounts are first-class citizens
```

> **🎯 Winner for IAM:** Azure if you're an enterprise with existing Active Directory. GCP for simplicity. AWS for granularity.

## 💰 Pricing Comparison

This is one of the most common questions: **"Which cloud is cheapest?"**

### The Honest Answer:

```
There is NO universally cheapest cloud. It depends on:
  ├── What services you use
  ├── How much data transfer (egress) you have
  ├── Your commitment level (reserved vs on-demand)
  ├── Your region
  └── Your negotiation skills (for enterprise contracts)
```

### General Pricing Patterns

| Factor | AWS | Azure | GCP |
|--------|-----|-------|-----|
| **Compute (VMs)** | Competitive | 5-15% cheaper for Windows workloads | Often 5-20% cheaper than AWS |
| **Storage** | S3 is well-priced | Competitive | Slightly cheaper |
| **Data Egress** | Most expensive | Similar to AWS | Cheapest (often 50%+ less) |
| **Discounts** | Reserved Instances, Savings Plans | Reserved VMs, Azure Hybrid Benefit | Committed Use Discounts, Sustained Use Discounts |
| **Free Tier** | 12 months + always free | 12 months + always free | Always Free + $300 credit (90 days) |
| **Pricing Calculator** | [calculator.aws](https://calculator.aws) | [azure.com/pricing/calculator](https://azure.microsoft.com/pricing/calculator) | [cloud.google.com/products/calculator](https://cloud.google.com/products/calculator) |

#### Key Pricing Features

```
AWS:
  ├── Reserved Instances (1yr or 3yr commitment → up to 72% discount)
  ├── Savings Plans (flexible commitment-based discounts)
  ├── Spot Instances (up to 90% off, but can be interrupted)
  └── Free Tier: t2.micro EC2 for 12 months, 5GB S3, etc.

Azure:
  ├── Azure Hybrid Benefit → Use existing Windows Server / SQL licenses
  │   (This alone can save 40-80% on Windows workloads!)
  ├── Reserved VMs (1yr or 3yr)
  ├── Spot VMs
  └── Free Tier: B1s VM for 12 months, 5GB Blob, etc.

GCP:
  ├── Sustained Use Discounts → AUTOMATIC discount if VM runs 25%+ of month
  │   (No commitment required! This is unique to GCP)
  ├── Committed Use Discounts (1yr or 3yr)
  ├── Spot VMs
  ├── Per-second billing (AWS and Azure also do this now)
  └── Free Tier: e2-micro VM always free, 5GB Cloud Storage, etc.
```

> **🎯 Cost Tip:** If you're running Windows workloads with existing licenses, Azure Hybrid Benefit is hard to beat. If you want automatic discounts without commitments, GCP's Sustained Use Discounts are fantastic.

| Strengths | Weaknesses |
|-----------|------------|

| Cleanest UI/UX among the three | Smaller community & fewer tutorials |
| Superior networking (global VPC, premium tier) | Enterprise support historically weaker |
| Simplest IAM model | Can feel like a "developer's cloud" — less enterprise-friendly |

## 🔧 CLI Tools Comparison

Each cloud has its own CLI. Here's a quick comparison:

### AWS CLI

```bash
# Install
curl "https://awscli.amazonaws.com/awscli-exe-linux-x86_64.zip" -o "awscliv2.zip"
unzip awscliv2.zip
sudo ./aws/install

# Configure
aws configure
# Enter: Access Key, Secret Key, Region, Output Format

# Example: List S3 buckets
aws s3 ls

# Example: Launch an EC2 instance
aws ec2 run-instances \
  --image-id ami-0abcdef1234567890 \
  --instance-type t3.micro \
  --key-name my-key \
  --count 1
```

### Azure CLI

```bash
# Install
curl -sL https://aka.ms/InstallAzureCLIDeb | sudo bash

# Login
az login

# Example: List resource groups
```

## 🌐 Multi-Cloud & Hybrid Strategies

In the real world, many companies use **more than one cloud.** Here's why and how:

### Why Multi-Cloud?

```
1. Avoid Vendor Lock-in
   └── Don't put all eggs in one basket

2. Best-of-Breed Services
   └── Use GCP for BigQuery + AWS for everything else

3. Compliance Requirements
   └── Data residency laws may require specific regions/providers

4. Redundancy
   └── If one cloud goes down, others keep running

5. Mergers & Acquisitions
   └── Company A uses AWS, Company B uses Azure → now you're multi-cloud!
```

### Multi-Cloud Tools

```
These tools work across ALL three clouds:

Terraform       → Infrastructure as Code (one language, all clouds)
Kubernetes      → Container orchestration (runs on EKS, AKS, GKE, or self-managed)
Pulumi          → IaC with real programming languages
Crossplane      → Kubernetes-native infrastructure management
Ansible         → Configuration management (works everywhere)
GitLab CI/CD    → CI/CD that deploys to any cloud
GitHub Actions  → CI/CD that deploys to any cloud
Datadog         → Monitoring across all clouds
Grafana         → Dashboards for metrics from any source
HashiCorp Vault → Secrets management (cloud-agnostic)
```

## 🧪 Interview Questions — AWS vs Azure vs GCP

Here are some common interview questions on this topic:

### Q1: Your company uses Active Directory and Office 365. Which cloud would you recommend and why?

**Answer:**
> Azure is the natural choice. It integrates natively with Azure AD (Entra ID), which is the cloud extension of on-premises Active Directory. This gives seamless SSO, hybrid identity, and leverages existing Microsoft enterprise agreements. Azure Hybrid Benefit can also save costs on Windows Server and SQL Server licensing.

### Q2: You need to set up a data analytics pipeline processing petabytes of data. Which cloud?

**Answer:**
> GCP with BigQuery. BigQuery is a serverless, petabyte-scale data warehouse that can query massive datasets in seconds using standard SQL. It charges per query (not per cluster), so you don't need to manage or size any infrastructure. Combined with Dataflow for ETL and Pub/Sub for real-time streaming, GCP provides the strongest analytics stack.

### Q3: What's the main networking difference between the three clouds?

**Answer:**
> The biggest difference is VPC scope. GCP's VPC is **global** — one VPC spans all regions automatically. AWS and Azure VPCs/VNets are **regional** — you need peering or transit gateways to connect across regions. GCP's global VPC simplifies multi-region architectures significantly.

### Q4: You're running a Kubernetes-based microservices architecture. Which cloud?

**Answer:**
> GCP with GKE (Google Kubernetes Engine). Google invented Kubernetes (from their internal Borg system), and GKE is considered the most mature, feature-rich, and easiest-to-use managed Kubernetes service. It offers features like Autopilot mode (fully managed nodes), built-in Istio service mesh, and binary authorization out of the box.

### Q5: How do the three clouds handle cost optimization differently?

**Answer:**
> - **AWS:** Reserved Instances and Savings Plans require upfront commitments for discounts (up to 72%).
> - **Azure:** Azure Hybrid Benefit lets you reuse existing Windows/SQL licenses (saving 40-80%). Reserved VMs for commitment discounts.
> - **GCP:** Sustained Use Discounts are **automatic** — if a VM runs 25%+ of the month, you get a discount with NO commitment required. This is unique to GCP.

### Q6: Your team wants to use a cloud-agnostic approach. What tools should you use?

**Answer:**
> Use **Terraform** for infrastructure as code (works with all three clouds using providers), **Kubernetes** for container orchestration (runs on EKS, AKS, GKE), **GitHub Actions or GitLab CI** for CI/CD, and **Datadog or Grafana** for monitoring. This stack is portable across all clouds.
