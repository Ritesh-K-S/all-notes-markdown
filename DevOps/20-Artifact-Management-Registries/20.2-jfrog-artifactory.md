# 20.2 — JFrog Artifactory — Universal Artifact Manager

> **One Line Answer:** JFrog Artifactory is the **industry's leading universal artifact repository** — it can store, manage, and distribute **any type of artifact** (Docker images, Maven JARs, npm packages, Python wheels, Helm charts, and 30+ more formats) all in **one single platform**.

---

## 🤔 Why Do We Need Artifactory?

Let's say your company builds multiple applications:

```
App A — Java backend      → produces JAR files
App B — React frontend    → produces npm packages
App C — Python ML service  → produces Python wheels
App D — Kubernetes app     → produces Docker images + Helm charts
```

Without Artifactory, you'd need **separate repositories** for each:

```
❌ The Messy Way:
├── Docker Hub          ← for Docker images
├── Maven Central       ← for Java JARs
├── npmjs.com           ← for npm packages
├── PyPI                ← for Python packages
├── ChartMuseum         ← for Helm charts
└── 😫 Managing 5 different tools, 5 different credentials, 5 different UIs
```

With Artifactory, **one tool handles everything:**

```
✅ The Clean Way:
└── JFrog Artifactory
    ├── Docker images    ← all in one place
    ├── Maven JARs       ← all in one place
    ├── npm packages     ← all in one place
    ├── Python wheels    ← all in one place
    ├── Helm charts      ← all in one place
    └── 🎉 One tool, one UI, one set of credentials, one set of policies
```

## 📂 Repository Types — The Core Concept

This is the **most important thing** to understand about Artifactory. There are **4 types of repositories:**

### 1. Local Repository

> **Your own private storage.** Artifacts YOU build are uploaded here.

```
┌──────────────────────┐
│   Local Repository   │
│                      │
│  Your CI/CD pipeline │
│  pushes artifacts    │───→  myapp-1.0.0.jar
│  here after building │───→  myapp:v1.2.3 (Docker)
│                      │───→  my-lib-2.0.0.tgz (npm)
└──────────────────────┘
```

**Example Names:**
- `libs-release-local` — for released Java libraries
- `libs-snapshot-local` — for snapshot/development Java builds
- `docker-local` — for your Docker images
- `npm-local` — for your npm packages

### 2. Remote Repository

> **A proxy/cache for external public repositories.** Instead of downloading from the internet every time, Artifactory caches them locally.

```
┌──────────────────────┐         ┌──────────────────┐
│  Remote Repository   │         │  Public Registry  │
│  (Proxy + Cache)     │ ──────→ │  (npm, Maven,     │
│                      │ ←────── │   Docker Hub,     │
│  First request:      │         │   PyPI, etc.)     │
│  fetches from remote │         └──────────────────┘
│                      │
│  Subsequent requests:│
│  served from cache ⚡│
└──────────────────────┘
```

**Why This Is Awesome:**

| Benefit | Explanation |
|---------|-------------|
| **Speed** | Dependencies are cached locally — no more slow downloads from internet |
| **Reliability** | If npmjs.com goes down, your builds still work (from cache) |
| **Security** | You control what enters your network — can block vulnerable packages |
| **Bandwidth** | Saves internet bandwidth — download once, serve many times |

**Example Names:**
- `npm-remote` — proxy for npmjs.com
- `maven-remote` — proxy for Maven Central
- `docker-remote` — proxy for Docker Hub
- `pypi-remote` — proxy for pypi.org

### 3. Virtual Repository

> **A single URL that combines Local + Remote repos.** Developers use ONE URL and Artifactory figures out where to find the artifact.

```
┌────────────────────────────────────┐
│       Virtual Repository           │
│       (npm-virtual)                │
│                                    │
│   Combines:                        │
│   ├── npm-local   (your packages)  │
│   └── npm-remote  (public npm)     │
│                                    │
│   Developer uses ONE URL:          │
│   https://artifactory.com/npm-virtual/
└────────────────────────────────────┘
```

**How Resolution Works:**

```
Developer runs: npm install lodash

Artifactory Virtual Repo (npm-virtual):
  1. Check npm-local → Not found (you didn't build lodash)
  2. Check npm-remote → Found! (cached from npmjs.com)
  3. Return to developer ✅

Developer runs: npm install @mycompany/my-lib

Artifactory Virtual Repo (npm-virtual):
  1. Check npm-local → Found! (your team built this)
  2. Return to developer ✅
```

**This is the beauty of virtual repos** — developers don't need to know if a package is internal or external. They use one URL for everything.

### 4. Federated Repository

> **Replicate repositories across multiple Artifactory instances** (multi-site/multi-region setups).

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│  Artifactory     │     │  Artifactory     │     │  Artifactory     │
│  US-East         │ ←──→│  EU-West         │ ←──→│  AP-Southeast    │
│                  │     │                  │     │                  │
│  docker-federated│     │  docker-federated│     │  docker-federated│
│  (auto-sync)     │     │  (auto-sync)     │     │  (auto-sync)     │
└─────────────────┘     └─────────────────┘     └─────────────────┘
```

Used in large enterprises where teams are spread across the globe and need local, fast access to the same artifacts.

## 🚀 Installing Artifactory

### Option 1: Docker (Easiest — Great for Learning)

```bash
# Pull the OSS (Open Source) version
docker pull releases-docker.jfrog.io/jfrog/artifactory-oss:latest

# Run Artifactory
docker run -d \
  --name artifactory \
  -p 8081:8081 \
  -p 8082:8082 \
  -v artifactory-data:/var/opt/jfrog/artifactory \
  releases-docker.jfrog.io/jfrog/artifactory-oss:latest
```

```
After starting:
- Web UI:  http://localhost:8082
- Default login:
  - Username: admin
  - Password: password (change this immediately!)
```

### Option 2: Docker Compose (Better for Production-like Setup)

```yaml
# docker-compose.yml
version: '3'
services:
  artifactory:
    image: releases-docker.jfrog.io/jfrog/artifactory-pro:latest
    container_name: artifactory
    ports:
      - "8081:8081"
      - "8082:8082"
    volumes:
      - artifactory-data:/var/opt/jfrog/artifactory
    environment:
      - JF_SHARED_DATABASE_TYPE=postgresql
      - JF_SHARED_DATABASE_DRIVER=org.postgresql.Driver
    restart: always

  postgresql:
    image: postgres:15
    container_name: artifactory-db
    environment:
      POSTGRES_DB: artifactory
      POSTGRES_USER: artifactory
      POSTGRES_PASSWORD: your-secure-password
    volumes:
      - postgres-data:/var/lib/postgresql/data

volumes:
  artifactory-data:
  postgres-data:
```

```bash
docker compose up -d
```

### Option 3: Kubernetes (Helm Chart)

```bash
# Add JFrog Helm repository
helm repo add jfrog https://charts.jfrog.io
helm repo update

# Install Artifactory
helm install artifactory jfrog/artifactory \
  --namespace artifactory \
  --create-namespace \
  --set artifactory.persistence.size=100Gi
```

## ⚙️ Configuring Artifactory — Step by Step

### Step 1: Create a Local Repository (for your artifacts)

**Via Web UI:**
1. Go to **Administration → Repositories → Local**
2. Click **New Local Repository**
3. Select package type (e.g., Docker)
4. Name it (e.g., `docker-local`)
5. Click **Create**

**Via REST API:**

```bash
# Create a local Docker repository
curl -X PUT \
  -H "Content-Type: application/json" \
  -u admin:password \
  "http://localhost:8082/artifactory/api/repositories/docker-local" \
  -d '{
    "key": "docker-local",
    "rclass": "local",
    "packageType": "docker",
    "description": "Local Docker images repository"
  }'
```

### Step 2: Create a Remote Repository (proxy for Docker Hub)

```bash
# Create a remote Docker repository (proxy for Docker Hub)
curl -X PUT \
  -H "Content-Type: application/json" \
  -u admin:password \
  "http://localhost:8082/artifactory/api/repositories/docker-remote" \
  -d '{
    "key": "docker-remote",
    "rclass": "remote",
    "packageType": "docker",
    "url": "https://registry-1.docker.io",
    "description": "Proxy for Docker Hub"
  }'
```

### Step 3: Create a Virtual Repository (combine them)

```bash
# Create a virtual Docker repository
curl -X PUT \
  -H "Content-Type: application/json" \
  -u admin:password \
  "http://localhost:8082/artifactory/api/repositories/docker-virtual" \
  -d '{
    "key": "docker-virtual",
    "rclass": "virtual",
    "packageType": "docker",
    "repositories": ["docker-local", "docker-remote"],
    "defaultDeploymentRepo": "docker-local",
    "description": "Virtual Docker repository"
  }'
```

## 🐳 Using Artifactory as a Docker Registry

### Configure Docker to Use Artifactory

```bash
# Login to Artifactory Docker registry
docker login artifactory.mycompany.com
# Enter your Artifactory username and password/API key

# Tag your image for Artifactory
docker tag myapp:v1.0.0 artifactory.mycompany.com/docker-local/myapp:v1.0.0

# Push to Artifactory
docker push artifactory.mycompany.com/docker-local/myapp:v1.0.0

# Pull from Artifactory
docker pull artifactory.mycompany.com/docker-virtual/myapp:v1.0.0
```

### In a Dockerfile (Using Artifactory as Base Image Source)

```dockerfile
# Pull base image THROUGH Artifactory (uses the remote/virtual repo)
FROM artifactory.mycompany.com/docker-virtual/node:18-alpine

WORKDIR /app
COPY . .
RUN npm ci
CMD ["node", "server.js"]
```

## ☕ Using Artifactory for Maven (Java)

### Configure `settings.xml`:

```xml
<!-- ~/.m2/settings.xml -->
<settings>
  <servers>
    <server>
      <id>artifactory</id>
      <username>your-username</username>
      <password>your-api-key</password>
    </server>
  </servers>

  <mirrors>
    <mirror>
      <id>artifactory</id>
      <name>Artifactory Mirror</name>
      <url>https://artifactory.mycompany.com/artifactory/maven-virtual/</url>
      <mirrorOf>*</mirrorOf>
    </mirror>
  </mirrors>
</settings>
```

### Deploy a JAR to Artifactory:

```bash
mvn deploy -DaltDeploymentRepository=artifactory::default::https://artifactory.mycompany.com/artifactory/libs-release-local/
```

## 🔍 Searching Artifacts

### Using AQL (Artifactory Query Language)

AQL is Artifactory's powerful query language to find artifacts:

```bash
# Find all Docker images tagged with "latest"
curl -u admin:password \
  -X POST \
  "http://localhost:8082/artifactory/api/search/aql" \
  -H "Content-Type: text/plain" \
  -d 'items.find({
    "repo": "docker-local",
    "name": "manifest.json",
    "path": {"$match": "*/latest/*"}
  })'
```

```bash
# Find all JAR files larger than 10MB
curl -u admin:password \
  -X POST \
  "http://localhost:8082/artifactory/api/search/aql" \
  -H "Content-Type: text/plain" \
  -d 'items.find({
    "repo": "libs-release-local",
    "name": {"$match": "*.jar"},
    "size": {"$gt": 10485760}
  })'
```

### Quick Search via REST API:

```bash
# Search by name
curl -u admin:password \
  "http://localhost:8082/artifactory/api/search/artifact?name=myapp&repos=libs-release-local"

# Search by checksum (SHA256)
curl -u admin:password \
  "http://localhost:8082/artifactory/api/search/checksum?sha256=abc123def456..."
```

## 🔐 Security & Access Control

### Users and Groups

```bash
# Create a user
curl -X PUT \
  -H "Content-Type: application/json" \
  -u admin:password \
  "http://localhost:8082/artifactory/api/security/users/john" \
  -d '{
    "email": "john@company.com",
    "password": "secure-password",
    "admin": false,
    "groups": ["developers"]
  }'

# Create a group
curl -X PUT \
  -H "Content-Type: application/json" \
  -u admin:password \
  "http://localhost:8082/artifactory/api/security/groups/developers" \
  -d '{
    "description": "Development team",
    "autoJoin": false
  }'
```

### Permission Targets

```bash
# Create a permission target
curl -X PUT \
  -H "Content-Type: application/json" \
  -u admin:password \
  "http://localhost:8082/artifactory/api/v2/security/permissions/dev-team-perms" \
  -d '{
    "name": "dev-team-perms",
    "repo": {
      "repositories": ["docker-local", "npm-local"],
      "actions": {
        "groups": {
          "developers": ["read", "write", "annotate"]
        }
      }
    }
  }'
```

### Permission Levels Explained:

| Permission | What It Allows |
|------------|---------------|
| **Read** | Download/pull artifacts |
| **Annotate** | Add/modify metadata and properties |
| **Write** | Upload/push artifacts |
| **Delete** | Remove artifacts |
| **Manage** | Admin-level control over the repository |

## 🧹 Cleanup & Retention Policies

Artifacts accumulate fast. Without cleanup, you'll run out of storage.

### Cleanup Policies via REST API:

```bash
# Delete artifacts older than 90 days from snapshot repo
curl -u admin:password \
  -X POST \
  "http://localhost:8082/artifactory/api/search/aql" \
  -H "Content-Type: text/plain" \
  -d 'items.find({
    "repo": "libs-snapshot-local",
    "created": {"$before": "90d"}
  })' | jq -r '.results[].path' | while read path; do
    curl -u admin:password -X DELETE \
      "http://localhost:8082/artifactory/$path"
  done
```

### Using the Web UI:
1. Go to **Administration → Artifactory → Cleanup Policies**
2. Create a new policy
3. Set criteria: repository, age, download count, etc.
4. Schedule the cleanup (e.g., run every Sunday at 2 AM)

## 🔗 Artifactory in CI/CD Pipelines

### Jenkins Integration

```groovy
// Jenkinsfile
pipeline {
    agent any
    
    stages {
        stage('Build') {
            steps {
                sh 'mvn clean package'
            }
        }
        
        stage('Publish to Artifactory') {
            steps {
                script {
                    def server = Artifactory.server 'my-artifactory'
                    def uploadSpec = """{
                        "files": [{
                            "pattern": "target/*.jar",
                            "target": "libs-release-local/com/mycompany/myapp/1.0.0/"
                        }]
                    }"""
                    server.upload(uploadSpec)
                }
            }
        }
    }
}
```

### GitHub Actions Integration

```yaml
# .github/workflows/build.yml
name: Build and Publish
on:
  push:
    branches: [main]

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      
      - name: Build Docker Image
        run: docker build -t myapp:${{ github.sha }} .
      
      - name: Login to Artifactory
        run: |
          echo ${{ secrets.ARTIFACTORY_PASSWORD }} | \
          docker login artifactory.mycompany.com \
            -u ${{ secrets.ARTIFACTORY_USER }} \
            --password-stdin
      
      - name: Push to Artifactory
        run: |
          docker tag myapp:${{ github.sha }} \
            artifactory.mycompany.com/docker-local/myapp:${{ github.sha }}
          docker push \
            artifactory.mycompany.com/docker-local/myapp:${{ github.sha }}
```

### GitLab CI Integration

```yaml
# .gitlab-ci.yml
stages:
  - build
  - publish

build:
  stage: build
  script:
    - docker build -t myapp:${CI_COMMIT_SHA} .

    - docker tag myapp:${CI_COMMIT_SHA} artifactory.mycompany.com/docker-local/myapp:${CI_COMMIT_SHA}
    - docker push artifactory.mycompany.com/docker-local/myapp:${CI_COMMIT_SHA}
```

## 🔧 JFrog CLI — Command Line Tool

JFrog provides a powerful CLI tool called `jf` (formerly `jfrog`) for interacting with Artifactory:

### Installation:

```bash
# macOS
brew install jfrog-cli

# Linux
curl -fL https://install-cli.jfrog.io | sh

# Windows (PowerShell)
iwr https://install-cli.jfrog.io -UseBasicParsing | iex
```

### Configuration:

```bash
# Configure connection to Artifactory
jf config add my-server \
  --url=https://artifactory.mycompany.com \
  --user=admin \
  --password=your-password \
  --interactive=false
```

### Common Commands:

```bash
# Upload artifacts
jf rt upload "target/*.jar" libs-release-local/com/mycompany/myapp/1.0.0/

# Download artifacts
jf rt download "libs-release-local/com/mycompany/myapp/1.0.0/*.jar" ./local-dir/

# Search artifacts
jf rt search "libs-release-local/com/mycompany/myapp/*"

# Copy artifacts between repos (promotion)
jf rt copy \
  "libs-staging-local/com/mycompany/myapp/1.0.0/" \
  "libs-release-local/com/mycompany/myapp/1.0.0/"

# Delete artifacts
jf rt delete "libs-snapshot-local/com/mycompany/myapp/0.1-SNAPSHOT/"

# Set properties (metadata) on artifacts
jf rt set-props \
  "libs-release-local/com/mycompany/myapp/1.0.0/myapp-1.0.0.jar" \
  "build.name=myapp;build.number=42;quality.gate=passed"

# Build Docker image and push through Artifactory
jf docker build -t artifactory.mycompany.com/docker-local/myapp:v1.0.0 .
jf docker push artifactory.mycompany.com/docker-local/myapp:v1.0.0

# Scan artifacts for vulnerabilities (requires Xray)
jf scan "libs-release-local/com/mycompany/myapp/1.0.0/myapp-1.0.0.jar"
```

## 📝 Quick Summary

| Concept | Key Takeaway |
|---------|-------------|
| **What is Artifactory?** | Universal artifact repository — stores ALL artifact types in one place |
| **Local Repo** | Your private storage — where your build artifacts go |
| **Remote Repo** | Proxy + cache for public registries (Docker Hub, npm, Maven Central) |
| **Virtual Repo** | Combines Local + Remote under one URL — simplifies configuration |
| **Federated Repo** | Multi-site replication for global teams |
| **Security** | Users, groups, permissions, API keys, tokens |
| **JFrog CLI (`jf`)** | Command-line tool for uploads, downloads, searches, and CI/CD integration |
| **Editions** | OSS (free, limited), Pro (most features), Enterprise+ (multi-site, HA) |
