## 1. The Problem MLflow Solves

Imagine you're a data scientist or ML engineer. You've trained 50 different models over the past month. Now ask yourself:

- Which model gave the best accuracy?
- What hyperparameters did you use for that one run last Tuesday?
- Which dataset version was used?
- Which model is currently in production?
- Can you reproduce that result?

If your answer is "I have no idea" or "let me check my Jupyter notebook history" — **you have a problem**.

This is exactly what MLflow solves. It brings **order to the chaos of machine learning experimentation**.

### Without MLflow:
```
notebooks/
├── model_v1_final.ipynb
├── model_v1_final_FINAL.ipynb
├── model_v1_ACTUALLY_final.ipynb
├── model_v2_test.ipynb
├── model_v2_test_lr0.001.ipynb      ← What was the accuracy here??
└── model_v3_maybe_best.ipynb         ← No one remembers
```

### With MLflow:
```
MLflow UI → Experiments → All runs neatly tracked with:
  - Parameters (learning_rate=0.001, epochs=50)
  - Metrics (accuracy=0.94, loss=0.12)
  - Artifacts (saved model, plots, data snapshots)
  - Tags (author, dataset_version, git_commit)
  - Model versions (v1=staging, v2=production)
```

**Bottom line:** MLflow gives you **reproducibility**, **traceability**, and **collaboration** for ML workflows.

## 2. What is MLflow?

**MLflow** is an **open-source platform** (created by Databricks) for managing the **entire machine learning lifecycle**.

Think of it like this:

| What Git does for code... | MLflow does for ML experiments |
|---|---|
| Tracks code changes | Tracks experiment parameters, metrics, artifacts |
| Branches & versions | Model versions & stages |
| Collaboration (GitHub) | Shared tracking server |
| Reproducibility | MLflow Projects + environment tracking |

### Key Facts:

| Property | Detail |
|---|---|
| Created by | Databricks |
| First Release | 2018 |
| Language | Python (core), REST API for any language |
| License | Apache 2.0 (fully open source) |
| Runs on | Local machine, on-prem servers, cloud |
| Integrates with | Scikit-learn, TensorFlow, PyTorch, XGBoost, Spark, Kubernetes, Docker, AWS, Azure, GCP |

## 3. MLflow Architecture & Components Overview

MLflow has **four main components** (think of them as four pillars):

```
┌──────────────────────────────────────────────────────────┐
│                        MLflow                            │
│                                                          │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐   │
│  │   Tracking    │  │   Projects   │  │    Models     │   │
│  │              │  │              │  │              │   │
│  │ Log params,  │  │ Reproducible │  │ Package &    │   │
│  │ metrics,     │  │ ML code      │  │ deploy       │   │
│  │ artifacts    │  │ packaging    │  │ models       │   │
│  └──────────────┘  └──────────────┘  └──────────────┘   │
│                                                          │
│  ┌───────────────────────────────────────────────────┐   │
│  │               Model Registry                      │   │
│  │                                                   │   │
│  │  Central hub to manage model versions,            │   │
│  │  stage transitions, approvals                     │   │
│  └───────────────────────────────────────────────────┘   │
└──────────────────────────────────────────────────────────┘
```

### Quick Summary of Each Component:

| Component | What It Does | Analogy |
|---|---|---|
| **Tracking** | Logs parameters, metrics, and artifacts for each experiment run | Like Git commits for experiments |
| **Projects** | Packages ML code in a reproducible format | Like Dockerfiles for ML code |
| **Models** | Standardized format to save/load/deploy models | Like container images for ML models |
| **Model Registry** | Central store to version, stage, and manage models | Like a Docker Registry, but for ML models |

## 5. Component 1: MLflow Tracking

This is the **most used** component of MLflow. It lets you **log everything** about your experiments.

### 5.1 — Core Concepts

| Concept | Meaning |
|---|---|
| **Experiment** | A named group of runs (e.g., "fraud-detection-model") |
| **Run** | A single execution of your training code |
| **Parameters** | Input settings (learning_rate, batch_size, epochs) |
| **Metrics** | Output measurements (accuracy, loss, F1 score) |
| **Artifacts** | Output files (model files, plots, data snapshots) |
| **Tags** | Key-value metadata (author, team, dataset_version) |

Think of it like this:
```
Experiment: "Fraud Detection"
├── Run 1: lr=0.01, epochs=10 → accuracy=0.85
├── Run 2: lr=0.001, epochs=50 → accuracy=0.91
├── Run 3: lr=0.001, epochs=100 → accuracy=0.94  ← Best!
└── Run 4: lr=0.0001, epochs=100 → accuracy=0.88
```

### 5.2 — Basic Tracking Example

Here's a simple Python script that trains a model and logs everything to MLflow:

```python
import mlflow
import mlflow.sklearn
from sklearn.ensemble import RandomForestClassifier
from sklearn.datasets import load_iris
from sklearn.model_selection import train_test_split
from sklearn.metrics import accuracy_score, f1_score

# Load data
data = load_iris()
X_train, X_test, y_train, y_test = train_test_split(
    data.data, data.target, test_size=0.2, random_state=42
)

# Set the experiment name
mlflow.set_experiment("iris-classification")

# Start a new run
with mlflow.start_run(run_name="random-forest-v1"):

    # Define hyperparameters
    n_estimators = 100
    max_depth = 5

    # Log parameters (inputs)
    mlflow.log_param("n_estimators", n_estimators)
    mlflow.log_param("max_depth", max_depth)
    mlflow.log_param("random_state", 42)

    # Train model
    model = RandomForestClassifier(
        n_estimators=n_estimators,
        max_depth=max_depth,
        random_state=42
    )
    model.fit(X_train, y_train)

    # Predict
    predictions = model.predict(X_test)

    # Calculate metrics
    accuracy = accuracy_score(y_test, predictions)
    f1 = f1_score(y_test, predictions, average='weighted')

    # Log metrics (outputs)
    mlflow.log_metric("accuracy", accuracy)
    mlflow.log_metric("f1_score", f1)

    # Log the model as an artifact
    mlflow.sklearn.log_model(model, "random-forest-model")

    # Log a tag
    mlflow.set_tag("author", "ritesh")
    mlflow.set_tag("dataset", "iris")

    print(f"Accuracy: {accuracy:.4f}")
    print(f"F1 Score: {f1:.4f}")
```

**What happens when you run this?**

1. MLflow creates an experiment called `iris-classification`
2. Inside it, a run called `random-forest-v1` is created
3. Parameters (`n_estimators=100`, `max_depth=5`) are logged
4. Metrics (`accuracy`, `f1_score`) are logged
5. The trained model is saved as an artifact
6. Tags are added for searchability

Now run `mlflow ui` and you'll see everything nicely displayed in a web dashboard!

### 5.3 — Autologging (The Easy Way)

Don't want to manually log everything? MLflow can **automatically** log params, metrics, and models for supported frameworks:

```python
import mlflow
from sklearn.ensemble import RandomForestClassifier
from sklearn.datasets import load_iris
from sklearn.model_selection import train_test_split

# Enable autologging for sklearn
mlflow.sklearn.autolog()

# Load data
data = load_iris()
X_train, X_test, y_train, y_test = train_test_split(
    data.data, data.target, test_size=0.2, random_state=42
)

# Just train — MLflow logs EVERYTHING automatically!
with mlflow.start_run():
    model = RandomForestClassifier(n_estimators=100, max_depth=5)
    model.fit(X_train, y_train)
```

**Autolog supports:** scikit-learn, TensorFlow, Keras, PyTorch, XGBoost, LightGBM, Spark, and more!

What autolog captures automatically:
- All hyperparameters
- Training metrics
- The trained model
- Feature importance plots (for tree-based models)
- Confusion matrix
- Model signature (input/output schema)

### 5.4 — Logging Metrics Over Time

For deep learning, you often want to track metrics **at each epoch**:

```python
import mlflow

mlflow.set_experiment("deep-learning-tracking")

with mlflow.start_run():
    mlflow.log_param("learning_rate", 0.001)
    mlflow.log_param("epochs", 10)

    # Simulate training loop
    for epoch in range(10):
        train_loss = 1.0 / (epoch + 1)  # Simulated decreasing loss
        val_accuracy = 0.5 + (epoch * 0.05)  # Simulated increasing accuracy

        # Log metric at each step
        mlflow.log_metric("train_loss", train_loss, step=epoch)
        mlflow.log_metric("val_accuracy", val_accuracy, step=epoch)

        print(f"Epoch {epoch}: loss={train_loss:.4f}, accuracy={val_accuracy:.4f}")
```

This creates **beautiful line charts** in the MLflow UI showing how your metrics change over time.

### 5.5 — Logging Artifacts

Artifacts are any files you want to save alongside your run — plots, data snapshots, config files, etc.

```python
import mlflow
import matplotlib.pyplot as plt
import json
import os

with mlflow.start_run():

    # 1. Log a single file
    # Create a plot
    plt.figure(figsize=(10, 6))
    plt.plot([1, 2, 3, 4, 5], [0.8, 0.85, 0.9, 0.93, 0.95])
    plt.title("Accuracy Over Epochs")
    plt.xlabel("Epoch")
    plt.ylabel("Accuracy")
    plt.savefig("accuracy_plot.png")
    plt.close()

    mlflow.log_artifact("accuracy_plot.png")  # Logs the file

    # 2. Log a dictionary as JSON
    config = {
        "model": "RandomForest",
        "preprocessing": "StandardScaler",
        "feature_selection": "top_10"
    }
    with open("config.json", "w") as f:
        json.dump(config, f, indent=2)

    mlflow.log_artifact("config.json")

    # 3. Log an entire directory
    os.makedirs("plots", exist_ok=True)
    # ... (create multiple plot files in plots/)
    mlflow.log_artifacts("plots", artifact_path="training-plots")

    # Clean up local files
    os.remove("accuracy_plot.png")
    os.remove("config.json")
```

### 5.6 — Searching & Comparing Runs

MLflow provides a powerful search API:

```python
import mlflow

# Search for runs with accuracy > 0.9
runs = mlflow.search_runs(
    experiment_names=["iris-classification"],
    filter_string="metrics.accuracy > 0.9",
    order_by=["metrics.accuracy DESC"]
)

print(runs[["run_id", "params.n_estimators", "metrics.accuracy"]])
```

You can also search using the **MLflow UI**:
- Open the experiment
- Use the filter bar: `metrics.accuracy > 0.9 and params.max_depth = "5"`
- Select multiple runs → click "Compare" for side-by-side comparison

## 6. Component 2: MLflow Projects

### What is it?

An **MLflow Project** is a way to **package your ML code** so that anyone can reproduce your results. Think of it like a `Dockerfile` but specifically for ML code.

### Project Structure:

```
my-ml-project/
├── MLproject           ← Project definition file (like Dockerfile)
├── conda.yaml          ← Environment definition (dependencies)
├── train.py            ← Training script
└── data/
    └── dataset.csv
```

### The `MLproject` File:

```yaml
name: my-ml-project

# Define the environment
conda_env: conda.yaml

# Or use Docker instead of Conda:
# docker_env:
#   image: my-training-image:latest

# Define entry points (like commands you can run)
entry_points:
  main:
    parameters:
      learning_rate: {type: float, default: 0.01}
      epochs: {type: int, default: 10}
      data_path: {type: str, default: "data/dataset.csv"}
    command: "python train.py --lr {learning_rate} --epochs {epochs} --data {data_path}"

  validate:
    parameters:
      model_path: {type: str}
    command: "python validate.py --model {model_path}"
```

### The `conda.yaml` File:

```yaml
name: my-ml-env
channels:
  - defaults
  - conda-forge
dependencies:
  - python=3.10
  - scikit-learn=1.3.0
  - pandas=2.0.0
  - pip:
    - mlflow>=2.0
    - matplotlib
```

### Running an MLflow Project:

```bash
# Run locally
mlflow run . -P learning_rate=0.001 -P epochs=50

# Run from a Git repo (yes, directly from GitHub!)
mlflow run https://github.com/your-org/ml-project.git -P learning_rate=0.001

# Run a specific entry point
mlflow run . -e validate -P model_path="models/best_model"

# Run with Docker backend
mlflow run . --backend docker
```

### Why This Matters:

- **Reproducibility:** Anyone can clone the repo and run the exact same experiment
- **Environment isolation:** Conda/Docker ensures consistent dependencies
- **Parameterization:** Easy to sweep over hyperparameters
- **Remote execution:** Run on any machine or cloud service

## 7. Component 3: MLflow Models

### What is it?

MLflow Models provides a **standard format** to package ML models so they can be deployed **anywhere** — as a REST API, as a Docker container, on Kubernetes, on AWS SageMaker, etc.

### The MLmodel File:

When you save a model with MLflow, it creates a directory like this:

```
random-forest-model/
├── MLmodel              ← Metadata (flavors, signature, etc.)
├── model.pkl            ← The actual model file
├── conda.yaml           ← Environment for serving
├── requirements.txt     ← Python dependencies
└── input_example.json   ← Example input for testing
```

The `MLmodel` file looks like:

```yaml
artifact_path: random-forest-model
flavors:
  python_function:
    env: conda.yaml
    loader_module: mlflow.sklearn
    model_path: model.pkl
    python_version: 3.10.12
  sklearn:
    pickled_model: model.pkl
    serialization_format: cloudpickle
    sklearn_version: 1.3.0
signature:
  inputs: '[{"type": "double", "name": "sepal_length"}, ...]'
  outputs: '[{"type": "long"}]'
```

### What Are "Flavors"?

Flavors are **different ways to interact with the model**. Every model has at least the `python_function` flavor (a generic Python interface), plus framework-specific flavors:

| Flavor | Use Case |
|---|---|
| `python_function` | Generic — works everywhere |
| `sklearn` | Scikit-learn specific loading |
| `tensorflow` | TensorFlow/Keras specific |
| `pytorch` | PyTorch specific |
| `spark` | Apache Spark ML |
| `xgboost` | XGBoost specific |
| `onnx` | ONNX format (cross-framework) |

### Saving and Loading Models:

```python
import mlflow
import mlflow.sklearn
from sklearn.ensemble import RandomForestClassifier

# Train a model
model = RandomForestClassifier()
model.fit(X_train, y_train)

# Save the model
with mlflow.start_run():
    # Method 1: Framework-specific save
    mlflow.sklearn.log_model(model, "my-model")

    # Method 2: With a signature (recommended)
    from mlflow.models.signature import infer_signature
    signature = infer_signature(X_train, model.predict(X_train))
    mlflow.sklearn.log_model(model, "my-model", signature=signature)
```

```python
# Load the model later
# Using the run ID
model = mlflow.sklearn.load_model("runs:/abc123def456/my-model")

# Using the generic python_function flavor
model = mlflow.pyfunc.load_model("runs:/abc123def456/my-model")

# Predict
predictions = model.predict(X_test)
```

### Serving a Model as a REST API:

```bash
# Serve the model from a run
mlflow models serve -m "runs:/abc123def456/my-model" --port 5001

# Serve from the model registry (more on this below)
mlflow models serve -m "models:/fraud-detector/Production" --port 5001
```

Now you can make predictions via HTTP:

```bash
# Send a prediction request
curl -X POST http://localhost:5001/invocations \
  -H "Content-Type: application/json" \
  -d '{
    "inputs": [[5.1, 3.5, 1.4, 0.2], [6.7, 3.0, 5.2, 2.3]]
  }'

# Response:
# {"predictions": [0, 2]}
```

### Build a Docker Image:

```bash
# Build a Docker image for the model
mlflow models build-docker -m "runs:/abc123def456/my-model" -n "my-model-server"

# Run it
docker run -p 5001:8080 my-model-server
```

## 8. Component 4: MLflow Model Registry

This is the **most important component from a DevOps perspective**. The Model Registry is like a **version-controlled repository for ML models**.

### 8.1 — Why Do You Need a Model Registry?

Without a registry:
```
"Hey, which model is in production?"
"Um... I think it's the one John trained last week?"
"Which version?"
"The good one?"
```

With a Model Registry:
```
Model: fraud-detector
├── Version 1 → Stage: Archived   (accuracy: 0.85)
├── Version 2 → Stage: Staging    (accuracy: 0.91)
├── Version 3 → Stage: Production (accuracy: 0.94)  ← Currently serving
└── Version 4 → Stage: None       (accuracy: 0.89 — didn't pass tests)
```

### 8.2 — Model Stages

MLflow Model Registry has **lifecycle stages**:

| Stage | Meaning |
|---|---|
| **None** | Just registered, not promoted yet |
| **Staging** | Being tested / validated before production |
| **Production** | Currently serving real traffic |
| **Archived** | Old version, kept for reference |

```
Flow:  None → Staging → Production → Archived
```

### 8.3 — Registering a Model

```python
import mlflow
import mlflow.sklearn

# During training, register the model directly
with mlflow.start_run():
    model = RandomForestClassifier(n_estimators=100)
    model.fit(X_train, y_train)

    # Log and register in one step
    mlflow.sklearn.log_model(
        model,
        "model",
        registered_model_name="fraud-detector"  # ← This registers it!
    )
```

Or register an existing run's model:

```python
# Register a model from an existing run
result = mlflow.register_model(
    model_uri="runs:/abc123def456/model",
    name="fraud-detector"
)

print(f"Registered version: {result.version}")
```

### 8.4 — Managing Model Versions

```python
from mlflow.tracking import MlflowClient

client = MlflowClient()

# Transition a model version to Staging
client.transition_model_version_stage(
    name="fraud-detector",
    version=3,
    stage="Staging"
)

# After testing, promote to Production
client.transition_model_version_stage(
    name="fraud-detector",
    version=3,
    stage="Production"
)

# Archive the old production model
client.transition_model_version_stage(
    name="fraud-detector",
    version=2,
    stage="Archived"
)

# Add a description
client.update_model_version(
    name="fraud-detector",
    version=3,
    description="Random Forest with feature engineering v2. Accuracy: 0.94"
)
```

### 8.5 — Loading Models from the Registry

This is where it gets powerful — your serving code doesn't need to know run IDs!

```python
import mlflow

# Load the Production model (always gets the latest Production version)
model = mlflow.pyfunc.load_model("models:/fraud-detector/Production")

# Load a specific version
model = mlflow.pyfunc.load_model("models:/fraud-detector/3")

# Load the Staging model (for testing)
model = mlflow.pyfunc.load_model("models:/fraud-detector/Staging")

# Make predictions
predictions = model.predict(new_data)
```

### 8.6 — Model Registry via the UI

The MLflow UI provides a beautiful interface to:

- View all registered models
- See version history
- Compare metrics across versions
- Transition stages with one click
- Add descriptions and tags
- View lineage (which run created this version)

## 9. MLflow Tracking Server — Setting Up for Teams

On your local machine, MLflow stores everything in a local `./mlruns` directory. For **team collaboration**, you need a **Tracking Server**.

### 9.1 — Architecture

```
┌──────────────────┐     ┌──────────────────┐     ┌──────────────────┐
│   Data Scientist │     │  Data Scientist  │     │   ML Engineer    │
│   (Laptop)       │     │  (Laptop)        │     │  (Laptop)        │
└────────┬─────────┘     └────────┬─────────┘     └────────┬─────────┘
         │                        │                        │
         └────────────┬───────────┴───────────┬────────────┘
                      │                       │
              ┌───────▼───────┐       ┌───────▼───────┐
              │   MLflow       │       │  Artifact     │
              │   Tracking     │       │  Store        │
              │   Server       │       │  (S3/GCS/     │
              │   (metadata)   │       │   Azure Blob) │
              └───────┬───────┘       └───────────────┘
                      │
              ┌───────▼───────┐
              │  Backend DB    │
              │  (PostgreSQL/  │
              │   MySQL)       │
              └───────────────┘
```

### 9.2 — Setting Up with PostgreSQL & S3

```bash
# Start MLflow Tracking Server
mlflow server \
  --backend-store-uri postgresql://user:password@db-host:5432/mlflow \
  --default-artifact-root s3://my-mlflow-bucket/artifacts \
  --host 0.0.0.0 \
  --port 5000
```

Let's break this down:

| Flag | What It Does |
|---|---|
| `--backend-store-uri` | Where to store experiment metadata (params, metrics) — use a real database |
| `--default-artifact-root` | Where to store artifacts (models, plots) — use cloud storage |
| `--host 0.0.0.0` | Listen on all network interfaces |
| `--port 5000` | Port number |

### 9.3 — Docker Compose Setup

Here's a ready-to-use Docker Compose file for running MLflow with PostgreSQL:

```yaml
# docker-compose.yml
version: "3.8"

services:
  postgres:
    image: postgres:15
    environment:
      POSTGRES_DB: mlflow
      POSTGRES_USER: mlflow
      POSTGRES_PASSWORD: mlflow_password
    volumes:
      - postgres_data:/var/lib/postgresql/data
    ports:
      - "5432:5432"

  minio:
    image: minio/minio:latest
    command: server /data --console-address ":9001"
    environment:
      MINIO_ROOT_USER: minioadmin
      MINIO_ROOT_PASSWORD: minioadmin
    volumes:
      - minio_data:/data
    ports:
      - "9000:9000"
      - "9001:9001"  # MinIO Console

  mlflow:
    image: ghcr.io/mlflow/mlflow:latest
    depends_on:
      - postgres
      - minio
    environment:
      MLFLOW_S3_ENDPOINT_URL: http://minio:9000
      AWS_ACCESS_KEY_ID: minioadmin
      AWS_SECRET_ACCESS_KEY: minioadmin
    command: >
      mlflow server
        --backend-store-uri postgresql://mlflow:mlflow_password@postgres:5432/mlflow
        --default-artifact-root s3://mlflow-artifacts/
        --host 0.0.0.0
        --port 5000
    ports:
      - "5000:5000"

volumes:
  postgres_data:
  minio_data:
```

```bash
# Start everything
docker compose up -d

# Access MLflow UI at http://localhost:5000
# Access MinIO Console at http://localhost:9001
```

## 11. MLflow in CI/CD Pipelines

Here's how MLflow fits into an automated ML pipeline:

```
┌──────────┐     ┌──────────┐     ┌──────────┐     ┌──────────┐     ┌──────────┐
│  Code    │────▶│  Train   │────▶│  Evaluate│────▶│ Register │────▶│  Deploy  │
│  Push    │     │  Model   │     │  Model   │     │  Model   │     │  Model   │
│  (Git)   │     │  (MLflow │     │  (Check  │     │  (MLflow │     │  (K8s/   │
│          │     │  Tracking│     │  metrics)│     │  Registry│     │  Docker) │
└──────────┘     └──────────┘     └──────────┘     └──────────┘     └──────────┘
```

### GitHub Actions Example:

```yaml
# .github/workflows/ml-pipeline.yml
name: ML Training Pipeline

on:
  push:
    paths:
      - 'models/**'
      - 'data/**'

env:
  MLFLOW_TRACKING_URI: ${{ secrets.MLFLOW_TRACKING_URI }}

jobs:
  train-and-evaluate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: '3.10'

      - name: Install dependencies
        run: pip install -r requirements.txt

      - name: Train model
        run: python train.py --epochs 50 --lr 0.001

      - name: Evaluate model
        run: |
          python evaluate.py
          # This script checks if new model > current production model
          # If yes, it registers the new model in MLflow Registry

      - name: Promote to Staging
        run: python promote_model.py --stage Staging

  deploy:
    needs: train-and-evaluate
    runs-on: ubuntu-latest
    steps:
      - name: Deploy Staging Model
        run: |
          # Build Docker image with the Staging model
          mlflow models build-docker \
            -m "models:/fraud-detector/Staging" \
            -n "fraud-detector:staging"

          # Push to container registry
          docker push my-registry/fraud-detector:staging

          # Deploy to Kubernetes staging
          kubectl set image deployment/fraud-detector \
            model-server=my-registry/fraud-detector:staging
```

## 12. MLflow vs Other Tools

| Feature | MLflow | Weights & Biases (W&B) | Neptune.ai | DVC | ClearML |
|---|---|---|---|---|---|
| **Open Source** | Yes (fully) | No (free tier) | No (free tier) | Yes | Yes |
| **Experiment Tracking** | Yes | Yes (excellent) | Yes | Basic | Yes |
| **Model Registry** | Yes | Yes | Yes | No | Yes |
| **Model Serving** | Yes | No | No | No | Yes |
| **Data Versioning** | No | Yes | No | Yes (excellent) | Yes |
| **Self-Hosted** | Yes | Enterprise only | Enterprise only | Yes | Yes |
| **UI Quality** | Good | Excellent | Excellent | Basic | Good |
| **Learning Curve** | Low | Low | Low | Medium | Low |
| **Best For** | Full lifecycle, on-prem | Research, collaboration | Research | Data-heavy ML | End-to-end |

### When to Choose MLflow:
- You want a **fully open-source** solution
- You need **self-hosted** (on-prem or your own cloud)
- You want a **complete platform** (tracking + registry + serving)
- You're already using **Databricks** (MLflow is native there)
- You need to run in **air-gapped environments**

### 13.1 — Experiment Organization

### 13.2 — Always Log Model Signatures

```python
from mlflow.models.signature import infer_signature

# Infer signature from training data
signature = infer_signature(X_train, model.predict(X_train))

# Log model with signature
mlflow.sklearn.log_model(model, "model", signature=signature)
```

This helps catch data type mismatches during serving.

### 13.3 — Version Everything

```python
with mlflow.start_run():
    # Log code version
    mlflow.set_tag("mlflow.source.git.commit", get_git_hash())

    # Log data version
    mlflow.log_param("data_version", "v2.3")
    mlflow.log_param("data_hash", hash_dataset(training_data))

    # Log environment
    mlflow.log_artifact("requirements.txt")
    mlflow.log_artifact("conda.yaml")
```

### 13.4 — Use Nested Runs for Hyperparameter Tuning

```python
with mlflow.start_run(run_name="hyperparameter-sweep"):
    for lr in [0.001, 0.01, 0.1]:
        for depth in [3, 5, 10]:

                model = train_model(lr=lr, max_depth=depth)

                mlflow.log_param("lr", lr)
                mlflow.log_param("max_depth", depth)
                mlflow.log_metric("accuracy", accuracy)
```

## 14. Hands-On: Complete End-to-End Example

Let's build a complete MLOps workflow with MLflow:

```python
"""
Complete MLflow Pipeline Example
File: train_pipeline.py
"""

import mlflow
import mlflow.sklearn
from mlflow.models.signature import infer_signature
from mlflow.tracking import MlflowClient

from sklearn.ensemble import RandomForestClassifier, GradientBoostingClassifier
from sklearn.linear_model import LogisticRegression
from sklearn.datasets import load_breast_cancer
from sklearn.model_selection import train_test_split, cross_val_score
from sklearn.preprocessing import StandardScaler
from sklearn.metrics import accuracy_score, precision_score, recall_score, f1_score
import numpy as np


def load_and_prepare_data():
    """Load and preprocess the data."""
    data = load_breast_cancer()
    X_train, X_test, y_train, y_test = train_test_split(
        data.data, data.target, test_size=0.2, random_state=42
    )

    scaler = StandardScaler()
    X_train_scaled = scaler.fit_transform(X_train)
    X_test_scaled = scaler.transform(X_test)

    return X_train_scaled, X_test_scaled, y_train, y_test, data.feature_names


def train_and_evaluate(model, X_train, X_test, y_train, y_test):
    """Train model and return metrics."""
    model.fit(X_train, y_train)
    predictions = model.predict(X_test)

    return {
        "accuracy": accuracy_score(y_test, predictions),
        "precision": precision_score(y_test, predictions),
        "recall": recall_score(y_test, predictions),
        "f1_score": f1_score(y_test, predictions),
        "cv_score": np.mean(cross_val_score(model, X_train, y_train, cv=5))
    }


def run_experiment():
    """Run the full experiment with multiple models."""
    # Setup
    mlflow.set_tracking_uri("http://localhost:5000")
    mlflow.set_experiment("breast-cancer-detection")

    X_train, X_test, y_train, y_test, feature_names = load_and_prepare_data()

    # Define models to try
    models = {
        "LogisticRegression": LogisticRegression(max_iter=1000),
        "RandomForest": RandomForestClassifier(n_estimators=100, max_depth=10),
        "GradientBoosting": GradientBoostingClassifier(n_estimators=100, max_depth=5),
    }

    best_model_name = None
    best_accuracy = 0

    # Parent run for the experiment
    with mlflow.start_run(run_name="model-comparison"):
        mlflow.set_tag("experiment_type", "model_selection")
        mlflow.set_tag("dataset", "breast_cancer_sklearn")

        for model_name, model in models.items():
            # Child run for each model
            with mlflow.start_run(run_name=model_name, nested=True):
                # Log parameters
                mlflow.log_params(model.get_params())

                # Train and evaluate
                metrics = train_and_evaluate(model, X_train, X_test, y_train, y_test)

                # Log metrics
                mlflow.log_metrics(metrics)

                # Log model with signature
                signature = infer_signature(X_train, model.predict(X_train))
                mlflow.sklearn.log_model(model, "model", signature=signature)

                print(f"{model_name}: accuracy={metrics['accuracy']:.4f}")

                # Track best model
                if metrics["accuracy"] > best_accuracy:
                    best_accuracy = metrics["accuracy"]
                    best_model_name = model_name

        mlflow.log_param("best_model", best_model_name)
        mlflow.log_metric("best_accuracy", best_accuracy)

    print(f"\nBest model: {best_model_name} (accuracy: {best_accuracy:.4f})")


def register_best_model():
    """Register the best model in the Model Registry."""
    client = MlflowClient()

    # Find the best run
    experiment = client.get_experiment_by_name("breast-cancer-detection")
    runs = client.search_runs(
        experiment_ids=[experiment.experiment_id],
        filter_string="tags.experiment_type = 'model_selection'",
        order_by=["metrics.best_accuracy DESC"],
        max_results=1
    )

    if runs:
        best_run = runs[0]
        model_uri = f"runs:/{best_run.info.run_id}/model"

        # Register the model
        result = mlflow.register_model(model_uri, "cancer-detector")

        # Transition to Staging
        client.transition_model_version_stage(
            name="cancer-detector",
            version=result.version,
            stage="Staging"
        )

        print(f"Registered model version {result.version} → Staging")


if __name__ == "__main__":
    run_experiment()
    register_best_model()
```
