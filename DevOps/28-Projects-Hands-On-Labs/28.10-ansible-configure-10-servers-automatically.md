> **Goal:** Build a complete Ansible project that automatically configures 10 servers — installing packages, managing users, deploying apps, setting up firewalls, and more — all with a single command.

## 🚀 Step 1 — Set Up Your Lab Environment with Vagrant

If you don't have 10 servers lying around (who does?), let's create them with Vagrant + VirtualBox.

### Install Vagrant & VirtualBox

```bash
# Ubuntu/Debian
sudo apt update
sudo apt install -y virtualbox vagrant

# macOS
brew install --cask virtualbox vagrant

# Windows
# Download from https://www.vagrantup.com/downloads
# Download from https://www.virtualbox.org/wiki/Downloads
```

### Create Vagrantfile

Create a project directory and the Vagrantfile:

```bash
mkdir ~/ansible-10-servers && cd ~/ansible-10-servers
```

```ruby
# Vagrantfile
# This creates 10 VMs — 3 web, 4 app, 3 db

Vagrant.configure("2") do |config|
  config.vm.box = "ubuntu/jammy64"    # Ubuntu 22.04

  # --- Web Servers ---
  (1..3).each do |i|
    config.vm.define "web#{i}" do |web|
      web.vm.hostname = "web#{i}"
      web.vm.network "private_network", ip: "192.168.56.#{10 + i}"
      web.vm.provider "virtualbox" do |vb|
        vb.memory = "512"
        vb.cpus = 1
      end
    end
  end

  # --- App Servers ---
  (1..4).each do |i|
    config.vm.define "app#{i}" do |app|
      app.vm.hostname = "app#{i}"
      app.vm.network "private_network", ip: "192.168.56.#{20 + i}"
      app.vm.provider "virtualbox" do |vb|
        vb.memory = "1024"
        vb.cpus = 1
      end
    end
  end

  # --- DB Servers ---
  (1..3).each do |i|
    config.vm.define "db#{i}" do |db|
      db.vm.hostname = "db#{i}"
      db.vm.network "private_network", ip: "192.168.56.#{30 + i}"
      db.vm.provider "virtualbox" do |vb|
        vb.memory = "1024"
        vb.cpus = 1
      end
    end
  end
end
```

**What this does:**
- Creates **10 Ubuntu 22.04 VMs** with private network IPs
- Web servers: `192.168.56.11-13` (512 MB RAM each)
- App servers: `192.168.56.21-24` (1 GB RAM each)
- DB servers: `192.168.56.31-33` (1 GB RAM each)

### Boot All VMs

```bash
vagrant up
```

> ⏰ This will take a few minutes the first time (downloads the base image).

### Set Up SSH Keys

```bash
# Generate SSH key on your control node (if you don't have one)
ssh-keygen -t ed25519 -C "ansible" -f ~/.ssh/ansible_key -N ""

# Copy the key to all 10 VMs
for ip in 192.168.56.{11..13} 192.168.56.{21..24} 192.168.56.{31..33}; do
    ssh-copy-id -i ~/.ssh/ansible_key.pub vagrant@$ip
done
# Default Vagrant password is: vagrant
```

### Verify Connectivity

```bash
# Quick test — can we reach all 10?
for ip in 192.168.56.{11..13} 192.168.56.{21..24} 192.168.56.{31..33}; do
    ssh -i ~/.ssh/ansible_key vagrant@$ip "hostname"
done
```

You should see all 10 hostnames printed. If yes — we're ready! 🎉

## 🚀 Step 4 — Ansible Configuration File

```ini
# ansible.cfg
[defaults]
inventory = inventories/production/hosts.ini
remote_user = vagrant
private_key_file = ~/.ssh/ansible_key
host_key_checking = False
retry_files_enabled = False
roles_path = roles

# Speed up Ansible with pipelining
[ssh_connection]
pipelining = True

[privilege_escalation]
become = True
become_method = sudo
become_ask_pass = False
```

**What each setting does:**

| Setting | Explanation |
|---------|-------------|
| `inventory` | Default location of our server list |
| `remote_user` | SSH as this user on remote servers |
| `private_key_file` | SSH key to use |
| `host_key_checking = False` | Don't ask "Are you sure you want to connect?" every time |
| `pipelining = True` | Makes Ansible faster by reducing SSH operations |
| `become = True` | Auto-escalate to root (sudo) on remote servers |

## 🚀 Step 5 — Inventory File (Our 10 Servers)

```ini
# inventories/production/hosts.ini

# ============================================
# Web Servers — These run Nginx
# ============================================
[webservers]
web1 ansible_host=192.168.56.11
web2 ansible_host=192.168.56.12
web3 ansible_host=192.168.56.13

# ============================================
# Application Servers — These run our Node.js app
# ============================================
[appservers]
app1 ansible_host=192.168.56.21
app2 ansible_host=192.168.56.22
app3 ansible_host=192.168.56.23
app4 ansible_host=192.168.56.24

# ============================================
# Database Servers — These run MySQL
# ============================================
[dbservers]
db1 ansible_host=192.168.56.31
db2 ansible_host=192.168.56.32
db3 ansible_host=192.168.56.33

# ============================================
# Group of Groups — All servers
# ============================================
[all_servers:children]
webservers
appservers
dbservers
```

**What's happening here?**

- We have **3 groups**: `webservers`, `appservers`, `dbservers`
- Each server has a **friendly name** (web1, app1, db1) and its **IP address**
- `all_servers:children` — creates a parent group that includes all three groups

## 🚀 Step 8 — Create Ansible Roles

Now the fun part! Let's create our roles.

### 8a — Common Role (Base Configuration for ALL Servers)

```bash
# Create role structure
ansible-galaxy init roles/common
```

This creates:

```
roles/common/
├── defaults/main.yml    # Default variables
├── handlers/main.yml    # Event handlers
├── tasks/main.yml       # Main task list
├── templates/           # Jinja2 templates
└── files/               # Static files
```

#### Tasks (`roles/common/tasks/main.yml`)

```yaml
# roles/common/tasks/main.yml
---
# ========================================
# Task 1: Update apt cache
# ========================================
- name: Update apt cache
  apt:
    update_cache: yes
    cache_valid_time: 3600    # Don't update if cache is less than 1 hour old
  tags: packages

# ========================================
# Task 2: Install common packages
# ========================================
- name: Install common packages
  apt:
    name: "{{ common_packages }}"
    state: present
  tags: packages

# ========================================
# Task 3: Set timezone
# ========================================
- name: Set timezone
  timezone:
    name: "{{ timezone }}"
  tags: system

# ========================================
# Task 4: Configure NTP
# ========================================
- name: Configure NTP servers
  template:
    src: ntp.conf.j2
    dest: /etc/ntp.conf
    owner: root
    group: root
    mode: '0644'
  notify: restart ntp
  tags: system

# ========================================
# Task 5: Create admin users
# ========================================
- name: Create admin users
  user:
    name: "{{ item.name }}"
    groups: "{{ item.groups }}"
    shell: "{{ item.shell }}"
    create_home: yes
    state: present
  loop: "{{ admin_users }}"
  tags: users

# ========================================
# Task 6: Set up SSH keys for users
# ========================================
- name: Add SSH authorized keys
  authorized_key:
    user: "{{ item.name }}"
    key: "{{ item.ssh_key }}"
    state: present
  loop: "{{ admin_users }}"
  tags: users

# ========================================
# Task 7: Set hostname
# ========================================
- name: Set hostname
  hostname:
    name: "{{ inventory_hostname }}"
  tags: system

# ========================================
# Task 8: Update /etc/hosts
# ========================================
- name: Add all servers to /etc/hosts
  lineinfile:
    path: /etc/hosts
    line: "{{ hostvars[item].ansible_host }} {{ item }}"
    state: present
  loop: "{{ groups['all'] }}"
  tags: system

# ========================================
# Task 9: Configure journald (log management)
# ========================================
- name: Configure journald max disk usage
  lineinfile:
    path: /etc/systemd/journald.conf
    regexp: '^#?SystemMaxUse='
    line: 'SystemMaxUse=500M'
  notify: restart journald
  tags: logging

# ========================================
# Task 10: Set up log rotation
# ========================================
- name: Configure logrotate for syslog
  template:
    src: logrotate-syslog.j2
    dest: /etc/logrotate.d/syslog
    owner: root
    group: root
    mode: '0644'
  tags: logging
```

**Let's understand what's happening here:**

| Task | What It Does | Why It Matters |
|------|-------------|----------------|
| Update apt cache | Refreshes package list | Ensures we install latest versions |
| Install packages | Installs vim, curl, htop, etc. | Standard tools every server needs |
| Set timezone | Sets server clock | Consistent log timestamps |
| Configure NTP | Syncs time with NTP servers | Accurate time across all servers |
| Create users | Creates admin accounts | Standardized access management |
| SSH keys | Adds authorized keys | Password-less secure access |
| Set hostname | Names each server | Easy identification |
| Update /etc/hosts | All servers know each other | Name resolution without DNS |

#### Handlers (`roles/common/handlers/main.yml`)

```yaml
# roles/common/handlers/main.yml
---
- name: restart ntp
  service:
    name: ntp
    state: restarted

- name: restart journald
  service:
    name: systemd-journald
    state: restarted
```

> **What are handlers?**
> Handlers are tasks that only run when **notified**. If we change the NTP config, we need to restart NTP. But if the config didn't change, no restart needed. Handlers make this automatic.

#### Template (`roles/common/templates/ntp.conf.j2`)

```jinja2
# /etc/ntp.conf — Managed by Ansible
# Do NOT edit manually!

driftfile /var/lib/ntp/ntp.drift

statistics loopstats peerstats clockstats
filegen loopstats file loopstats type day enable

{% for server in ntp_servers %}
server {{ server }} iburst
{% endfor %}

restrict -4 default kod notrap nomodify nopeer noquery limited
restrict -6 default kod notrap nomodify nopeer noquery limited
restrict 127.0.0.1
restrict ::1
```

**What's this Jinja2 stuff?**
- `{% for server in ntp_servers %}` — loops through our NTP server list from variables
- `{{ server }}` — inserts the current server address
- This means if we change the NTP servers in `group_vars/all.yml`, the config file updates automatically!

### 8b — Security Role (Firewall + SSH Hardening)

```bash
ansible-galaxy init roles/security
```

#### Tasks (`roles/security/tasks/main.yml`)

```yaml
# roles/security/tasks/main.yml
---
# ========================================
# SSH Hardening
# ========================================
- name: Harden SSH — Disable root login
  lineinfile:
    path: /etc/ssh/sshd_config
    regexp: '^#?PermitRootLogin'
    line: "PermitRootLogin {{ ssh_permit_root_login }}"
  notify: restart sshd
  tags: security

- name: Harden SSH — Disable password authentication
  lineinfile:
    path: /etc/ssh/sshd_config
    regexp: '^#?PasswordAuthentication'
    line: "PasswordAuthentication {{ ssh_password_authentication }}"
  notify: restart sshd
  tags: security

- name: Harden SSH — Set SSH port
  lineinfile:
    path: /etc/ssh/sshd_config
    regexp: '^#?Port'
    line: "Port {{ ssh_port }}"
  notify: restart sshd
  tags: security

# ========================================
# UFW Firewall
# ========================================
- name: Install UFW
  apt:
    name: ufw
    state: present
  tags: firewall

- name: UFW — Set default deny incoming
  ufw:
    direction: incoming
    policy: deny
  tags: firewall

- name: UFW — Set default allow outgoing
  ufw:
    direction: outgoing
    policy: allow
  tags: firewall

- name: UFW — Allow common ports
  ufw:
    rule: allow
    port: "{{ item.split('/')[0] }}"
    proto: "{{ item.split('/')[1] }}"
  loop: "{{ firewall_allowed_ports }}"
  tags: firewall

# Allow group-specific ports (web, app, or db)
- name: UFW — Allow web server ports
  ufw:
    rule: allow
    port: "{{ item.split('/')[0] }}"
    proto: "{{ item.split('/')[1] }}"
  loop: "{{ web_firewall_ports | default([]) }}"
  when: "'webservers' in group_names"
  tags: firewall

- name: UFW — Allow app server ports
  ufw:
    rule: allow
    port: "{{ item.split('/')[0] }}"
    proto: "{{ item.split('/')[1] }}"
  loop: "{{ app_firewall_ports | default([]) }}"
  when: "'appservers' in group_names"
  tags: firewall

- name: UFW — Allow database ports
  ufw:
    rule: allow
    port: "{{ item.split('/')[0] }}"
    proto: "{{ item.split('/')[1] }}"
  loop: "{{ db_firewall_ports | default([]) }}"
  when: "'dbservers' in group_names"
  tags: firewall

- name: UFW — Enable firewall
  ufw:
    state: enabled
  tags: firewall

# ========================================
# Fail2Ban (Brute-force protection)
# ========================================
- name: Install fail2ban
  apt:
    name: fail2ban
    state: present
  tags: security

- name: Configure fail2ban for SSH
  template:
    src: jail.local.j2
    dest: /etc/fail2ban/jail.local
    owner: root
    group: root
    mode: '0644'
  notify: restart fail2ban
  tags: security

- name: Start and enable fail2ban
  service:
    name: fail2ban
    state: started
    enabled: yes
  tags: security
```

#### Handlers (`roles/security/handlers/main.yml`)

```yaml
# roles/security/handlers/main.yml
---
- name: restart sshd
  service:
    name: sshd
    state: restarted

- name: restart fail2ban
  service:
    name: fail2ban
    state: restarted
```

#### Template (`roles/security/templates/jail.local.j2`)

```jinja2
# /etc/fail2ban/jail.local — Managed by Ansible

[DEFAULT]
bantime  = 3600
findtime = 600
maxretry = 5
backend  = systemd

[sshd]
enabled  = true
port     = {{ ssh_port }}
filter   = sshd
logpath  = /var/log/auth.log
maxretry = 3
bantime  = 7200
```

**What does Fail2Ban do?**
- Watches SSH login attempts
- If someone fails **3 times in 10 minutes** → they get **banned for 2 hours**
- Protects your servers from brute-force attacks 🛡️

### 8c — Web Server Role (Nginx)

```bash
ansible-galaxy init roles/webserver
```

#### Tasks (`roles/webserver/tasks/main.yml`)

```yaml
# roles/webserver/tasks/main.yml
---
- name: Install Nginx
  apt:
    name: nginx
    state: present
  tags: webserver

- name: Deploy Nginx main config
  template:
    src: nginx.conf.j2
    dest: /etc/nginx/nginx.conf
    owner: root
    group: root
    mode: '0644'
  notify: reload nginx
  tags: webserver

- name: Deploy site config (reverse proxy to app servers)
  template:
    src: myapp.conf.j2
    dest: /etc/nginx/sites-available/myapp
    owner: root
    group: root
    mode: '0644'
  notify: reload nginx
  tags: webserver

- name: Enable site
  file:
    src: /etc/nginx/sites-available/myapp
    dest: /etc/nginx/sites-enabled/myapp
    state: link
  notify: reload nginx
  tags: webserver

- name: Remove default site
  file:
    path: /etc/nginx/sites-enabled/default
    state: absent
  notify: reload nginx
  tags: webserver

- name: Start and enable Nginx
  service:
    name: nginx
    state: started
    enabled: yes
  tags: webserver
```

#### Template (`roles/webserver/templates/myapp.conf.j2`)

```nginx
# /etc/nginx/sites-available/myapp — Managed by Ansible

# Upstream — load balance across app servers
upstream app_backend {
{% for backend in app_backends %}
    server {{ backend }};
{% endfor %}
}

server {
    listen 80;
    server_name {{ server_name }};

    # Access & Error logs
    access_log /var/log/nginx/myapp_access.log;
    error_log  /var/log/nginx/myapp_error.log;

    # Proxy settings
    location / {
        proxy_pass http://app_backend;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }

    # Health check endpoint
    location /health {
        access_log off;
        return 200 'OK';
        add_header Content-Type text/plain;
    }
}
```

**What this does:**

```
User Request → Nginx (web1/web2/web3)
                 ↓
         Load Balancer picks one
                 ↓
         ┌───────┼───────┬───────┐
         ↓       ↓       ↓       ↓
        app1    app2    app3    app4
```

Nginx **distributes traffic** across all 4 app servers — if one goes down, the others handle it!

#!/bin/bash
# MySQL Backup Script — Managed by Ansible

BACKUP_DIR="/var/backups/mysql"
DATE=$(date +%Y%m%d_%H%M%S)
RETENTION_DAYS=7

# Backup all databases
mysqldump --all-databases --single-transaction --routines --triggers \
    > "${BACKUP_DIR}/all_databases_${DATE}.sql"

# Compress
gzip "${BACKUP_DIR}/all_databases_${DATE}.sql"

# Remove backups older than retention period
find ${BACKUP_DIR} -name "*.sql.gz" -mtime +${RETENTION_DAYS} -delete

echo "[$(date)] Backup completed: all_databases_${DATE}.sql.gz"

## 🔑 Key Concepts Reinforced

| Concept | What You Learned |
|---------|-----------------|
| **Inventory** | Organize servers into groups (web, app, db) |
| **Roles** | Reusable, modular configuration units |
| **Variables** | `group_vars/` and `host_vars/` for flexible configuration |
| **Templates (Jinja2)** | Dynamic config files that adapt per server/group |
| **Handlers** | Only restart services when their config changes |
| **Vault** | Encrypt secrets — never put passwords in plain text |
| **Tags** | Run specific parts of your playbook |
| **Idempotency** | Run it 100 times — same result, no side effects |

## 🚨 Common Issues & Troubleshooting

| Problem | Solution |
|---------|----------|
| `UNREACHABLE` error | Check SSH keys, verify IP addresses, ensure VMs are running |
| `Permission denied` | Ensure `become: yes` is set, check sudo access |
| `Module not found` | Install missing Python modules on target (e.g., `python3-mysqldb`) |
| `Vault password wrong` | Re-create vault file or use `ansible-vault rekey` |
| `Service failed to start` | Check syntax of config templates, look at journalctl logs |
| Slow execution | Enable pipelining in `ansible.cfg`, use `--forks 10` |
