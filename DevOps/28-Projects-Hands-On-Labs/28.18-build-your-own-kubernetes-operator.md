## 🤔 What is a Kubernetes Operator?

Let me explain this with a story.

Imagine you're the **database admin** at a company. Every time someone needs a PostgreSQL database, they come to you:

- "Hey, can you create a database?"
- "Can you set up replication?"
- "Can you take a backup?"
- "Can you restore from last night's backup?"
- "Can you scale it up? We have more traffic."

You're doing the same tasks **over and over**. Wouldn't it be great if you could teach a robot to do all of this?

**That's exactly what a Kubernetes Operator is — a robot (software) that automates the human knowledge of managing an application.**

> **Operator** = Custom Controller + Custom Resource Definition (CRD) that encodes operational knowledge into software.

### The Official Definition

An Operator is a method of packaging, deploying, and managing a Kubernetes application. It extends Kubernetes to automate the management of complex, stateful applications by using **Custom Resources** to represent the desired state and **Controllers** to reconcile that state.

## 🧩 Key Concepts Before We Start

### 1. Custom Resource Definition (CRD)

Kubernetes comes with built-in resources: Pod, Deployment, Service, ConfigMap, etc.

A **CRD** lets you create your **OWN** resource type. For example:

```yaml
# Built-in Kubernetes resource
apiVersion: apps/v1
kind: Deployment          # ← Built-in kind
metadata:
  name: my-app

# YOUR custom resource (after creating a CRD)
apiVersion: mycompany.com/v1
kind: WebApp              # ← YOUR custom kind!
metadata:
  name: my-webapp
spec:
  image: nginx:1.25
  replicas: 3
  enableSSL: true
```

With a CRD, you can make Kubernetes understand **your** applications natively.

### 2. Controller

A **Controller** is a program that watches for changes to resources and takes action to make the **actual state** match the **desired state**.

```
                    ┌──────────────────┐
                    │  Desired State   │
                    │  (What you want) │
                    │                  │
                    │  WebApp:         │
                    │    replicas: 3   │
                    │    image: nginx  │
                    └────────┬─────────┘
                             │
                             ▼
                    ┌──────────────────┐
                    │   Controller     │
                    │   (Reconciler)   │◄──── Watches for changes
                    │                  │
                    │  "Hmm, desired   │
                    │   is 3 replicas  │
                    │   but actual is  │
                    │   2. Let me      │
                    │   create 1 more" │
                    └────────┬─────────┘
                             │
                             ▼
                    ┌──────────────────┐
                    │  Actual State    │
                    │  (What exists)   │
                    │                  │
                    │  2 pods running  │
                    │  → creates 1    │
                    │  → now 3 pods ✅│
                    └──────────────────┘
```

This is called the **Reconciliation Loop** — the heart of every operator.

### 3. Operator = CRD + Controller

| Component | What It Does |
|-----------|-------------|
| **CRD** | Defines a new resource type (like `WebApp`) |
| **Controller** | Watches the CRD and takes action to reconcile |
| **Operator** | The combined package (CRD + Controller + logic) |

### Real-World Operators

| Operator | What It Manages |
|----------|----------------|
| **Prometheus Operator** | Creates/manages Prometheus instances via `Prometheus` CRD |
| **Cert-Manager** | Manages TLS certificates via `Certificate` CRD |
| **Strimzi** | Manages Kafka clusters via `Kafka` CRD |
| **Zalando Postgres Operator** | Manages PostgreSQL clusters via `PostgresCluster` CRD |
| **ArgoCD** | Manages application deployments via `Application` CRD |

## 📋 What We're Building

We'll build a **WebApp Operator** that:

1. Accepts a `WebApp` custom resource with fields like `image`, `replicas`, and `port`
2. Automatically creates a **Deployment** for the web app
3. Automatically creates a **Service** to expose it
4. Handles **scaling** — when you change `replicas` in the CR, the operator updates the Deployment
5. Handles **image updates** — when you change `image`, the operator does a rolling update
6. Handles **deletion** — when you delete the `WebApp` CR, the operator cleans up everything

### The End Result

Instead of writing this:

```yaml
# Without operator — you need to write ALL of this:
apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-app
spec:
  replicas: 3
  selector:
    matchLabels:
      app: my-app
  template:
    metadata:
      labels:
        app: my-app
    spec:
      containers:
        - name: app
          image: nginx:1.25
          ports:
            - containerPort: 80
---
apiVersion: v1
kind: Service
metadata:
  name: my-app
spec:
  selector:
    app: my-app
  ports:
    - port: 80
      targetPort: 80
```

You just write this:

```yaml
# With YOUR operator — just this!
apiVersion: apps.mycompany.com/v1alpha1
kind: WebApp
metadata:
  name: my-app
spec:
  image: nginx:1.25
  replicas: 3
  port: 80
```

The operator automatically creates the Deployment, Service, and everything else! That's the power of operators.

## 🚀 Step 3 — Define the CRD (Custom Resource Definition)

Open `api/v1alpha1/webapp_types.go` and define what a `WebApp` resource looks like:

### `api/v1alpha1/webapp_types.go`

```go
/*

Licensed under the Apache License, Version 2.0 (the "License");
...
*/

package v1alpha1

import (
	metav1 "k8s.io/apimachinery/pkg/apis/meta/v1"
)

// ─────────────────────────────────────────────
// WebAppSpec — This is what the user specifies
// It's the "desired state" — what the user WANTS
// ─────────────────────────────────────────────
type WebAppSpec struct {
	// Image is the container image for the web application
	// Example: "nginx:1.25", "my-app:v2.0"
	// +kubebuilder:validation:Required
	Image string `json:"image"`

	// Replicas is the number of pod replicas to run
	// +kubebuilder:validation:Minimum=1
	// +kubebuilder:validation:Maximum=10
	// +kubebuilder:default=1
	Replicas int32 `json:"replicas,omitempty"`

	// Port is the container port the application listens on
	// +kubebuilder:validation:Minimum=1
	// +kubebuilder:validation:Maximum=65535
	// +kubebuilder:default=80
	Port int32 `json:"port,omitempty"`

	// EnableService controls whether a Kubernetes Service is created
	// +kubebuilder:default=true
	EnableService bool `json:"enableService,omitempty"`

	// ServiceType is the type of Kubernetes Service (ClusterIP, NodePort, LoadBalancer)
	// +kubebuilder:validation:Enum=ClusterIP;NodePort;LoadBalancer
	// +kubebuilder:default=ClusterIP
	ServiceType string `json:"serviceType,omitempty"`

	// Resources defines CPU/Memory limits for the container
	Resources *ResourceRequirements `json:"resources,omitempty"`

	// Env is a list of environment variables for the container
	Env []EnvVar `json:"env,omitempty"`
}

// ResourceRequirements defines compute resources
type ResourceRequirements struct {
	// CPULimit is the CPU limit (e.g., "500m", "1")
	CPULimit string `json:"cpuLimit,omitempty"`

	// MemoryLimit is the memory limit (e.g., "128Mi", "1Gi")
	MemoryLimit string `json:"memoryLimit,omitempty"`

	// CPURequest is the CPU request (e.g., "100m")
	CPURequest string `json:"cpuRequest,omitempty"`

	// MemoryRequest is the memory request (e.g., "64Mi")
	MemoryRequest string `json:"memoryRequest,omitempty"`
}

// EnvVar defines an environment variable
type EnvVar struct {
	// Name of the environment variable
	Name string `json:"name"`

	// Value of the environment variable
	Value string `json:"value"`
}

// ─────────────────────────────────────────────
// WebAppStatus — This is what the operator reports back
// It's the "actual state" — what CURRENTLY exists
// ─────────────────────────────────────────────
type WebAppStatus struct {
	// AvailableReplicas is the number of pods that are running and ready
	AvailableReplicas int32 `json:"availableReplicas,omitempty"`

	// Conditions represent the latest available observations of the WebApp's state
	Conditions []metav1.Condition `json:"conditions,omitempty"`

	// URL is the endpoint where the application can be accessed
	URL string `json:"url,omitempty"`

	// Phase represents the current lifecycle phase (Pending, Running, Failed)
	// +kubebuilder:validation:Enum=Pending;Running;Failed;Unknown
	Phase string `json:"phase,omitempty"`
}

// ─────────────────────────────────────────────
// WebApp — The main CRD struct
// This ties Spec and Status together
// ─────────────────────────────────────────────

// +kubebuilder:object:root=true
// +kubebuilder:subresource:status
// +kubebuilder:printcolumn:name="Image",type=string,JSONPath=`.spec.image`
// +kubebuilder:printcolumn:name="Replicas",type=integer,JSONPath=`.spec.replicas`
// +kubebuilder:printcolumn:name="Available",type=integer,JSONPath=`.status.availableReplicas`
// +kubebuilder:printcolumn:name="Phase",type=string,JSONPath=`.status.phase`
// +kubebuilder:printcolumn:name="Age",type="date",JSONPath=".metadata.creationTimestamp"
type WebApp struct {
	metav1.TypeMeta   `json:",inline"`
	metav1.ObjectMeta `json:"metadata,omitempty"`

	Spec   WebAppSpec   `json:"spec,omitempty"`
	Status WebAppStatus `json:"status,omitempty"`
}

// +kubebuilder:object:root=true
// WebAppList contains a list of WebApp
type WebAppList struct {
	metav1.TypeMeta `json:",inline"`
	metav1.ListMeta `json:"metadata,omitempty"`
	Items           []WebApp `json:"items"`
}

func init() {
	SchemeBuilder.Register(&WebApp{}, &WebAppList{})
}
```

**Let me explain the important parts:**

### The Spec (What the User Wants)

```go
type WebAppSpec struct {
    Image     string `json:"image"`        // What container to run
    Replicas  int32  `json:"replicas"`     // How many copies
    Port      int32  `json:"port"`         // What port to expose
    // ...
}
```

This maps directly to what the user writes in YAML:

```yaml
spec:
  image: nginx:1.25     → WebAppSpec.Image
  replicas: 3           → WebAppSpec.Replicas
  port: 80              → WebAppSpec.Port
```

### The Status (What Actually Exists)

```go
type WebAppStatus struct {
    AvailableReplicas int32  `json:"availableReplicas"`
    Phase             string `json:"phase"`
    URL               string `json:"url"`
}
```

The operator updates the status to show the current state. You can check it with:

```bash
kubectl get webapp my-app -o yaml
# status:
#   availableReplicas: 3
#   phase: Running
#   url: http://my-app.default.svc.cluster.local
```

### Kubebuilder Markers (The `// +kubebuilder:...` Comments)

These are **not** regular comments — they're special annotations that Kubebuilder uses to generate CRD YAML:

| Marker | What It Does |
|--------|-------------|
| `+kubebuilder:validation:Required` | Field is mandatory |
| `+kubebuilder:validation:Minimum=1` | Minimum allowed value |
| `+kubebuilder:validation:Maximum=10` | Maximum allowed value |
| `+kubebuilder:default=1` | Default value if not specified |
| `+kubebuilder:validation:Enum=A;B;C` | Only these values are allowed |
| `+kubebuilder:subresource:status` | Enable status subresource |
| `+kubebuilder:printcolumn` | Custom columns in `kubectl get` output |

### Generate the CRD

After defining the types, generate the CRD YAML and Go code:

```bash
# Generate deep copy functions and CRD manifests
make generate
make manifests

# This creates:
# config/crd/bases/apps.mycompany.com_webapps.yaml  ← The CRD YAML!
```

## 🚀 Step 4 — Write the Controller (The Brain!)

This is where the magic happens. The controller watches for `WebApp` resources and creates/updates Deployments and Services.

### `internal/controller/webapp_controller.go`

```go
/*
Copyright 2024.

Licensed under the Apache License, Version 2.0 (the "License");
...
*/

package controller

import (
	"context"
	"fmt"
	"time"

	appsv1 "k8s.io/api/apps/v1"
	corev1 "k8s.io/api/core/v1"
	"k8s.io/apimachinery/pkg/api/errors"
	"k8s.io/apimachinery/pkg/api/resource"
	metav1 "k8s.io/apimachinery/pkg/apis/meta/v1"
	"k8s.io/apimachinery/pkg/runtime"
	"k8s.io/apimachinery/pkg/types"
	"k8s.io/apimachinery/pkg/util/intstr"
	ctrl "sigs.k8s.io/controller-runtime"
	"sigs.k8s.io/controller-runtime/pkg/client"
	"sigs.k8s.io/controller-runtime/pkg/controller/controllerutil"
	"sigs.k8s.io/controller-runtime/pkg/log"

	appsv1alpha1 "github.com/mycompany/webapp-operator/api/v1alpha1"
)

// ─────────────────────────────────────────────
// WebAppReconciler — The main controller struct
// It holds the Kubernetes client and scheme
// ─────────────────────────────────────────────
type WebAppReconciler struct {
	client.Client
	Scheme *runtime.Scheme
}

// ─────────────────────────────────────────────
// RBAC Markers — These generate ClusterRole rules
// The controller needs permissions to manage these resources
// ─────────────────────────────────────────────

//+kubebuilder:rbac:groups=apps.mycompany.com,resources=webapps,verbs=get;list;watch;create;update;patch;delete
//+kubebuilder:rbac:groups=apps.mycompany.com,resources=webapps/status,verbs=get;update;patch
//+kubebuilder:rbac:groups=apps.mycompany.com,resources=webapps/finalizers,verbs=update
//+kubebuilder:rbac:groups=apps,resources=deployments,verbs=get;list;watch;create;update;patch;delete
//+kubebuilder:rbac:groups="",resources=services,verbs=get;list;watch;create;update;patch;delete
//+kubebuilder:rbac:groups="",resources=events,verbs=create;patch

// ─────────────────────────────────────────────
// Reconcile — THE HEART OF THE OPERATOR
//
// This function is called every time:
// 1. A WebApp resource is created
// 2. A WebApp resource is updated
// 3. A WebApp resource is deleted
// 4. A child resource (Deployment/Service) changes
//
// Our job: Make the ACTUAL state match the DESIRED state
// ─────────────────────────────────────────────
func (r *WebAppReconciler) Reconcile(ctx context.Context, req ctrl.Request) (ctrl.Result, error) {
	logger := log.FromContext(ctx)
	logger.Info("🔄 Reconciling WebApp", "name", req.NamespacedName)

	// ─── Step 1: Fetch the WebApp resource ───
	webapp := &appsv1alpha1.WebApp{}
	err := r.Get(ctx, req.NamespacedName, webapp)
	if err != nil {
		if errors.IsNotFound(err) {
			// The WebApp was deleted — nothing to do
			// (Kubernetes garbage collection handles child resources)
			logger.Info("WebApp not found — probably deleted. Ignoring.")
			return ctrl.Result{}, nil
		}
		logger.Error(err, "Failed to fetch WebApp")
		return ctrl.Result{}, err
	}

	// ─── Step 2: Set initial status ───
	if webapp.Status.Phase == "" {
		webapp.Status.Phase = "Pending"
		if err := r.Status().Update(ctx, webapp); err != nil {
			logger.Error(err, "Failed to update initial status")
			return ctrl.Result{}, err
		}
	}

	// ─── Step 3: Reconcile the Deployment ───
	deployment, err := r.reconcileDeployment(ctx, webapp)
	if err != nil {
		logger.Error(err, "Failed to reconcile Deployment")
		webapp.Status.Phase = "Failed"
		_ = r.Status().Update(ctx, webapp)
		return ctrl.Result{}, err
	}

	// ─── Step 4: Reconcile the Service (if enabled) ───
	if webapp.Spec.EnableService {
		if err := r.reconcileService(ctx, webapp); err != nil {
			logger.Error(err, "Failed to reconcile Service")
			webapp.Status.Phase = "Failed"
			_ = r.Status().Update(ctx, webapp)
			return ctrl.Result{}, err
		}
	}

	// ─── Step 5: Update status ───
	webapp.Status.AvailableReplicas = deployment.Status.AvailableReplicas
	webapp.Status.URL = fmt.Sprintf("http://%s.%s.svc.cluster.local:%d",
		webapp.Name, webapp.Namespace, webapp.Spec.Port)

	if deployment.Status.AvailableReplicas == webapp.Spec.Replicas {
		webapp.Status.Phase = "Running"
	} else {
		webapp.Status.Phase = "Pending"
	}

	if err := r.Status().Update(ctx, webapp); err != nil {
		logger.Error(err, "Failed to update WebApp status")
		return ctrl.Result{}, err
	}

	logger.Info("✅ Reconciliation complete",
		"replicas", webapp.Spec.Replicas,
		"available", deployment.Status.AvailableReplicas,
		"phase", webapp.Status.Phase,
	)

	// If not all replicas are available, requeue after 10 seconds
	if deployment.Status.AvailableReplicas != webapp.Spec.Replicas {
		return ctrl.Result{RequeueAfter: 10 * time.Second}, nil
	}

	return ctrl.Result{}, nil
}

// ─────────────────────────────────────────────
// reconcileDeployment — Creates or updates the Deployment
// ─────────────────────────────────────────────
func (r *WebAppReconciler) reconcileDeployment(ctx context.Context, webapp *appsv1alpha1.WebApp) (*appsv1.Deployment, error) {
	logger := log.FromContext(ctx)

	// Define the desired Deployment
	deployment := &appsv1.Deployment{
		ObjectMeta: metav1.ObjectMeta{
			Name:      webapp.Name,
			Namespace: webapp.Namespace,
		},
	}

	// CreateOrUpdate — This is the magic function!
	// It checks if the Deployment exists:
	//   - If it doesn't exist → CREATE it
	//   - If it exists → UPDATE it with the new spec
	result, err := controllerutil.CreateOrUpdate(ctx, r.Client, deployment, func() error {
		// ─── Define the Deployment spec ───
		labels := map[string]string{
			"app":                          webapp.Name,
			"app.kubernetes.io/name":       webapp.Name,
			"app.kubernetes.io/managed-by": "webapp-operator",
		}

		deployment.Spec = appsv1.DeploymentSpec{
			Replicas: &webapp.Spec.Replicas,
			Selector: &metav1.LabelSelector{
				MatchLabels: labels,
			},
			Template: corev1.PodTemplateSpec{
				ObjectMeta: metav1.ObjectMeta{
					Labels: labels,
				},
				Spec: corev1.PodSpec{
					Containers: []corev1.Container{
						{
							Name:  "webapp",
							Image: webapp.Spec.Image,
							Ports: []corev1.ContainerPort{
								{
									ContainerPort: webapp.Spec.Port,
									Protocol:      corev1.ProtocolTCP,
								},
							},
							// Add resource limits if specified
							Resources: r.buildResourceRequirements(webapp),
							// Add environment variables
							Env: r.buildEnvVars(webapp),
							// Add a readiness probe
							ReadinessProbe: &corev1.Probe{
								ProbeHandler: corev1.ProbeHandler{
									TCPSocket: &corev1.TCPSocketAction{
										Port: intstr.FromInt32(webapp.Spec.Port),
									},
								},
								InitialDelaySeconds: 5,
								PeriodSeconds:       10,
							},
						},
					},
				},
			},
		}

		// Set the owner reference — THIS IS CRITICAL!
		// It tells Kubernetes: "This Deployment is owned by this WebApp"
		// When the WebApp is deleted, Kubernetes automatically deletes the Deployment too!
		return controllerutil.SetControllerReference(webapp, deployment, r.Scheme)
	})

	if err != nil {
		return nil, fmt.Errorf("failed to reconcile Deployment: %w", err)
	}

	logger.Info("Deployment reconciled", "result", result)

	// Fetch the latest deployment status
	latestDeployment := &appsv1.Deployment{}
	err = r.Get(ctx, types.NamespacedName{
		Name:      webapp.Name,
		Namespace: webapp.Namespace,
	}, latestDeployment)
	if err != nil {
		return nil, err
	}

	return latestDeployment, nil
}

// ─────────────────────────────────────────────
// reconcileService — Creates or updates the Service
// ─────────────────────────────────────────────
func (r *WebAppReconciler) reconcileService(ctx context.Context, webapp *appsv1alpha1.WebApp) error {
	logger := log.FromContext(ctx)

	service := &corev1.Service{
		ObjectMeta: metav1.ObjectMeta{
			Name:      webapp.Name,
			Namespace: webapp.Namespace,
		},
	}

	result, err := controllerutil.CreateOrUpdate(ctx, r.Client, service, func() error {
		labels := map[string]string{
			"app": webapp.Name,
		}

		// Determine service type
		serviceType := corev1.ServiceTypeClusterIP
		switch webapp.Spec.ServiceType {
		case "NodePort":
			serviceType = corev1.ServiceTypeNodePort
		case "LoadBalancer":
			serviceType = corev1.ServiceTypeLoadBalancer
		}

		service.Spec = corev1.ServiceSpec{
			Selector: labels,
			Type:     serviceType,
			Ports: []corev1.ServicePort{
				{
					Port:       webapp.Spec.Port,
					TargetPort: intstr.FromInt32(webapp.Spec.Port),
					Protocol:   corev1.ProtocolTCP,
				},
			},
		}

		// Set owner reference
		return controllerutil.SetControllerReference(webapp, service, r.Scheme)
	})

	if err != nil {
		return fmt.Errorf("failed to reconcile Service: %w", err)
	}

	logger.Info("Service reconciled", "result", result)
	return nil
}

// ─────────────────────────────────────────────
// buildResourceRequirements — Convert our CRD's resource spec
// into Kubernetes resource requirements
// ─────────────────────────────────────────────
func (r *WebAppReconciler) buildResourceRequirements(webapp *appsv1alpha1.WebApp) corev1.ResourceRequirements {
	resources := corev1.ResourceRequirements{}

	if webapp.Spec.Resources != nil {
		res := webapp.Spec.Resources

		if res.CPULimit != "" || res.MemoryLimit != "" {
			resources.Limits = corev1.ResourceList{}
			if res.CPULimit != "" {
				resources.Limits[corev1.ResourceCPU] = resource.MustParse(res.CPULimit)
			}
			if res.MemoryLimit != "" {
				resources.Limits[corev1.ResourceMemory] = resource.MustParse(res.MemoryLimit)
			}
		}

		if res.CPURequest != "" || res.MemoryRequest != "" {
			resources.Requests = corev1.ResourceList{}
			if res.CPURequest != "" {
				resources.Requests[corev1.ResourceCPU] = resource.MustParse(res.CPURequest)
			}
			if res.MemoryRequest != "" {
				resources.Requests[corev1.ResourceMemory] = resource.MustParse(res.MemoryRequest)
			}
		}
	}

	return resources
}

// ─────────────────────────────────────────────
// buildEnvVars — Convert our CRD's env vars into K8s format
// ─────────────────────────────────────────────
func (r *WebAppReconciler) buildEnvVars(webapp *appsv1alpha1.WebApp) []corev1.EnvVar {
	var envVars []corev1.EnvVar
	for _, env := range webapp.Spec.Env {
		envVars = append(envVars, corev1.EnvVar{
			Name:  env.Name,
			Value: env.Value,
		})
	}
	return envVars
}

// ─────────────────────────────────────────────
// SetupWithManager — Register the controller with the manager
// This tells the controller framework:
// "Watch WebApp resources AND Deployments/Services owned by WebApp"
// ─────────────────────────────────────────────
func (r *WebAppReconciler) SetupWithManager(mgr ctrl.Manager) error {
	return ctrl.NewControllerManagedBy(mgr).
		For(&appsv1alpha1.WebApp{}).                // Watch WebApp CRDs
		Owns(&appsv1.Deployment{}).                  // Also watch owned Deployments
		Owns(&corev1.Service{}).                     // Also watch owned Services
		Complete(r)
}
```

```
                         WebApp Created/Updated/Deleted
                                    │
                                    ▼
                    ┌───────────────────────────────┐
                    │ Step 1: Fetch the WebApp CR   │
                    │ "Does this resource exist?"   │
                    └──────────────┬────────────────┘
                                   │
                          ┌────────┴────────┐
                          │                 │
                     Not Found           Found
                     (deleted)              │
                          │                 ▼
                     Return nil   ┌─────────────────────────┐
                     (done!)      │ Step 2: Set initial      │
                                  │ status to "Pending"      │
                                  └────────────┬────────────┘
                                               │
                                               ▼
                                  ┌─────────────────────────┐
                                  │ Step 3: Reconcile        │
                                  │ Deployment               │
                                  │ (Create or Update)       │
                                  └────────────┬────────────┘
                                               │
                                               ▼
                                  ┌─────────────────────────┐
                                  │ Step 4: Reconcile        │
                                  │ Service                  │
                                  │ (Create or Update)       │
                                  └────────────┬────────────┘
                                               │
                                               ▼
                                  ┌─────────────────────────┐
                                  │ Step 5: Update Status    │
                                  │ (availableReplicas,      │
                                  │  phase, URL)             │
                                  └─────────────────────────┘
```

### Key Patterns Explained

**1. Owner References (Garbage Collection)**

```go
controllerutil.SetControllerReference(webapp, deployment, r.Scheme)
```

This is like telling Kubernetes: "This Deployment belongs to this WebApp." When the WebApp is deleted, Kubernetes automatically cleans up the Deployment and Service. No manual cleanup needed!

**2. CreateOrUpdate Pattern**

```go
controllerutil.CreateOrUpdate(ctx, r.Client, deployment, func() error {
    // Define what the deployment should look like
    deployment.Spec = ...
    return nil
})
```

This is an incredibly useful function:
- If the Deployment **doesn't exist** → It creates it
- If the Deployment **exists** → It updates it with the new spec
- You don't have to write separate Create and Update logic!

**3. Requeue Pattern**

```go
return ctrl.Result{RequeueAfter: 10 * time.Second}, nil
```

If not all replicas are ready yet, we tell the controller: "Check again in 10 seconds." This is how operators wait for things to become ready without blocking.

## 🚀 Step 5 — Write Tests

Good operators need tests! Let's write a test for our controller:

### `internal/controller/webapp_controller_test.go`

```go
package controller

import (
	"context"
	"time"

	. "github.com/onsi/ginkgo/v2"
	. "github.com/onsi/gomega"

	appsv1 "k8s.io/api/apps/v1"
	corev1 "k8s.io/api/core/v1"
	metav1 "k8s.io/apimachinery/pkg/apis/meta/v1"
	"k8s.io/apimachinery/pkg/types"

	appsv1alpha1 "github.com/mycompany/webapp-operator/api/v1alpha1"
)

var _ = Describe("WebApp Controller", func() {
	const (
		WebAppName      = "test-webapp"
		WebAppNamespace = "default"
		timeout         = time.Second * 30
		interval        = time.Millisecond * 250
	)

	Context("When creating a WebApp", func() {
		It("Should create a Deployment and Service", func() {
			ctx := context.Background()

			// ─── Create a WebApp resource ───
			webapp := &appsv1alpha1.WebApp{
				TypeMeta: metav1.TypeMeta{
					APIVersion: "apps.mycompany.com/v1alpha1",
					Kind:       "WebApp",
				},
				ObjectMeta: metav1.ObjectMeta{
					Name:      WebAppName,
					Namespace: WebAppNamespace,
				},
				Spec: appsv1alpha1.WebAppSpec{
					Image:         "nginx:1.25",
					Replicas:      2,
					Port:          80,
					EnableService: true,
					ServiceType:   "ClusterIP",
				},
			}
			Expect(k8sClient.Create(ctx, webapp)).Should(Succeed())

			// ─── Verify Deployment is created ───
			deploymentKey := types.NamespacedName{
				Name:      WebAppName,
				Namespace: WebAppNamespace,
			}
			createdDeployment := &appsv1.Deployment{}

			// Wait for the Deployment to be created
			Eventually(func() bool {
				err := k8sClient.Get(ctx, deploymentKey, createdDeployment)
				return err == nil
			}, timeout, interval).Should(BeTrue())

			// Verify Deployment spec
			Expect(*createdDeployment.Spec.Replicas).Should(Equal(int32(2)))
			Expect(createdDeployment.Spec.Template.Spec.Containers[0].Image).Should(Equal("nginx:1.25"))

			// ─── Verify Service is created ───
			serviceKey := types.NamespacedName{
				Name:      WebAppName,
				Namespace: WebAppNamespace,
			}
			createdService := &corev1.Service{}

			Eventually(func() bool {
				err := k8sClient.Get(ctx, serviceKey, createdService)
				return err == nil
			}, timeout, interval).Should(BeTrue())

			Expect(createdService.Spec.Ports[0].Port).Should(Equal(int32(80)))
		})

		It("Should update Deployment when WebApp spec changes", func() {
			ctx := context.Background()

			// Fetch the existing WebApp
			webapp := &appsv1alpha1.WebApp{}
			Expect(k8sClient.Get(ctx, types.NamespacedName{
				Name:      WebAppName,
				Namespace: WebAppNamespace,
			}, webapp)).Should(Succeed())

			// Update replicas from 2 to 4
			webapp.Spec.Replicas = 4
			Expect(k8sClient.Update(ctx, webapp)).Should(Succeed())

			// Verify the Deployment is updated
			deployment := &appsv1.Deployment{}
			Eventually(func() int32 {
				k8sClient.Get(ctx, types.NamespacedName{
					Name:      WebAppName,
					Namespace: WebAppNamespace,
				}, deployment)
				return *deployment.Spec.Replicas
			}, timeout, interval).Should(Equal(int32(4)))
		})
	})
})
```

## 🧠 Key Concepts Recap

| Concept | What It Means | Analogy |
|---------|--------------|---------|
| **CRD** | Custom Resource Definition — defines a new resource type | Creating a new form template |
| **CR** | Custom Resource — an instance of a CRD | A filled-out form |
| **Controller** | Watches CRs and takes action | A worker who reads forms and does the work |
| **Reconcile** | Make actual state match desired state | Checking your to-do list and completing tasks |
| **Owner Reference** | Links parent → child resources | A parent-child relationship (delete parent = delete children) |
| **Operator** | CRD + Controller packaged together | A complete automation worker |
| **Finalizer** | Code that runs before deletion | Cleanup tasks before throwing something away |
| **Status Subresource** | Reports current state back to user | A progress update |

## 🔄 The Complete Lifecycle

```
1. User creates WebApp YAML
   └── kubectl apply -f my-webapp.yaml

2. Kubernetes API Server stores the WebApp object
   └── Stored in etcd

3. Controller detects the new WebApp
   └── Watch event triggers Reconcile()

4. Reconcile creates Deployment + Service
   └── With owner references pointing back to WebApp

5. User updates WebApp (e.g., more replicas)
   └── kubectl patch webapp ... replicas=5

6. Controller detects the change
   └── Reconcile updates the Deployment

7. User deletes WebApp
   └── kubectl delete webapp my-awesome-app

8. Kubernetes sees owner references
   └── Automatically deletes Deployment + Service (garbage collection)
```

## ⚠️ Common Mistakes & How to Avoid Them

| Mistake | Fix |
|---------|-----|
| Not setting owner references | Always use `SetControllerReference()` for garbage collection |
| Not handling "not found" errors | Check `errors.IsNotFound()` — it means the resource was deleted |
| Reconciling too aggressively | Use `RequeueAfter` instead of tight loops |
| Not using `CreateOrUpdate` | This pattern simplifies create vs update logic enormously |
| Hardcoding namespaces | Always use `req.Namespace` from the reconcile request |
| Not generating manifests after type changes | Always run `make generate && make manifests` after editing types |
| Testing in production | Always test with `make run` locally first |
